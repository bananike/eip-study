1과목 · 소프트웨어 설계

⭐ 응집도 (Cohesion) — 강할수록 좋음

약 → 강 : 우 논 시 절 교(통) 순 기
순서응집도뜻
1 (약)우연적(Coincidental)관련 없는 요소끼리 우연히 모임
2논리적(Logical)유사 성격(입력/출력 등)끼리 묶음
3시간적(Temporal)같은 시간대에 실행되는 것끼리 (초기화 등)
4절차적(Procedural)순서대로 수행
5교환/통신적(Communicational)같은 자료를 사용
6순차적(Sequential)앞 출력이 뒤 입력
7 (강)기능적(Functional)단일 기능만 수행 (가장 이상적)

⭐ 결합도 (Coupling) — 약할수록 좋음

약 → 강 : 자 스 제 외 공 내
순서결합도뜻
1 (약)자료(Data)파라미터로 값만 전달 (가장 좋음)
2스탬프(Stamp)자료구조 전체를 전달
3제어(Control)제어 플래그를 전달
4외부(External)외부 전역변수/장치 공유
5공통(Common)전역변수 공유
6 (강)내용(Content)다른 모듈 내부를 직접 참조 (가장 나쁨)
시험 단골: "가장 좋은 결합도 = 자료(Data)", "가장 좋은 응집도 = 기능적". 반대로 "가장 나쁜 = 내용/우연적".

⭐ 럼바우(Rumbaugh) 객체지향 분석 3모델링

객 동 기 = 객체 · 동적 · 기능
  • 객체(Object) 모델링: 객체 다이어그램으로 정적 구조 표현 (= 정보 모델링)
  • 동적(Dynamic) 모델링: 상태 다이어그램, 시간 흐름·상태 변화
  • 기능(Functional) 모델링: 자료흐름도(DFD), 데이터 변환 과정

⭐ 디자인 패턴 GoF 23 (생성5 / 구조7 / 행위11)

생성 5 "추빌팩프싱"Abstract Factory(추상팩토리), Builder(빌더), Factory Method(팩토리메소드), Prototype(프로토타입), Singleton(싱글톤)
구조 7 "어브컴 데퍼플프"Adapter(어댑터), Bridge(브리지), Composite(컴포지트), Decorator(데코레이터), Facade(퍼사드), Flyweight(플라이웨이트), Proxy(프록시)
행위 11Chain of Responsibility, Command, Interpreter, Iterator, Mediator, Memento, Observer(옵서버), State, Strategy(전략), Template Method, Visitor
자주 물음: 개수(생성5·구조7·행위11), Singleton=인스턴스 1개 보장, Observer=1:N 상태변화 통보, Adapter=인터페이스 변환, Facade=단순화된 통합 창구.

개발 방법론 & 애자일

  • 폭포수: 고전적·선형순차. 단계 명확, 변경 어려움
  • 프로토타입: 시제품으로 요구 확인
  • 나선형(Boehm): 위험분석 포함, 4단계 반복 (계획→위험분석→개발→평가)
  • V모델: 폭포수 확장, 단계별 테스트 대응
  • XP 5가치 "용단의피존": 용기·단순성·의사소통·피드백·존중
  • 스크럼: PO(제품책임자)·SM(스크럼마스터)·개발팀, 스프린트(2~4주), 데일리스크럼
  • 애자일 = 짧은 반복, 변화 수용, 동작 SW 중시

UML

  • 구조(정적) 다이어그램: 클래스, 객체, 컴포넌트, 배치, 복합체, 패키지
  • 행위(동적) 다이어그램: 유스케이스, 순차(Sequence), 통신, 상태, 활동, 타이밍
  • 관계: 연관·집합(Aggregation)·포함(Composition)·일반화(상속)·의존·실체화
함정: 순차 다이어그램 = 행위(동적) 다이어그램. "정적 측면 모델링"이라 하면 틀린 보기 (2022년 4월 기출).

미들웨어 종류

  • MOM(Message-Oriented): 비동기 메시지, 메시지 큐 (즉각응답보다 안정)
  • RPC: 원격 프로시저 호출  ·  ORB: 객체 요청  ·  TP monitor: 트랜잭션  ·  WAS: 웹앱  ·  DB 미들웨어(ODBC 등)

2과목 · 소프트웨어 개발

⭐ 소프트웨어 테스트 (최다 출제)

블랙박스 (명세 기반, 출력 검증)
동 경 원 비 오
  • 동치 분할(Equivalence)
  • 경계값 분석(Boundary)
  • 원인-효과 그래프
  • 비교 검사
  • 오류 예측
화이트박스 (내부 로직·경로)
  • 기초 경로(Basis Path)
  • 조건 / 결정 커버리지
  • 구문(문장) 커버리지
  • 데이터 흐름 검사
  • 루프 검사
테스트 레벨(V모델)단위 → 통합 → 시스템 → 인수
통합 방식하향식(Stub 필요) / 상향식(Driver 필요) / 빅뱅 / 샌드위치
테스트 원리살충제 패러독스(같은 테스트 반복→결함 못 찾음), 오류-부재의 궤변(결함 0이어도 요구 불충족이면 무의미), 결함 집중(파레토)
McCabe 순환복잡도 = 화살표(E) − 노드(N) + 2  (또는 폐구역+1). 계산문제 단골.

자료구조

  • 스택(Stack): LIFO. 후위표기·재귀·인터럽트
  • 큐(Queue): FIFO
  • 트리: 순회 = 전위(root먼저)/중위/후위
  • 트리 차수 = 자식 최대 수
  • 이진트리 최대 노드 = 2ᵏ−1
  • 그래프: 무방향 간선 최대 n(n−1)/2
중위→후위 변환 단골: A+B*C → A B C * + (곱셈 먼저).

형상관리 (SCM)

활동: 식별 → 통제 → 감사 → 기록(보고)
  • 형상 식별 → 형상 통제(변경 제어) → 형상 감사 → 형상 기록
  • 버전관리 도구: Git(분산), SVN·CVS(중앙집중)
  • 목적: 변경 추적·통제 (성능 향상 아님 ← 함정)

3과목 · 데이터베이스 구축

⭐ 정규화 단계 (최다 출제)

순서: 도 부 이 결 다 조
단계제거 대상
1NF (도메인)모든 속성값이 원자값(더 못 쪼갬)
2NF (부분)부분 함수 종속 제거
3NF (이행)이행 함수 종속 제거
BCNF (결정자)결정자가 후보키가 아닌 종속 제거
4NF (다치)다치(다중값) 종속 제거
5NF (조인)조인 종속 제거
이상현상(Anomaly) 3종: 삽입·삭제·갱신 이상. 정규화는 이 이상을 제거하려는 것.

⭐ 무결성 & 트랜잭션

무결성(Integrity)
  • 개체 무결성: 기본키 NOT NULL·유일
  • 참조 무결성: 외래키=참조값 또는 NULL
  • 도메인 무결성: 속성값이 정의역 내
트랜잭션 ACID 원 일 격 지
  • 원자성(Atomicity): 전부 or 전무
  • 일관성(Consistency)
  • 격리성(Isolation)
  • 지속성(Durability)

키(Key) & 관계대수

키슈퍼키(유일성○ 최소성✕) / 후보키(유일+최소) / 기본키 / 대체키 / 외래키
순수 관계연산SELECT(σ, 행) · PROJECT(π, 열) · JOIN(⋈) · DIVISION(÷)
일반 집합연산합집합 ∪ · 교집합 ∩ · 차집합 − · 곱(카티션) ×

SQL 분류

DDLCREATE · ALTER · DROP · TRUNCATE (구조 정의)
DMLSELECT · INSERT · UPDATE · DELETE (조작)
DCLGRANT · REVOKE · COMMIT · ROLLBACK (제어)
GRANT/REVOKE=권한, COMMIT/ROLLBACK=트랜잭션 제어. DELETE(DML) vs DROP(DDL) 구별 함정 주의.

4과목 · 프로그래밍 언어 · OS · 네트워크

⭐ OSI 7계층

하위→상위: 물 데 네 전 세 표 응
계층단위/장비·프로토콜
1 물리(Physical)비트 · 리피터/허브
2 데이터링크프레임 · 브리지/스위치, MAC
3 네트워크패킷 · 라우터, IP
4 전송(Transport)세그먼트 · TCP/UDP
5 세션 / 6 표현 / 7 응용데이터 · HTTP·FTP·SMTP·DNS

운영체제 — 교착상태 4조건 & 스케줄링

교착 4조건: 상 점 비 환 = 상호배제·점유대기·비선점·환형대기
  • 선점: RR(라운드로빈), SRT, 다단계큐
  • 비선점: FCFS, SJF, 우선순위, HRN
  • HRN 우선순위 = (대기+서비스)/서비스
  • 페이지 교체: FIFO·LRU·LFU·OPT·NUR
  • 벨레이디 모순: FIFO에서 프레임↑인데 부재↑

네트워크 라우팅 / IP

  • 라우팅: RIP(거리벡터, 홉≤15) · OSPF(링크상태) · BGP(AS간)
  • IPv4=32비트, IPv6=128비트(16진수, 헤더단순·보안강화)
  • TCP=연결형·신뢰성, UDP=비연결·빠름

언어 핵심

  • C: 배열명=포인터(주소), call by value vs 포인터(reference)
  • Java: 상속·오버라이딩(재정의, 시그니처 같음) vs 오버로딩(중복, 매개변수 다름)
  • Python: 들여쓰기 블록, 동적타이핑, 리스트/튜플/딕셔너리/셋

5과목 · 정보시스템 구축관리 (보안·방법론)

⭐ 암호화 (최다 출제)

대칭키 (비밀키)같은 키로 암·복호. 빠름. DES·AES·SEED·ARIA·IDEA·RC4
비대칭키 (공개키)공개키 암호/개인키 복호. 느림·키분배 쉬움. RSA·ECC·DSA·디피헬만
해시 (단방향)복호 불가, 무결성. MD5·SHA·HAVAL
함정: RSA=비대칭. AES/DES/SEED/ARIA=대칭. "대칭키는 키 개수 n(n−1)/2".
