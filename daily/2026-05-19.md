### 문제 4 [출처: 기출 유형 복원 — 2022년 3회 필기 반복문/range]

### 문제 5 [출처: 기출 유형 복원 — 2023년 1회 필기 자료형 특성]
Python 자료형에 대한 설명으로 옳지 않은 것은?
① list는 가변(mutable) 자료형이며 순서를 가진다.
② tuple은 불변(immutable) 자료형이며 인덱싱이 가능하다.
③ set은 중복을 허용하지 않으며 인덱싱이 가능하다.
④ dictionary는 키(key)의 중복을 허용하지 않는다.

### 문제 6 [출처: 기출 유형 복원 — 2024년 1회 필기 식별자 규칙]
Python 변수(식별자) 명명 규칙으로 옳지 않은 것은?
① 영문자, 숫자, 언더스코어(_)를 사용할 수 있다.
② 첫 글자는 숫자로 시작할 수 없다.
③ 대소문자를 구분하지 않는다.
④ 예약어(keyword)는 변수명으로 사용할 수 없다.

### 문제 7 [출처: 기출 유형 복원 — 2023년 2회 필기 클래스/self]
다음 Python 클래스 코드의 실행 결과로 옳은 것은?
```python
class Cls:
    cnt = 0
    def __init__(self, v):
        self.v = v
        Cls.cnt += 1

a = Cls(10)
b = Cls(20)
c = Cls(30)
print(Cls.cnt, b.v)
```
① 1 10
② 2 20
③ 3 20
④ 3 30

### 문제 8 [출처: 기출 유형 복원 — 2024년 2회 필기 예외처리]
다음 Python 코드의 출력 결과로 옳은 것은?
```python
try:
    x = 10 / 0
    print("A")
except ZeroDivisionError:
    print("B")
except Exception:
    print("C")
finally:
    print("D")
```
① A
② B D
③ C D
④ B C D

## 💡 상세 해설 (자기 설명과 대조)

### 문제 1
정답: ②
- 초기 a = {A,B,C,D} (원소 4개).
- add('E') → {A,B,C,D,E} (5개).
- add('A') → 이미 존재, 변화 없음 (set은 중복 불허).
- remove('B') → {A,C,D,E} (4개).
- len(a) = 4.

### 문제 2
정답: ①
- 슬라이싱 `a[start:stop:step]`. start=1, stop=5, step=2.
- 인덱스 1(20) → 인덱스 3(40) → 인덱스 5는 stop 미포함이라 종료.
- 결과: [20, 40].

### 문제 3
정답: ③
- 초기 d = {'a':1, 'b':2, 'c':3}.
- d['b'] = 20 → 키 b의 값을 2 → 20으로 갱신.
- d['d'] = 4 → 새 키-값 추가.
- 최종 values = [1, 20, 3, 4], sum = 28.

### 문제 4
정답: ②
- range(1, 11, 2) → 1, 3, 5, 7, 9 (stop=11 미포함).
- 합 = 1+3+5+7+9 = 25.

### 문제 5
정답: ③
- ③ set은 순서가 없으므로 인덱싱(`s[0]`)이 불가능하다. 따라서 옳지 않음.
- ① list: 가변, 순서 있음 → 옳음.
- ② tuple: 불변, 인덱싱 가능 → 옳음.
- ④ dictionary: key 중복 불가, value는 중복 가능 → 옳음.

### 문제 6
정답: ③
- ③ Python은 **대소문자를 구분**한다 (`Var` ≠ `var`). 따라서 옳지 않음.
- ①, ②, ④는 모두 Python 식별자 규칙으로 옳음.

### 문제 7
정답: ③
- `cnt`는 클래스 변수. 인스턴스 a, b, c가 생성될 때마다 `Cls.cnt += 1` 실행.
- 따라서 Cls.cnt = 3.
- `b.v`는 b 객체의 인스턴스 변수 v로 20.
- 결과: `3 20`.

### 문제 8
정답: ②
- `10 / 0` 실행 시 ZeroDivisionError 발생.
- 첫 except 절(ZeroDivisionError)이 매칭 → "B" 출력.
- 두 번째 except(Exception)는 실행되지 않음.
- finally는 예외 발생 여부와 무관하게 항상 실행 → "D" 출력.
- 결과: `B`, `D` (두 줄).

## 📚 역추적 참조 (사후 검증용 — 학습 자료 아님)

### 핵심 용어
| 용어 | 정의 |
|---|---|
| mutable | 객체 생성 후 내부 상태 변경 가능 (list, dict, set) |
| immutable | 객체 생성 후 변경 불가 (tuple, str, int, float, bool, frozenset) |
| 슬라이싱 | `[start:stop:step]` 문법으로 시퀀스의 부분 추출, stop 미포함 |
| 컴프리헨션 | `[식 for 변수 in 반복 if 조건]` 형식의 리스트/딕트/셋 생성 표현식 |
| self | 인스턴스 메서드의 첫 매개변수, 호출 시점의 인스턴스 객체를 가리킴 |
| 클래스 변수 | 클래스 자체에 속한 변수, 모든 인스턴스가 공유 |
| 인스턴스 변수 | 각 객체별로 독립적으로 존재하는 변수 |

### 핵심 개념 (오늘 문제에 등장)
- set 연산: add(원소), remove(원소, 없으면 KeyError), discard(원소, 없어도 무시), update(이터러블), pop(임의 원소 제거).
- 슬라이싱 step 음수: `a[::-1]` 역순.
- dict 메서드: keys(), values(), items(), get(키, 기본값), pop(키), update(다른 dict).
- range(start, stop, step): stop 미포함, step은 양수/음수 가능.
- 예외 처리 실행 순서: try → (예외 발생 시) 매칭되는 첫 except → finally. 예외 없으면 try → else(있다면) → finally.
- 식별자 규칙: 영문자/숫자/언더스코어, 첫 글자는 숫자 불가, 대소문자 구분, 예약어 불가.
- Python 예약어 33개 (3.x 기준 일부): False, None, True, and, as, assert, async, await, break, class, continue, def, del, elif, else, except, finally, for, from, global, if, import, in, is, lambda, nonlocal, not, or, pass, raise, return, try, while, with, yield.
