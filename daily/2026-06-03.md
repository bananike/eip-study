## 🔍 오답 역추적 개념 발췌

오늘 문제 풀이 시 헷갈릴 가능성이 높은 키워드(설명 없음, 정답표·해설 참조).
- SEED — KISA 1999년 개발, 128비트 블록/키, Feistel 16라운드
- ARIA — 학(Academy)·연(Research Institute)·관(Agency) 협력 개발, 128비트 블록, 128/192/256비트 키, SPN 구조
- HIGHT — 경량(저전력·RFID·USN용) 64비트 블록, 128비트 키
- LEA — 고속 경량 블록 암호, 128비트 블록, 128/192/256비트 키
- DES — 64비트 블록, 56비트 유효 키, Feistel
- 3DES — DES 3회 적용(168비트 유효 키)
- AES — 128비트 블록, 128/192/256비트 키, SPN, Rijndael
- IDEA — 64비트 블록, 128비트 키
- RC4 — 스트림 암호
- RSA 기반 = 소인수분해
- ECC 기반 = 타원곡선 이산대수
- ElGamal/DH/DSA 기반 = 이산대수
- 배낭(Knapsack) — Merkle-Hellman 공개키 (해독됨)
- 해시 출력 비트수: MD5=128, SHA-1=160, HAS-160=160, SHA-224=224, SHA-256=256, SHA-384=384, SHA-512=512
- 해시 충돌(Collision) — 비둘기집 원리상 이론적으로 반드시 존재. "절대 없도록 보장" = 거짓
- 접근통제: MAC(강제, 보안등급) / DAC(임의, 소유자 권한 위임) / RBAC(역할) / ABAC(속성)
- Bell-LaPadula — 기밀성 모델(No Read Up, No Write Down)
- Biba — 무결성 모델(No Read Down, No Write Up)
- Clark-Wilson — 무결성(상용 환경)
- 정보보안 3요소 CIA = Confidentiality + Integrity + Availability
- 부인방지(Non-repudiation)는 CIA 확장 요소 (3요소 아님)
- AAA = Authentication + Authorization + Accounting (Availability 아님)
- RADIUS / TACACS+ / DIAMETER — AAA 프로토콜

## 💡 상세 해설 (자기 설명과 대조)

### 문제 1
정답: ②
- 제공 해설: SEED는 1999년 한국인터넷진흥원(KISA)이 개발한 국산 대칭키 블록 암호. 128비트 평문 블록과 128비트 키, Feistel 구조 16라운드. 국내 표준(TTAS.KO-12.0004)으로 전자상거래·금융·무선통신에 광범위하게 사용.
- 오답 이유:
  - ①: AES — 미국 NIST가 Rijndael 알고리즘을 표준으로 채택. SPN 구조. 국내 개발 아님.
  - ③: RSA — 비대칭키(공개키) 암호. 블록 암호 아님.
  - ④: IDEA — 스위스 ETH 개발(1991년). 64비트 블록, 128비트 키. PGP에서 사용. 국내 개발 아님.
- 함정: "국내 개발 + 128비트 + Feistel" = SEED. 같은 국산이지만 ARIA는 SPN 구조 / HIGHT는 64비트 블록 / LEA는 SPN+ARX 구조로 변별.

### 문제 2
정답: ①
- 제공 해설: RSA(Rivest-Shamir-Adleman, 1978)는 두 개의 큰 소수의 곱 N = p × q를 만들면 N을 다시 p, q로 인수분해하기 매우 어렵다는 사실에 기반한다. 공개키 e와 개인키 d는 오일러 함수 φ(N) = (p-1)(q-1)을 이용해 e·d ≡ 1 (mod φ(N))을 만족하도록 생성.
- 오답 이유:
  - ②: 이산대수 — Diffie-Hellman, ElGamal, DSA의 기반. RSA 아님.
  - ③: 타원곡선 이산대수 — ECC(Elliptic Curve Cryptography)의 기반.
  - ④: 배낭(Knapsack) — Merkle-Hellman 공개키 암호의 기반. 1984년 Shamir에 의해 해독되어 현재 사용 안 함.
- 함정: 비대칭키 알고리즘마다 수학적 난제 다름. RSA=소인수분해 / DH·ElGamal·DSA=이산대수 / ECC=타원곡선 이산대수. 필수 암기.

### 문제 3
정답: ③
- 제공 해설: 대칭키(비밀키) 암호는 암·복호화에 동일한 키를 사용. DES, 3DES, AES, SEED, ARIA, HIGHT, LEA, IDEA, Blowfish, RC4/RC5/RC6 등이 해당. RSA는 비대칭키(공개키) 암호로 암호화 키(공개키)와 복호화 키(개인키)가 다름.
- 오답 이유:
  - ①: DES — 64비트 블록 대칭키 (NIST 표준).
  - ②: AES — 128비트 블록 대칭키 (NIST 표준).
  - ④: SEED — 128비트 블록 대칭키 (KISA 국산).
- 함정: 대칭키와 비대칭키 분류는 거의 매 회차 출제. RSA·ECC·ElGamal·DH·DSA = 비대칭 / 나머지 = 대칭 (해시 제외).

### 문제 4
정답: ④
- 제공 해설: 해시 함수는 무한한 입력 공간을 유한한 출력 공간으로 매핑하므로 비둘기집 원리에 의해 충돌은 반드시 존재한다. 단지 충돌을 찾기가 계산적으로 어려울 뿐(강한 충돌 저항성). "절대 발생하지 않도록 수학적으로 보장"은 거짓.
- 오답 이유:
  - ①: 가변 입력 → 고정 출력은 해시의 기본 성질로 옳음.
  - ②: 일방향성(원본 복원 불가)은 해시의 핵심 성질로 옳음.
  - ③: MD5=128비트, SHA-1=160비트, SHA-256=256비트로 옳음.
- 함정: 해시 충돌은 이론적으로 반드시 존재. MD5와 SHA-1은 실제로 충돌 사례가 발견되어 deprecated. "절대" "100%" "보장" 같은 강한 단정 보기는 의심.

### 문제 5
정답: ②
- 제공 해설: MAC(강제적 접근 통제)는 시스템(관리자)이 사용자(주체)와 자원(객체)에 보안 등급(라벨)을 부여하고, 등급 비교를 통해 접근 권한을 강제로 결정. 군·정부 기관 기밀 보호용. Bell-LaPadula(기밀성), Biba(무결성)가 대표 모델.
- 오답 이유:
  - ①: DAC(임의적 접근 통제) — 객체 소유자가 자율적으로 권한 부여(파일 시스템 chmod). 신원(Identity) 기반.
  - ③: RBAC(역할 기반 접근 통제) — 사용자 역할(직무)에 따라 권한 부여. 기업 환경에 적합.
  - ④: ABAC(속성 기반 접근 통제) — 사용자·자원·환경 속성과 정책 규칙을 조합. 가장 세분화된 통제.
- 함정: "보안 등급(라벨) + 강제" = MAC. "소유자가 권한 위임" = DAC. "역할" = RBAC. 키워드로 즉시 변별.

### 문제 6
정답: ④
- 제공 해설: 정보보안의 3대 요소(CIA)는 기밀성(Confidentiality), 무결성(Integrity), 가용성(Availability). 부인방지(Non-repudiation)는 인증·책임성과 함께 CIA를 확장한 추가 요소이며 3대 요소에는 포함되지 않는다.
- 오답 이유:
  - ①: 기밀성 — 허가된 자만 정보에 접근. CIA 1요소.
  - ②: 무결성 — 정보가 인가되지 않은 변경으로부터 보호. CIA 2요소.
  - ③: 가용성 — 인가된 사용자가 필요할 때 자원 접근 가능. CIA 3요소.
- 함정: 부인방지(전자서명·로그)는 보안 확장 요소이지만 "3대 요소"에 묶이지 않음. CIA 암기 필수.

### 문제 7
정답: ④
- 제공 해설: AAA는 Authentication(인증), Authorization(권한 부여), Accounting(과금/감사) 3단계. 가용성(Availability)은 CIA의 한 축이지 AAA에 포함되지 않는다. AAA 프로토콜: RADIUS, TACACS+, DIAMETER.
- 오답 이유:
  - ①: 인증(누구인가) — AAA 1단계로 옳음.
  - ②: 권한 부여(무엇을 할 수 있는가) — AAA 2단계로 옳음.
  - ③: 과금/감사(무엇을 했는가) — AAA 3단계로 옳음.
- 함정: A로 시작하는 단어 함정. AAA의 세 번째 A는 Availability가 아니라 Accounting. Availability는 CIA 소속.

### 문제 8
정답: ②
- 제공 해설: 비대칭키(공개키) 암호는 암호화/복호화 키가 다른 한 쌍(공개키·개인키)을 사용. RSA(소인수분해), ECC(타원곡선 이산대수), ElGamal(이산대수), DH(이산대수, 키교환), DSA(이산대수, 전자서명)가 대표.
- 오답 이유:
  - ①: DES, AES, SEED — 모두 대칭키 블록 암호.
  - ③: MD5, SHA-256, HMAC — 해시 함수 / 메시지 인증 코드(암호화 아님).
  - ④: ARIA, HIGHT, LEA — 모두 국산 대칭키 블록 암호.
- 함정: 해시(MD5/SHA)는 암호화가 아닌 무결성 검증. HMAC은 해시 기반 MAC. "비대칭" 분류에서 제외.

### 핵심 개념
- **대칭 vs 비대칭 한눈 정리:**
  - 대칭: 동일 키, 빠름, 키 분배 어려움 → 대용량 데이터 암호화
  - 비대칭: 다른 키 쌍, 느림, 키 분배 용이 → 키 교환·전자서명
  - 실제: 비대칭으로 세션키 교환 → 대칭으로 데이터 암호화 (하이브리드)
- **국산 암호 4총사:**
  - SEED(1999, 128/128, Feistel) — 범용
  - HIGHT(2005, 64/128, 경량) — RFID/USN
  - ARIA(2003, 128/128·192·256, SPN) — 정부 표준 후계
  - LEA(2013, 128/128·192·256, 고속 경량) — IoT
- **비대칭키 수학적 난제:**
  - 소인수분해 → RSA
  - 이산대수 → DH, ElGamal, DSA
  - 타원곡선 이산대수 → ECC, ECDSA, ECDH
  - 배낭 → Merkle-Hellman (해독됨)
- **해시 4대 성질:**
  - ① 압축성(가변 입력 → 고정 출력)
  - ② 일방향성(역상 저항성, Pre-image resistance)
  - ③ 약한 충돌 저항성(2nd Pre-image resistance)
  - ④ 강한 충돌 저항성(Collision resistance) — 충돌 자체는 존재
- **접근통제 정책 핵심 구분:**
  - MAC: 시스템이 강제, 보안 라벨 비교(군·정부)
  - DAC: 객체 소유자가 자율 위임(파일 시스템 ACL)
  - RBAC: 역할(직무)에 권한 매핑(기업)
  - ABAC: 다중 속성·정책 엔진(가장 유연)
- **보안 모델 핵심 규칙:**
  - Bell-LaPadula(기밀성): No Read Up(상위 등급 못 읽음), No Write Down(하위 등급에 못 씀)
  - Biba(무결성): No Read Down(하위 등급 못 읽음), No Write Up(상위 등급에 못 씀)
  - Biba는 BLP의 정반대 방향
- **CIA vs AAA 혼동 주의:**
  - CIA(보안 목표 3요소): Confidentiality, Integrity, Availability
  - AAA(접근 통제 절차): Authentication, Authorization, Accounting
  - 세 번째 A가 헷갈림 — AAA는 Accounting(과금/감사), CIA는 Availability(가용성)
