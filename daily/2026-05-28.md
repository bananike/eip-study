## 🎯 오늘의 기출 8문제

### 문제 1 [빈출 유형: 정규화의 목적]
데이터베이스 정규화(Normalization)의 목적으로 가장 거리가 먼 것은?
① 데이터의 중복(Redundancy)을 최소화한다.
② 삽입·삭제·갱신 이상(Anomaly)의 발생 가능성을 줄인다.
③ 모든 질의(Query)의 검색 속도를 무조건 향상시킨다.
④ 데이터 구조의 안정성과 일관성을 유지한다.

### 문제 2 [빈출 유형: 제1정규형(1NF)의 조건]
릴레이션의 모든 속성(Attribute)이 더 이상 분해할 수 없는 원자값(Atomic Value)만으로 구성되어 있을 때 만족하는 정규형은?
① 제1정규형(1NF)
② 제2정규형(2NF)
③ 제3정규형(3NF)
④ 보이스/코드 정규형(BCNF)

### 문제 3 [빈출 유형: 제2정규형(2NF) 도달 조건]
제1정규형(1NF)을 만족하는 릴레이션에서 기본키에 대한 부분 함수 종속(Partial Functional Dependency)을 제거하면 만족하게 되는 정규형은?
① 제1정규형(1NF)
② 제2정규형(2NF)
③ 제3정규형(3NF)
④ 보이스/코드 정규형(BCNF)

### 문제 4 [빈출 유형: 제3정규형(3NF) 도달 조건]
제2정규형(2NF)을 만족하는 릴레이션에서 기본키가 아닌 모든 속성이 기본키에 대해 이행적 함수 종속(Transitive Functional Dependency)되지 않도록 제거하면 만족하게 되는 정규형은?
① 제1정규형(1NF)
② 제2정규형(2NF)
③ 제3정규형(3NF)
④ 제4정규형(4NF)

### 문제 5 [빈출 유형: 보이스/코드 정규형(BCNF)의 정의]
보이스/코드 정규형(BCNF, Boyce-Codd Normal Form)에 대한 설명으로 옳은 것은?
① 릴레이션의 모든 속성이 원자값을 갖는 상태이다.
② 릴레이션의 모든 결정자(Determinant)가 후보키(Candidate Key)인 상태이다.
③ 다치 종속(Multi-Valued Dependency)을 제거한 상태이다.
④ 조인 종속(Join Dependency)을 제거한 상태이다.

### 문제 6 [빈출 유형: 정규화 단계 ↔ 제거 종속 매칭]
정규화 단계와 각 단계에서 제거되는 대상의 연결로 옳지 않은 것은?
① 제1정규형(1NF) — 원자값이 아닌(다중값) 속성 제거
② 제2정규형(2NF) — 부분 함수 종속 제거
③ 제3정규형(3NF) — 이행적 함수 종속 제거
④ 제4정규형(4NF) — 결정자이면서 후보키가 아닌 함수 종속 제거

### 문제 7 [빈출 유형: 이상 현상(Anomaly)의 종류]
정규화가 충분하지 않은 릴레이션에서 발생할 수 있는 이상(Anomaly) 현상에 해당하지 않는 것은?
① 삽입 이상(Insertion Anomaly)
② 삭제 이상(Deletion Anomaly)
③ 갱신 이상(Update Anomaly)
④ 검색 이상(Retrieval Anomaly)

### 문제 8 [빈출 유형: 반정규화(De-normalization)]
정규화된 데이터베이스에서 조회 성능 향상 등을 목적으로, 의도적으로 데이터의 중복을 허용하거나 테이블을 통합·분할·추가하는 기법은?
① 정규화(Normalization)
② 반정규화(De-normalization)
③ 무결성 제약(Integrity Constraint)
④ 인덱싱(Indexing)

## 🔍 오답 역추적 개념 발췌

- 정규화 단계 순서 암기: 도(원자값) → 부(부분 함수 종속) → 이(이행 함수 종속) → 결(결정자=후보키) → 다(다치 종속) → 조(조인 종속)
- 1NF: 모든 속성이 원자값(분해 불가, 다중값/반복 그룹 없음)
- 2NF: 1NF + 부분 함수 종속 제거 (기본키의 일부에만 종속되는 속성 없음)
- 3NF: 2NF + 이행적 함수 종속 제거 (X→Y, Y→Z ⇒ X→Z 제거)
- BCNF: 3NF 강화, 모든 결정자가 후보키
- 4NF: BCNF + 다치 종속(MVD) 제거
- 5NF(PJNF): 4NF + 조인 종속(JD) 제거
- 함수 종속 X→Y: X=결정자(Determinant), Y=종속자(Dependent)
- 완전 함수 종속 vs 부분 함수 종속 vs 이행 함수 종속
- 이상 현상 3종: 삽입 이상·삭제 이상·갱신 이상 ("검색 이상"은 존재하지 않음)
- 반정규화(De-normalization): 성능을 위해 의도적으로 중복 허용 (정규화의 역과정)
- 정규화는 조인 증가로 검색 성능을 저하시킬 수 있음 → "성능 무조건 향상"은 오답

## 💡 상세 해설 (자기 설명과 대조)

### 문제 1 — 정답 ③
- **정규화는 검색(질의) 성능을 무조건 향상시키지 않는다.** 정규화로 테이블이 분할되면 조인(Join) 연산이 늘어 오히려 조회 성능이 저하될 수 있고, 이를 보완하기 위해 반정규화를 사용한다. 따라서 ③이 "거리가 먼" 보기.
- ①: 중복 최소화는 정규화의 핵심 목적. 정답 설명.
- ②: 삽입·삭제·갱신 이상 제거가 정규화의 주된 동기. 정답 설명.
- ④: 데이터 구조의 안정성·일관성 유지도 정규화의 목적. 정답 설명.

### 문제 2 — 정답 ①
- **제1정규형(1NF)**: 모든 속성이 원자값(더 이상 분해 불가)만 갖는 상태. 다중값 속성·반복 그룹이 없어야 한다.
- ②: 2NF = 1NF + 부분 함수 종속 제거.
- ③: 3NF = 2NF + 이행적 함수 종속 제거.
- ④: BCNF = 모든 결정자가 후보키.

### 문제 3 — 정답 ②
- **제2정규형(2NF)**: 1NF를 만족하면서 기본키에 대한 부분 함수 종속을 제거한 상태(모든 비키 속성이 기본키에 완전 함수 종속).
- 예: 기본키가 (학번, 과목번호)일 때 학번만으로 결정되는 '학년'은 부분 함수 종속 → 분리해야 2NF.
- ①: 1NF는 원자값 조건만 만족.
- ③: 3NF는 이행 종속까지 제거.
- ④: BCNF는 결정자=후보키 조건.

### 문제 4 — 정답 ③
- **제3정규형(3NF)**: 2NF를 만족하면서 기본키가 아닌 속성들 사이의 이행적 함수 종속(X→Y, Y→Z ⇒ X→Z)을 제거한 상태.
- ①②: 1NF·2NF는 이행 종속을 제거하기 전 단계.
- ④: 4NF는 다치 종속(MVD) 제거 단계.

### 문제 5 — 정답 ②
- **BCNF**: 제3정규형을 더 강화한 정규형으로, **릴레이션의 모든 결정자(Determinant)가 후보키(Candidate Key)** 인 상태. 즉 "결정자이면서 후보키가 아닌" 함수 종속을 제거한 것.
- ①: 원자값 조건은 1NF.
- ③: 다치 종속 제거는 4NF.
- ④: 조인 종속 제거는 5NF(PJNF).

### 문제 6 — 정답 ④
- **"결정자이면서 후보키가 아닌 함수 종속 제거"는 BCNF의 조건**이다. 제4정규형(4NF)은 **다치 종속(Multi-Valued Dependency)** 을 제거하는 단계이므로 ④가 잘못된 연결.
- ①: 1NF = 원자값(다중값 속성 제거). 정답 연결.
- ②: 2NF = 부분 함수 종속 제거. 정답 연결.
- ③: 3NF = 이행적 함수 종속 제거. 정답 연결.

### 문제 7 — 정답 ④
- **이상 현상(Anomaly)은 삽입 이상·삭제 이상·갱신 이상의 3종**이다. "검색 이상(Retrieval Anomaly)"이라는 용어는 정규화 이론에 존재하지 않으므로 ④가 정답.
- ①: 삽입 이상 = 원하지 않는 자료까지 함께 삽입해야만 하는 문제.
- ②: 삭제 이상 = 한 튜플 삭제 시 함께 보관되던 다른 정보까지 손실되는 문제.
- ③: 갱신 이상 = 중복 저장된 값 일부만 변경되어 데이터 불일치가 생기는 문제.

### 문제 8 — 정답 ②
- **반정규화(De-normalization)**: 정규화된 모델에서 시스템 성능(특히 조회) 향상, 개발·운영 편의를 위해 의도적으로 정규화 원칙을 위배해 중복을 허용하거나 테이블을 병합·분할·추가하는 기법.
- ①: 정규화는 중복을 제거하는 정반대 과정.
- ③: 무결성 제약은 데이터 정확성 보장 규칙(반정규화 기법 아님).
- ④: 인덱싱은 검색 속도 향상 기법이나, 테이블 구조의 의도적 중복 허용과는 다른 개념.

## 📚 역추적 참조 (사후 검증용 — 학습 자료 아님)

### 핵심 용어
| 용어 | 정의 |
|---|---|
| 정규화(Normalization) | 이상 현상을 제거하기 위해 종속성에 따라 릴레이션을 분해하는 과정 |
| 함수 종속(FD) | 속성 X 값이 정해지면 속성 Y 값이 유일하게 결정되는 관계(X→Y) |
| 결정자(Determinant) | 함수 종속 X→Y에서 왼쪽 항 X |
| 종속자(Dependent) | 함수 종속 X→Y에서 오른쪽 항 Y |
| 완전 함수 종속 | 기본키 전체에 종속(키 일부만으로는 결정 불가) |
| 부분 함수 종속 | 기본키의 일부에만 종속 |
| 이행적 함수 종속 | X→Y, Y→Z일 때 X→Z가 성립 |
| 다치 종속(MVD) | 한 속성값에 다른 속성의 여러 값이 독립적으로 대응 |
| 조인 종속(JD) | 분해된 릴레이션을 조인하면 원래 릴레이션이 복원되는 종속 |
| 후보키(Candidate Key) | 튜플을 유일하게 식별하는 최소 속성 집합 |
| 이상 현상(Anomaly) | 중복으로 인해 삽입·삭제·갱신 시 발생하는 부작용 |
| 반정규화(De-normalization) | 성능을 위해 의도적으로 중복을 허용하는 역과정 |

### 정규화 단계 정리 (사후 검증용)
- 비정규 릴레이션 → 1NF(원자값) → 2NF(부분 함수 종속 제거) → 3NF(이행적 함수 종속 제거) → BCNF(결정자=후보키) → 4NF(다치 종속 제거) → 5NF(조인 종속 제거)
- 암기법: **도·부·이·결·다·조** (도메인 원자값 / 부분 / 이행 / 결정자 / 다치 / 조인)
- 고차 정규형일수록 제약이 강하고, 한 정규형을 만족하면 그 하위 정규형은 자동으로 만족

### 함수 종속 예시 (사후 검증용)
- 릴레이션 수강(학번, 과목번호, 성적, 학년), 기본키 (학번, 과목번호)
- 성적: (학번, 과목번호) → 성적 ⇒ 완전 함수 종속
- 학년: 학번 → 학년 ⇒ 부분 함수 종속(기본키 일부에만 종속) → 2NF 위반
- 이행 예: 학번 → 학과, 학과 → 학과사무실 ⇒ 학번 → 학과사무실(이행) → 3NF 위반

### 이상 현상 정리 (사후 검증용)
- 삽입 이상(Insertion): 불필요한 데이터 없이는 삽입 불가
- 삭제 이상(Deletion): 삭제 시 연쇄적으로 필요한 정보까지 손실
- 갱신 이상(Update): 중복 데이터 일부만 갱신되어 불일치 발생

### 반정규화 기법 (사후 검증용)
- 테이블: 통합(병합)·분할(수직/수평)·추가(중복·통계·이력·부분 테이블)
- 컬럼 중복, 파생(계산) 컬럼 추가, 관계 중복(중복 관계 추가)
- 트레이드오프: 조회 성능 ↑ / 데이터 무결성·일관성 관리 부담 ↑
