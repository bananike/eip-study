## 🔍 오답 역추적 개념 발췌

- **슬라이싱 `a[start:stop:step]`**:
  - `stop`은 포함되지 않음(반개구간).
  - 음수 인덱스: `a[-1]`은 마지막 원소, `a[-5]`는 끝에서 5번째.
  - `step`이 양수면 좌→우, 음수면 우→좌. `a[::-1]`은 전체 역순.
  - `len([10,20,30,40,50,60,70])`에서 `a[1:5:2] = [20, 40]` — 인덱스 1,3 선택.
- **리스트 컴프리헨션 이중 루프**: `[expr for x in A for y in B]`은 `for x in A: for y in B:` 순서. 조건 `if`는 가장 안쪽 `for` 뒤에 위치.
- **딕셔너리 핵심 메소드**:
  - `d.get(k, default)` — 키 없을 때 default 반환(예외 없음)
  - `d.update(other)` — 다른 dict 병합(같은 키 덮어쓰기)
  - `d.pop(k, default)` — 키 제거 후 값 반환(없으면 default)
  - `d.items()` — `(key, value)` 튜플의 view
  - `d.setdefault(k, v)` — 키 없을 때만 v로 설정
- **문자열 메소드 정리**:
  - `s.strip()` — 양쪽 공백 제거. `lstrip`/`rstrip`은 한쪽만.
  - `s.split(sep)` — 구분자로 분할 → 리스트. `sep` 생략 시 공백 기준 + 빈 문자열 제거.
  - `sep.join(iterable)` — 이터러블의 문자열을 sep으로 연결.
  - `s.replace(old, new)` — 부분 문자열 치환.
  - `s[::-1]` — 문자열 역순.
- **클래스/상속 + super()**:
  - `__init__` — 생성자. 첫 인자 `self`는 객체 자신.
  - `super().__init__(...)` — 부모 생성자 명시 호출(자동 호출 X, 자바와 다름).
  - **오버라이딩**: 자식이 같은 이름의 메소드를 재정의하면, 자식 객체에서는 자식 메소드가 호출됨(동적 디스패치).
  - 인스턴스 변수는 `self.x` 형태, 클래스 변수는 클래스 직속 선언.
- **try/except/else/finally 흐름**:
  - 예외 발생 X → `try` → `else` → `finally` 순서
  - 예외 발생 O → `try` 중단 → 매칭되는 `except` → `finally`
  - `try` 또는 `except`에 `return`이 있어도 **finally는 반환 직전에 실행**된다.
  - 한 함수 내에서 print 순서를 추적할 때는 **return 전에 finally의 print가 먼저 출력**됨에 유의.
- **람다·map/filter/zip**:
  - `lambda x, y: x+y` — 익명 함수.
  - `map(f, a, b)` — 같은 인덱스끼리 f 적용. 짧은 쪽 길이까지.
  - `filter(pred, it)` — pred가 True인 원소만.
  - `zip(a, b)` — 같은 인덱스끼리 튜플로 묶음. 짧은 쪽 기준.
  - `range(start, stop, step)` — `stop` 미포함.
- **set 연산자**:
  - `a | b` — 합집합(union)
  - `a & b` — 교집합(intersection)
  - `a - b` — 차집합(difference)
  - `a ^ b` — 대칭차집합(symmetric difference) = `(a|b)-(a&b)`
  - set은 **순서 없음·중복 없음·해시 가능 원소만**.

## 💡 상세 해설

### 문제 1
**정답: ① [4, 6]**

- 리스트 `a = [1, 2, 3, 4, 5, 6, 7, 8]` (길이 8, 인덱스 0~7 또는 -8~-1)
- `a[-5:-1:2]` 해석:
  - `start = -5` → 인덱스 `8 + (-5) = 3` → 값 `4`
  - `stop = -1` → 인덱스 `8 + (-1) = 7` → 값 `8` (미포함)
  - `step = 2` → 2씩 건너뜀
- 선택 인덱스: 3, 5 → 값: **4, 6**
- 결과: `[4, 6]`

**오답 분석**
- ② [4, 6, 8] — stop이 포함된다고 잘못 본 오류(stop은 미포함)
- ③ [5, 7] — start 인덱스를 -5 = 5(값)로 착각한 오류
- ④ [3, 5, 7] — 음수 인덱스를 1-based로 환산한 오류

> 핵심: **음수 인덱스 = `len + 음수값`, stop은 미포함.**

### 문제 2
**정답: ④ 36**

- 이중 컴프리헨션 `for x in range(1,4) for y in range(1,4)` → x∈{1,2,3}, y∈{1,2,3} 총 **9쌍**
- 각 쌍의 `x*y`:
  - x=1: 1*1=1, 1*2=2, 1*3=3 → 합 6
  - x=2: 2*1=2, 2*2=4, 2*3=6 → 합 12
  - x=3: 3*1=3, 3*2=6, 3*3=9 → 합 18
- 총합 = 6 + 12 + 18 = **36**
- 또는 `(1+2+3) * (1+2+3) = 6 * 6 = 36` (분배 법칙)

**오답 분석**
- ① 18 — x=3행만 계산한 오류(또는 (1+2+3)*3)
- ② 24 — 일부 쌍 누락 오류
- ③ 30 — 대각선(x=y) 합 6+12+18에서 누락 계산 오류

> 핵심: **이중 컴프리헨션은 `for x in A: for y in B: expr`의 펼친 형태. 모든 쌍에 expr 적용.**

### 문제 3
**정답: ① 1 27**

- `d = {'a':1, 'b':2, 'c':3}`
- `d.update({'b':20, 'd':4})`:
  - `'b'`는 이미 존재 → **덮어쓰기**: b = 20
  - `'d'`는 없음 → 추가: d = 4
  - 결과: `{'a':1, 'b':20, 'c':3, 'd':4}`
- `d.pop('a', 99)`:
  - 키 `'a'`가 존재 → 값 `1`을 반환하고 a 제거(default 99는 무시)
  - 결과: `{'b':20, 'c':3, 'd':4}`
- `v = 1`, `sum(d.values()) = 20 + 3 + 4 = 27`
- 출력: `1 27`

**오답 분석**
- ② 1 26 — 합 계산 오류
- ③ 99 26 — pop의 default가 우선이라고 본 오류(키 존재 시 default 무시)
- ④ 1 30 — update에서 기존 b=2가 유지된다고 본 오류(덮어쓰기 누락)

> 핵심: **`update`는 같은 키 덮어쓰기, `pop(k, default)`는 키 존재 시 값 반환 + 삭제, 키 없을 때만 default 반환.**

### 문제 4
**정답: ② Python-World-Hello**

- `s = "  Hello,World,Python  "`
- `s.strip()` → 양쪽 공백 제거 → `"Hello,World,Python"`
- `.split(',')` → `['Hello', 'World', 'Python']`
- `parts[::-1]` → `['Python', 'World', 'Hello']`
- `'-'.join(...)` → `"Python-World-Hello"`

**오답 분석**
- ① Hello-World-Python — `[::-1]` 역순을 누락한 오류
- ③ -Python-World-Hello- — join이 양쪽에도 sep을 붙인다고 본 오류(붙이지 않음)
- ④ Python,World,Hello — join의 sep가 원래 `,`로 유지된다고 본 오류

> 핵심: **`split`은 구분자 기준 분할 후 리스트, `join`은 sep + 이터러블 요소 + sep + ... 형태가 아니라 요소들 사이에만 sep 삽입.**

### 문제 5
**정답: ② B:10**

- `b = B(3, 7)`:
  - `B.__init__(self, 3, 7)` 호출
  - `super().__init__(3)` → `A.__init__(self, 3)` → `self.x = 3`
  - `self.y = 7`
- `b.show()` — **B가 show를 오버라이딩**했으므로 `B.show` 실행 → `"B:" + str(3+7)` = `"B:10"`

**오답 분석**
- ① A:3 — 부모의 show가 호출됐다고 본 오류(오버라이딩되면 자식 메소드 우선)
- ③ B:37 — 문자열 연결로 본 오류(x+y는 정수 덧셈)
- ④ A:3 B:10 — show가 두 번 호출됐다고 본 오류

> 핵심: **Python에서 자식 클래스의 메소드가 부모와 같은 이름이면 자동으로 오버라이딩. `super().__init__()`는 명시적으로 호출해야 함(자바와 달리 자동 삽입 없음).**

### 문제 6
**정답: ② OF5\nEF-1**

- `f(2)`:
  - `try` → `r = 10 // 2 = 5` (예외 없음)
  - 예외 미발생 → `else` 진입 → `print("O", end='')` → "O"
  - `else`의 `return 5` 실행 직전, **`finally` 먼저 실행** → `print("F", end='')` → "F"
  - 5 반환
- `print(f(2))`: 위에서 "OF" 출력 후, 반환된 5를 print → "5" + 줄바꿈 → "5\n"
- `f(0)`:
  - `try` → `10 // 0` → `ZeroDivisionError`
  - `except` 매칭 → `print("E", end='')` → "E"
  - `except`의 `return -1` 직전 **`finally` 먼저 실행** → `print("F", end='')` → "F"
  - -1 반환
- `print(f(0))`: "EF" 출력 후, -1 print → "-1" + 줄바꿈 → "-1\n"

**전체 출력**: `OF5\nEF-1\n`

**오답 분석**
- ① OF5EF-1 — `print()`의 기본 줄바꿈 누락
- ③ O5FE-1F — finally를 return 후로 본 오류
- ④ 5O-1E — print(end='')와 실행 순서 혼동

> 핵심: **`else`는 예외 없을 때만 실행, `finally`는 항상 실행되며 `return` 직전에 끼어든다. 함수 내부 print는 함수 외부 `print(f(...))`보다 먼저 출력됨.**

### 문제 7
**정답: ① 66**

- `map(lambda x,y: x+y, a, b)` → 같은 인덱스 합 → `[11, 22, 33, 44, 55]`
- `filter(lambda v: v%2==0, r)` → 짝수만 → `[22, 44]`
  - 11(홀), 22(짝), 33(홀), 44(짝), 55(홀)
- `sum([22, 44])` = **66**

**오답 분석**
- ② 99 — 홀수만 필터링한 오류(33+...)
- ③ 132 — 짝수 잘못 선택(22+44+66...)
- ④ 165 — 전체 합 11+22+33+44+55

> 핵심: **`map`은 같은 인덱스끼리 함수 적용, `filter`는 predicate True인 원소만.**

### 문제 8
**정답: ② [1, 2, 3, 6, 7, 8] 6**

- `a = {1,2,3,4,5}`, `b = {4,5,6,7,8}`
- `a | b` (합집합) = `{1,2,3,4,5,6,7,8}`
- `a & b` (교집합) = `{4,5}`
- `(a|b) - (a&b)` = `{1,2,3,4,5,6,7,8} - {4,5}` = `{1,2,3,6,7,8}` (= 대칭차 `a^b`)
- `sorted(c)` → `[1,2,3,6,7,8]`
- `len(c)` = 6

**오답 분석**
- ① [1,...,8] 8 — `a&b` 차감을 누락한 오류
- ③ [4,5] 2 — 합집합과 교집합을 바꿔본 오류
- ④ [1,2,3] 3 — `a - b`만 계산한 오류

> 핵심: **`(a|b) - (a&b) == a ^ b` (대칭차집합).**

### 참조 5 — Python 클래스/상속 정리
- `__init__(self, ...)` — 생성자. 인스턴스 변수는 `self.속성`.
- 상속: `class Child(Parent):` — Parent의 메소드/속성 모두 상속.
- `super().__init__(...)` — **명시적 호출 필요**(Python은 자바와 달리 자동 호출 X).
- 오버라이딩: 자식이 같은 이름의 메소드를 정의하면 자식 객체에서는 자식 메소드 실행.
- 다중 상속: `class C(A, B):` 가능. MRO(C3 선형화)로 메소드 탐색 순서 결정.

### 참조 6 — 람다·고차함수
- `lambda 인자: 식` — 익명 함수. 단일 식만 반환.
- `map(f, *iterables)` — 각 이터러블의 같은 인덱스에 f 적용. **이터레이터 반환** → `list()`로 변환 필요.
- `filter(pred, it)` — pred가 True인 원소만. 이터레이터 반환.
- `zip(*iterables)` — 같은 인덱스끼리 튜플. 짧은 쪽 기준.
- `enumerate(it, start=0)` — `(인덱스, 값)` 튜플 생성.
- `sorted(it, key=, reverse=)` — 새 리스트 반환(원본 불변).
