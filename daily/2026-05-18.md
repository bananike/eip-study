### 문제 4 [유사 기출: 정보처리기사 필기 2022~2024 — 코드 추적 정형 패턴]
다음 Java 코드의 실행 결과로 옳은 것은?

```java
class Parent {
    void show() { System.out.print("A"); }
}
class Child extends Parent {
    void show() { System.out.print("B"); }
}
public class Test {
    public static void main(String[] args) {
        Parent p = new Child();
        p.show();
    }
}
```

① A
② B
③ AB
④ 컴파일 오류

### 문제 5 [유사 기출: 정보처리기사 필기 2021~2023 — 예외처리 정형 패턴]
Java의 예외 처리(exception handling)에 대한 설명으로 옳지 않은 것은?

① try 블록 내부에서 발생한 예외는 catch 블록에서 처리된다.
② finally 블록은 예외 발생 여부와 관계없이 항상 실행된다.
③ 하나의 try 블록에는 다중 catch 블록을 사용할 수 없다.
④ throw 키워드는 예외를 강제로 발생시킬 때 사용된다.

### 문제 6 [유사 기출: 정보처리기사 필기 2020·2022 반복 출제 — 정형 패턴]
객체지향의 4대 특성 중 다음 설명에 해당하는 것은?

> 데이터(속성)와 그 데이터를 처리하는 함수(메소드)를 하나의 단위(클래스)로 묶고, 외부에서 내부 데이터에 직접 접근하지 못하도록 정보를 은닉(information hiding)하는 것.

① 상속(Inheritance)
② 캡슐화(Encapsulation)
③ 다형성(Polymorphism)
④ 추상화(Abstraction)

---

### 문제 7 [유사 기출: 정보처리기사 필기 2022~2024 — 추상클래스/인터페이스 비교 정형 패턴]
Java의 인터페이스(interface)에 대한 설명으로 옳지 않은 것은? (Java 7 기준)

① 인터페이스 내의 모든 메소드는 묵시적으로 public abstract이다.
② 인터페이스 내의 모든 필드는 묵시적으로 public static final이다.
③ 클래스는 다중 상속(extends)은 불가능하지만 인터페이스는 다중 구현(implements)이 가능하다.
④ 인터페이스 내부에는 일반(non-abstract) 메소드 본문을 자유롭게 정의할 수 있다.

---

### 문제 8 [유사 기출: 정보처리기사 필기 2021~2023 — 오버로딩/오버라이딩 정형 패턴]
Java에서 메소드 오버로딩(Overloading)의 조건으로 옳은 것은?

① 메소드 이름, 매개변수 개수, 매개변수 타입이 모두 동일해야 한다.
② 메소드 이름은 같고, 매개변수의 개수 또는 타입이 달라야 한다.
③ 상위 클래스의 메소드를 하위 클래스에서 동일 시그니처로 재정의해야 한다.
④ 반환형(return type)만 다르면 오버로딩이 성립한다.

## 💡 상세 해설 (자기 설명과 대조)

### 문제 1
정답: ② 클래스(Class)
- 제공 해설: 공통 속성·연산을 갖는 객체의 집합 = 클래스. 객체를 만들어내는 틀 역할.
- 오답 이유:
  - ① 메소드: 객체의 연산(행위)을 정의하는 함수 자체.
  - ③ 메시지: 객체 간 통신/메소드 호출 요청.
  - ④ 인스턴스: 클래스로부터 실제로 생성된 객체(메모리에 적재된 실체).

### 문제 2
정답: ④ private
- 제공 해설: private는 동일 클래스 내부에서만 접근 가능.
- 오답 이유:
  - ① public: 어디서나 접근 가능 (제한 없음).
  - ② protected: 동일 패키지 + 다른 패키지의 상속 자식 클래스에서 접근 가능.
  - ③ default(package-private): 동일 패키지 내 모든 클래스에서 접근 가능.

### 문제 3
정답: ③ float
- 제공 해설: float는 4byte 실수형(부동소수점). 정수형은 byte/short/int/long.
- 오답 이유:
  - ① byte: 1byte 정수형.
  - ② short: 2byte 정수형.
  - ④ long: 8byte 정수형.

### 문제 4
정답: ② B
- 제공 해설: Parent 타입 참조 변수 p가 실제로는 Child 객체를 가리킨다(상향 형변환). Java의 메소드는 기본적으로 가상(virtual)이며 동적 바인딩되므로 실제 객체 타입(Child)의 show()가 호출되어 "B" 출력.
- 오답 이유:
  - ① A: 정적 바인딩 가정 시 답이지만, Java 일반 메소드는 동적 바인딩이 기본.
  - ③ AB: 두 메소드가 모두 호출되지 않음. Parent의 show는 오버라이딩으로 가려짐.
  - ④ 컴파일 오류: 상향 형변환은 묵시적 허용 → 오류 없음.

### 문제 5
정답: ③ (옳지 않은 것)
- 제공 해설: 하나의 try 블록에 catch는 여러 개 사용할 수 있다(다중 catch). 발생한 예외 타입에 따라 분기 처리. ③의 진술이 잘못됨.
- 오답 이유 (옳은 보기들):
  - ① try 블록의 예외를 catch가 잡는다 — 옳음.
  - ② finally는 예외 발생 여부와 무관하게 실행 — 옳음(단 System.exit() 등 예외).
  - ④ throw는 예외 객체를 강제로 발생시킴 — 옳음.

### 문제 6
정답: ② 캡슐화(Encapsulation)
- 제공 해설: 데이터+메소드 묶음 + 정보 은닉 = 캡슐화. private 필드 + public getter/setter가 대표 패턴.
- 오답 이유:
  - ① 상속: 상위 클래스의 속성/연산을 하위 클래스가 물려받음.
  - ③ 다형성: 같은 메시지(메소드 호출)가 객체 타입에 따라 다르게 동작.
  - ④ 추상화: 공통 속성/기능을 추출해 단순화. 캡슐화의 "묶음·은닉"과 구분.

### 문제 7
정답: ④ (옳지 않은 것)
- 제공 해설: Java 7까지 인터페이스 내부에는 메소드 본문을 정의할 수 없다(모두 abstract). Java 8부터 default 메소드와 static 메소드로 본문 허용이 추가됐지만, "자유롭게 정의 가능"은 여전히 거짓. 문제 단서가 Java 7 기준이므로 명백히 거짓.
- 오답 이유 (옳은 보기들):
  - ① 인터페이스 메소드는 묵시적으로 public abstract — 옳음.
  - ② 인터페이스 필드는 묵시적으로 public static final (상수) — 옳음.
  - ③ 클래스 다중 상속 불가, 인터페이스 다중 구현 가능 — 옳음.

### 문제 8
정답: ② 메소드 이름은 같고 매개변수 개수/타입이 달라야 함
- 제공 해설: 오버로딩은 동일 클래스 내에서 같은 이름의 메소드를 매개변수 시그니처(개수·타입·순서)를 달리하여 정의하는 것.
- 오답 이유:
  - ① 모두 동일하면 중복 정의 컴파일 오류.
  - ③ 상위→하위 동일 시그니처 재정의는 오버라이딩(Overriding).
  - ④ 반환형만 달라서는 오버로딩 성립 안 함 (컴파일러가 호출 시점에 반환형으로 식별 불가).

| 객체(Object) / 인스턴스(Instance) | 클래스로부터 생성되어 메모리에 적재된 실체. |
| 메시지(Message) | 객체 간 통신 수단. 일반적으로 메소드 호출 형태. |
| 캡슐화(Encapsulation) | 데이터+메소드 묶음 + 외부로부터 내부 은닉. |
| 상속(Inheritance) | 상위 클래스의 속성/메소드를 하위 클래스가 물려받음. |
| 다형성(Polymorphism) | 동일 인터페이스에 대해 객체별로 다른 동작. 오버로딩·오버라이딩으로 실현. |
| 추상화(Abstraction) | 공통적 본질만 추출하여 단순화. |
| 오버로딩(Overloading) | 동일 이름 + 다른 시그니처. 컴파일 시점 정적 바인딩. |
| 오버라이딩(Overriding) | 상위 메소드를 하위에서 동일 시그니처로 재정의. 실행 시점 동적 바인딩. |
| 추상 클래스(abstract class) | 추상 메소드를 1개 이상 가질 수 있는 클래스. 단일 상속. |
| 인터페이스(interface) | 모든 메소드 묵시적 abstract(Java 7), 모든 필드 묵시적 public static final. 다중 구현 가능. |

### 핵심 개념 (사후 검증용)

**Java 접근 제어자 범위표**
| 제어자 | 동일 클래스 | 동일 패키지 | 자식(다른 패키지) | 외부 |
|---|---|---|---|---|
| public | O | O | O | O |
| protected | O | O | O | X |
| default | O | O | X | X |
| private | O | X | X | X |

**Java 기본 자료형 8종**
- 정수형: byte(1) / short(2) / int(4) / long(8)
- 실수형: float(4) / double(8)
- 문자형: char(2)
- 논리형: boolean(1, JVM 구현에 따라)

**try-catch-finally 흐름**
- 정상: try → finally
- 예외 발생 & 처리: try(예외 발생) → 일치하는 catch → finally
- 예외 발생 & 미처리: try(예외 발생) → finally → 호출자에게 예외 전파

**오버로딩 vs 오버라이딩**
| 항목 | 오버로딩 | 오버라이딩 |
|---|---|---|
| 이름 | 같음 | 같음 |
| 매개변수 | 달라야 함 | 같아야 함 |
| 반환형 | 무관 | 같거나 공변(covariant) |
| 위치 | 동일 클래스 | 상속 관계 (하위가 상위 재정의) |
| 바인딩 | 정적(컴파일) | 동적(런타임) |

**추상 클래스 vs 인터페이스 (Java 7 기준)**
| 항목 | 추상 클래스 | 인터페이스 |
|---|---|---|
| 키워드 | abstract class | interface |
| 사용 측 | extends | implements |
| 다중 | 단일 상속 | 다중 구현 가능 |
| 필드 | 일반 필드 가능 | public static final 상수만 |
| 생성자 | 있음 | 없음 |
| 메소드 본문 | 일반 메소드 가능 | 모두 abstract (Java 8+ default/static 예외) |

**Java 컬렉션 프레임워크 핵심**
- List: 순서O, 중복O — ArrayList, LinkedList, Vector
- Set: 순서X, 중복X — HashSet, TreeSet, LinkedHashSet
- Queue: FIFO — LinkedList, PriorityQueue, ArrayDeque
- Map: key-value, key 중복X — HashMap, TreeMap, LinkedHashMap, Hashtable
