# Day 7 · [소프트웨어 개발] 정렬·탐색 알고리즘

## 📚 오늘의 주제
어제(Day 6) 다룬 자료구조·Big-O를 토대로, 정보처리기사 2과목에서 **회차당 2~4문제** 출제되는 정렬·탐색 알고리즘을 정복한다. 오늘 목표는 ① 정렬 8종(버블·선택·삽입·쉘·합병·퀵·힙·기수)의 **시간복잡도/안정성/특징** 표 암기, ② **N회전 결과 추적**(버블/선택/삽입) 문제 유형 풀이, ③ 탐색 3종(순차·이진·보간) + 해싱(충돌해결 4기법) 정리.

## 🔑 핵심 용어 (10개)
| 용어 | 한 줄 정의 | 핵심 키워드 |
|---|---|---|
| **버블 정렬 (Bubble Sort)** | 인접한 두 원소를 비교·교환하며 가장 큰 값을 뒤로 보냄 | 평균/최악 O(n²), 안정, 비교횟수 n(n-1)/2 |
| **선택 정렬 (Selection Sort)** | 매 회전마다 최솟값(또는 최댓값)을 찾아 앞쪽에 배치 | 평균/최악 O(n²), **불안정**, 교환횟수 최소 |
| **삽입 정렬 (Insertion Sort)** | i번째 원소를 그 이전 정렬 구간의 적절한 위치에 삽입 | 평균/최악 O(n²), **최선 O(n)**, 안정 |
| **쉘 정렬 (Shell Sort)** | 일정 간격(gap)으로 나눠 부분 삽입정렬 후 간격을 줄여나감 | 평균 O(n^1.5) 수준, 삽입정렬 개선판 |
| **합병 정렬 (Merge Sort)** | 배열을 반으로 분할→정렬→병합 (분할정복) | **모든 경우 O(n log n)**, 안정, 추가공간 O(n) |
| **퀵 정렬 (Quick Sort)** | 피벗(pivot) 기준으로 작은/큰 그룹 분할 후 재귀 | 평균 O(n log n), **최악 O(n²)**, 불안정 |
| **힙 정렬 (Heap Sort)** | 최대힙(또는 최소힙) 구성 후 루트를 꺼내며 정렬 | **모든 경우 O(n log n)**, 불안정, 제자리(in-place) |
| **기수 정렬 (Radix Sort)** | 자릿수별로 버킷에 분배·수집 반복 (비교 없음) | O(dn), d=자릿수, 안정, 키가 정수일 때만 |
| **이진 탐색 (Binary Search)** | **정렬된** 데이터에서 중간값과 비교하며 범위를 절반씩 축소 | O(log₂ n), 정렬 전제 필수 |
| **해싱 (Hashing)** | 해시 함수로 키를 주소(버킷)에 매핑. 평균 O(1) 탐색 | 충돌(Collision)·동의어(Synonym)·체이닝·개방주소법 |

## 📖 핵심 개념 정리

### 1. 정렬 알고리즘 시간복잡도 총정리 (★★★ 최대 빈출)

| 알고리즘 | 최선 | 평균 | 최악 | 안정성 | 공간 | 비고 |
|---|---|---|---|---|---|---|
| **버블** | O(n) | O(n²) | O(n²) | ✅ 안정 | O(1) | 인접 비교, 구현 단순 |
| **선택** | O(n²) | O(n²) | O(n²) | ❌ 불안정 | O(1) | 교환 횟수 최소 |
| **삽입** | **O(n)** | O(n²) | O(n²) | ✅ 안정 | O(1) | 거의 정렬된 데이터에 유리 |
| **쉘** | O(n) | O(n^1.5) | O(n^1.5)~O(n²) | ❌ 불안정 | O(1) | 삽입정렬 개선 |
| **합병** | O(n log n) | O(n log n) | **O(n log n)** | ✅ 안정 | **O(n)** | 분할정복, 외부정렬에 유리 |
| **퀵** | O(n log n) | O(n log n) | **O(n²)** | ❌ 불안정 | O(log n) | 평균 가장 빠름, 피벗 의존 |
| **힙** | O(n log n) | O(n log n) | **O(n log n)** | ❌ 불안정 | O(1) | 우선순위큐, 제자리 정렬 |
| **기수** | O(dn) | O(dn) | O(dn) | ✅ 안정 | O(n+k) | 비교 X, 정수/문자열 키 |

⚠ **시험 빈출 포인트** (반드시 암기)
- **평균 O(n log n)인 정렬**: 합병, 퀵, 힙 (3개)
- **모든 경우 O(n log n)인 정렬**: 합병, 힙 (퀵은 최악 O(n²)이므로 제외)
- **최악 O(n²)인 정렬**: 버블, 선택, 삽입, **퀵** (퀵 함정 주의!)
- **안정 정렬**: 버블, 삽입, 합병, 기수 (암기법: **「버삽합기」**)
- **불안정 정렬**: 선택, 쉘, 퀵, 힙 (암기법: **「선쉘퀵힙」**)
- **거의 정렬된 데이터에 가장 유리**: 삽입 정렬 (최선 O(n))

### 2. 버블 정렬 (Bubble Sort) — 회전별 추적

**원리**: 1회전에 인접한 두 원소를 (n-1)번 비교·교환 → 가장 큰 값이 맨 뒤로.

**예제**: [8, 3, 4, 9, 7] 오름차순 정렬

| 회전 | 비교·교환 과정 | 결과 |
|---|---|---|
| 시작 | — | 8, 3, 4, 9, 7 |
| 1회전 | (8,3)↔ → (8,4)↔ → (8,9)그대로 → (9,7)↔ | **3, 4, 8, 7, 9** |
| 2회전 | (3,4) → (4,8) → (8,7)↔ → 끝(9 고정) | 3, 4, 7, 8, 9 |
| 3회전 | 변경 없음 → 종료 | 3, 4, 7, 8, 9 |

총 비교횟수: **n(n-1)/2** = 5·4/2 = 10회

### 3. 선택 정렬 (Selection Sort) — 회전별 추적

**원리**: 매 회전마다 미정렬 구간의 **최솟값**을 찾아 맨 앞과 교환.

**예제**: [8, 5, 6, 2, 4] 오름차순

| 회전 | 최솟값 찾기 → 교환 | 결과 |
|---|---|---|
| 1회전 | 최솟값 2 ↔ 첫 원소 8 | **2, 5, 6, 8, 4** |
| 2회전 | 미정렬[5,6,8,4]의 최솟값 4 ↔ 5 | 2, 4, 6, 8, 5 |
| 3회전 | 미정렬[6,8,5]의 최솟값 5 ↔ 6 | 2, 4, 5, 8, 6 |
| 4회전 | 미정렬[8,6]의 최솟값 6 ↔ 8 | 2, 4, 5, 6, 8 |

### 4. 삽입 정렬 (Insertion Sort) — 회전별 추적

**원리**: i번째(i=2부터) 원소를 Key로 잡고, 그 앞 정렬구간을 뒤에서부터 비교하며 적절한 위치에 삽입.

**예제**: [8, 5, 6, 2, 4]

| 회전 | Key | 비교·삽입 | 결과 |
|---|---|---|---|
| 1회전 | 5 | 5 < 8 → 5를 8 앞에 삽입 | **5, 8, 6, 2, 4** |
| 2회전 | 6 | 6 < 8, 6 > 5 → 5와 8 사이 | **5, 6, 8, 2, 4** |
| 3회전 | 2 | 2가 모두보다 작음 → 맨 앞 | 2, 5, 6, 8, 4 |
| 4회전 | 4 | 4 < 5, 4 > 2 → 2와 5 사이 | 2, 4, 5, 6, 8 |

### 5. 퀵 정렬 (Quick Sort) — 분할정복

1. **피벗(Pivot)** 선택 (보통 첫·끝·중간 원소)
2. 피벗보다 작은 원소는 왼쪽, 큰 원소는 오른쪽으로 분할
3. 좌·우 부분배열에 대해 재귀 호출

**시간복잡도**
- 평균/최선: O(n log n) — 피벗이 거의 중간값일 때
- 최악: **O(n²)** — 이미 정렬된 데이터에서 첫 원소를 피벗으로 잡으면 한쪽으로만 분할

⚠ "다음 중 평균 O(n log n)이지만 **최악 O(n²)** 인 정렬은?" → **퀵** (단골 함정)

### 6. 합병 정렬 (Merge Sort) — 분할정복

1. 배열을 절반으로 **분할**(재귀)
2. 각 부분배열을 **정렬**
3. 두 정렬된 배열을 **병합(merge)**

- 모든 경우 **O(n log n)** 보장
- 추가 공간 **O(n)** 필요 (in-place 아님)
- **외부정렬(External Sort)** 에 적합 — 대용량 데이터 처리

### 7. 힙 정렬 (Heap Sort)

1. 입력 배열로 **최대 힙** 구성 (O(n))
2. 루트(최댓값)와 마지막 원소 교환 → 힙 크기 -1 → 재구성 (O(log n))
3. n번 반복 → 총 O(n log n)

특징: **제자리 정렬**(공간 O(1)), 최악도 O(n log n) 보장

### 8. 탐색 알고리즘 정리

| 알고리즘 | 시간복잡도 | 전제 조건 | 특징 |
|---|---|---|---|
| **순차(선형) 탐색** | O(n) | 없음 | 처음부터 끝까지 차례로 비교 |
| **이진(이분) 탐색** | **O(log₂ n)** | **정렬 필수** | 중간값과 비교, 범위 반씩 축소 |
| **보간 탐색 (Interpolation)** | 평균 O(log log n) | 정렬 + 균등분포 | 찾는 값의 위치를 비례식으로 추정 |
| **피보나치 탐색** | O(log n) | 정렬 | 피보나치 수열로 분할 |
| **블록(색인순차) 탐색** | O(√n) | 정렬+색인 | 블록 단위 색인 후 블록 내 순차 |
| **해싱 (Hashing)** | 평균 **O(1)** | 해시 테이블 | 해시 함수로 주소 직접 계산 |

**이진 탐색 비교 횟수 공식**: ⌊log₂ n⌋ + 1
- 예: n=1000개 → 약 10회로 탐색 완료 (2¹⁰ = 1024)

### 9. 해싱 (Hashing) 상세

**해시 함수 종류**
1. **제산법(Division)**: h(k) = k mod m (m은 보통 소수)
2. **제곱법(Mid-Square)**: 키를 제곱한 후 중간 비트 사용
3. **폴딩법(Folding)**: 키를 일정 자릿수로 나눠 합산
4. **숫자 분석법(Digit Analysis)**: 분포가 고른 자릿수 선택
5. **기수 변환법**: 다른 진수로 변환
6. **무작위법(Random)**: 난수 발생

**충돌(Collision) 해결 4가지** (★ 빈출)
| 방식 | 설명 |
|---|---|
| **체이닝 (Chaining)** | 같은 버킷에 충돌 시 연결 리스트로 연결 |
| **개방주소법 - 선형조사 (Linear Probing)** | 충돌 시 다음 빈 칸을 순차 탐색 (h+1, h+2, ...) |
| **개방주소법 - 이차조사 (Quadratic Probing)** | 충돌 시 h+1², h+2², h+3² ... 으로 이동 |
| **개방주소법 - 이중해싱 (Double Hashing)** | 다른 해시 함수로 다시 계산 |

**용어 구분**
- **동의어(Synonym)**: 같은 해시값을 갖는 서로 다른 키들의 집합
- **충돌(Collision)**: 서로 다른 키가 같은 버킷에 할당되는 현상
- **오버플로(Overflow)**: 버킷에 빈 슬롯이 없는데 새 키가 들어오는 상황

## 💡 상세 해설

### 문제 1 해설 — 정답 ④
버블 정렬 오름차순 1회전: [8, 3, 4, 9, 7]
- (8,3) 교환 → [3, 8, 4, 9, 7]
- (8,4) 교환 → [3, 4, 8, 9, 7]
- (8,9) 그대로 → [3, 4, 8, 9, 7]
- (9,7) 교환 → [3, 4, 8, 7, 9]
∴ **3, 4, 8, 7, 9** (가장 큰 9가 맨 뒤로 고정)

### 문제 2 해설 — 정답 ④
- 합병/퀵/힙 정렬: 평균 O(n log n) ✓
- 선택 정렬: 평균 **O(n²)** ✗
선택 정렬은 매 회전마다 (n-i)번 비교하므로 등차수열 합으로 n(n-1)/2 = O(n²).

### 문제 3 해설 — 정답 ①
선택 정렬 PASS 1: 미정렬 [8,5,6,2,4]의 **최솟값 2**를 찾아 첫 자리 8과 교환.
→ [**2**, 5, 6, 8, 4]

### 문제 4 해설 — 정답 ②
이진 탐색은 매 단계마다 탐색 범위가 절반으로 줄어들므로 시간복잡도는 **O(log₂ N)**. n=1000일 때 약 10회만에 탐색 완료.

### 문제 5 해설 — 정답 ②
- 합병/힙: 모든 경우 O(n log n) 보장
- 기수: O(dn) — log n 형태 아님
- **퀵 정렬**: 평균 O(n log n)이지만 피벗 선택이 나쁘면(정렬된 입력에 첫 원소를 피벗) 한쪽으로만 분할되어 **최악 O(n²)**

### 문제 6 해설 — 정답 ①
삽입 정렬:
- PASS 1: Key=5 → [**5**, 8, 6, 2, 4]
- PASS 2: Key=6 → 8보다 작고 5보다 크므로 5와 8 사이에 삽입 → [**5, 6, 8**, 2, 4]
∴ **5, 6, 8, 2, 4**

### 문제 7 해설 — 정답 ③
이진 탐색의 **전제 조건은 데이터가 정렬되어 있어야 함**. 정렬되지 않은 데이터에는 적용 불가. 나머지는 모두 옳음.

### 문제 8 해설 — 정답 ②
- **충돌(Collision)**: 서로 다른 키가 같은 해시 주소를 갖는 현상
- **동의어(Synonym)**: 같은 해시 주소를 갖는 키들의 집합 (현상 X, 키 집합 O)
- 오버플로: 버킷이 꽉 차서 새 키를 넣을 수 없는 상태
- 슬롯: 버킷 내 한 자리

### 문제 9 해설 — 정답 ②
- **제곱법(Mid-Square)**: 키를 제곱하고 중간 자릿수를 추출
- 제산법: k mod m
- 폴딩법: 키를 분할 후 합산
- 숫자 분석법: 분포가 균등한 자릿수 선택

### 문제 10 해설 — 정답 ②

|---|---|---|
| 합병 | O(n log n) | **O(n log n)** ✓ |

|---|---|
| 순차 | O(n) |
| 이진 | O(log n) |

### 문제 20 해설 — 정답 ②

### 문제 21 해설 — 정답 ③

- ①②③은 모두 옳음

## 📌 시험 직전 1분 요약

1. **평균 O(n log n) 정렬 3종**: 합병·퀵·힙. 이 중 **모든 경우 O(n log n) 보장은 합병·힙** (퀵은 최악 O(n²)).
2. **안정 정렬**: 「버삽합기」 (버블·삽입·합병·기수) ↔ **불안정**: 「선쉘퀵힙」 (선택·쉘·퀵·힙).
3. **이진 탐색 = O(log₂ n), 정렬 전제 필수**. n=1024 → 최대 약 11회 비교.
4. **해싱 충돌 해결 4가지**: 체이닝, 선형조사, 이차조사, 이중해싱. **폴딩법은 해시 함수**이지 충돌해결법 아님!
5. 회전 추적 핵심: **버블**=인접 비교로 큰 값이 뒤로 / **선택**=최솟값 찾아 앞으로 / **삽입**=Key를 앞쪽 정렬구간에 끼워넣기.
