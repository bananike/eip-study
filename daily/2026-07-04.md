③ **명세 → 도출 → 분석 → 검증**

### 문제 2 [유사 기출: 2023년 2회 1과목 — 애자일 방법론(XP 12 실천항목) 정형 패턴]
다음 중 **XP(eXtreme Programming)** 의 **12가지 실천 항목(Practices)** 에 해당하지 **않는** 것은?

① **Pair Programming(짝 프로그래밍)** — 두 명이 한 대의 컴퓨터로 함께 코딩
② **Test-Driven Development(TDD, 테스트 주도 개발)** — 테스트 코드 먼저, 구현 코드 나중
③ **Refactoring(리팩토링)** — 기능 변경 없이 내부 구조 개선
④ **Big Design Up Front(BDUF, 사전 대규모 설계)** — 개발 전 모든 설계를 완료 후 순차 진행

### 문제 3 [유사 기출: 2023년 3회 1과목 — MVC 아키텍처 패턴 정형 패턴]
다음 중 **MVC(Model-View-Controller)** 패턴에서 각 구성 요소의 역할 설명으로 **옳지 않은** 것은?

① **Model** — 데이터(도메인 객체)와 비즈니스 로직을 담당하며, DB와 상호작용한다.
② **View** — 사용자에게 데이터를 시각적으로 표시(HTML·화면)하며, 사용자 입력을 받는다.
③ **Controller** — View의 입력을 받아 Model을 갱신하고, 결과에 따라 View를 선택·제어한다.
④ **View는 Model을 거치지 않고 직접 데이터베이스에 SQL을 발행**해 화면을 렌더링한다.

### 문제 4 [유사 기출: 2023년 1회 1과목 — SOLID(SRP) 정형 패턴]
다음 중 객체지향 설계 원칙 **SOLID** 중 **SRP(Single Responsibility Principle, 단일 책임 원칙)** 에 대한 설명으로 가장 옳은 것은?

① 클래스는 **변경의 이유(Reason to Change)가 오직 하나**여야 하며, 하나의 책임만 가져야 한다.
② 소프트웨어는 확장에는 열려 있고, 변경에는 닫혀 있어야 한다.
③ 자식 클래스는 부모 클래스로 대체 가능해야 한다.
④ 상위 모듈은 하위 모듈에 의존하지 말고, 둘 다 추상에 의존해야 한다.

### 문제 5 [유사 기출: 2023년 2회 2과목 — 큐(Queue) 자료구조 정형 패턴]
다음 중 **큐(Queue)** 에 대한 설명으로 **옳지 않은** 것은?

① **FIFO(First In First Out, 선입선출)** 구조로, 먼저 들어온 데이터가 먼저 나간다.
② 삽입은 **rear(뒤)** 에서, 삭제는 **front(앞)** 에서 이루어진다.
③ **BFS(너비 우선 탐색)** ·**프린터 인쇄 대기열** ·**CPU 라운드 로빈 스케줄링** ·**메시지 큐(Kafka·RabbitMQ)** 에 사용된다.
④ **한쪽 끝(top)** 에서만 삽입·삭제가 일어나며, 함수 호출 복귀 주소 저장에 사용되는 **LIFO(Last In First Out)** 구조이다.

### 문제 6 [유사 기출: 2023년 3회 2과목 — 통합 테스트(하향식/상향식/샌드위치/빅뱅) 정형 패턴]
다음 중 **통합 테스트(Integration Test)** 방식에 대한 설명으로 **옳지 않은** 것은?

① **하향식 통합(Top-Down)** — 상위 모듈부터 통합하며, **하위 모듈 대신 스텁(Stub)** 을 사용한다.
② **상향식 통합(Bottom-Up)** — 하위 모듈부터 통합하며, **상위 모듈 대신 드라이버(Driver)** 를 사용한다.
③ **샌드위치 통합(Sandwich)** — 하향식과 상향식을 동시에 수행하며, **스텁과 드라이버 모두** 사용한다.
④ **빅뱅 통합(Big-Bang)** — 상위·하위 모듈을 **단계별 점진 통합**하며, 통합 순서상 반드시 스텁만 사용하고 드라이버는 사용하지 않는다.

### 문제 7 [유사 기출: 2023년 1회 2과목 — Git 명령어(add/commit/push/merge) 정형 패턴]
다음 중 **Git** 형상관리 명령어에 대한 설명으로 **옳지 않은** 것은?

① `git add <파일>` — 작업 디렉토리의 변경 사항을 **스테이징 영역(Staging Area, Index)** 에 추가
② `git commit -m "메시지"` — 스테이징 영역의 스냅샷을 **로컬 저장소(HEAD)** 에 기록
③ `git push origin main` — 로컬 저장소의 커밋을 **원격 저장소(origin)** 로 업로드
④ `git merge`는 **원격 저장소에서 데이터를 가져오기만 하고 로컬 브랜치에 병합은 절대 수행하지 않는다** (병합은 오직 `git fetch`가 수행)

### 문제 9 [유사 기출: 2023년 3회 3과목 — 무결성 제약 조건(4종) 정형 패턴]
다음 중 관계형 데이터베이스의 **무결성 제약 조건**에 대한 설명으로 **옳지 않은** 것은?

① **개체 무결성(Entity Integrity)** — **기본 키(Primary Key)** 는 **NULL을 가질 수 없고 중복될 수 없다**.
② **참조 무결성(Referential Integrity)** — **외래 키(Foreign Key)** 는 **참조하는 테이블의 기본 키 값과 일치**하거나 NULL이어야 한다.
③ **도메인 무결성(Domain Integrity)** — 속성 값은 **정의된 도메인(자료형·범위·형식)** 에 속해야 한다.
④ **사용자 정의 무결성(User-Defined Integrity)은 기본 키·외래 키만 강제하며**, 업무 규칙(예: 나이≥0, 급여>최저임금)은 무결성 대상에 포함되지 않는다.

### 문제 10 [유사 기출: 2023년 1회 3과목 — SQL 조인(6종) 정형 패턴]
다음 중 **SQL 조인(JOIN)** 에 대한 설명으로 **옳지 않은** 것은?

① **INNER JOIN(내부 조인)** — 양쪽 테이블에 **매칭되는 행만** 반환 (기본 JOIN)
② **LEFT OUTER JOIN(왼쪽 외부 조인)** — 왼쪽 테이블의 **모든 행** + 오른쪽에서 매칭되는 행 (매칭 없으면 NULL)
③ **CROSS JOIN(교차 조인)** — 두 테이블의 **모든 조합(m × n)** 을 반환 (카티션 곱)
④ **FULL OUTER JOIN(완전 외부 조인)은 양쪽 테이블 모두에서 매칭되는 행만 반환**하며, 매칭되지 않는 행은 결과에서 완전히 제외된다.

### 문제 11 [유사 기출: 2023년 2회 3과목 — 정규화 2NF(부분 함수 종속 제거) 정형 패턴]
다음 중 **제2정규형(2NF)** 에 대한 설명으로 가장 옳은 것은?

① **1NF를 만족**하고, 모든 **비주요 속성**이 **기본 키에 대해 완전 함수 종속**(부분 함수 종속 제거)
② **1NF를 만족**하고, 모든 **비주요 속성**이 기본 키에 대해 **이행 함수 종속을 갖지 않음**
③ **BCNF를 만족**하고, **다치 종속(Multi-Valued Dependency)** 을 제거
④ 모든 **속성 값이 원자 값(Atomic)** 을 가지며, 반복 그룹이 없는 상태

### 문제 12 [유사 기출: 2023년 3회 3과목 — 인덱스(B-Tree/Hash/Bitmap) 정형 패턴]
다음 중 **인덱스(Index)** 유형별 특징으로 **옳지 않은** 것은?

① **B-Tree 인덱스** — 범위 검색(BETWEEN, <, >), 정렬 결과에 강함. 대부분의 RDBMS 기본 인덱스.
② **Hash 인덱스** — **등가 비교(=)** 에만 사용 가능. 범위 검색·정렬 불가.
③ **Bitmap 인덱스** — **카디널리티(고유값 수)가 낮은** 컬럼(성별·상태값)에 유리. OLAP·DW에서 자주 사용.
④ **B-Tree 인덱스는 등가 비교(=)만 지원**하며 범위 검색(BETWEEN, <, >)에는 사용할 수 없다. 범위 검색은 오직 Hash 인덱스만 가능하다.

### 문제 14 [유사 기출: 2023년 2회 4과목 — Java 예외처리(try-catch-finally 흐름) 정형 패턴]

```java
public class Ex {
    public static void main(String[] args) {
        try {
            int[] arr = {1, 2, 3};
            System.out.println(arr[5]);
            System.out.println("A");
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("B");
        } catch (Exception e) {
            System.out.println("C");
        } finally {
            System.out.println("D");
        }
        System.out.println("E");
    }
}
```

① `A B D E`
② `B D E`
③ `B C D E`
④ `B D`

### 문제 16 [유사 기출: 2023년 1회 4과목 — 서브넷 마스크 & IPv4 클래스 정형 패턴]
다음 중 **IPv4 서브넷 마스크(Subnet Mask)** 에 대한 설명으로 **옳지 않은** 것은?

① C 클래스 기본 서브넷 마스크는 **255.255.255.0 (/24)** 이며, 호스트 비트 8개로 254개 호스트를 표현한다.
② `192.168.10.0/26` — 서브넷 마스크는 **255.255.255.192**, 호스트 비트 **6개**, 서브넷당 **62개 사용 가능 호스트**(2⁶−2)
③ **VLSM(Variable Length Subnet Mask)** — 서브넷마다 **다른 길이의 마스크**를 적용해 IP 낭비를 줄이는 기법이다.
④ `/24` 는 서브넷 마스크 **255.255.0.0**을 의미하며, **호스트 비트 16개**로 65534개 호스트를 표현한다.

### 문제 17 [유사 기출: 2023년 2회 5과목 — COCOMO 3유형(Organic/Semi-Detached/Embedded) 정형 패턴]
다음 중 **COCOMO(Constructive Cost Model)** 의 **3가지 프로젝트 유형**에 대한 설명으로 **옳지 않은** 것은?

① **Organic(조직형)** — 소규모(**5만 라인 이하**) 응용 프로그램(급여계산·재고관리 등)에 적합
② **Semi-Detached(반결합형)** — 중규모(**30만 라인 이하**) 트랜잭션 처리 시스템·컴파일러·DBMS에 적합
③ **Embedded(내장형)** — 대규모(**30만 라인 초과**) 실시간 제어·항공·미사일 등 하드웨어에 밀접한 시스템에 적합
④ COCOMO는 **기능 점수(FP, Function Point)** 를 입력값으로 사용하며, **원시 코드 라인(LOC)** 은 사용하지 않는다.

### 문제 18 [유사 기출: 2023년 3회 5과목 — 해시 함수(MD5/SHA-1/SHA-256/HAS-160) 정형 패턴]
다음 중 **해시 함수(Hash Function)** 와 **출력 비트 수** 매칭으로 **옳지 않은** 것은?

① **MD5** — 128 비트 출력. 이미 충돌 취약점 발견으로 보안 용도 폐기 권고.
② **SHA-1** — 160 비트 출력. Google이 2017년 SHAttered 충돌 공격 성공, 현재 폐기 권고.
③ **SHA-256** — **256 비트** 출력. SHA-2 계열, 현재 산업 표준.
④ **HAS-160은 한국형 해시 함수로 512 비트**를 출력하며, SEED와 함께 국제 표준으로 채택되었다.

### 문제 19 [유사 기출: 2023년 1회 5과목 — 랜섬웨어(Ransomware) 대응 정형 패턴]
다음 중 **랜섬웨어(Ransomware)** 및 그 대응 방법에 대한 설명으로 **옳지 않은** 것은?

① 랜섬웨어는 사용자의 **파일을 암호화**한 후 **복호화 키를 대가로 금전(암호화폐)** 을 요구하는 악성코드이다.
② 대표 사례로 **WannaCry(2017, EternalBlue 취약점 이용)** ·**Petya/NotPetya** ·**Ryuk** ·**Locky** 등이 있다.
③ **대응 방법** — 정기 **오프라인 백업(3-2-1 규칙)** ·**최신 보안 패치** ·**의심 첨부파일·매크로 차단** ·**엔드포인트 EDR/XDR** ·**네트워크 세그멘테이션**.
④ 랜섬웨어에 감염되면 **가장 확실한 복구 방법은 즉시 몸값(암호화폐)을 지불하는 것**이며, 지불 시 복호화 키를 반드시 받을 수 있으므로 백업보다 우선 권고된다.

### 문제 20 [유사 기출: 2023년 2회 5과목 — Zero Trust(제로 트러스트) 보안 모델 정형 패턴]
다음 중 **Zero Trust(제로 트러스트) 보안 모델**에 대한 설명으로 **옳지 않은** 것은?

① 핵심 원칙은 **"Never Trust, Always Verify"** — 네트워크 내부·외부를 구분하지 않고 **모든 접근 요청을 검증**한다.
② **경계 기반 보안(Perimeter Security, 성곽 모델)** 의 한계(내부자 위협·클라우드 확산·재택근무)를 극복하기 위해 등장.
③ 주요 구성 요소 — **ID/디바이스 인증(MFA)** ·**최소 권한(Least Privilege)** ·**마이크로 세그멘테이션** ·**지속적 모니터링(SIEM/UEBA)**.
④ Zero Trust는 **사내 네트워크 안에 있는 사용자는 모두 신뢰**하며, 오직 **외부 인터넷에서 들어오는 트래픽만 검증**하면 되는 모델이다.

## 💡 상세 해설

### 문제 1
**정답 ①** — **요구공학(Requirements Engineering) 4단계 = 도출 → 분석 → 명세 → 검증**. **① 도출(Elicitation)** = 이해관계자 인터뷰·워크숍·설문으로 요구 발굴 → **② 분석(Analysis)** = 요구 분류·우선순위·충돌 해결 → **③ 명세(Specification)** = SRS(Software Requirements Specification) 문서화 → **④ 검증(Verification/Validation)** = 요구가 정확·완전·일관·검증가능한지 확인.
- 함정: 순서 뒤바꾸기. **"분석 먼저" ·"명세 먼저" ·"검증 먼저"** 모두 정답 아님. **"도-분-명-검"** 암기.

### 문제 2
**정답 ④** — **BDUF(Big Design Up Front)** 는 **폭포수(Waterfall) 계열**의 사전 설계 방식이며, **애자일 XP의 정반대 개념**. XP 12실천 = **① Planning Game ② Small Release ③ Metaphor ④ Simple Design ⑤ Test-Driven Development ⑥ Refactoring ⑦ Pair Programming ⑧ Collective Ownership ⑨ Continuous Integration ⑩ 40-hour Week ⑪ On-site Customer ⑫ Coding Standards**.
- 함정: XP 5대 가치 = **의사소통·단순성·피드백·용기·존중**. XP는 **YAGNI(You Aren't Gonna Need It)** ·**Simple Design**을 강조하므로 BDUF와 정반대. "BDUF = 애자일 실천" 뒤바꾸기 빈출.

### 문제 3
**정답 ④** — **View는 Model을 통해서만 데이터에 접근**해야 하며, DB에 직접 SQL을 발행하면 MVC 원칙 위반(관심사 분리 실패). 정상 흐름 = **① 사용자 입력 → Controller → ② Controller가 Model 갱신 → ③ Model이 데이터/로직 처리 → ④ Controller가 View 선택 → ⑤ View가 Model 데이터 표시**.
- 함정: MVC의 핵심은 **관심사의 분리(Separation of Concerns)**. **View = 표시만, Model = 데이터만, Controller = 흐름 제어만**. "View가 DB 직접 접근" 반대 진술.

### 문제 4
**정답 ①** — **SRP(Single Responsibility Principle)** = "**변경의 이유는 오직 하나**여야 한다" (Robert C. Martin). 클래스가 여러 책임을 가지면 한 책임 변경이 다른 책임에 영향을 주어 응집도가 떨어짐.
- 함정: ② OCP · ③ LSP · ④ DIP 정의. SOLID 5원칙 = **S**RP · **O**CP · **L**SP · **I**SP · **D**IP.

### 문제 5
**정답 ④** — 보기 ④는 **스택(Stack)** 의 정의(LIFO). 큐는 **양쪽 끝에서 각각 삽입·삭제**(rear/front)가 일어나는 **FIFO** 구조.
- 함정: 스택·큐 정의 뒤바꾸기 초빈출. **큐 = FIFO = BFS·프린터·라운드로빈·메시지 큐** / **스택 = LIFO = 함수 콜·DFS·후위표기·괄호 검사**. 반드시 구분.

### 문제 6
**정답 ④** — **빅뱅 통합(Big-Bang)** 은 **모든 모듈을 한꺼번에 통합**하는 방식으로, **점진 통합이 아니며 스텁·드라이버를 사용하지 않는다**. 오류 위치 파악이 어려워 실무에서는 잘 쓰지 않음.
- 함정: 통합 테스트 4방식 = **① 하향식(Top-Down)= Stub** / **② 상향식(Bottom-Up)= Driver** / **③ 샌드위치= Stub+Driver 둘 다** / **④ 빅뱅= 한꺼번에·보조 모듈 없음**. "빅뱅이 점진 통합" 정반대 진술.

### 문제 7
**정답 ④** — **완전 뒤바꾸기**. **`git merge`** = 두 브랜치를 **병합**(로컬 작업 브랜치에 통합). **`git fetch`** = 원격 저장소에서 데이터를 **가져오기만 하고 병합은 하지 않음**(수동 병합 필요). **`git pull` = fetch + merge** 합친 명령.
- 함정: `git fetch` vs `git merge` vs `git pull` 역할 정확 구분. "**fetch = 가져오되 병합 X**" / "**merge = 병합만**" / "**pull = fetch + merge**".

### 문제 8
**정답 ③** — **선택 정렬(Selection Sort)** 1회전: **① 전체에서 최솟값 찾기 → ② 맨 앞과 교환**. `[7,2,5,1,4]` 에서 최솟값 = **1**(인덱스 3). 인덱스 0(값 7)과 교환 → **`[1, 7, 5, 2, 4]`**.
- 함정: **① 삽입 정렬 결과**(1을 앞으로 삽입), **② 버블 정렬 1패스 결과**, **④ 다른 교환 방식**. **선택 정렬 = 최솟값 찾아 맨 앞과 교환** 규칙 준수.

### 문제 9
**정답 ④** — **사용자 정의 무결성(User-Defined Integrity)** = 업무 규칙(비즈니스 룰)을 **CHECK 제약** ·**트리거** ·**저장 프로시저** 등으로 강제하는 것. 예: "나이 ≥ 0", "급여 > 최저임금", "이메일 형식 검증".
- 함정: 무결성 4종 = **① 개체(Entity) = PK NOT NULL + UNIQUE** / **② 참조(Referential) = FK가 참조 PK 일치 or NULL** / **③ 도메인(Domain) = 자료형/범위/형식** / **④ 사용자 정의 = 업무 규칙(CHECK)**. "사용자 정의 무결성이 PK/FK만" 정반대 진술.

### 문제 10
**정답 ④** — 정반대. **FULL OUTER JOIN** = 양쪽 테이블의 **모든 행을 반환**하며, 매칭되지 않는 쪽은 **NULL**로 채운다(LEFT OUTER + RIGHT OUTER 결합).
- 함정: 조인 6종 = **① INNER = 교집합** / **② LEFT OUTER = 왼쪽 전부 + 오른쪽 매칭** / **③ RIGHT OUTER = 오른쪽 전부 + 왼쪽 매칭** / **④ FULL OUTER = 양쪽 전부(합집합)** / **⑤ CROSS = m×n** / **⑥ SELF = 자기 자신과 조인**. "FULL OUTER가 교집합만" 정반대 진술.

### 문제 11
**정답 ①** — **2NF 정의 = 1NF + 부분 함수 종속(Partial FD) 제거**. 부분 함수 종속 = 복합 기본 키의 **일부**에만 종속되는 상태. 예: PK = {학번, 과목코드}, 속성 "학생이름"이 학번에만 종속 → 부분 종속 → 2NF 위반.
- 함정: ② 3NF(이행 종속 제거) · ③ 4NF(다치 종속 제거) · ④ 1NF(원자값). 도-부-이-결-다-조 = **1NF 원자값 → 2NF 부분종속 → 3NF 이행종속 → BCNF 결정자=후보키 → 4NF 다치종속 → 5NF 조인종속**.

### 문제 12
**정답 ④** — 정반대. **B-Tree 인덱스**는 **등가(=)뿐만 아니라 범위 검색(BETWEEN, <, >, <=, >=)과 정렬(ORDER BY)에 모두 강함**. Hash 인덱스가 등가 전용.
- 함정: 인덱스 유형별 사용처 매트릭스 뒤바꾸기 초빈출. **"B-Tree = 범위+등가+정렬 모두"** / **"Hash = 등가 전용"** / **"Bitmap = 카디널리티 낮은 컬럼 · OLAP 강함"**.

### 문제 13
**정답 ①** — 배열 `a = {10,20,30,40,50}`, 포인터 `p = a`. **① `*(p+2) = a[2] = 30`** / **② `p[3] = a[3] = 40`** / **③ `*(a+4) = a[4] = 50`** → 출력 **`30 40 50`**.
- 함정: C에서 **`p[i] ≡ *(p+i) ≡ *(a+i) ≡ a[i]`** 4가지 표기 완전 등가. "`a[3] = 30`" 같은 인덱스 착오 함정.

### 문제 14
**정답 ②** — `arr[5]` 접근 → **ArrayIndexOutOfBoundsException 발생** → "A" 출력 스킵 → **catch(ArrayIndexOutOfBoundsException) 진입 → "B" 출력** → 두 번째 catch 스킵 → **finally → "D" 출력** → 마지막 문장 **"E" 출력** → 최종 **`B D E`**.
- 함정: **finally는 예외 발생 여부와 무관하게 반드시 실행**(단, `System.exit()` 예외). catch 블록은 **가장 먼저 매칭되는 하나만** 실행. "A B D E"(A까지 실행 착오) · "B C D E"(catch 두 개 다 실행 착오) 등이 함정.

### 문제 15
**정답 ①** — `range(1,8) = [1,2,3,4,5,6,7]` → **3의 배수(3,6) 제외** → `[1,2,4,5,7]` → 각 x*2 → **`[2, 4, 8, 10, 14]`**.
- 함정: ② if 조건 무시(모든 x*2) · ③ 조건 반대(3의 배수만) · ④ x*2 안 함(원본 그대로). **컴프리헨션 구조 = `[표현식 for 변수 in 이터러블 if 조건]`** 순서 정확 파악.

### 문제 16
**정답 ④** — **`/24` = 255.255.255.0** (호스트 비트 **8개**, 254 호스트). **`255.255.0.0`은 `/16`** 에 해당. **CIDR 표기법**에서 슬래시 뒤 숫자는 **네트워크 비트 수**(왼쪽부터 1의 개수).
- 함정: CIDR 대응 암기: **`/24` = 255.255.255.0** / **`/16` = 255.255.0.0** / **`/8` = 255.0.0.0** / **`/25` = 255.255.255.128** / **`/26` = 255.255.255.192** / **`/27` = 255.255.255.224**. "`/24 = 255.255.0.0`" 정반대 진술.

### 문제 17
**정답 ④** — 완전 뒤바꾸기. **COCOMO의 입력값은 LOC(Line of Code, 원시 코드 라인 수)** 이며, 개발 노력 = a·(KLOC)^b·(개인적 요인) 공식. FP(Function Point)는 **다른 모델**(FP 산정법)이며 COCOMO 계열이 아님.
- 함정: 비용 산정 모델 3종 = **① LOC 기반 = COCOMO(3유형 Organic 5만↓ / Semi 30만↓ / Embedded 30만↑) · Putnam(Rayleigh 곡선)** / **② FP 기반 = Albrecht FP · IFPUG · ESTIMACS(가중치 기반, 언어 독립)** / **③ 전문가 판단 = 델파이(Delphi)**. **"COCOMO = LOC 기반"** 필수 암기. "COCOMO = FP" 뒤바꾸기 초빈출.

### 문제 18
**정답 ④** — **HAS-160은 한국형 해시 함수로 160비트**(SHA-1과 동일 출력 길이) 출력. **512 비트 출력은 SHA-512**. SEED는 **대칭키 블록 암호** (해시 함수 아님).
- 함정: 해시 출력 비트 수 매트릭스: **MD5 = 128** / **SHA-1 = 160** / **SHA-224 = 224** / **SHA-256 = 256** / **SHA-384 = 384** / **SHA-512 = 512** / **HAS-160 = 160(한국형)** / **RIPEMD-160 = 160**. "HAS-160 = 512" 뒤바꾸기 함정. SEED/ARIA/HIGHT/LEA는 한국형 **대칭키 블록 암호**이지 해시 아님.

### 문제 19
**정답 ④** — **몸값 지불은 절대 권고되지 않음**. ① 지불해도 복호화 키를 받지 못하는 경우 다수(50% 이상) ② 지불 사실이 알려지면 다른 랜섬웨어 표적이 됨(재감염) ③ 범죄 조직 자금 지원 ④ 국제 표준(No More Ransom 프로젝트·FBI·경찰청) 모두 **"지불하지 말고 백업 복구·전문가 신고"** 권고.
- 함정: "지불 = 확실한 복구" 정반대 진술. **정답 = 오프라인 백업(3-2-1 규칙 = 3부 복사본, 2개 매체, 1개 오프사이트)** + **패치** + **매크로/첨부 차단** + **EDR/XDR** + **네트워크 세그멘테이션**. WannaCry(2017)·NotPetya(2017)·Colonial Pipeline(2021, DarkSide) 사례 빈출.

### 문제 20
**정답 ④** — 정반대. **Zero Trust의 핵심 = "내부 신뢰 없음"**. 사내 네트워크에 있는 사용자·디바이스라도 **모든 요청마다 인증·인가·컨텍스트 검증**. 경계 기반 모델(성곽 모델·Castle-and-Moat)의 "내부는 안전, 외부만 위험" 가정을 **완전히 폐기**한 것이 Zero Trust.
- 함정: 완전 반대 정의. Zero Trust 3원칙 = **① Verify Explicitly(명시적 검증)** / **② Use Least Privilege Access(최소 권한)** / **③ Assume Breach(침해 가정)**. NIST SP 800-207 표준. 관련 신기술 = **BeyondCorp(Google)** · **ZTNA(Zero Trust Network Access)** · **SASE(Secure Access Service Edge)** · **SDP(Software-Defined Perimeter)**.

## 📊 자주 틀리는 함정 5가지 (2023년 회독 1/4 정리)

1. **요구공학 4단계 순서 착오** — **"도-분-명-검"**(도출→분석→명세→검증). 순서 뒤바꾸기가 4가지 보기에 두루 배치되므로 첫 글자 암기 필수.
2. **XP 12실천 vs 폭포수 원칙 뒤바꾸기** — **BDUF·문서 우선·순차 단계**는 폭포수/전통 방법론. XP는 **YAGNI · Simple Design · 반복·짧은 배포**. "BDUF가 XP 실천"이라는 정반대 진술이 오답 함정.
3. **B-Tree vs Hash 인덱스 범위 검색** — **B-Tree = 등가+범위+정렬 모두** / **Hash = 등가 전용**. "B-Tree = 등가만"이라는 뒤바꾸기 함정 초빈출. 실무에서 대부분 RDBMS의 기본 인덱스는 B-Tree/B+Tree.
4. **CIDR 표기법 (`/24` 등)** — `/24 = 255.255.255.0`, `/16 = 255.255.0.0`, `/8 = 255.0.0.0`. 슬래시 뒤 숫자가 **네트워크 비트 수**임을 잊고 서브넷 마스크 옥텟 값으로 혼동. 반드시 이진 매핑으로 계산.
5. **랜섬웨어 = "지불하면 복구"** 오해 — **절대 권고되지 않음**. 정답은 **오프라인 백업(3-2-1) + 패치 + 매크로 차단 + EDR/XDR + 세그멘테이션**. 지불은 재감염·법적 문제·범죄 자금 지원 3중 리스크.
