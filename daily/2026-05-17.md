# Day 6 · [소프트웨어 개발] 자료구조 + 알고리즘 복잡도 (Big-O)
> 날짜: 2026-05-17 (KST) | D-82 (목표 2026-08-07)

## 📚 오늘의 주제
1과목(소프트웨어 설계)을 마치고 **2과목(소프트웨어 개발)의 첫 단원**으로 진입한다. 자료구조와 알고리즘 복잡도(Big-O)는 정보처리기사 2과목에서 **매 회차 3~5문제**가 출제되는 최대 빈출 단원이며, 실기에도 자주 등장한다. 오늘 목표는 ① 자료구조 11종(선형 5 + 비선형 4 + 파일 2) 분류, ② 스택·큐·트리의 핵심 연산 및 공식 암기, ③ Big-O 7단계(O(1)~O(n!)) 비교, ④ 트리 순회(전위/중위/후위) 절차 체득.

## 🔑 핵심 용어 (10개)
| 용어 | 한 줄 정의 | 핵심 키워드 |
|---|---|---|
| **선형 자료구조 (Linear Data Structure)** | 데이터를 일렬로 나열한 구조. 한 원소 뒤에 단 하나의 원소가 옴 | 배열·리스트·스택·큐·데크 |
| **비선형 자료구조 (Non-linear Data Structure)** | 한 원소 뒤에 여러 원소가 올 수 있는 계층/네트워크 구조 | 트리·그래프 |
| **스택 (Stack)** | 후입선출(LIFO, Last In First Out). 한쪽 끝(top)에서만 삽입/삭제 | push·pop·peek·재귀·괄호검사·후위표기 |
| **큐 (Queue)** | 선입선출(FIFO, First In First Out). rear에서 삽입, front에서 삭제 | enqueue·dequeue·작업 스케줄링·버퍼 |
| **데크 (Deque, Double-ended Queue)** | 양쪽 끝에서 모두 삽입·삭제 가능. 스택+큐 통합 | scroll(입력 한쪽)·shelf(출력 한쪽) |
| **이진 트리 (Binary Tree)** | 각 노드의 차수(degree)가 **2 이하**인 트리 | 단말 노드 공식: n₀ = n₂ + 1 |
| **트리 순회 (Tree Traversal)** | 전위(Preorder: 루트→좌→우), 중위(Inorder: 좌→루트→우), 후위(Postorder: 좌→우→루트) | DLR / LDR / LRD |
| **그래프 (Graph)** | 정점(Vertex) + 간선(Edge). 방향/무방향, 가중치 유무 | 인접 행렬(O(V²) 공간), 인접 리스트(O(V+E)) |
| **해싱 (Hashing)** | 해시 함수로 키를 주소로 변환. 평균 O(1) 탐색 | 충돌(Collision)·체이닝·개방주소(선형/이차/이중) |
| **Big-O 표기법** | 알고리즘의 **최악(upper bound) 시간복잡도** 표기 | O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) < O(n!) |

## 📖 핵심 개념 정리

### 1. 자료구조 분류 (필수 암기)

```
자료구조
├── 선형 (Linear)        : 배열, 선형리스트(연결리스트 포함), 스택, 큐, 데크
├── 비선형 (Non-linear)  : 트리, 그래프
└── 파일 (File)          : 순차파일, 색인파일, 직접파일
```

⚠ **빈출 함정**: "다음 중 비선형 자료구조는?" → 트리·그래프만 비선형. 스택·큐·데크는 모두 **선형**.

암기법: 「**선·배·연·스·큐·데** / **비·트·그**」 (선형은 배열·연결리스트·스택·큐·데크 / 비선형은 트리·그래프)

### 2. 스택 (Stack) — LIFO

**핵심 연산**
- `push(x)`: top에 x 추가 → top++
- `pop()`: top 원소 제거·반환 → top--
- `peek()/top()`: top 원소 확인만 (제거 X)
- `isEmpty()`: top == -1 (또는 -1로 초기화) 시 true

**응용 분야 (빈출)**
1. **함수 호출(서브루틴) 복귀 주소 저장** ← 콜 스택
2. **재귀 호출 (Recursion)**
3. **수식 표기법 변환** (중위 ↔ 후위 ↔ 전위)
4. **괄호 검사** (Balanced Parentheses)
5. **인터럽트 처리**, **DFS(깊이우선탐색)**, **Undo 기능**

**오버플로/언더플로**
- 스택 오버플로(Stack Overflow): push할 자리가 없음 (top == size-1)
- 스택 언더플로(Stack Underflow): pop할 원소가 없음 (top == -1)

### 3. 큐 (Queue) — FIFO

**핵심 연산**: enqueue (rear에 추가) / dequeue (front에서 제거)

**원형 큐(Circular Queue)**: 일반 큐는 dequeue 후 앞부분이 비어도 재사용 불가. 원형 큐는 `(rear+1) % size`로 끝과 처음을 연결해 메모리 재활용.

**응용**: 프로세스 작업 스케줄링, 프린터 대기열, 키보드 버퍼, BFS(너비우선탐색).

### 4. 데크 (Deque)

양쪽 끝에서 삽입·삭제 모두 가능. **입출력 제한** 변형:
- **Scroll 데크**: 입력은 한쪽만, 출력은 양쪽
- **Shelf 데크**: 입력은 양쪽, 출력은 한쪽

### 5. 트리 (Tree) 핵심 공식

| 용어 | 정의 |
|---|---|
| **노드의 차수(Degree of Node)** | 그 노드의 **자식 수** |
| **트리의 차수(Degree of Tree)** | 트리 내 노드 차수 중 **최댓값** |
| **단말(Terminal/Leaf) 노드** | 차수가 **0**인 노드 |
| **레벨(Level)** | 루트가 1(또는 0), 자식으로 갈수록 +1 |
| **트리의 높이(Height)** | 가장 깊은 레벨 값 |

**이진 트리 핵심 공식 (암기)**
- 레벨 i의 최대 노드 수: **2^(i-1)** (루트 레벨=1 기준)
- 높이 h인 이진트리 최대 노드 수: **2^h - 1**
- 단말 노드 수와 차수 2 노드 수의 관계: **n₀ = n₂ + 1**

**트리 순회 (DFS 계열)**
| 순회 | 순서 | 약어 |
|---|---|---|
| 전위(Preorder) | **루트(D) → 좌(L) → 우(R)** | DLR |
| 중위(Inorder) | 좌(L) → 루트(D) → 우(R) | LDR |
| 후위(Postorder) | 좌(L) → 우(R) → 루트(D) | LRD |

암기법: **루트가 어디 있는지로 구분** — 앞(전위)·가운데(중위)·뒤(후위).

### 6. 그래프 (Graph)

- **방향(Directed) / 무방향(Undirected)** 그래프
- **인접 행렬(Adjacency Matrix)**: V×V 2차원 배열, 공간 O(V²), 간선 존재 확인 O(1)
- **인접 리스트(Adjacency List)**: 각 정점마다 연결된 정점 리스트, 공간 O(V+E), 희소 그래프에 유리
- **최단경로**: 다익스트라(Dijkstra, 음수 간선 X), 벨만-포드(음수 허용), 플로이드-워셜(전 쌍, O(V³))
- **최소 신장 트리(MST)**: 프림(Prim), 크루스칼(Kruskal)

### 7. 해싱 (Hashing)

**해시 함수 종류**: 제산법(Division, key % p), 제곱법(Mid-Square), 폴딩법(Folding), 기수 변환(Radix), 무작위법(Random), 숫자 분석법(Digit Analysis).

**충돌(Collision) 해결**
1. **체이닝(Chaining, 개방 해싱)**: 같은 버킷에 연결 리스트로 연결
2. **개방 주소법(Open Addressing, 폐쇄 해싱)**:
   - 선형 조사(Linear Probing): h(k)+1, h(k)+2, …
   - 이차 조사(Quadratic Probing): h(k)+1², h(k)+2², …
   - 이중 해싱(Double Hashing): 두 번째 해시 함수 사용

### 8. 알고리즘 시간복잡도 (Big-O) — 핵심 7단계

| Big-O | 명칭 | 대표 알고리즘 | 입력 n=1,000일 때 대략 |
|---|---|---|---|
| **O(1)** | 상수 시간 | 배열 인덱스 접근, 해시 평균 탐색 | 1 |
| **O(log n)** | 로그 시간 | 이진 탐색, 균형 BST 탐색 | 10 |
| **O(n)** | 선형 시간 | 선형 탐색, 단순 순회 | 1,000 |
| **O(n log n)** | 선형로그 시간 | 병합·힙·퀵(평균) 정렬 | 10,000 |
| **O(n²)** | 이차 시간 | 버블·선택·삽입 정렬 | 1,000,000 |
| **O(2ⁿ)** | 지수 시간 | 부분집합 생성, 재귀 피보나치 | 우주의 입자보다 큼 |
| **O(n!)** | 팩토리얼 시간 | 외판원 완전탐색(TSP brute-force) | 측정 불가 |

⚠ **빅O 규칙**
1. 최고차항만 남기고 계수·하위항 제거 → `3n²+5n+7 = O(n²)`
2. **최악의 경우** 상한 표기 (Worst-case upper bound)
3. 비교: **Ω(오메가)=하한**, **Θ(세타)=평균/타이트 바운드**

### 9. 정렬 알고리즘 복잡도 요약

| 알고리즘 | 평균 | 최악 | 최선 | 안정성 | 제자리 |
|---|---|---|---|---|---|
| 버블(Bubble) | O(n²) | O(n²) | O(n) | ○ | ○ |
| 선택(Selection) | O(n²) | O(n²) | O(n²) | ✕ | ○ |
| 삽입(Insertion) | O(n²) | O(n²) | **O(n)** | ○ | ○ |
| 셸(Shell) | O(n^1.5) | O(n²) | O(n) | ✕ | ○ |
| 퀵(Quick) | **O(n log n)** | O(n²) | O(n log n) | ✕ | ○ |
| 병합(Merge) | O(n log n) | O(n log n) | O(n log n) | ○ | ✕ |
| 힙(Heap) | O(n log n) | O(n log n) | O(n log n) | ✕ | ○ |
| 기수(Radix) | O(dn) | O(dn) | O(dn) | ○ | ✕ |

## 💡 상세 해설

### 문제 1 해설
**정답 ④ 트리.** 선형 = 한 원소 뒤에 단 하나의 원소만 옴(스택·큐·데크·배열·리스트). 비선형 = 한 원소가 여러 자식·이웃을 가짐(트리·그래프). 스택·큐·데크는 모두 **선형**이라는 점을 혼동하지 말 것.

### 문제 2 해설
**정답 ③ 운영체제의 작업 스케줄링.** 작업 스케줄링은 먼저 들어온 작업을 먼저 처리하는 **큐(FIFO)** 응용이다. 나머지(함수 복귀, 재귀, 후위표기 변환)는 모두 스택의 대표 응용.

### 문제 3 해설
**정답 ④ ENQUEUE.** ENQUEUE는 큐의 삽입 연산. 스택(LIFO)의 연산은 PUSH(삽입)/POP(삭제)/TOP·PEEK(확인).

### 문제 4 해설
**정답 ④.** 함수 호출의 복귀 주소 저장은 **스택**의 역할(콜 스택). 큐는 FIFO + rear 삽입/front 삭제 + 스케줄링 응용이 맞다.

### 문제 5 해설
**정답 ②.** 일반 배열 기반 큐는 dequeue 후 앞부분이 비어도 다시 사용 불가 → 메모리 낭비. 원형 큐는 `(rear+1) % size`로 끝과 처음을 연결해 빈 공간을 재활용한다.

### 문제 6 해설
**정답 ③ 데크(Deque).** Double-Ended Queue의 약자. 양쪽에서 모두 삽입·삭제 가능. 스택과 큐의 기능을 모두 수행할 수 있다.

### 문제 7 해설
**정답 ③ 5개.** 단말 노드(자식이 없는 노드) = D, H, I, F, G → 5개. E는 자식(H, I)이 있으므로 단말이 아님.

### 문제 8 해설
**정답 ② 2.** 트리의 차수 = 모든 노드의 자식 수 중 **최댓값**. A·B·C·E 모두 자식이 2개씩이고, 그보다 많은 자식을 가진 노드가 없으므로 트리 차수 = 2.

### 문제 9 해설
**정답 ② n₀ = n₂ + 1.** 이진 트리의 유명한 공식. 도출: 총 노드 수 = n₀+n₁+n₂, 총 간선 수 = 0·n₀+1·n₁+2·n₂. 그리고 간선 수 = 노드 수 − 1. 두 식을 풀면 n₀ = n₂ + 1.

### 문제 10 해설
**정답 ② + * A B D.** 전위(DLR) = 루트→좌→우. (+) → 좌(*→A→B) → 우(D) = **+ * A B D**.

### 문제 11 해설
**정답 ① A B * D +.** 후위(LRD) = 좌→우→루트. 좌(A→B→*) → 우(D) → 루트(+) = **A B * D +**.

### 문제 12 해설
**정답 ③ 트리, 그래프.** 비선형은 트리와 그래프만. 나머지(배열·연결리스트·스택·큐·데크·원형큐)는 모두 선형.

### 문제 13 해설
**정답 ③ 트리.** 트리는 부모-자식의 **계층(hierarchical)** 관계를 표현한다. 스택·큐·데크는 선형이며 평면적이다.

### 문제 14 해설
**정답 ④ 버블 정렬.** 버블 정렬은 정렬 알고리즘이지 해시 충돌 해결 기법이 아니다. 해시 충돌 해결: ① 체이닝(분리연결법) ② 개방주소법(선형/이차/이중해싱).

### 문제 15 해설
**정답 ② O(n log n).** 빅O 크기 순(작을수록 빠름): O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) < O(n!). 보기 중 O(n log n)이 가장 작다.

### 문제 16 해설
**정답 ② O(log n).** 이진 탐색은 매 단계마다 탐색 범위를 절반으로 줄이므로 log₂n 단계 만에 종료. 단, 사전에 **정렬된 배열**이 전제 조건.

### 문제 17 해설
**정답 ④ 병합 정렬.** 병합 정렬은 최선·평균·최악 모두 O(n log n). 버블·선택·삽입은 모두 최악 O(n²).

### 문제 18 해설
**정답 ③.** Big-O는 알고리즘 수행 시간의 **상한(Upper bound)** = 최악의 경우. 평균은 보통 Θ(세타), 최선은 Ω(오메가)로 표기.

### 문제 19 해설
**정답 ② Big-Omega(Ω).** Ω = 하한(lower bound, 최선), O = 상한(upper bound, 최악), Θ = 타이트 바운드(평균/정확). Big-Pi(Π)는 표준 표기법이 아님.

### 문제 20 해설
**정답 ③ 인접 트리.** 그래프 표현 방식: 인접 행렬, 인접 리스트, 인접 다중 리스트. "인접 트리"는 표준 그래프 표현 방식이 아니다(트리는 그래프의 특수 형태일 뿐).

### 문제 21 해설
**정답 ④ O(V²).** 인접 행렬은 V×V 2차원 배열을 사용하므로 공간 O(V²). 반면 인접 리스트는 O(V+E)로 희소 그래프(E ≪ V²)에서 유리.

### 문제 22 해설
**정답 ① A B C * + D -.** 연산자 우선순위에 따라 `*`가 `+`, `-`보다 먼저:
- `A + B * C - D` = `((A + (B*C)) - D)`
- 좌결합 트리화 후 후위 순회(LRD): A, B, C, *, +, D, - = **A B C * + D -**

다른 보기 검증:
- ③ `A B + C * D -` = `((A+B)*C) - D` → 다른 수식이므로 오답.
- ④ `A B C D + * -` = 스택 시뮬 시 피연산자 부족으로 잘못된 표기.

### 문제 23 해설
**정답 ③ 2^h - 1.** 완전 이진트리 높이 h(루트 레벨 1, 최대 깊이 = h)일 때 노드 수 = 1+2+4+…+2^(h-1) = 2^h - 1.

### 문제 24 해설
**정답 ④ 트리 — 환형 구조.** 트리는 **사이클(환)이 없는** 연결 그래프다. 환형이 있으면 그래프이지 트리가 아님.

## 📌 시험 직전 1분 요약 (5줄)

1. **비선형 = 트리·그래프**, 나머지(스택·큐·데크 포함)는 **선형**.
2. 스택 = LIFO (PUSH/POP, 재귀·함수복귀·후위표기), 큐 = FIFO (ENQUEUE/DEQUEUE, 스케줄링·BFS).
3. 이진 트리 핵심 공식: **n₀ = n₂ + 1**, 높이 h 최대 노드 = **2^h − 1**, 순회는 루트 위치로 구분(전·중·후위).
4. Big-O 7단계: **1 < log n < n < n log n < n² < 2ⁿ < n!** — 작을수록 빠름. Big-O=최악, Ω=최선, Θ=평균.
5. 해시 충돌 해결 = 체이닝 / 개방주소(선형·이차·이중해싱). 그래프 표현 = 인접 행렬 O(V²) / 인접 리스트 O(V+E).
