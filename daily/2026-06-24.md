## 🎯 오늘의 기출 8문제 (Day 23·24의 2회독 — 분류·매칭 각도 강화)

### 문제 1 [유사 기출: 정처기 필기 2021년 1회·2023년 2회·2025년 1회 반복 — 대칭/비대칭키 분류 정형 패턴]
다음 **암호 알고리즘** 중 **대칭키(Symmetric Key)** 알고리즘이 **아닌** 것은?

① **AES(Advanced Encryption Standard)** — DES의 후속 미국 표준 블록 암호로, **128/192/256bit 키**를 지원하는 **대칭키 블록 암호**이다.
② **SEED** — 한국인터넷진흥원(KISA)이 개발한 **128bit 블록 길이·128bit 키**의 **국산 대칭키 블록 암호**로, 금융권·전자정부 표준이다.
③ **ARIA** — 학계·국정원·ETRI가 공동 개발한 **128bit 블록·128/192/256bit 키**의 **국산 대칭키 블록 암호**로, 국가표준(KS) 및 IETF RFC 5794에 등재되어 있다.
④ **ECC(Elliptic Curve Cryptography)** — **타원곡선의 이산대수 문제**의 어려움에 기반한 **공개키 암호**로, RSA보다 짧은 키 길이로 동등한 보안 강도를 제공하는 **대표적인 대칭키 블록 암호**이다.

### 문제 2 [유사 기출: 정처기 필기 2020년 2회·2022년 3회·2024년 1회 반복 — 해시 함수 출력 비트수 정형 패턴]
다음 **해시 함수**와 **출력 다이제스트(Digest) 길이**의 매칭으로 옳지 **않은** 것은?

① **MD5(Message-Digest 5)** — Ron Rivest가 설계한 해시 함수로, **128bit** 다이제스트를 출력한다(현재 충돌 공격 발견되어 보안 용도 사용 비권장).
② **SHA-1(Secure Hash Algorithm 1)** — NIST·NSA가 표준화한 해시로, **160bit** 다이제스트를 출력한다(2017년 SHAttered 충돌 공격으로 deprecate).
③ **SHA-256** — SHA-2 계열의 대표 해시로, **256bit** 다이제스트를 출력하며 비트코인 작업증명(PoW)에도 사용된다.
④ **HAS-160** — 한국정보통신기술협회(TTA)가 표준화한 **국산 해시 함수**이나, 출력 다이제스트 길이는 **512bit**로 SHA-512와 동일하다.

### 문제 3 [유사 기출: 정처기 필기 2021년 3회·2023년 1회·2025년 2회 반복 — 접근통제 4정책 정형 패턴]
다음 **접근통제(Access Control) 정책**에 대한 설명으로 가장 옳지 **않은** 것은?

① **MAC(Mandatory Access Control, 강제적 접근통제)** — **시스템(관리자)이 사전에 정해진 보안 등급·레이블**에 따라 접근 권한을 **강제로 결정**하며, 사용자가 임의로 변경할 수 없다(BLP·Biba 모델, 군·정부 기밀 시스템).
② **DAC(Discretionary Access Control, 임의적 접근통제)** — **객체(파일·자원)의 소유자(Owner)** 가 다른 사용자에게 접근 권한을 **임의로 부여·회수**할 수 있다(UNIX의 rwx 파일 권한).
③ **RBAC(Role-Based Access Control, 역할 기반 접근통제)** — 사용자에게 직접 권한을 부여하지 않고, **역할(Role, 직책·직무)에 권한을 묶어 할당**하고 사용자를 역할에 배정한다(대규모 조직 권한 관리 용이).
④ **ABAC(Attribute-Based Access Control, 속성 기반 접근통제)** — **사용자 본인의 임의 판단** 없이, 오직 **사용자의 IP 주소 단일 속성**만을 이용해 접근을 통제하는 방식으로, **MAC의 별칭**으로 분류된다.

### 문제 4 [유사 기출: 정처기 필기 2020년 4회·2022년 1회·2024년 3회 반복 — AAA + 인증 3요소 정형 패턴]
다음 **인증(Authentication) 관련 개념**에 대한 설명으로 옳지 **않은** 것은?

① **AAA**는 **Authentication(인증, "당신은 누구인가")** + **Authorization(인가/권한부여, "무엇을 할 수 있는가")** + **Accounting(과금/감사, "무엇을 했는가")** 의 3가지 보안 기능을 의미한다.
② **AAA 프로토콜**로는 **RADIUS, DIAMETER, TACACS+** 등이 있으며, RADIUS는 UDP 기반, DIAMETER는 RADIUS의 후속으로 TCP/SCTP를 사용한다.
③ **사용자 인증의 3요소**는 **지식 기반(Knowledge, 비밀번호·PIN)** + **소유 기반(Possession, OTP 토큰·스마트카드·공동인증서)** + **존재 기반(Biometric, 지문·홍채·정맥)** 이며, 둘 이상을 결합하면 다중 인증(MFA)이다.
④ **Kerberos**는 **공개키(비대칭키) 기반 인증 프로토콜**로, **CA가 발급한 X.509 인증서**를 사용해 사용자·서버 상호 인증을 수행하며, **티켓(Ticket) 개념을 사용하지 않는** SSL/TLS의 한국형 표준 명칭이다.

### 문제 5 [유사 기출: 정처기 필기 2021년 2회·2023년 3회·2025년 1회 반복 — DoS 변종 식별 정형 패턴]
다음 **서비스 거부(DoS) 공격**의 정의로 옳지 **않은** 것은?

① **SYN Flood** — TCP 3-way 핸드셰이크에서 **SYN 패킷만 대량 송신**하고 서버의 SYN+ACK 응답에 ACK로 응답하지 않아, **서버의 백로그(연결 대기) 큐를 고갈**시킨다.
② **Smurf Attack** — **ICMP Echo Request(ping) 패킷의 출발지 IP를 피해자 IP로 위장**한 뒤 네트워크 **브로드캐스트 주소**로 송신해, 다수의 호스트가 동시에 피해자에게 응답을 보내도록 한다.
③ **Land Attack** — **출발지 IP와 목적지 IP를 동일하게(피해자 자신으로) 위조**한 패킷을 보내, 피해자 시스템이 자기 자신과 무한 통신하도록 만들어 자원을 고갈시킨다.
④ **Teardrop Attack** — **사용자의 비밀번호 사전(Dictionary)** 을 무차별로 대입해 계정을 탈취하는 **온라인 패스워드 추측 공격**으로, DoS와는 무관한 **인증 우회** 기법이다.

### 문제 6 [유사 기출: 정처기 필기 2020년 3회·2022년 2회·2024년 2회 반복 — 웹·사회공학 공격 분류 정형 패턴]
다음 **웹·사회공학 공격**에 대한 설명으로 옳지 **않은** 것은?

① **SQL Injection** — 사용자 입력값에 `' OR 1=1 --` 등 **SQL 구문을 삽입**하여 인증을 우회하거나 DB 정보를 탈취하는 공격으로, **Prepared Statement·입력값 검증**으로 방어한다.
② **XSS(Cross-Site Scripting)** — 게시판·댓글 등에 **악성 JavaScript를 저장하거나 URL 파라미터로 주입**하여, **다른 사용자의 브라우저에서 스크립트가 실행**되도록 하는 공격이다.
③ **CSRF(Cross-Site Request Forgery)** — **이미 로그인된 사용자의 권한(세션 쿠키)을 도용**하여, 사용자가 의도하지 않은 요청(글 작성·계정 변경·송금 등)을 서버에 전송하도록 유도하는 공격이다.
④ **파밍(Pharming)** — 공격자가 **휴대전화 SMS(문자메시지)**에 악성 URL을 삽입해 피해자가 클릭하면 악성 앱을 설치하거나 개인정보를 탈취하는 공격으로, **SMS와 피싱(Phishing)의 합성어**이다.

### 문제 7 [유사 기출: 정처기 필기 2021년 1회·2023년 2회·2025년 2회 반복 — 보안 솔루션 매칭 정형 패턴]
다음 **보안 솔루션**에 대한 설명으로 옳지 **않은** 것은?

① **방화벽(Firewall)** — **출발지·목적지 IP, 포트, 프로토콜 등 패킷 헤더 정보**를 기준으로 트래픽을 허용·차단하는 솔루션으로, 일반적으로 OSI 3·4계층에서 동작한다.
② **IDS(Intrusion Detection System)** vs **IPS(Intrusion Prevention System)** — **IDS는 침입을 탐지·경보만 발생**시키는 **수동(passive)** 솔루션이고, **IPS는 탐지 + 자동 차단(drop·session reset)** 까지 수행하는 **능동(inline)** 솔루션이다.
③ **WAF(Web Application Firewall)** — **HTTP/HTTPS 트래픽의 페이로드(payload)** 를 검사하여 **SQL Injection·XSS·CSRF 등 웹 애플리케이션 계층(OSI 7계층) 공격을 전문적으로 탐지·차단**하는 솔루션이다.
④ **VPN(Virtual Private Network)** — 공중망 위에 **암호화 터널**을 만들어 가상 사설망을 구현하는 기술로, **IPSec VPN은 응용 계층에서, SSL/TLS VPN은 데이터링크 계층에서** 각각 동작하여 두 방식은 **계층상 완전히 동일**하다.

### 문제 8 [유사 기출: 정처기 필기 2020년 1회·2022년 3회·2024년 1회 반복 — 가짜 약어·정의 충돌 함정 정형 패턴]
다음 보안 약어·개념의 정의로 **가장 옳은 것**은?

① **DSA(Digital Signature Algorithm)** 는 **디지털 서명 전용 표준 알고리즘**으로, 메시지의 무결성·부인방지를 보장하나 **그 자체로 메시지를 암호화하는 용도로는 사용하지 않는다**.
② **DH(Diffie-Hellman)** 는 **공개키 암호의 대표 알고리즘**으로, RSA처럼 **메시지 암·복호화와 전자서명 모두**에 사용되며, 키 교환 용도로는 사용할 수 없다.
③ **MAC(Message Authentication Code, 메시지 인증코드)** 는 접근통제 정책 **MAC(Mandatory Access Control)** 의 **공식 줄임말**로, 두 용어는 의미·약어가 **완전히 동일**하며 시험에서는 혼용해도 무방하다.
④ **PKI(Public Key Infrastructure)** 는 **대칭키 분배**를 위한 인프라로, **CA·RA·CRL 등을 사용하지 않으며** 오직 **SEED·ARIA 등 국산 대칭키 블록암호의 키 관리만을 담당**한다.

## 🔍 오답 역추적 개념 발췌
> 함정 보기 4개의 출처·키워드를 정리해, "이 함정을 봤을 때 어떤 정답 개념으로 즉답해야 하는가" 매핑.

- **"ECC = 대표적 대칭키 블록 암호"** (문제 1 ④) → **ECC는 비대칭(공개키) 암호** · 타원곡선 이산대수 기반. 함정 즉답: **"공개키·개인키 쌍" = 비대칭** → RSA·ECC·ElGamal·DH·DSA 5개 외워두기.
- **"HAS-160 출력 = 512bit"** (문제 2 ④) → **HAS-160의 "160"이 곧 출력 비트수** · 국산 해시지만 SHA-512와 무관. 함정 즉답: **해시명에 숫자가 붙으면 그 숫자가 출력 비트수**(SHA-256=256, SHA-512=512, HAS-160=160). 단, MD5=128bit만 예외 암기.
- **"ABAC = IP 단일 속성만 사용 + MAC의 별칭"** (문제 3 ④) → ABAC는 **사용자·자원·환경 다양한 속성 조합 정책** · MAC과 별개의 4번째 정책. 함정 즉답: **MAC=강제(레이블), DAC=임의(소유자), RBAC=역할, ABAC=속성 조합** 1자 매칭. "별칭"·"동일" 단어는 함정 신호.
- **"Kerberos = X.509 인증서 기반·티켓 미사용·SSL/TLS 한국형 표준"** (문제 4 ④) → Kerberos는 **MIT 개발 대칭키 기반·티켓(TGT) 사용 인증 프로토콜**. **공개키·X.509·SSL과 무관**. 함정 즉답: **Kerberos = 티켓(Ticket) + 대칭키 + MIT** 3종 외워두기.
- **"Teardrop = 사전 공격(Dictionary Attack)"** (문제 5 ④) → Teardrop은 **IP 단편화 offset 조작 재조립 오류 DoS**. 사전 공격은 별개 인증 우회 기법. 함정 즉답: **Teardrop = 단편(Fragment) + offset** 1자 매칭.
- **"파밍 = SMS + 피싱 합성어"** (문제 6 ④) → SMS+피싱은 **스미싱(Smishing)** · 파밍은 **DNS·hosts 파일 변조**해 정상 URL → 가짜 사이트로 유도. 함정 즉답: **파밍=DNS/hosts 변조, 스미싱=SMS, 피싱=메일·사이트** 3종 1자 매칭.
- **"IPSec VPN = 응용 계층 / SSL VPN = 데이터링크 계층 / 두 방식 계층 동일"** (문제 7 ④) → **IPSec = 네트워크 계층(L3, AH/ESP) / SSL/TLS VPN = 전송·응용 사이(L4~L7)** — 계층 다름. 함정 즉답: **IPSec=L3, SSL=L4~L7** 1자 매칭.
- **"MAC(메시지인증코드) = MAC(접근통제)의 공식 줄임말·동일"** (문제 8 ③) → **같은 약어 다른 개념** — Message Authentication Code vs Mandatory Access Control. 시험은 둘 다 별개로 출제. 함정 즉답: **약어가 같아도 풀네임이 다르면 별개 개념**.

## 💡 상세 해설

### 문제 1
**정답 ④ ECC = 대표적인 대칭키 블록 암호.**
- **ECC(Elliptic Curve Cryptography)** 는 **타원곡선상의 이산대수 문제(ECDLP)** 의 어려움에 기반한 **비대칭(공개키) 암호**. RSA와 같은 공개키·개인키 쌍 구조. **대칭키 블록 암호가 아님**.
- ECC의 장점: **RSA 2048bit ≈ ECC 224bit** 정도로, **짧은 키 길이로 동등한 보안 강도** → 모바일·IoT 환경에 유리. 비트코인·이더리움 서명(ECDSA)에도 사용.
- ① **AES** = 미국 표준 대칭키 블록 암호(128bit 블록, 128/192/256 키). ✔
- ② **SEED** = 한국인터넷진흥원(KISA, 1999) 개발 국산 대칭키. 금융·전자정부 표준. ✔
- ③ **ARIA** = 학계·국정원·ETRI 공동 개발 국산 대칭키. KS·RFC 5794. ✔
- 정형 함정: **국산 대칭키 4종 "SEED·ARIA·HIGHT·LEA" + AES·DES·3DES·IDEA** 사이에 **RSA/ECC/ElGamal/DH/DSA** 같은 **비대칭키**를 끼워넣어 분류 혼란 유도. **"공개키·개인키 쌍·이산대수·소인수분해·타원곡선" 키워드가 등장하면 비대칭**으로 즉답.

### 문제 2
**정답 ④ HAS-160 출력 = 512bit (SHA-512와 동일).**
- **HAS-160(Hash Algorithm Standard - 160)** 은 한국정보통신기술협회(TTA) 표준 **국산 해시 함수**로, **명칭의 "160"이 곧 출력 다이제스트 길이 = 160bit**. SHA-1과 동일한 길이지만 별개 알고리즘.
- ① **MD5 = 128bit**(Ron Rivest, 1992). 충돌 공격(2004 Wang) 발견되어 보안 용도 비권장. ✔
- ② **SHA-1 = 160bit**(NIST FIPS 180-1, 1995). 2017 SHAttered 충돌 공격 PoC. ✔
- ③ **SHA-256 = 256bit**(SHA-2 계열, FIPS 180-2). 비트코인 PoW·TLS 인증서 서명. ✔
- 정형 함정: **해시명의 숫자 ↔ 출력 비트수**가 일치하는 규칙(SHA-256=256, SHA-512=512, HAS-160=160)을 **거꾸로 매칭**(HAS-160=512, SHA-256=512 등)하는 단골. **MD5만 "5"가 버전 번호** → 128bit 별도 암기.

### 문제 3
**정답 ④ ABAC = IP 단일 속성만 사용 + MAC의 별칭.**
- **ABAC(Attribute-Based Access Control)** 는 **사용자 속성(부서·직급) + 자원 속성(분류 등급) + 환경 속성(시간·위치) + 행동 속성** 등 **다양한 속성 조합**으로 정책(Policy)을 작성하는 4번째 접근통제 모델. **IP 단일 속성 X**, **MAC의 별칭 X**.
- **MAC·DAC·RBAC·ABAC는 모두 별개의 4정책** — MAC = 강제(시스템·레이블) / DAC = 임의(소유자) / RBAC = 역할(직무) / ABAC = 속성(조합).
- ① **MAC** = 시스템·관리자가 보안 레벨(Top Secret·Secret·Confidential)·레이블로 강제 결정. BLP(기밀성)·Biba(무결성). ✔
- ② **DAC** = 객체 소유자가 임의 권한 부여. UNIX rwx, Windows ACL. ✔
- ③ **RBAC** = Role에 권한 묶음 + 사용자를 Role에 배정. NIST RBAC 표준. ✔
- 정형 함정: **ABAC을 "MAC의 별칭·동의어"** 로 묶거나, **DAC을 "강제적"**, **MAC을 "임의적"** 으로 정의를 뒤바꾸는 패턴. **"강제·임의·역할·속성" 4단어 ↔ MAC·DAC·RBAC·ABAC** 1:1 매칭으로 즉답.

### 문제 4
**정답 ④ Kerberos = X.509 인증서 기반·티켓 미사용·SSL/TLS 한국형 표준.**
- **Kerberos(케르베로스)** 는 **MIT가 개발한 대칭키 기반 네트워크 인증 프로토콜**로, **티켓(Ticket) 개념의 핵심**: **TGT(Ticket Granting Ticket)·TGS(Ticket Granting Server)·AS(Authentication Server)** 3자 구조. **X.509 인증서 X**, **SSL/TLS와 무관**, **한국형 표준이 아닌 MIT 미국 표준**.
- ① **AAA = Authentication + Authorization + Accounting** 3종(인증·인가·과금). ✔
- ② **RADIUS**(RFC 2865, UDP 1812/1813) / **DIAMETER**(RFC 6733, TCP/SCTP, RADIUS 후속) / **TACACS+**(Cisco) — 대표 AAA 프로토콜. ✔
- ③ **인증 3요소** = **What you know(지식, 비밀번호)** / **What you have(소유, OTP·스마트카드)** / **What you are(존재, 생체)**. 2종 이상 결합 = MFA(Multi-Factor Authentication). ✔
- 정형 함정: **Kerberos를 공개키·X.509·SSL과 묶어 정의 변조**, 혹은 **"한국 표준"** 같은 가짜 출처 끼워넣기. **Kerberos = 티켓 + 대칭키 + MIT**, **PKI = X.509 + CA + 공개키** 명확 구분.

### 문제 5
**정답 ④ Teardrop = 사전 공격(Dictionary Attack) · 인증 우회.**
- **Teardrop Attack**은 **IP 단편화(Fragmentation) 시 단편 offset 필드를 의도적으로 겹치게(overlap)·잘못 조작**해 보낸 후, 수신측이 단편을 재조립할 때 **버퍼 오류·시스템 다운**을 유발하는 **DoS 공격**(Windows 95/NT, Linux 2.0.x 취약). **사전 공격과 무관**.
- **사전 공격(Dictionary Attack)** 은 미리 준비한 단어 사전을 무차별 대입해 비밀번호를 추측하는 **온라인 패스워드 추측 공격** — 인증·계정 탈취 영역.
- ① **SYN Flood** = TCP 3-way의 첫 SYN만 대량 보내고 ACK 미응답 → SYN_RECV 큐 고갈. SYN Cookie로 방어. ✔
- ② **Smurf** = ICMP Echo Request의 출발지 IP를 피해자로 위조 + 브로드캐스트 전송 → 다수가 피해자에게 응답 폭주(증폭 공격). ✔
- ③ **Land** = 출발지·목적지 IP 동일(피해자 자신) 위조 → 자기 자신과 무한 통신 자원 고갈. ✔
- 정형 함정: **DoS 5종(SYN Flood / Smurf / Land / Teardrop / Ping of Death)** 의 정의를 셔플하거나, **Teardrop·Ping of Death** 자리에 **사전공격·SQLi** 같은 **다른 영역 공격** 끼워넣기. **Teardrop = 단편(Fragment)** 1자 매칭.

### 문제 6
**정답 ④ 파밍 = SMS + 피싱 합성어.**
- **SMS + Phishing = 스미싱(Smishing)** — 문자메시지(SMS)로 악성 URL/앱 유포. **파밍이 아님**.
- **파밍(Pharming)** = **DNS 서버·로컬 hosts 파일 변조**로 정상 URL 입력 시에도 **가짜(공격자) 사이트로 자동 리다이렉트** → 사용자가 URL을 정확히 입력해도 속는 고난도 공격. 어원: **Phishing + farming(농사)**.
- ① **SQL Injection** = 입력값에 SQL 구문 삽입(' OR 1=1 --, UNION 활용) → 인증 우회·DB 덤프. 방어: **Prepared Statement·바인딩 변수·입력값 화이트리스트**. ✔
- ② **XSS** = 게시판·댓글·URL 파라미터에 `<script>` 삽입 → **다른 사용자 브라우저**에서 실행(쿠키 탈취·세션 하이재킹). Stored/Reflected/DOM 3유형. 방어: **출력 인코딩·CSP**. ✔
- ③ **CSRF** = 로그인된 사용자 권한(세션 쿠키 자동 첨부)을 도용해 의도치 않은 요청 송신. 방어: **CSRF 토큰·SameSite Cookie·Referer 검증**. ✔
- 정형 함정: **피싱·스미싱·파밍·비싱(Vishing, Voice+Phishing)** 4종 사회공학 + **SQLi·XSS·CSRF** 웹 공격 정의 셔플. **"SMS=스미싱, DNS/hosts=파밍, 음성=비싱, 이메일/사이트=피싱"** 4종 1자 매칭.

### 문제 7
**정답 ④ IPSec VPN = 응용 계층 / SSL/TLS VPN = 데이터링크 계층 / 두 방식 계층 동일.**
- **IPSec VPN** = **OSI 3계층(네트워크 계층)** 에서 동작. **AH(Authentication Header, 인증)** + **ESP(Encapsulating Security Payload, 암호화+인증)** 사용. **Transport 모드**(IP 페이로드만)·**Tunnel 모드**(IP 헤더 포함 전체) 지원. 사이트 간(Site-to-Site) VPN 표준.
- **SSL/TLS VPN** = **OSI 4~7계층 사이(전송~응용 계층)** 에서 동작. **웹 브라우저 기반 클라이언트리스** VPN 가능. 원격 사용자 접근(Remote Access)에 유리.
- → **두 방식은 동작 계층·구조 자체가 다름**, 동일하지 않음.
- ① **방화벽** = IP·포트·프로토콜 헤더 기반 필터링, OSI 3·4계층(Stateful Inspection 포함). ✔
- ② **IDS(수동·passive·alert) vs IPS(능동·inline·차단)** 명확 구분. ✔
- ③ **WAF** = HTTP/HTTPS 페이로드 분석, OSI 7계층, **SQLi·XSS·CSRF·File Upload** 등 웹 공격 전문 차단(ModSecurity, AWS WAF 등). ✔
- 정형 함정: **IPSec·SSL VPN의 동작 계층을 뒤바꾸거나 "동일"이라고 묶는 단골**. **IPSec=L3 / SSL=L4~L7** 명확 매칭. 또한 **방화벽 vs WAF**(L3·4 vs L7) 계층 차이도 자주 출제.

### 문제 8
**정답 ① DSA = 디지털 서명 전용 표준, 메시지 암호화 용도로 사용하지 않음.**
- **DSA(Digital Signature Algorithm)** 는 **NIST FIPS 186 표준의 전자서명 전용 알고리즘** — **이산대수 문제** 기반, 메시지의 **무결성·부인방지·인증** 보장. **메시지를 직접 암·복호화하는 용도로는 사용하지 않음**(RSA는 서명·암호화 둘 다 가능하나 DSA는 서명만).
- ② DH(Diffie-Hellman)는 **키 교환(Key Exchange) 전용**(1976) — 두 통신자가 공개된 채널에서 공유 비밀키를 안전하게 합의. **암·복호화·서명 용도로는 사용하지 않음**. ✘ (정의 정반대)
- ③ **MAC(Message Authentication Code) ≠ MAC(Mandatory Access Control)** — 같은 약어, 다른 풀네임, **다른 개념**. 메시지 인증코드는 **공유 비밀키 + 메시지 → MAC 태그**(HMAC, CMAC) 생성으로 무결성+인증 보장. 접근통제 MAC은 보안 정책. ✘
- ④ **PKI(Public Key Infrastructure)** 는 **공개키(비대칭키) 인프라** — **CA(Certificate Authority, 인증기관)** + **RA(등록기관)** + **Repository** + **CRL(인증서 폐기 목록)** + **OCSP** + **X.509 인증서**로 구성. **대칭키 분배가 아닌 공개키 인증서 발급·검증·관리** 인프라. SEED·ARIA는 대칭키로 PKI 대상 X. ✘
- 정형 함정: **DSA = 서명 + RSA = 서명/암호화 둘 다 + DH = 키 교환 전용 + ECC = 서명/암호화/키교환 다용도** 4종 비대칭키의 **용도 구분**을 셔플하거나, **MAC 약어 충돌**·**PKI 정의 변조**(대칭키 분배) 패턴이 단골. **"~ 전용" 키워드가 정답의 신호**.
