① 순차 탐색(Sequential Search)
② 이진 탐색(Binary Search)
③ 보간 탐색(Interpolation Search)
④ 블록 탐색(Block Search)

### 문제 2 [유사 기출: 정보처리기사 필기 — 이진탐색 비교 횟수 정형 패턴]
1부터 100까지의 정렬된 정수가 저장된 배열에서 이진 탐색(Binary Search)으로 특정 값을 찾을 때, 최악의 경우 필요한 최대 비교 횟수로 옳은 것은? (비교 횟수는 ⌊log₂N⌋ + 1로 계산)

① 5회
② 7회
③ 10회
④ 50회

### 문제 3 [유사 기출: 정보처리기사 필기 — 보간/피보나치 탐색 정형 패턴]
다음에서 설명하는 탐색 방법으로 옳은 것은?

> 정렬된 데이터에서 찾으려는 키 값의 위치를 사전(辭典)에서 단어를 찾듯이 비례식으로 예측하여 탐색하는 방법. 데이터가 균등하게 분포할수록 효율적이다.

① 이진 탐색(Binary Search)
② 보간 탐색(Interpolation Search)
③ 순차 탐색(Sequential Search)
④ 깊이 우선 탐색(DFS)

### 문제 4 [유사 기출: 정보처리기사 필기 — 해시 함수(제산법) 계산 정형 패턴]
해시 함수(Hash Function)로 제산법(나눗셈법, Division Method)을 사용하고 버킷(bucket) 크기가 13일 때, 키 값 25의 홈 주소(home address)로 옳은 것은? (해시 주소 = 키 mod 버킷 크기)

① 1
② 12
③ 13
④ 25

### 문제 5 [유사 기출: 정보처리기사 필기 — 해시 충돌 해결 정형 패턴]
해싱(Hashing)에서 서로 다른 두 키가 동일한 해시 주소를 갖는 충돌(collision)이 발생했을 때, 충돌이 일어난 자리 이외의 비어 있는 다른 버킷을 찾아 저장하는 충돌 해결 방법은?

① 개방 주소법(Open Addressing)
② 체이닝(Chaining)
③ 직접 접근(Direct Access)
④ 폴딩법(Folding)

---

### 문제 6 [유사 기출: 정보처리기사 필기 — 해시 용어/선형조사 정형 패턴]
해싱(Hashing)에 대한 설명으로 옳지 않은 것은?

① 서로 다른 키가 같은 해시 주소로 사상되는 것을 충돌(collision)이라 한다.
② 같은 해시 주소를 갖는 서로 다른 레코드의 집합을 동의어(synonym)라 한다.
③ 선형 조사법(Linear Probing)은 충돌 발생 시 그 다음 비어 있는 버킷을 순차적으로 찾는다.
④ 해시 테이블에서 적재율(load factor)이 높아질수록 충돌 가능성은 낮아진다.

### 문제 7 [유사 기출: 정보처리기사 필기 — 그래프 순회(BFS/DFS) 정형 패턴]
그래프 탐색 방법 중 너비 우선 탐색(BFS, Breadth First Search)을 구현할 때 사용하는 자료 구조로 가장 적절한 것은?

① 스택(Stack)
② 큐(Queue)
③ 트리(Tree)
④ 힙(Heap)

---

### 문제 8 [유사 기출: 정보처리기사 필기 — 그래프 표현(인접행렬/인접리스트) 정형 패턴]
그래프(Graph)의 표현 방법인 인접 행렬(Adjacency Matrix)과 인접 리스트(Adjacency List)에 대한 설명으로 옳지 않은 것은?

① 정점(vertex)이 N개인 그래프의 인접 행렬은 N×N 크기의 배열로 표현된다.
② 인접 행렬은 두 정점 간 간선(edge)의 존재 여부를 O(1)에 확인할 수 있다.
③ 간선(edge)이 적은 희소(sparse) 그래프에는 인접 리스트가 메모리 면에서 유리하다.
④ 인접 리스트는 간선이 적든 많든 항상 N×N 크기의 메모리를 고정으로 사용한다.

- 순차 탐색(Sequential Search) vs 이진 탐색(Binary Search) — 정렬 전제 여부

- 충돌(Collision) / 동의어(Synonym) / 적재율(Load factor)
- 충돌 해결: 개방 주소법(Open Addressing) — 선형 조사·이차 조사·이중 해싱 / 체이닝(Chaining)
- 그래프 순회: BFS(큐, FIFO) vs DFS(스택/재귀, LIFO)
- 그래프 표현: 인접 행렬(N×N, 밀집 그래프 유리) vs 인접 리스트(희소 그래프 유리)

## 💡 상세 해설 (자기 설명과 대조)

### 문제 1
정답: ② 이진 탐색(Binary Search)
- 제공 해설: 정렬된 데이터에서 중앙값과 키를 비교해 탐색 범위를 매번 절반으로 줄이는 것은 이진 탐색의 정의. 시간복잡도 O(log₂N).
- 오답 이유:
  - ① 순차 탐색: 정렬 여부와 무관하게 처음부터 하나씩 차례로 비교. O(N).
  - ③ 보간 탐색: 비례식으로 위치를 예측하지 범위를 단순히 1/2로 나누지 않음.
  - ④ 블록 탐색: 데이터를 블록 단위로 나눠 대표값으로 블록을 찾은 뒤 블록 내부를 순차 탐색하는 방식(절반 분할이 핵심이 아님).

### 문제 2
정답: ② 7회
- 제공 해설: N=100. 이진 탐색 최악 비교 횟수 = ⌊log₂100⌋ + 1. log₂100 ≈ 6.64이므로 ⌊6.64⌋ = 6, 여기에 +1 하면 7회. (범위가 100 → 50 → 25 → 13 → 7 → 4 → 2 → 1로 약 7단계 안에 1로 줄어듦.)
- 오답 이유:
  - ① 5회: 2⁵=32로 100을 다 덮지 못함(부족).
  - ③ 10회: 2¹⁰=1024로 과도하게 큼. log₂100은 10이 아님.
  - ④ 50회: 순차 탐색의 평균(N/2) 개념과 혼동한 값. 이진 탐색과 무관.

### 문제 3
정답: ② 보간 탐색(Interpolation Search)
- 제공 해설: 사전에서 단어 찾듯 키 값의 상대 위치를 비례식(보간 공식)으로 예측해 접근하는 것이 보간 탐색. 데이터가 균등 분포일 때 평균 O(log log N)까지 빨라질 수 있음.
- 오답 이유:
  - ① 이진 탐색: 항상 중앙(1/2 지점)을 보지, 키 값에 비례한 위치를 예측하지 않음.
  - ③ 순차 탐색: 비례 예측 없이 처음부터 순서대로 비교.
  - ④ DFS(깊이 우선 탐색): 그래프/트리 순회 기법으로 정렬 배열 탐색과 무관.

### 문제 4
정답: ② 12
- 제공 해설: 제산법(나눗셈법) 해시 주소 = 키 mod 버킷 크기 = 25 mod 13. 25 = 13×1 + 12 이므로 나머지는 12. 홈 주소는 12.
- 오답 이유:
  - ① 1: 25 ÷ 13의 몫(1)을 답으로 착각한 값. 해시 주소는 나머지를 사용.
  - ③ 13: 버킷 크기 자체. 나머지는 0~12 범위이므로 13은 나올 수 없음.
  - ④ 25: 키 값 그대로. 해시 함수를 적용하지 않은 값.

### 문제 5
정답: ① 개방 주소법(Open Addressing)
- 제공 해설: 충돌 시 동일 테이블 내의 다른 빈 버킷을 조사(probing)하여 저장하는 방법이 개방 주소법. 선형 조사·이차 조사·이중 해싱이 여기에 속한다.
- 오답 이유:
  - ② 체이닝: 같은 주소에 연결 리스트(linked list)를 달아 충돌 레코드를 매다는 방식(폐쇄 주소법). 빈 다른 버킷을 찾아가지 않음.
  - ③ 직접 접근: 키를 곧바로 주소로 쓰는 방식으로 충돌 해결 기법이 아님.
  - ④ 폴딩법: 키를 몇 부분으로 나눠 더하는 해시 함수 생성 기법이지 충돌 해결 기법이 아님.

### 문제 6
정답: ④ (옳지 않은 것)
- 제공 해설: 적재율(load factor = 저장된 키 수 / 버킷 수)이 높아질수록 빈 자리가 줄어 충돌 가능성은 오히려 높아진다. ④는 정반대로 진술되어 거짓.
- 오답 이유 (옳은 보기들):
  - ① 충돌(collision) 정의 — 옳음.
  - ② 동의어(synonym): 같은 해시 주소를 갖는 레코드 집합 — 옳음.
  - ③ 선형 조사법(Linear Probing): 충돌 시 바로 다음 버킷부터 순차 탐색 — 옳음.

### 문제 7
정답: ② 큐(Queue)
- 제공 해설: BFS(너비 우선 탐색)는 가까운 정점부터 레벨 순서로 방문하므로 FIFO(First In First Out)인 큐를 사용한다. 반대로 DFS는 스택(또는 재귀)을 사용.
- 오답 이유:
  - ① 스택(Stack): LIFO 구조로 DFS(깊이 우선 탐색)에 사용.
  - ③ 트리(Tree): 자료 구조 자체이지 BFS 구현 보조 구조가 아님.
  - ④ 힙(Heap): 우선순위 큐/정렬에 쓰이며 일반 BFS의 핵심 구조가 아님.

### 문제 8
정답: ④ (옳지 않은 것)
- 제공 해설: 항상 N×N 메모리를 고정으로 쓰는 것은 인접 행렬(Adjacency Matrix)이다. 인접 리스트는 실제 간선 수(E)에 비례해 O(N+E) 메모리만 사용하므로 ④의 진술이 거짓.
- 오답 이유 (옳은 보기들):
  - ① 정점 N개 그래프의 인접 행렬은 N×N 배열 — 옳음.
  - ② 인접 행렬에서 간선 존재 확인은 배열 인덱스 접근으로 O(1) — 옳음.
  - ③ 희소 그래프(간선 적음)에서는 인접 리스트가 메모리 면에서 유리 — 옳음.

## 📚 역추적 참조 (사후 검증용 — 학습 자료 아님)

### 핵심 용어
| 용어 | 정의 |
|---|---|
| 순차 탐색(Sequential Search) | 처음부터 끝까지 하나씩 차례로 비교. 정렬 불필요. 평균 O(N). |
| 이진 탐색(Binary Search) | 정렬 데이터에서 중앙값 비교로 범위를 1/2씩 축소. O(log₂N). |
| 보간 탐색(Interpolation Search) | 키 값의 비례 위치를 예측해 접근. 균등 분포 시 O(log log N). |
| 피보나치 탐색(Fibonacci Search) | 피보나치 수로 범위를 분할. 곱셈·나눗셈 없이 가감산만 사용. |
| 해시 함수(Hash Function) | 키를 해시 주소로 변환하는 함수. 제산법이 대표. |
| 충돌(Collision) | 서로 다른 키가 같은 해시 주소로 사상되는 현상. |
| 동의어(Synonym) | 같은 해시 주소를 갖는 서로 다른 레코드(키)들의 집합. |
| 적재율(Load Factor) | 저장된 키 수 ÷ 전체 버킷 수. 높을수록 충돌 증가. |
| 개방 주소법(Open Addressing) | 충돌 시 동일 테이블의 다른 빈 버킷을 조사. (선형/이차/이중) |
| 체이닝(Chaining) | 같은 주소에 연결 리스트로 충돌 레코드를 매다는 폐쇄 주소법. |
| BFS(Breadth First Search) | 너비 우선 탐색. 큐(FIFO) 사용. 레벨 순 방문. |
| DFS(Depth First Search) | 깊이 우선 탐색. 스택(LIFO)/재귀 사용. 한 경로 끝까지 방문. |

### 핵심 개념 (사후 검증용)

**탐색 방법 비교**
| 탐색 | 정렬 전제 | 평균 시간복잡도 | 핵심 |
|---|---|---|---|
| 순차 탐색 | X | O(N) | 처음부터 순차 비교 |
| 이진 탐색 | O | O(log₂N) | 중앙값 기준 1/2 분할 |
| 보간 탐색 | O | O(log log N) | 비례식 위치 예측(균등 분포) |
| 피보나치 탐색 | O | O(log₂N) | 피보나치 수 범위 분할(가감산) |

**이진 탐색 비교 횟수**
- 최악 비교 횟수 = ⌊log₂N⌋ + 1
- N=100 → log₂100 ≈ 6.64 → ⌊6.64⌋ + 1 = 7회
- 매 비교마다 탐색 범위가 절반으로 줄어듦 (100→50→25→13→7→4→2→1)

**해시 함수 생성법**
- 제산법(Division): 주소 = 키 mod 버킷크기. 버킷 크기는 보통 소수(prime)로 설정.
- 중간제곱법(Mid-square): 키를 제곱한 뒤 중간 자릿수를 주소로 사용.
- 폴딩법(Folding): 키를 여러 부분으로 나눠 더한(또는 XOR) 값을 주소로 사용.
- 숫자분석법(Digit analysis): 키의 각 자릿수 분포를 분석해 고른 자리를 주소로 사용.

**해시 충돌 해결**
| 분류 | 기법 | 동작 |
|---|---|---|
| 개방 주소법 | 선형 조사(Linear Probing) | 충돌 시 (h+1, h+2 …) 다음 버킷을 순차 조사 |
| 개방 주소법 | 이차 조사(Quadratic Probing) | (h+1², h+2² …) 제곱 간격으로 조사 |
| 개방 주소법 | 이중 해싱(Double Hashing) | 두 번째 해시 함수로 조사 간격 결정 |
| 폐쇄 주소법 | 체이닝(Chaining) | 같은 주소에 연결 리스트로 매닮 |

- 제산법 계산 예: 25 mod 13 = 12 (25 = 13×1 + 12)
- 적재율↑ → 충돌↑ → 탐색 성능↓

**BFS vs DFS**
| 항목 | BFS | DFS |
|---|---|---|
| 자료 구조 | 큐(Queue, FIFO) | 스택(Stack, LIFO) / 재귀 |
| 방문 순서 | 가까운 정점부터 레벨 순 | 한 경로 끝까지 깊이 우선 |
| 비가중 최단 경로 | 보장 | 미보장 |
| 메모리 경향 | 너비만큼 큐에 보관 | 깊이만큼 스택에 보관 |

**그래프 표현: 인접 행렬 vs 인접 리스트**
| 항목 | 인접 행렬 | 인접 리스트 |
|---|---|---|
| 메모리 | O(N²) 고정 | O(N+E) |
| 간선 존재 확인 | O(1) | O(차수) |
| 유리한 경우 | 밀집(dense) 그래프 | 희소(sparse) 그래프 |
| 정점 차수 계산 | 한 행 전체 스캔 O(N) | 리스트 길이 O(차수) |
