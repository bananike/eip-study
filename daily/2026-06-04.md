## 🎯 오늘의 기출 8문제

### 문제 1 [출처: 2021년 2회 정보처리기사 필기 — 유사 기출]
다음 설명에 해당하는 서비스 거부(DoS) 공격은?

```
- 공격자가 출발지 IP를 공격 대상의 IP로 위조(Spoofing)
- 브로드캐스트 주소로 다량의 ICMP Echo Request 패킷 전송
- 네트워크 내 모든 호스트가 ICMP Echo Reply를 공격 대상에 보내 트래픽 폭주
- 직접 공격이 아닌 증폭(Amplification) 공격의 대표 사례
```
① SYN Flooding
② Smurf Attack
③ Land Attack
④ Teardrop Attack

### 문제 2 [출처: 2022년 1회 정보처리기사 필기 — 유사 기출]
TCP의 3-way handshake 과정의 취약점을 이용해 다량의 SYN 패킷을 보내고 SYN/ACK 응답에 대한 ACK를 보내지 않아 서버의 백로그 큐를 가득 채워 정상 접속을 차단하는 공격은?
① Smurf Attack
② SYN Flooding
③ Ping of Death
④ Tear Drop

### 문제 3 [출처: 2020년 4회 정보처리기사 필기 — 유사 기출]
IPSec(IP Security)에 대한 설명으로 옳지 않은 것은?
① OSI 7계층 중 네트워크 계층(L3)에서 동작한다.
② AH(Authentication Header)는 데이터 무결성과 인증을 제공하지만 기밀성은 제공하지 않는다.
③ ESP(Encapsulating Security Payload)는 기밀성, 무결성, 인증을 모두 제공할 수 있다.
④ IKE(Internet Key Exchange)는 데이터 패킷을 직접 암호화하는 기능을 담당한다.

### 문제 4 [출처: 2021년 3회 정보처리기사 필기 — 유사 기출]
침입 탐지 시스템(IDS)과 침입 방지 시스템(IPS)의 차이로 옳은 것은?
① IDS는 인라인(In-line)으로 설치되어 트래픽을 직접 차단하고, IPS는 미러링된 트래픽을 분석만 한다.
② IDS는 탐지 후 알림만 수행하고, IPS는 탐지된 공격 트래픽을 실시간으로 차단할 수 있다.
③ IDS는 시그니처 기반 탐지만 가능하고, IPS는 이상 탐지(Anomaly)만 가능하다.
④ IDS는 호스트 기반에서만 동작하고, IPS는 네트워크 기반에서만 동작한다.

### 문제 5 [출처: 2022년 3회 정보처리기사 필기 — 유사 기출]
다음 설명에 해당하는 공격 기법은?

```
- 특정 목표(정부·기업·기관)를 장기간에 걸쳐 은밀하게 침투·정찰·확산하는 지능형 지속 공격
- 스피어 피싱·제로데이 취약점·악성코드를 복합적으로 활용
- 단계: 정찰(Reconnaissance) → 침투(Initial Compromise) → 거점 확보(Foothold) → 권한 상승(Privilege Escalation) → 내부 정찰(Internal Recon) → 횡적 이동(Lateral Movement) → 목표 달성(Exfiltration)
```
① DDoS (Distributed Denial of Service)
② APT (Advanced Persistent Threat)
③ Drive-by Download
④ Watering Hole

### 문제 6 [출처: 2023년 1회 정보처리기사 필기 — 유사 기출]
SSL/TLS 프로토콜에 대한 설명으로 옳지 않은 것은?
① 전송 계층(L4)과 응용 계층(L7) 사이에서 동작하며 응용 계층 데이터를 암호화한다.
② Handshake 단계에서 서버 인증서를 통한 서버 인증과 세션키 합의가 이루어진다.
③ POODLE 공격은 TLS 1.3의 키 교환 취약점을 이용한 공격이다.
④ HTTPS는 HTTP 위에 SSL/TLS를 적용한 프로토콜이다.

### 문제 7 [출처: 2021년 1회 정보처리기사 필기 — 유사 기출]
방화벽(Firewall) 방식에 대한 설명으로 옳지 않은 것은?
① 패킷 필터링(Packet Filtering) 방화벽은 IP 주소, 포트 번호, 프로토콜 정보를 기반으로 필터링한다.
② 상태 기반(Stateful Inspection) 방화벽은 연결 상태 테이블을 유지하여 세션 단위로 패킷을 검사한다.
③ 응용 게이트웨이(Application Gateway, Proxy) 방화벽은 응용 계층에서 동작하며 사용자 인증과 콘텐츠 검사가 가능하다.
④ 패킷 필터링 방화벽은 응용 계층 페이로드를 분석하여 바이러스·악성코드를 탐지·차단할 수 있다.

### 문제 8 [출처: 2023년 2회 정보처리기사 필기 — 유사 기출]
무선 LAN 보안 프로토콜에 대한 설명으로 옳지 않은 것은?
① WEP(Wired Equivalent Privacy)는 RC4 스트림 암호와 짧은 IV(24비트)를 사용해 보안 취약점이 있다.
② WPA는 TKIP(Temporal Key Integrity Protocol)를 도입해 WEP의 키 재사용 문제를 보완하였다.
③ WPA2는 CCMP(AES 기반)를 표준으로 채택하여 WPA보다 강화된 보안을 제공한다.
④ WPA3는 RC4 알고리즘을 표준으로 사용하며, WPA2보다 키 길이가 짧아 성능이 우수하다.

## 🔍 오답 역추적 개념 발췌

오늘 문제 풀이 시 헷갈릴 가능성이 높은 키워드(설명 없음, 정답표·해설 참조).
- Smurf — ICMP Echo Request + 브로드캐스트 + IP Spoofing → 증폭 공격
- Fraggle — Smurf의 UDP 버전 (UDP Echo 사용)
- DRDoS — 분산 반사 DoS, 증폭률 큰 프로토콜(DNS/NTP/Memcached) 악용
- SYN Flooding — 3-way handshake 백로그 큐 고갈
- Land Attack — 출발지 IP·포트 = 목적지 IP·포트로 위조 (자기 자신과 통신)
- Teardrop — IP 단편화(Fragment) Offset 조작으로 재조립 오류 유발
- Ping of Death — 비정상적으로 큰 ICMP 패킷(65535B 초과)
- Slowloris — HTTP 헤더를 천천히 보내 연결 유지, 워커 고갈 (L7 DDoS)
- IPSec — 네트워크 계층(L3) 보안 프로토콜
- AH(Authentication Header) — 무결성·인증만 (기밀성 X)
- ESP(Encapsulating Security Payload) — 기밀성+무결성+인증
- IKE(Internet Key Exchange) — SA(Security Association) 협상·키 교환
- IPSec 모드: 전송(Transport) / 터널(Tunnel)
- IDS — 탐지·알림 (Passive, Out-of-band, 미러링)
- IPS — 탐지·차단 (Active, In-line)
- 오용 탐지(Misuse/Signature) — 알려진 공격 패턴 매칭, 오탐 적음, 신종 공격 못 잡음
- 이상 탐지(Anomaly) — 정상 행위 모델 학습, 신종 공격 탐지 가능, 오탐 많음
- HIDS(Host IDS) / NIDS(Network IDS)
- 방화벽 세대: 1세대 패킷필터(L3/4) → 2세대 상태기반(Stateful, 세션) → 3세대 응용게이트웨이(L7 Proxy) → 차세대 방화벽(NGFW, DPI+IPS+앱식별)
- DPI(Deep Packet Inspection) — 페이로드까지 검사
- APT(Advanced Persistent Threat) — 지능형 지속 공격, 장기 잠복
- 스피어 피싱 — 특정 표적 맞춤 피싱
- Watering Hole — 표적이 자주 방문하는 사이트 감염
- Drive-by Download — 웹 방문만으로 악성코드 다운로드
- SSL/TLS 동작 계층 — L4와 L7 사이 (세션 계층 근처)
- TLS 핸드셰이크: ClientHello → ServerHello + 인증서 → 키 교환 → Finished
- POODLE — SSL 3.0 패딩 오라클 공격 (TLS를 SSL 3.0으로 다운그레이드)
- BEAST — SSL 3.0/TLS 1.0 CBC 모드 IV 취약점
- Heartbleed — OpenSSL Heartbeat 확장 버퍼 오버리드 (CVE-2014-0160)
- CRIME / BREACH — TLS 압축 사이드채널
- WEP — RC4 + 24비트 IV (취약)
- WPA — TKIP + RC4 (과도기)
- WPA2 — CCMP + AES (현재 표준)
- WPA3 — SAE(Simultaneous Authentication of Equals), 192비트 보안, PMF 강제
- VPN 프로토콜 계층: PPTP(L2) / L2TP(L2) / IPSec(L3) / SSL VPN(L4~L7)
- L2TP + IPSec — 보안 강화 조합 (L2TP 자체는 암호화 없음)

## 💡 상세 해설 (자기 설명과 대조)

### 문제 1
정답: ②
- 제공 해설: Smurf 공격은 ICMP Echo Request 패킷의 출발지 IP를 공격 대상으로 위조(Spoofing)하고 브로드캐스트 주소로 송신해, 네트워크 내 모든 호스트가 공격 대상에게 ICMP Echo Reply를 보내도록 유도하는 증폭형 DoS. 대응: 라우터에서 Directed Broadcast 차단, ICMP 응답 제한.
- 오답 이유:
  - ①: SYN Flooding — TCP 3-way handshake의 SYN/ACK 응답에 ACK를 안 보내 백로그 큐 고갈. ICMP 아님.
  - ③: Land Attack — 출발지·목적지 IP/포트를 동일하게 위조해 자기 루프 발생.
  - ④: Teardrop — IP 단편화(Fragment) Offset 값을 겹치게 조작해 재조립 오류 유발.
- 함정: "ICMP + 브로드캐스트 + 출발지 위조" 3박자 = Smurf. UDP Echo로 동일 패턴이면 Fraggle.

### 문제 2
정답: ②
- 제공 해설: SYN Flooding은 TCP 3-way handshake(SYN → SYN/ACK → ACK)의 두 번째 단계에서 ACK를 응답하지 않아 서버의 백로그 큐(half-open 연결 큐)를 가득 채워 정상 사용자의 연결 요청을 거부시키는 공격. 대응: SYN Cookie, 백로그 큐 확장, 첫 RTT 타임아웃 단축.
- 오답 이유:
  - ①: Smurf — ICMP 브로드캐스트 증폭 공격.
  - ③: Ping of Death — 65535바이트 초과 ICMP 패킷으로 재조립 오버플로 유발.
  - ④: Tear Drop — IP Fragment Offset 조작.
- 함정: 3-way handshake / 백로그 큐 / half-open 키워드 = SYN Flood.

### 문제 3
정답: ④
- 제공 해설: IKE(Internet Key Exchange)는 SA(Security Association) 협상과 키 교환을 담당하는 키 관리 프로토콜이지, 데이터 패킷 자체를 암호화하지 않는다. 실제 패킷 암호화는 ESP가 수행. IKE는 ISAKMP + Oakley + SKEME 기반.
- 오답 이유:
  - ①: IPSec은 L3(네트워크 계층)에서 동작 — 옳음.
  - ②: AH는 인증·무결성만 제공, 기밀성 X — 옳음.
  - ③: ESP는 기밀성+무결성+인증 모두 가능 — 옳음.
- 함정: AH(무결성), ESP(기밀성+무결성), IKE(키 교환) 역할 분리 필수. "IKE가 암호화" = 거짓.

### 문제 4
정답: ②
- 제공 해설: IDS(Intrusion Detection System)는 트래픽 미러링으로 패킷을 복사받아 분석하고 탐지 시 알림만 수행하는 Passive 시스템. IPS(Intrusion Prevention System)는 트래픽 경로 위(in-line)에 설치되어 탐지된 공격을 실시간으로 차단하는 Active 시스템.
- 오답 이유:
  - ①: 정반대. IPS가 인라인, IDS가 미러링.
  - ③: IDS·IPS 모두 시그니처 기반(오용 탐지)과 이상 탐지(Anomaly) 둘 다 사용 가능.
  - ④: IDS·IPS 모두 호스트 기반(HIDS/HIPS), 네트워크 기반(NIDS/NIPS) 모두 존재.
- 함정: IDS=탐지, IPS=차단. IPS는 지연 영향이 있어 고가용성·성능 설계 필요.

### 문제 5
정답: ②
- 제공 해설: APT(Advanced Persistent Threat)는 특정 표적을 장기간 은밀하게 침투·정찰·확산하는 지능형 지속 공격. 스피어 피싱·제로데이·맞춤 악성코드를 결합. Kill Chain: 정찰 → 무기화 → 전달 → 익스플로잇 → 설치 → 명령·통제(C2) → 목표 달성.
- 오답 이유:
  - ①: DDoS — 다수 좀비로 대상 서비스 마비. 단기·트래픽 기반, 정보 탈취 목적 아님.
  - ③: Drive-by Download — 사용자가 악성 사이트 방문만으로 악성코드 자동 설치되는 단발 공격.
  - ④: Watering Hole — 표적이 자주 가는 사이트를 감염시켜 표적이 방문하길 기다리는 공격 (APT의 침투 수단 중 하나일 수는 있음).
- 함정: "장기간 + 은밀 + 특정 표적 + 복합 기법" = APT. Watering Hole/Drive-by는 단일 기법으로 APT의 부분집합.

### 문제 6
정답: ③
- 제공 해설: POODLE(Padding Oracle On Downgraded Legacy Encryption)은 SSL 3.0의 CBC 패딩 오라클 취약점을 악용하는 공격으로, TLS 연결을 SSL 3.0으로 다운그레이드한 뒤 암호문을 해독한다. TLS 1.3과 무관하며 TLS 1.3은 SSL/TLS 다운그레이드 공격을 차단하도록 설계됨.
- 오답 이유:
  - ①: SSL/TLS는 L4(전송)와 L7(응용) 사이 동작 — 옳음.
  - ②: 핸드셰이크에서 인증서 검증·세션키 합의 — 옳음.
  - ④: HTTPS = HTTP + SSL/TLS — 옳음.
- 함정: POODLE = SSL 3.0 다운그레이드. BEAST = SSL 3.0/TLS 1.0 CBC IV. Heartbleed = OpenSSL Heartbeat 버퍼 오버리드. CVE 구별.

### 문제 7
정답: ④
- 제공 해설: 패킷 필터링 방화벽(1세대)은 L3/L4 헤더(IP, 포트, 프로토콜)만 검사하며 페이로드(데이터부)는 검사하지 않는다. 페이로드까지 검사해 악성코드를 탐지·차단하려면 DPI(Deep Packet Inspection)나 응용 게이트웨이, NGFW(차세대 방화벽)가 필요.
- 오답 이유:
  - ①: 패킷 필터링이 IP·포트·프로토콜 기반 — 옳음.
  - ②: Stateful Inspection이 세션 테이블 유지 — 옳음.
  - ③: 응용 게이트웨이(Proxy)가 L7 동작·사용자 인증 가능 — 옳음.
- 함정: 1세대 패킷필터는 "헤더만" 본다. "페이로드·바이러스 검사" = NGFW/DPI/응용 게이트웨이.

### 문제 8
정답: ④
- 제공 해설: WPA3는 RC4가 아닌 AES 기반 GCMP/CCMP를 사용하며, 192비트 보안 모드(Enterprise)와 SAE(Simultaneous Authentication of Equals, 일명 Dragonfly) 키 합의로 오프라인 사전 공격을 차단한다. PMF(Protected Management Frame) 강제. RC4 사용은 WEP/WPA 시절 유물.
- 오답 이유:
  - ①: WEP는 RC4 + 24비트 IV로 IV 재사용 공격에 취약 — 옳음.
  - ②: WPA는 TKIP 도입으로 WEP의 키 재사용 문제 보완 — 옳음.
  - ③: WPA2는 CCMP(AES-128 기반)를 표준으로 채택 — 옳음.
- 함정: WPA3 = AES + SAE + PMF. "RC4 사용", "키 짧음" = 거짓.

### 핵심 개념
- **방화벽 4세대 한눈 정리:**
  - 1세대 패킷필터(L3/L4): IP·포트·프로토콜만 검사. 빠름, 응용 공격 무력
  - 2세대 Stateful: 세션 테이블 유지, 동적 응답 패킷 허용
  - 3세대 응용 게이트웨이(Proxy): L7 동작, 사용자 인증·콘텐츠 검사. 느림
  - NGFW: DPI + IPS + 앱 식별 + 사용자 식별 + URL 필터링 통합
- **IDS vs IPS 변별:**
  - IDS: 미러링·Passive·탐지+알림 → 지연 영향 없음, 차단 불가
  - IPS: 인라인·Active·탐지+차단 → 지연 영향 있음, 오탐 시 정상 트래픽 차단 위험
  - 탐지 방식(오용/이상)은 IDS/IPS 모두 동일하게 사용 가능
- **IPSec 구성:**
  - AH: 무결성+인증 (기밀성 X), Replay 방어
  - ESP: 기밀성+무결성+인증 (AH 기능 포함)
  - IKE: SA 협상·키 교환 (IKEv1/IKEv2), Phase 1/Phase 2
  - 모드: Transport(엔드포인트), Tunnel(게이트웨이, VPN)
- **SSL/TLS 핸드셰이크 단계:**
  1. ClientHello (지원 암호 스위트·랜덤)
  2. ServerHello + Certificate + ServerKeyExchange + ServerHelloDone
  3. ClientKeyExchange + ChangeCipherSpec + Finished
  4. Server ChangeCipherSpec + Finished
  5. 이후 Application Data 암호화 전송
- **TLS 버전 보안 차이:**
  - SSL 2.0/3.0: 폐기 (POODLE)
  - TLS 1.0/1.1: 폐기 (BEAST, CBC 취약)
  - TLS 1.2: 현재 광범위 사용
  - TLS 1.3: 0-RTT 지원, RSA 키 교환 폐기, 핸드셰이크 단축
- **DoS/DDoS 분류:**
  - 대역폭 소진: UDP Flood, ICMP Flood, Smurf, Fraggle, DRDoS
  - 자원 고갈: SYN Flood, Slowloris, Connection exhaustion
  - 비정상 패킷: Land, Teardrop, Ping of Death
- **APT 공격 단계(Cyber Kill Chain):**
  1. Reconnaissance(정찰)
  2. Weaponization(무기화)
  3. Delivery(전달, 스피어 피싱 등)
  4. Exploitation(취약점 익스플로잇)
  5. Installation(악성코드 설치)
  6. Command & Control(C2)
  7. Actions on Objectives(목표 달성, 데이터 탈취)
- **무선 보안 변천:**
  - WEP(1997) → WPA(2003) → WPA2(2004) → WPA3(2018)
  - 알고리즘: RC4 → TKIP+RC4 → CCMP+AES → GCMP+AES+SAE
  - 키 길이: 64/128 → 128 → 128/256 → 128/192
- **VPN 프로토콜 동작 계층:**
  - L2: PPTP, L2TP, L2F
  - L3: IPSec (AH/ESP)
  - L4~L7: SSL VPN, OpenVPN, WireGuard(UDP 기반)
  - 클라이언트리스 VPN = SSL VPN (브라우저로 접속)
