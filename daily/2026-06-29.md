## 🎯 오늘의 기출문제 — 2021년 정처기 필기 20문제 (5과목 골골루 · 회독 4/4)

### 문제 1 [유사 기출: 2021년 1회 1과목 — 미들웨어(Middleware) 분류 정형 패턴]
다음 중 **미들웨어(Middleware)** 의 종류에 대한 설명으로 옳지 **않은** 것은?

① **MOM(Message-Oriented Middleware)** — 비동기 **메시지 큐(Message Queue)** 기반으로 분산 시스템 간 메시지를 송수신(Kafka·RabbitMQ·ActiveMQ).
② **RPC(Remote Procedure Call)** — 원격지의 프로시저(함수)를 **로컬 함수 호출처럼** 사용하게 해주는 동기식 미들웨어.
③ **ORB(Object Request Broker)** — CORBA 표준 기반의 **분산 객체** 간 통신을 중개하는 미들웨어.
④ **DBMS(Database Management System)** — 미들웨어의 일종으로 **응용 프로그램과 사용자 사이**에 위치해 화면 표시(GUI) 기능을 담당한다.

### 문제 2 [유사 기출: 2021년 2회 1과목 — UI 설계 원칙 정형 패턴]
다음 중 **사용자 인터페이스(UI)** 의 **4대 설계 원칙**에 해당하지 **않는** 것은?

① **직관성(Intuitiveness)** — 사용자가 별도 학습 없이도 쉽게 이해하고 사용할 수 있어야 한다.
② **유효성(Efficiency)** — 사용자의 목적을 **정확하고 빠르게** 달성할 수 있어야 한다.
③ **학습성(Learnability)** — 초보자도 쉽게 배우고 익숙해질 수 있어야 한다.
④ **이식성(Portability)** — UI가 다른 운영체제·플랫폼으로 **쉽게 이전** 가능해야 한다.

### 문제 3 [유사 기출: 2021년 3회 1과목 — 코드 설계 종류 정형 패턴]
다음 중 **코드(Code) 설계 방식**의 설명으로 옳지 **않은** 것은?

① **순차 코드(Sequence Code)** — 발생 순서·크기 순으로 일련번호 부여(1, 2, 3, …). 간단하나 분류·확장에 약함.
② **블록 코드(Block Code)** — 자릿수를 블록 단위로 나누어 구간별 의미를 부여(예: 100~199 = 부서A).
③ **십진 코드(Decimal Code)** — 0~9의 10진수를 계층적으로 분류해 부여(도서 분류법 등).
④ **연상 코드(Mnemonic Code)** — 코드 자릿수와 무관하게 **무작위 난수**를 생성해 보안용으로 사용(연상 코드의 정의와 정반대).

### 문제 4 [유사 기출: 2021년 1회 1과목 — CBD(Component Based Development) 정형 패턴]
**CBD(Component Based Development)** 방법론에 대한 설명으로 가장 옳은 것은?

① 폭포수(Waterfall) 모델의 일종으로 **요구분석·설계·구현·테스트를 순차적으로** 진행하며 컴포넌트는 부수적 개념이다.
② 재사용 가능한 **컴포넌트(Component)** 단위로 시스템을 구축하여 **재사용성·생산성·유지보수성**을 향상시키는 방법론. 컴포넌트는 독립적으로 배포·교체 가능.
③ XP의 일종으로 페어 프로그래밍과 TDD를 핵심 가치로 한다.
④ 객체지향과 무관한 **절차지향** 방법론으로, 함수 라이브러리를 재사용하는 데 초점을 둔다.

### 문제 5 [유사 기출: 2021년 2회 2과목 — 큐(Queue) 자료구조 정형 패턴]
**큐(Queue)** 자료구조에 대한 설명으로 옳지 **않은** 것은?

① 큐는 **FIFO(First In First Out)** 구조로, 먼저 입력된 데이터가 먼저 출력된다.
② **삽입(enqueue)** 은 **rear(후단)** 에서, **삭제(dequeue)** 는 **front(전단)** 에서 일어난다.
③ **원형 큐(Circular Queue)** 는 배열의 끝과 시작을 연결해 **선형 큐의 메모리 낭비 문제**를 해결한다.
④ **덱(Deque, Double Ended Queue)** 은 **한쪽 끝에서만 삽입·삭제**가 일어나는 큐의 변형으로 스택과 동일한 동작을 한다.

### 문제 6 [유사 기출: 2021년 1회·2회 2과목 — 동치 분할(Equivalence Partitioning) 정형 패턴]
**블랙박스 테스트(Black-Box Test)** 기법 중 **동치 분할(Equivalence Partitioning)** 에 대한 설명으로 가장 옳은 것은?

① 입력 자료의 **경계값(Boundary Value)** 부근에서 오류가 자주 발생한다는 점에 근거해 경계값을 중심으로 테스트 케이스를 설계(경계값 분석의 설명).
② 입력 도메인을 **동등한 클래스(Equivalence Class)** 로 분할하고, 각 클래스에서 **대표값 하나씩**을 선택해 테스트 케이스로 활용(중복 테스트 제거).
③ 프로그램 내부 **제어 흐름·조건문**을 모두 검사하는 테스트(화이트박스 기법의 일종).
④ 입력값을 무작위로 생성해 시스템이 다운되는지를 검증하는 **퍼징(Fuzzing)** 기법.

### 문제 7 [유사 기출: 2021년 3회 2과목 — Git Flow 브랜치 전략 정형 패턴]
**Git Flow** 브랜치 전략에서 각 브랜치의 역할에 대한 설명으로 옳지 **않은** 것은?

① **master(main) 브랜치** — 운영 환경에 배포되는 안정된 릴리즈 코드를 보관. 태그(Tag)로 버전 관리.
② **develop 브랜치** — 다음 릴리즈를 위한 **통합 개발** 브랜치. feature 브랜치들이 머지되는 기준 브랜치.
③ **feature 브랜치** — 새로운 기능 개발용. develop에서 분기·develop으로 머지.
④ **hotfix 브랜치** — develop 브랜치에서 분기되며, 다음 정기 릴리즈 일정에 맞춰 통합되는 **장기 기능 개발** 용도 브랜치이다.

### 문제 8 [유사 기출: 2021년 1회 2과목 — 인터페이스 데이터 표준(JSON/XML) 정형 패턴]
다음 중 **인터페이스 구현**에 사용되는 데이터 형식·통신 방식 설명으로 옳지 **않은** 것은?

① **JSON(JavaScript Object Notation)** — 키-값(key-value) 쌍의 경량 데이터 교환 형식으로 가독성이 높고 XML보다 파싱 속도가 빠르다.
② **XML(eXtensible Markup Language)** — 사용자 정의 태그 기반 마크업 언어로, 데이터 구조 표현과 스키마(XSD) 검증이 가능.
③ **REST(REpresentational State Transfer)** — HTTP 메서드(GET·POST·PUT·DELETE)와 URI 자원 기반의 아키텍처 스타일.
④ **SOAP(Simple Object Access Protocol)** — **JSON 전용**의 경량 RPC 프로토콜로 XML을 지원하지 않으며, WS-Security 등 표준 규약이 없다.

### 문제 9 [유사 기출: 2021년 2회 3과목 — E-R 다이어그램 표기 정형 패턴]
**E-R(Entity-Relationship) 다이어그램**의 표기 기호로 옳지 **않은** 것은?

① **개체(Entity)** = **사각형(□)** — 현실 세계의 사물·개념.
② **관계(Relationship)** = **마름모(◇)** — 개체 간 연관 관계.
③ **속성(Attribute)** = **타원(○)** — 개체나 관계의 특성. 기본키 속성은 밑줄.
④ **연결선(Link)** = **이중선(=)** — 약한 개체(Weak Entity)와 강한 개체 간 식별 관계는 **단일선(─)** 으로 표기한다.

### 문제 10 [유사 기출: 2021년 1회·3회 3과목 — BCNF(Boyce-Codd Normal Form) 정형 패턴]
다음 중 **BCNF(Boyce-Codd Normal Form)** 에 대한 설명으로 가장 옳은 것은?

① BCNF는 1NF의 부분집합으로, **원자값(Atomic Value)** 만 만족하면 BCNF가 된다.
② 3NF를 만족하면서, 모든 **결정자(Determinant)** 가 **후보키(Candidate Key)** 인 정규형. 3NF보다 강한 조건.
③ **다치 종속(Multi-Valued Dependency)** 을 제거한 정규형(4NF의 정의).
④ **조인 종속(Join Dependency)** 을 제거한 정규형(5NF의 정의).

### 문제 11 [유사 기출: 2021년 2회 3과목 — 병행제어 2단계 잠금규약(2PL) 정형 패턴]
**2단계 잠금규약(2-Phase Locking, 2PL)** 에 대한 설명으로 옳지 **않은** 것은?

① **확장 단계(Growing Phase)** — 트랜잭션이 **새로운 락(Lock)을 획득할 수만** 있고 해제는 불가.
② **축소 단계(Shrinking Phase)** — 트랜잭션이 **획득한 락을 해제할 수만** 있고 새 락 획득 불가.
③ 2PL은 **직렬가능성(Serializability)** 을 보장하지만, **교착상태(Deadlock)** 가 발생할 수 있다.
④ 2PL은 모든 트랜잭션의 **교착상태를 원천적으로 방지**하고 **기아(Starvation)도 발생하지 않음**을 수학적으로 증명한 규약이다.

### 문제 12 [유사 기출: 2021년 1회 3과목 — 데이터 모델링 3단계 정형 패턴]
**데이터 모델링 3단계**의 순서와 산출물 매칭으로 가장 옳은 것은?

① **개념적(Conceptual) → 물리적(Physical) → 논리적(Logical)** 순서로 진행되며, 산출물은 ERD·테이블 스키마·인덱스 정의서 순.
② **개념적 모델링** = 업무 요구사항을 ERD로 표현 / **논리적 모델링** = DBMS 독립적 관계형 스키마(테이블·관계) 설계 / **물리적 모델링** = 특정 DBMS에 맞춘 인덱스·테이블스페이스·파티션 등 성능 구현 설계.
③ 3단계는 모두 **특정 DBMS(예: Oracle)에 종속**되므로 DBMS 변경 시 처음부터 다시 수행해야 한다.
④ 물리적 모델링 단계에서는 **정규화**를 수행하고, 논리적 모델링에서는 **반정규화**를 수행한다(역순).

### 문제 13 [유사 기출: 2021년 2회 4과목 — C 문자열 함수 strncpy 정형 패턴]
다음 C 코드의 출력 결과로 옳은 것은?
```c
#include <stdio.h>
#include <string.h>
int main(void) {
    char dest[10] = "AAAAAAAAA";
    char src[]  = "BCD";
    strncpy(dest, src, 3);
    printf("%s\n", dest);
    return 0;
}
```

① **BCD** (널 종료문자 자동 추가, 4번째 이후 자동 절단)
② **BCDAAAAAA** (strncpy는 정확히 n바이트만 복사하므로 4번째 위치에 널이 자동 삽입되지 **않음** — dest[0..2]만 'B','C','D'로 덮어쓰고 dest[3]부터는 기존 'A' 유지)
③ **AAA** (역방향 복사)
④ 컴파일 오류 (strncpy 사용 불가)

### 문제 14 [유사 기출: 2021년 3회 4과목 — Java 컬렉션 프레임워크 정형 패턴]
**Java 컬렉션 프레임워크(Collection Framework)** 의 인터페이스 분류로 옳지 **않은** 것은?

① **List** — **순서 있음·중복 허용**. 구현체: ArrayList·LinkedList·Vector.
② **Set** — **순서 없음·중복 불가**. 구현체: HashSet·LinkedHashSet·TreeSet.
③ **Map** — **키-값 쌍**으로 저장하며 **키는 중복 불가, 값은 중복 허용**. 구현체: HashMap·LinkedHashMap·TreeMap.
④ **Queue** 인터페이스는 Java 컬렉션 프레임워크에 포함되지 **않으며**, java.util 패키지에도 존재하지 않는다.

### 문제 16 [유사 기출: 2021년 3회 4과목 — TCP/IP 4계층 vs OSI 7계층 매핑 정형 패턴]
**TCP/IP 4계층 모델**과 **OSI 7계층 모델**의 계층 매핑으로 옳지 **않은** 것은?

① TCP/IP **응용 계층(Application)** ↔ OSI **응용·표현·세션 계층**(7+6+5층).
② TCP/IP **전송 계층(Transport)** ↔ OSI **전송 계층**(4층). TCP·UDP 동작.
③ TCP/IP **인터넷 계층(Internet)** ↔ OSI **네트워크 계층**(3층). IP·ICMP·ARP 동작.
④ TCP/IP **네트워크 액세스 계층(Network Access)** ↔ OSI **세션 계층(5층)** 만 일대일 매핑되며 데이터링크·물리 계층과는 무관하다.

### 문제 17 [유사 기출: 2021년 1회 5과목 — COCOMO 모델 정형 패턴]
**COCOMO(COnstructive COst MOdel)** 모델 3종에 대한 설명으로 옳지 **않은** 것은?

① **Organic(조직형)** — 5만 라인(50 KLOC) **이하**의 비교적 단순한 소형 프로젝트(중소 규모 비즈니스·과학 응용).
② **Semi-detached(반분리형)** — 30만 라인(300 KLOC) **이하**의 중간 규모 프로젝트(트랜잭션 시스템·DBMS).
③ **Embedded(내장형)** — 30만 라인 **초과**의 대규모·고난도 프로젝트(실시간 처리·미사일 제어·OS·항공 우주).
④ COCOMO는 **기능 점수(FP, Function Point)** 를 입력 변수로 사용하는 비용 산정 모델로, LOC 기반은 사용하지 않는다.

### 문제 18 [유사 기출: 2021년 2회 5과목 — DDoS 공격 유형 정형 패턴]
다음 중 **DDoS(Distributed Denial of Service) 공격**에 해당하지 **않는** 것은?

① **SYN Flood** — TCP 3-way handshake의 SYN 패킷을 대량 전송해 서버의 **반-연결(half-open) 큐**를 가득 채우는 공격.
② **UDP Flood** — 임의 포트로 대량의 UDP 패킷을 전송해 대상 시스템의 자원을 고갈.
③ **Smurf Attack** — 공격자가 출발지 IP를 피해자로 위조한 ICMP Echo 요청을 브로드캐스트로 전송, 응답이 피해자에게 폭주.
④ **SQL Injection** — 웹 입력값 검증 부족을 악용해 DB에서 비인가 데이터를 조회·조작하는 공격으로, **DDoS와는 공격 목적·방식이 다른** 데이터 변조·유출형 웹 공격.

### 문제 19 [유사 기출: 2021년 3회 5과목 — 클라우드 서비스 모델 정형 패턴]
**클라우드 컴퓨팅 서비스 모델**에 대한 설명으로 옳지 **않은** 것은?

① **IaaS(Infrastructure as a Service)** — 가상 서버·스토리지·네트워크 등 **인프라**를 서비스로 제공(AWS EC2·S3, GCP Compute Engine).
② **PaaS(Platform as a Service)** — 애플리케이션 **개발·실행 플랫폼**(런타임·DB·미들웨어)을 서비스로 제공(Heroku·Google App Engine).
③ **SaaS(Software as a Service)** — 완성된 **소프트웨어**를 인터넷으로 제공(Gmail·Salesforce·Office 365).
④ **DaaS(Desktop as a Service)** 는 **IaaS·PaaS·SaaS 3대 모델 중 하나**로 분류되며, 클라우드 표준 3대 모델에 해당한다.

### 문제 20 [유사 기출: 2021년 1회·3회 5과목 — IDS/IPS/방화벽/WAF 비교 정형 패턴]
**보안 장비**에 대한 설명으로 옳지 **않은** 것은?

① **방화벽(Firewall)** — IP·포트·프로토콜 기반 정책(ACL)으로 트래픽을 **허용·차단**. L3·L4 동작이 일반적.
② **IDS(Intrusion Detection System)** — 침입을 **탐지·경보**만 수행하며 차단은 하지 않는다(수동적). 탐지 방식: **오용 탐지(Misuse·시그니처 기반)** + **이상 탐지(Anomaly·통계·임계값 기반)**.
③ **IPS(Intrusion Prevention System)** — 침입을 **탐지+차단**까지 수행(능동적). 인라인(in-line) 구성.
④ **WAF(Web Application Firewall)** 는 OSI **3계층(네트워크)** 에서 IP·포트만 검사하는 장비로, HTTP 페이로드·SQL 구문은 분석하지 못한다.

### 문제 8
**정답 ④** — **SOAP = XML 기반** RPC 프로토콜. WS-Security·WS-ReliableMessaging 등 표준 규약 존재. JSON 전용이 아님.
- **REST vs SOAP**: REST = 경량·HTTP·JSON 주로 / SOAP = 무거움·XML·표준 규약 풍부·트랜잭션/보안 강함(금융·엔터프라이즈).
- 함정: 최근 출제 트렌드는 **REST API + JSON**이 대세이나 SOAP의 XML·표준 지원도 시험 빈출.

### 문제 9
**정답 ④** — **연결선(Link)** = **단일선(─)**. **약한 개체(Weak Entity)** 와 강한 개체 간 **식별 관계는 이중선(=) 또는 이중 마름모(◇◇)** 로 표기.
- E-R 표기 정리: 개체(□)·약한 개체(이중 □)·관계(◇)·식별 관계(이중 ◇)·속성(○)·기본키 속성(밑줄 ○)·유도 속성(점선 ○)·다중값 속성(이중 ○)·연결(─).
- 함정: 약한 개체와 강한 개체의 표기를 교묘히 바꾸어 출제.

### 문제 10
**정답 ②** — **BCNF**: 3NF + **모든 결정자가 후보키**여야 함. 3NF보다 강한 정규형.
- 정규형 진행: **1NF(원자값) → 2NF(부분 함수종속 제거) → 3NF(이행 함수종속 제거) → BCNF(결정자=후보키) → 4NF(다치 종속 제거) → 5NF(조인 종속 제거)**.
- BCNF 예: (학생, 과목, 교수)에서 "교수→과목"이 성립할 때 교수가 후보키가 아니면 BCNF 위반.

### 문제 11
**정답 ④** — **2PL은 직렬가능성을 보장하지만 교착상태(Deadlock) 발생 가능**. 교착 방지를 위해 **타임스탬프(Timestamp) 기법·Wait-Die·Wound-Wait** 등 별도 기법 필요.
- 2PL 단계: **확장(Growing, 락 획득만)** → **축소(Shrinking, 락 해제만)** — 한 번이라도 해제하면 새 락 획득 불가.
- 변형: **엄격 2PL(Strict 2PL)** = 모든 X-락을 트랜잭션 종료까지 유지 → 캐스케이딩 롤백 방지.

### 문제 12
**정답 ②** — **데이터 모델링 3단계**: **개념적 → 논리적 → 물리적**.
- **개념적(Conceptual)** = 업무 요구 → ERD로 표현. DBMS 독립적·추상적.
- **논리적(Logical)** = 관계형 모델로 변환. 테이블·관계·정규화. DBMS 독립적.
- **물리적(Physical)** = 특정 DBMS(Oracle·MySQL 등)에 맞춰 인덱스·테이블스페이스·파티션·반정규화. DBMS 종속적.
- 함정: 정규화는 **논리 단계**, 반정규화는 **물리 단계**에서 성능 위해 수행(④ 역순).

### 문제 13
**정답 ②** — **strncpy(dest, src, n)** 은 **정확히 n바이트만 복사하며 널 종료문자를 자동 추가하지 않는다**(n바이트 안에 src의 널이 포함되어야 함). src="BCD"는 3바이트이므로 dest[0]='B', dest[1]='C', dest[2]='D'까지만 덮어쓰고 dest[3]부터는 기존 'A' 그대로 → **"BCDAAAAAA"** + 마지막 dest[9]는 원래 널 종료문자였으므로 "BCDAAAAAA"로 출력.
- **strcpy vs strncpy**: strcpy는 src의 널까지 자동 복사(버퍼 오버플로 위험), strncpy는 길이 제한(널 누락 위험) — **둘 다 위험, strncpy_s/strlcpy 권장**.

### 문제 14
**정답 ④** — Java **Queue 인터페이스는 java.util 패키지에 존재**. 구현체로 **LinkedList·PriorityQueue·ArrayDeque** 등. Deque 인터페이스도 별도 존재.
- 컬렉션 프레임워크 구조: **Collection**(Set·List·Queue) + **Map**(별도 계층). Map은 Collection 인터페이스 상속하지 않음(키-값 구조라 별도).
- 동기화: **HashMap → ConcurrentHashMap** / **ArrayList → Vector or Collections.synchronizedList()**.

### 문제 15
**정답 ②** — `[x*x for x in range(1,6) if x%2==0]` = range(1,6)=[1,2,3,4,5] 중 짝수 → 2, 4 → 각각 제곱 → **[4, 16]**.
- 리스트 컴프리헨션 일반형: `[<식> for <변수> in <iterable> if <조건>]`. 조건 없으면 모든 요소.
- 중첩: `[[i*j for j in range(3)] for i in range(3)]` → 2차원 리스트 생성. 함정: range(1,6)은 6 미포함(1~5).

### 문제 16
**정답 ④** — TCP/IP **네트워크 액세스 계층 ↔ OSI 데이터링크(2층) + 물리(1층)** 두 계층을 합친 매핑. 세션 계층(5층)은 응용 계층에 매핑.
- **TCP/IP 4 vs OSI 7 정확 매핑**: 응용(TCP/IP) = 응용+표현+세션(OSI 5-7) / 전송 = 전송(4) / 인터넷 = 네트워크(3) / 네트워크 액세스 = 데이터링크+물리(2+1).
- 5계층 모델(하이브리드): 응용·전송·네트워크·데이터링크·물리 — 표현·세션을 응용에 합쳐 5층으로 단순화.

### 문제 17
**정답 ④** — **COCOMO는 LOC(Line of Code) 기반** 비용 산정 모델. FP 기반은 ESTIMACS·SLIM·FP-Analysis.
- COCOMO 3종(Boehm, 1981): **Organic(≤50KLOC, 단순)·Semi-detached(≤300KLOC, 중간)·Embedded(>300KLOC, 고난도 실시간)**.
- 노력(MM) = a × (KLOC)^b × EAF(노력 조정 인수). a·b는 모드별 상수.
- 함정: COCOMO II는 더 정교화되어 FP·객체 포인트도 입력 가능하지만, **고전 COCOMO = LOC 기반**.

### 문제 18
**정답 ④** — **SQL Injection은 데이터 유출·변조형 웹 공격**이지 **DoS(서비스 거부) 공격이 아님**. DDoS는 다수의 좀비PC가 동시에 서비스 자원을 고갈시키는 공격.
- **DDoS 4대 유형**: SYN Flood·UDP Flood·ICMP Flood·Smurf(ICMP 증폭).
- 응용 계층 DoS: **Slowloris**(HTTP 헤더 천천히 전송) / **HTTP Get Flood**(대량 GET 요청).
- 방어: **CDN·DDoS 차단 장비·Rate Limiting·SYN Cookie·Anycast 분산**.

### 문제 19
**정답 ④** — **클라우드 표준 3대 모델 = IaaS·PaaS·SaaS**. **DaaS(Desktop as a Service)·BaaS(Backend)·FaaS(Function, 서버리스)** 등은 **확장/세부 모델**로 3대 모델에 포함되지 않음.
- 책임 분담(공유 책임 모델): IaaS = OS·런타임·앱은 사용자 책임 / PaaS = 앱·데이터만 사용자 책임 / SaaS = 데이터만 사용자.
- 함정: FaaS(AWS Lambda)는 PaaS의 발전형으로 분류되지만 별도 표기 시 **서버리스(Serverless)** 라 함.

### 문제 20
**정답 ④** — **WAF(Web Application Firewall)** 는 **OSI 7계층(응용)** 에서 동작하며 **HTTP 페이로드·SQL·XSS·CSRF 등 웹 공격을 탐지·차단**. SQL Injection·XSS 등 OWASP Top 10 방어가 주 목적.
- 보안 장비 비교:
  - **방화벽(Firewall)** = L3/L4, IP·포트.
  - **IDS** = 탐지만(수동).
  - **IPS** = 탐지+차단(능동).
  - **WAF** = L7, HTTP/HTTPS 웹 트래픽.
  - **UTM(Unified Threat Management)** = 방화벽·IDS/IPS·VPN·안티바이러스 통합.
- 배치: WAF는 일반적으로 **웹 서버 앞단**에 인라인 또는 프록시 모드로 구성.

## 📊 자주 틀리는 함정 5가지 (2021년 회독 4/4 최종 정리)

1. **CBD ≠ XP/Waterfall** — CBD는 "컴포넌트 재사용 중심" 방법론. 다른 방법론과 혼동 주의.
2. **BCNF의 핵심 = "결정자가 후보키"** — 단순히 "3NF + α"가 아닌 **결정자(Determinant) 검사**가 핵심. 모든 결정자가 후보키가 아니면 BCNF 위반.
3. **2PL은 직렬가능성 보장 ≠ 교착상태 방지** — 두 개념 분리. 2PL의 한계는 교착상태 발생 가능성 → 별도 교착 처리 필요.
4. **strncpy는 널 종료문자 자동 추가 X** — n바이트만 정확히 복사. src가 n보다 짧으면 나머지는 널 패딩, src가 n 이상이면 dest에 널이 없을 수 있음(매우 위험).
5. **WAF = L7 응용계층** — 방화벽(L3/L4)과 혼동 빈출. WAF는 HTTP 페이로드를 파싱해 SQL·XSS·CSRF 탐지.
