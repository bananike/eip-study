```
사용자가 열지 못하게 하고, 금전을 요구하는 악성 프로그램
```

① Smishing  
② Ransomware  
③ Phishing  
④ XSS  

### 문제 2 [출처: 2021년 1회 필기]
시스템 내의 정보는 오직 인가된 사용자만 수정할 수 있는 보안 요소는?

① 기밀성(Confidentiality)  
② 부인 방지(Non-repudiation)  
③ 가용성(Availability)  
④ 무결성(Integrity)  

### 문제 3 [출처: 2023년 2회 필기 / 유사 기출]
DDoS 공격과 관련한 설명으로 틀린 것은?

① 자신의 IP 주소를 속이는 spoofing 기법을 사용한다.  
② Trinoo, TFN과 같은 도구가 사용된다.  
③ 일반적으로 공격은 DDoS 마스터(Master) - 좀비(Zombie) - 핸들러(Handler) - 에이전트(Agent) 순으로 이루어진다.  
④ 분산된 공격 지점에서 다수의 패킷을 발생시켜 서비스 가용성을 떨어뜨린다.  

### 문제 4 [출처: 2022년 2회 필기]
SQL Injection 공격에 대한 설명으로 옳지 않은 것은?

① 사용자의 입력값을 검증하지 않는 경우에 발생한다.  
② 데이터베이스의 데이터를 조작하거나 인증을 우회할 수 있다.  
③ Prepared Statement(매개변수화 쿼리)를 사용하면 방어할 수 있다.  
④ 클라이언트 측 스크립트를 삽입하여 다른 사용자의 세션을 탈취하는 공격이다.  

### 문제 5 [출처: 2024년 1회 필기 / 유사 기출]
다음에서 설명하는 접근 통제 모델은?

```
주체와 객체에 보안 등급(Security Label)을 부여하고,
시스템이 강제적으로 등급을 비교해 접근 여부를 결정한다.
대표적으로 Bell-LaPadula 모델이 있다.
```

① DAC (Discretionary Access Control)  
② MAC (Mandatory Access Control)  
③ RBAC (Role-Based Access Control)  
④ ABAC (Attribute-Based Access Control)  

### 문제 6 [출처: 2021년 3회 필기]
Secure SDLC에 대한 설명으로 옳지 않은 것은?

① 소프트웨어 개발 생명주기 전체 단계에 걸쳐 보안 활동을 수행한다.  
② 대표 방법론으로 MS-SDL, Seven Touchpoints, CLASP, OWASP SAMM 등이 있다.  
③ 보안 활동은 코딩(구현) 단계에서만 집중적으로 수행된다.  
④ 요구사항·설계·구현·테스트·유지보수 단계에서 각각 보안 활동을 정의한다.  

### 문제 7 [출처: 2022년 3회 필기 / 유사 기출]
다음에서 설명하는 악성 프로그램은?

```
정상 프로그램으로 위장하여 사용자에게 설치되며,
일단 실행되면 백도어 설치·정보 유출 등의 악성 행위를 한다.
스스로 복제하지는 않는다.
```

① Worm  
② Virus  
③ Trojan Horse  
④ Rootkit  

### 문제 8 [출처: 2023년 1회 필기 / 유사 기출]
Cross-Site Scripting(XSS) 공격에 대한 방어 방법으로 가장 적절한 것은?

① 데이터베이스 쿼리에 Prepared Statement를 적용한다.  
② 입력값에 대한 HTML 특수문자(`<`, `>`, `&`, `"`)를 이스케이프(escape) 처리한다.  
③ 패스워드 저장 시 단방향 해시 함수(SHA-256 등)를 사용한다.  
④ 세션 ID에 랜덤 토큰(CSRF Token)을 부여하여 함께 검증한다.

## 🔍 오답 역추적 개념 발췌
- **랜섬웨어(Ransomware)**: 파일 암호화 + 금전(비트코인) 요구. WannaCry, CryptoLocker.
- **CIA 3대 요소**: Confidentiality(기밀성·노출 차단), Integrity(무결성·변조 차단), Availability(가용성·중단 차단).
- **DDoS**: Attacker → Master(C&C) → Handler → Agent(Zombie) → Victim. 좀비/마스터/핸들러는 공격자측, "Zombie-Handler-Agent" 순서 오답 주의.
- **SQL Injection ↔ XSS**: SQLi는 DB쿼리 조작, XSS는 클라이언트 스크립트 실행. 둘을 섞은 보기 함정.
- **접근통제 모델**: DAC(소유자 재량) / MAC(보안등급 강제, Bell-LaPadula·Biba) / RBAC(역할 기반) / ABAC(속성 기반).
- **Secure SDLC**: 코딩단계만 아님 — 전 생명주기에 보안활동 분산.
- **악성코드 분류**: 자가복제O(Virus·Worm) / 자가복제X(Trojan·Spyware) / 은닉(Rootkit) / 후속침입로(Backdoor).
- **XSS 방어**: 출력 인코딩(이스케이프)·CSP·HttpOnly Cookie. SQLi 방어와 혼동 금지.

## 💡 상세 해설

### 문제 1
정답: ② Ransomware
- Ransom(몸값) + ware. 파일을 암호화하고 복호화 키 대가로 금전(주로 암호화폐) 요구.
- 오답 ① Smishing: SMS + Phishing, 문자 메시지로 악성 링크 유도.
- 오답 ③ Phishing: 정상 사이트로 위장한 가짜 사이트로 자격증명 탈취.
- 오답 ④ XSS: 웹 페이지에 악성 스크립트 삽입.

### 문제 2
정답: ④ 무결성(Integrity)
- "인가된 사용자만 수정" → 무결성. 비인가 변조 차단.
- 오답 ① 기밀성: 인가된 사용자만 "열람" 가능 (Read).
- 오답 ② 부인 방지: 송수신 사실 부인 불가 (전자서명 등).
- 오답 ③ 가용성: 필요할 때 자원에 접근 가능.

### 문제 3
정답: ③
- 정상 순서는 Attacker → Master(C&C) → Handler → Agent(Zombie) → Victim. 보기 ③의 "Master - 좀비 - 핸들러 - 에이전트"는 순서가 뒤섞임 (Zombie와 Agent를 분리하면서 핸들러를 뒤로 보냄).
- 오답 ① IP Spoofing은 출발지 위조로 추적 회피, DDoS의 기본 기법.
- 오답 ② Trinoo, TFN(Tribe Flood Network), Stacheldraht는 대표 DDoS 도구.
- 오답 ④ 분산된 다수 좀비에서 트래픽 폭주 → 가용성 침해, DDoS 정의 그대로.

### 문제 4
정답: ④
- ④는 XSS(Cross-Site Scripting)의 설명. SQLi와 XSS를 섞은 함정.
- 오답 ① 입력값 미검증이 SQLi 근본 원인.
- 오답 ② DB 데이터 조작·인증 우회는 SQLi의 대표 영향.
- 오답 ③ Prepared Statement(파라미터 바인딩)는 SQLi의 가장 효과적 대응.

### 문제 5
정답: ② MAC
- 보안등급(Security Label) + 시스템 강제 + Bell-LaPadula → MAC.
- 오답 ① DAC: 객체 소유자가 재량으로 권한 부여(Unix chmod, ACL).
- 오답 ③ RBAC: 역할(Role) 기반 권한 할당, 조직 직무 변경에 유연.
- 오답 ④ ABAC: 사용자/객체/환경 속성(Attribute) 기반.

### 문제 6
정답: ③
- Secure SDLC의 핵심은 "전 생명주기 분산". 코딩 단계만 집중은 틀림.
- 오답 ① 요구사항~유지보수 전 단계 보안 활동 — 정의 그대로.
- 오답 ② MS-SDL(Microsoft), Seven Touchpoints(McGraw), CLASP(OWASP), SAMM(OWASP) — 대표 4가지.
- 오답 ④ 각 단계별 보안 산출물(위협 모델링·보안 설계 검토·정적 분석·동적 테스트·취약점 패치).

### 문제 7
정답: ③ Trojan Horse
- 정상 프로그램 위장 + 자가복제 X + 백도어/정보유출 → Trojan.
- 오답 ① Worm: 자가복제 O, 숙주 불필요, 네트워크 전파.
- 오답 ② Virus: 자가복제 O, 숙주 필요.
- 오답 ④ Rootkit: 침입 후 권한 유지·은닉 도구 모음.

### 문제 8
정답: ②
- XSS는 클라이언트 측 스크립트가 브라우저에서 실행되는 취약점. 출력 시 HTML 특수문자 이스케이프가 핵심 방어.
- 오답 ① Prepared Statement → SQL Injection 방어.
- 오답 ③ 단방향 해시 → 패스워드 안전 저장(Storage).
- 오답 ④ CSRF Token → CSRF 방어.

## 📚 역추적 참조 (사후 검증용)

### 핵심 용어
| 용어 | 정의 |
|---|---|
| 버퍼오버플로우(Buffer Overflow) | 할당된 버퍼 경계를 넘는 데이터 입력으로 인접 메모리(복귀주소 등) 덮어쓰는 취약점 |
| ASLR | Address Space Layout Randomization. 프로세스 주소공간 무작위화로 익스플로잇 어렵게 함 |
| DEP/NX | Data Execution Prevention / No-eXecute. 데이터 영역의 코드 실행 차단 |
| Stack Canary | 함수 복귀주소 앞에 임의값(Canary) 삽입, 복귀 전 변조 여부 검증 |
| SetUID | Unix 권한 비트. 실행 시 파일 소유자 권한으로 동작 (passwd 등). 악용 시 권한상승 |
| Rootkit | 관리자(root) 권한 획득 후 백도어·로그 은닉·프로세스 숨김 도구 모음 |
| Backdoor | 정상 인증 우회 비밀 통로. 개발자 의도/공격자 설치 모두 포함 |
| Ransomware | 파일 암호화 후 복호화 대가로 금전 요구 (WannaCry, CryptoLocker) |
| Keylogger | 키보드 입력 가로채 탈취 (소프트웨어/하드웨어형) |
| Spyware | 사용자 모르게 정보 수집·전송 |
| Worm | 자가복제 + 네트워크 자체 전파, 숙주 불필요 |
| Virus | 자가복제 + 숙주 파일 필요 |
| Trojan Horse | 정상 위장, 자가복제 X, 백도어·정보유출 수행 |
| DAC | Discretionary Access Control. 객체 소유자 재량으로 권한 부여 (chmod, ACL) |
| MAC | Mandatory Access Control. 보안등급 기반 시스템 강제 (Bell-LaPadula, Biba) |
| RBAC | Role-Based Access Control. 역할별 권한 할당 |
| ABAC | Attribute-Based Access Control. 주체·객체·환경 속성 기반 |
| Bell-LaPadula | 기밀성 모델. No Read Up(상위등급 읽기 금지), No Write Down(하위등급 쓰기 금지) |
| Biba | 무결성 모델. No Read Down, No Write Up (Bell-LaPadula의 반대) |
| Secure SDLC | 개발 전 생명주기에 보안활동 통합 |
| MS-SDL | Microsoft Security Development Lifecycle |
| Seven Touchpoints | McGraw 제안, 보안 강화 7가지 활동(코드리뷰·위험분석·침투테스트 등) |
| CLASP | Comprehensive Lightweight Application Security Process, OWASP |
| OWASP SAMM | Software Assurance Maturity Model |
| Secure Coding | 구현 단계에서 보안약점 제거 (KISA 가이드 - 한국 SW개발보안 의무화 2012년 12월) |
| SQL Injection | 입력값으로 SQL 쿼리 조작 → DB 데이터/인증 우회 |
| XSS | Cross-Site Scripting. 웹 페이지에 악성 스크립트 삽입 → 다른 사용자 브라우저에서 실행 |
| Stored XSS | 서버 DB에 저장된 스크립트 (게시판) |
| Reflected XSS | URL 파라미터 등 입력값이 즉시 반영 |
| DOM-based XSS | 클라이언트 DOM 조작 |
| CSRF | Cross-Site Request Forgery. 인증된 사용자 권한으로 의도하지 않은 요청 전송 |
| SSRF | Server-Side Request Forgery. 서버가 공격자 지정 URL로 요청 |

### 핵심 개념
- **CIA 3원칙 + 확장 5원칙**: 기밀성·무결성·가용성 + 인증(Authentication)·부인방지(Non-repudiation).
- **인증(Authentication) vs 인가(Authorization) vs 식별(Identification)**: 누구인가 주장(식별) → 검증(인증) → 무엇 가능(인가).
- **보안기능 7대 보안약점 영역** (KISA Secure Coding):
  1. 입력데이터 검증 및 표현 (SQLi, XSS, CSRF 등)
  2. 보안 기능 (취약 암호, 적절치 않은 인가)
  3. 시간 및 상태 (경쟁조건, 종료되지 않은 반복문)
  4. 에러 처리 (오류 메시지로 정보 노출)
  5. 코드 오류 (널 포인터, 부적절한 자원 해제)
  6. 캡슐화 (내부 데이터 노출)
  7. API 오용 (위험한 API 사용)
- **버퍼오버플로우 ↔ 대응 매트릭스**: ASLR(주소 무작위) + DEP(실행 차단) + Canary(변조 탐지) + SSP(Stack Smashing Protector) 컴파일러 옵션.
- **공격 단계 일반화 (Kill Chain)**: 정찰 → 무기화 → 전달 → 익스플로잇 → 설치 → C&C → 목표 행동.
- **OWASP Top 10 2021**: Broken Access Control / Cryptographic Failures / Injection / Insecure Design / Security Misconfiguration / Vulnerable Components / Identification & Auth Failures / Software & Data Integrity Failures / Logging Failures / SSRF.
