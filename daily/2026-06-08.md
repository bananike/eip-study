① 버킷(Bucket) — 하나의 주소를 갖는 파일의 한 구역으로, 여러 개의 슬롯을 가질 수 있다.

④ 오버플로(Overflow) — 해시함수가 키를 해시 주소로 **변환하지 못해** 발생하는 오류이다.

### 문제 7 [유사 기출: 정보처리기사 필기 2021·2023 반복 — 그래프 표현 비교 정형 패턴]
그래프를 **인접 행렬(Adjacency Matrix)**과 **인접 리스트(Adjacency List)**로 표현하는 방식에 대한 설명으로 옳지 **않은** 것은?

① 인접 행렬은 정점이 n개일 때 n×n 크기의 2차원 배열로 표현되며, **공간 복잡도는 O(n²)** 이다.
② 인접 리스트는 정점 수 V, 간선 수 E에 대해 **공간 복잡도가 O(V + E)** 로, **희소(Sparse) 그래프**에 유리하다.
③ 인접 행렬에서 두 정점 u, v 사이의 **간선 존재 여부 확인**은 O(1) 시간에 가능하다.
④ 인접 리스트는 **밀집(Dense) 그래프**에서 인접 행렬보다 항상 **공간을 적게** 사용한다.

## 🔍 오답 역추적 개념 발췌
- **탐색 알고리즘 평균 시간복잡도**: 순차 O(n) / 이진 O(log n) / 보간 O(log log n) — 평균, 최악은 O(n) / 해시 O(1) — 평균
- **이진 탐색 전제**: 데이터가 **정렬되어 있어야** 함. 중간값(mid = (low+high)/2) 기준으로 좌/우 반으로 분할 반복
- **해시함수 5종**: 제산법(나머지 연산), 제곱법(중간자리 추출), 폴딩법(분할 후 더하기/XOR), 숫자분석법(자릿수 분포 분석), 기수변환법(진법 변환)
- **충돌 해결 2분류**:
  - **개방 주소법(폐쇄 해싱)**: 충돌 시 **같은 테이블 내** 다른 빈자리 탐색 — 선형 조사, 이차 조사, 이중 해싱
  - **체이닝(개방 해싱)**: 충돌 시 **외부 연결 리스트**에 추가
- **해시 용어 — 오버플로 vs 충돌**:
  - 충돌(Collision): 두 키가 같은 해시 주소로 사상되는 현상
  - 오버플로(Overflow): 충돌 발생 시 **버킷의 모든 슬롯이 차서** 더 이상 저장할 수 없는 상태
  - 동의어(Synonym): 같은 해시 주소를 갖는 키들의 집합
- **그래프 G=(V,E)**: V = 정점 집합, E = 간선 집합
- **차수 합**: 무방향 그래프의 모든 정점 차수의 합 = 2 × 간선 수
- **무방향 그래프 최대 간선**: n(n−1)/2 / **방향 그래프 최대 간선**: n(n−1)
- **인접 행렬 vs 인접 리스트**:
  - 인접 행렬: O(n²) 공간, 간선 존재 확인 O(1), 밀집 그래프 유리
  - 인접 리스트: O(V+E) 공간, 간선 존재 확인 O(degree), 희소 그래프 유리
- **DFS vs BFS**: DFS는 **스택(재귀)**, BFS는 **큐**
- **그래프 알고리즘 분류**:
  - **MST**: 크루스칼(간선 정렬+Union-Find), 프림(정점 확장)
  - **최단 경로**: 다익스트라(단일 시작점, 음수 간선 X), 벨만-포드(음수 허용), 플로이드(모든 쌍)
  - **순회**: DFS, BFS

## 💡 상세 해설

### 문제 1
정답: ③
- 보간 탐색(Interpolation Search)의 **평균 시간복잡도는 O(log log n)** 이며, **최악**일 때 O(n)이다. O(n²)은 **잘못된 값**.
- ① 순차 탐색: 처음부터 하나씩 비교 → 평균 O(n) ✔
- ② 이진 탐색: 정렬된 데이터에서 절반씩 분할 → O(log n) ✔
- ④ 해시 탐색: 해시함수로 직접 주소 계산 → 평균 O(1), 최악 O(n) ✔
- 함정: 보간 탐색은 **이진 탐색보다 빠른** 평균 성능을 갖는데, 잘못된 보기에서는 O(n²)로 과장한다.

### 문제 2
정답: ①
- 데이터: `3, 5, 7, 9, 11, 14, 17, 21, 25` (인덱스 1~9), 찾을 값 = 14
- **1회차**: low=1, high=9 → mid=(1+9)/2=**5**, A[5]=**11** → 14 > 11 → low=6
- **2회차**: low=6, high=9 → mid=(6+9)/2=**7**(소수점 버림), A[7]=**17** → 14 < 17 → high=6
- **3회차**: low=6, high=6 → mid=**6**, A[6]=**14** → **일치, 탐색 종료**
- 비교 횟수 = **3회**

※ 함정 보정: 보기 ①을 "1회"가 아니라 "**3회**"로 변경 — 본 풀이는 **3회**가 정답. 실제 시험에서는 보기 배치가 달라질 수 있으므로 **추적 절차**가 핵심.

(채점 정답 ① — 정답표는 추적 결과를 그대로 ① 위치에 매핑. **풀이의 핵심은 "3회"라는 숫자**.)

### 문제 3
정답: ④
- 해시함수 종류 5가지: **제산법, 제곱법(중간제곱법), 폴딩법, 숫자분석법(계수분석법), 기수변환법** (+ 무작위법)
- ④ 이진 탐색법은 **탐색 알고리즘**이지 해시함수가 아님 → 정답
- ① 제산법: H(k) = k mod m (m은 소수가 좋음)
- ② 제곱법: 키를 제곱한 후 중간 자리 추출
- ③ 폴딩법: 키를 일정 길이로 나눠 합(또는 XOR)

### 문제 4
정답: ④
- **개방 주소법(Open Addressing) = 폐쇄 해싱**: 충돌 시 **같은 테이블의 다른 빈자리**를 탐색 → 선형/이차 조사, 이중 해싱
- **체이닝(Chaining) = 개방 해싱**: 충돌 시 **외부 연결 리스트**에 추가 → 개방 주소법 **아님**
- 용어 함정: "개방 주소법" ≠ "개방 해싱". 이름이 비슷해 혼동 주의.
  - 개방 주소법(Open Addressing) = 폐쇄 해싱(Closed Hashing)
  - 체이닝(Chaining) = 개방 해싱(Open Hashing)
- 정답은 ④ 체이닝

### 문제 5
정답: ④
- ④ 오버플로(Overflow)는 해시함수가 **변환을 못해서**가 아니라, **충돌 발생 후 버킷의 모든 슬롯이 가득 차서** 더 이상 레코드를 저장할 수 없는 상태를 말함. 설명이 틀림.
- ① 버킷: 한 해시 주소에 대응하는 저장 구역, 여러 슬롯 보유 가능 ✔
- ② 슬롯: 한 레코드 저장 단위 ✔
- ③ 동의어(Synonym): 같은 해시 주소로 사상되는 서로 다른 키들의 집합 ✔
- 핵심: **충돌 ≠ 오버플로**. 충돌이 일어나도 슬롯에 여유가 있으면 오버플로는 안 남.

### 문제 6
정답: ③
- 무방향 그래프: 두 정점 u, v 사이의 간선 (u,v)와 (v,u)는 **같은 간선**으로 취급
- n개의 정점에서 2개를 뽑는 조합 = **nC2 = n(n−1)/2**
- 방향 그래프(Directed): (u,v) ≠ (v,u) → 최대 간선 수 = **n(n−1)**
- ① n−1: **트리(Tree)** 의 간선 수
- ② n(n−1): **방향 그래프** 최대 간선 수
- ④ n²: 자기 자신 포함(루프 허용) 일반 행렬 크기

### 문제 7
정답: ④
- ④가 틀림. 인접 리스트가 **항상** 공간을 적게 쓰는 것은 아니다.
  - **희소(Sparse) 그래프** (E ≪ V²): 인접 리스트가 유리
  - **밀집(Dense) 그래프** (E ≈ V²): 인접 리스트도 O(V+E) ≈ O(V²)에 가까워지고, 연결리스트 노드 오버헤드(포인터 등) 때문에 **인접 행렬보다 더 많은 공간을 쓸 수 있음**
- ① 인접 행렬: V×V 2차원 배열 → O(V²) ✔
- ② 인접 리스트: 정점마다 인접 정점 리스트 → O(V+E) ✔
- ③ 인접 행렬에서 간선 (u,v) 존재 = A[u][v] ≠ 0 → O(1) ✔

### 문제 8
정답: ①
- **최소 신장 트리(MST)**: 크루스칼(Kruskal), 프림(Prim) — 둘 다 가중치 합 최소인 신장 트리 구성
- ② 다익스트라·플로이드: **최단 경로** 알고리즘 (MST 아님)
- ③ DFS·BFS: **그래프 순회** 알고리즘 (MST 아님, 단 비가중 그래프에서 BFS는 최단 경로 가능)
- ④ 벨만-포드·다익스트라: 둘 다 **단일 출발점 최단 경로**
- 분류 암기 필수:
  - **MST**: Kruskal, Prim
  - **단일 출발점 최단 경로**: Dijkstra(음수 X), Bellman-Ford(음수 O)
  - **모든 쌍 최단 경로**: Floyd-Warshall
  - **순회**: DFS, BFS

| 이차 조사법(Quadratic Probing) | 충돌 시 +1², +2², +3² ... 빈자리 이동 |
| 이중 해싱법(Double Hashing) | 충돌 시 두 번째 해시함수로 이동 간격 결정 |
| 체이닝(Chaining) | 충돌 시 외부 연결 리스트로 연결 (= 개방 해싱) |
| 정점(Vertex) | 그래프의 노드 |
| 간선(Edge) | 두 정점을 연결하는 선 |
| 차수(Degree) | 한 정점에 연결된 간선 수 |
| 진입 차수(In-degree) | 방향 그래프에서 정점으로 들어오는 간선 수 |

| 인접 행렬(Adjacency Matrix) | V×V 배열로 간선 표현, O(V²) 공간 |
| 인접 리스트(Adjacency List) | 정점별 인접 정점 리스트, O(V+E) 공간 |

### 핵심 개념
- **정렬과 탐색의 결합**: 이진 탐색은 정렬을 전제로 함. 정렬 비용(O(n log n))을 포함하면 일회성 탐색에는 비효율, 반복 탐색일수록 유리.
- **해시 충돌의 본질**: 입력 공간(가능한 키 전체) > 출력 공간(해시 테이블 크기) 이므로 충돌은 필연. 해결 전략을 어떻게 설계하느냐가 성능을 좌우.
- **개방 주소법 vs 체이닝 트레이드오프**:
  - 개방 주소법: 캐시 친화적, 메모리 효율적, 적재율(load factor)에 민감
  - 체이닝: 적재율 1 초과 가능, 포인터 오버헤드, 캐시 미스 증가
- **그래프 표현 선택 기준**: 간선 수 E의 크기에 따라 — E가 V²에 가까우면 인접 행렬, E ≪ V²이면 인접 리스트.
- **MST와 최단 경로의 차이**: MST는 **전체 가중치 합 최소**의 트리, 최단 경로는 **특정 두 정점 사이 경로 합 최소**. 같지 않다.
