## 🎯 오늘의 기출문제 — 운영체제(OS) 약점 집중 15문제 (Phase 4 약점보강 3일차 · Day 9 신규 비중복)

### 문제 1 [유사 기출: 2021년 1회 4과목 — 프로세스(Process) vs 스레드(Thread) 정형 구분]
**프로세스(Process)와 스레드(Thread)** 에 관한 설명 중 **가장 옳지 않은 것**은?

① **프로세스** — 실행 중인 프로그램의 인스턴스로, 독립된 **주소공간(코드·데이터·힙·스택)** 과 **PCB(Process Control Block)** 를 가지며, 프로세스 간 통신은 **IPC(파이프·메시지 큐·공유 메모리·소켓)** 를 통해서만 가능하다.
② **스레드** — 프로세스 내부의 실행 흐름 단위로, 같은 프로세스 내 스레드는 **코드·데이터·힙을 공유**하되 **스택·PC·레지스터는 각자** 가진다. 문맥 교환 비용이 프로세스보다 낮다.
③ **사용자 스레드 vs 커널 스레드** — 사용자 스레드는 라이브러리 수준에서 관리(빠름·이식성 높음·블로킹 시 전체 정지 위험), 커널 스레드는 OS가 관리(멀티코어 활용·문맥교환 비용 큼). 매핑 방식은 1:1·M:1·M:N.
④ **프로세스와 스레드는 완전히 동일한 개념**이며, **스레드가 오히려 독립된 주소공간을 가지고 IPC로만 통신**하며, **프로세스는 힙·스택을 모두 공유**하는 경량 실행 단위이다.

### 문제 2 [유사 기출: 2022년 2회 4과목 — PCB · 문맥 교환(Context Switching) 정형]
**PCB(Process Control Block) 와 문맥 교환(Context Switching)** 에 관한 설명 중 **가장 옳지 않은 것**은?

① **PCB 구성 요소** — ① 프로세스 상태(new/ready/running/waiting/terminated) ② PID(프로세스 식별자) ③ 프로그램 카운터(PC) ④ CPU 레지스터 값 ⑤ CPU 스케줄링 정보(우선순위 등) ⑥ 메모리 관리 정보(페이지 테이블·세그먼트 테이블) ⑦ 계정·I/O 정보.
② **문맥 교환** — CPU를 다른 프로세스에 넘길 때, 현 프로세스의 상태를 **PCB에 저장**하고 다음 프로세스의 상태를 **PCB에서 복원**하는 과정. 이 시간 동안 유용한 작업은 수행되지 않아 **오버헤드(overhead)** 로 간주된다.
③ **문맥 교환 발생 시점** — ① 인터럽트(하드웨어·타이머) ② 시스템 호출(I/O 요청) ③ 우선순위 변화 ④ 시간 할당량(Quantum) 만료(RR). 스레드 간 문맥 교환은 같은 주소공간이므로 **프로세스 간 교환보다 비용이 훨씬 낮다**.
④ **PCB는 프로세스가 아니라 CPU의 하드웨어 캐시(L1·L2·L3) 안에만 저장**되며, **문맥 교환은 CPU를 낭비하지 않고 오히려 처리량(throughput)을 극대화**하는 순수한 이득 작업으로, 오버헤드가 전혀 발생하지 않는다.

### 문제 3 [유사 기출: 2020년 3회 4과목 — CPU 스케줄링 알고리즘 정형 분류]
**CPU 스케줄링 알고리즘의 선점(Preemptive) vs 비선점(Non-preemptive) 분류**에 관한 설명 중 **가장 옳지 않은 것**은?

① **비선점 스케줄링** — 프로세스가 CPU를 자발적으로 반납(종료·I/O 대기)할 때까지 뺏지 않는다. 대표 알고리즘 = **FCFS(First-Come, First-Served) · SJF(Shortest Job First) · HRRN(Highest Response Ratio Next) · 우선순위(비선점형)**.
② **선점 스케줄링** — OS가 실행 중인 프로세스의 CPU를 강제로 회수할 수 있다. 대표 알고리즘 = **RR(Round Robin) · SRT(Shortest Remaining Time) · 우선순위(선점형) · MLQ/MLFQ(다단계 큐/피드백 큐)**.
③ **HRRN 응답률 공식** = **(대기 시간 + 서비스 시간) / 서비스 시간**. SJF의 짧은 작업 편애·긴 작업의 **기아(Starvation)** 문제를 완화하기 위해 대기가 길어질수록 우선순위가 자동으로 올라가는 **에이징(Aging) 효과**를 갖는다.
④ **비선점 스케줄링에는 RR·SRT가 포함**되고 **선점 스케줄링에는 FCFS·SJF가 포함**되며, **HRRN 응답률 공식은 서비스 시간 / (대기 시간 + 서비스 시간)** 으로 짧은 작업일수록 우선순위가 낮아진다.

### 문제 5 [유사 기출: 2022년 1회 4과목 — Round Robin 정형 시간 할당량]
**라운드 로빈(Round Robin, RR)** 스케줄링에 관한 설명 중 **가장 옳지 않은 것**은?

① **동작 원리** — 준비 큐(FIFO)에서 프로세스를 꺼내 **시간 할당량(Time Quantum·Time Slice)** 만큼 CPU를 부여하고, 할당량이 만료되면 타이머 인터럽트로 **선점**하여 준비 큐 끝으로 보낸다. 시분할(Time-sharing) 시스템의 표준 스케줄러.
② **시간 할당량 크기 트레이드오프** — 할당량이 **너무 크면** 사실상 **FCFS와 유사**해져 응답 시간이 나빠지고, **너무 작으면** 문맥 교환 오버헤드가 지나쳐 처리량이 낮아진다. 일반적으로 **10~100ms** 범위에서 튜닝한다.
③ **응답 시간 특성** — RR은 **응답 시간이 짧고 균등**하여 대화형(interactive) 워크로드에 적합하지만, 평균 대기 시간이 SJF·SRT보다 길 수 있고 CPU 편중이 심한 프로세스는 상대적으로 손해를 본다.
④ **RR은 비선점 스케줄링**이며, **시간 할당량이 만료되어도 프로세스가 자발적으로 반납할 때까지 CPU를 뺏지 않고** 대기시키는 방식으로, **문맥 교환이 전혀 발생하지 않는다**.

### 문제 6 [유사 기출: 2023년 2회 4과목 — 메모리 관리 · 페이징 vs 세그멘테이션 정형]
**메모리 관리 기법 페이징(Paging) vs 세그멘테이션(Segmentation)** 에 관한 설명 중 **가장 옳지 않은 것**은?

① **페이징** — 프로세스와 물리 메모리를 **고정 크기 블록**(페이지/프레임)으로 나누어 매핑한다. 주소 변환은 **페이지 번호 + 오프셋** → 페이지 테이블 → 프레임 번호. **외부 단편화는 없고 내부 단편화가 발생**.
② **세그멘테이션** — 프로세스를 논리적 단위(코드·데이터·스택·힙 등) **가변 크기 세그먼트**로 나눈다. 주소 변환은 **세그먼트 번호 + 오프셋** → 세그먼트 테이블(base·limit). **내부 단편화는 없고 외부 단편화가 발생**.
③ **혼합 기법** — **세그먼테이션 + 페이징**(x86, Multics)은 세그먼트로 논리 구분·보호를 하되, 각 세그먼트를 다시 페이지로 나누어 외부 단편화를 없앤다. TLB(Translation Lookaside Buffer)로 주소 변환 속도를 보완.
④ **페이징은 가변 크기·외부 단편화 발생**이 특징이고, **세그멘테이션은 고정 크기·내부 단편화 발생**이 특징이며, **TLB는 페이징에서는 사용되지 않고 오직 세그멘테이션에서만 사용**된다.

### 문제 9 [유사 기출: 2024년 1회 4과목 — Belady 이상현상 · Optimal 정형 개념]
**페이지 교체 알고리즘 이론**에 관한 설명 중 **가장 옳지 않은 것**은?

① **Optimal(OPT · Belady 알고리즘)** — 앞으로 **가장 오래 참조되지 않을 페이지**를 교체한다. 이론적 최소 페이지 부재를 보장하지만 **미래 참조를 알아야** 하므로 실제 구현 불가·성능 상한 벤치마크로만 사용.
② **LRU(Least Recently Used)** — 과거 **가장 오랫동안 참조되지 않은 페이지**를 교체. 시간 지역성(temporal locality)에 근거해 Optimal에 가까운 성능을 내며, **참조 시각 스탬프·이중 연결리스트+해시**로 구현.
③ **Belady 이상현상(Belady's Anomaly)** — **프레임 수를 늘렸는데 페이지 부재가 오히려 증가**하는 반직관적 현상. **FIFO에서 발생**하며, **LRU·Optimal 같은 스택 알고리즘(Stack Algorithm)에서는 발생하지 않는다**.
④ **Belady 이상현상은 LRU·Optimal에서도 자주 발생**하며, **FIFO에서는 절대 발생하지 않고** 오히려 프레임 수를 늘리면 반드시 페이지 부재가 감소한다는 것이 Belady의 정리이다.

### 문제 10 [유사 기출: 2022년 3회 4과목 — LFU · NUR(Clock) · 2차 기회 정형 분류]
**페이지 교체 알고리즘 계열별 분류**에 관한 설명 중 **가장 옳지 않은 것**은?

① **LFU(Least Frequently Used)** — 참조 **횟수가 가장 적은** 페이지를 교체. 초기 다량 참조 후 사용 안 되는 페이지가 계속 남는 **캐시 오염(Cache Pollution)** 문제가 있으며, **Aging(참조 카운터 감소)** 기법으로 완화한다.
② **NUR / Clock(2차 기회)** — 참조 비트(Reference Bit)를 이용해 최근 사용 여부만 확인. 참조 비트=1이면 0으로 리셋하고 다음으로 넘어가며, 0을 만나면 교체. LRU 근사·구현 단순·리눅스 커널 사용.
③ **MRU(Most Recently Used)** — 최근 참조된 페이지가 곧 다시 사용될 확률이 낮은 특수 워크로드(대용량 순차 스캔·DB Full Table Scan)에서 유리하며, 일반적인 시간 지역성이 강한 경우엔 LRU가 우수.
④ **LFU는 참조 시간 순서만 보고 참조 횟수는 무시**하며, **Clock 알고리즘은 참조 비트가 아니라 페이지 크기만 확인**하고, **MRU는 항상 LRU보다 성능이 우수**하다.

### 문제 11 [유사 기출: 2021년 1회 4과목 — 교착상태(Deadlock) 4대 필요조건 정형]
**교착상태(Deadlock)의 4대 필요조건**에 관한 설명 중 **가장 옳지 않은 것**은?

① **상호 배제(Mutual Exclusion)** — 자원은 한 번에 한 프로세스만 사용할 수 있어야 한다. 공유 가능한 자원(읽기 전용 파일 등)은 교착상태를 유발하지 않는다.
② **점유와 대기(Hold and Wait)** — 프로세스가 최소 하나 이상의 자원을 **보유(hold)** 한 상태에서 다른 프로세스가 점유한 자원을 **대기(wait)** 하고 있어야 한다.
③ **비선점(No Preemption)** — 다른 프로세스가 사용 중인 자원을 **강제로 뺏을 수 없어야** 한다. 자원을 자발적으로 반납할 때까지 기다린다. **환형 대기(Circular Wait)** = 프로세스와 자원이 원형 사슬을 이룬다.
④ **4대 필요조건은 상호 배제 · 점유 대기 · 선점(Preemption) · 순차 대기(Sequential Wait)** 이며, **비선점과 환형 대기는 교착상태와 무관한 조건**이다. 이 중 하나만 성립해도 교착상태가 반드시 발생한다.

### 문제 12 [유사 기출: 2023년 1회 4과목 — 교착상태 해결 4가지 (예방/회피/탐지/무시) 정형]
**교착상태 해결 전략(예방 · 회피 · 탐지·회복 · 무시)** 에 관한 설명 중 **가장 옳지 않은 것**은?

① **예방(Prevention)** — 4대 필요조건 중 **최소 하나를 성립하지 못하게 사전 차단**한다. 예: **자원 순서 부여**로 환형대기 차단, **작업 시작 전 필요한 모든 자원을 한꺼번에 할당**해 점유대기 차단.
② **회피(Avoidance) — 은행원 알고리즘(Banker's Algorithm)** — 자원 할당 요청 시 시스템이 **안전 상태(safe state)** 를 유지할 수 있는지 검사해, 안전 순차(safe sequence)가 존재할 때만 할당한다. 최대 요구량을 미리 알아야 한다.
③ **탐지·회복(Detection & Recovery)** — 자원 할당 그래프·대기 그래프(Wait-For Graph)의 **사이클 검출**로 교착상태를 발견하고, **프로세스 종료(strong·weak)** 또는 **자원 선점(rollback)** 으로 회복한다. **무시(타조 알고리즘)** 는 UNIX/Windows의 실용적 접근.
④ **은행원 알고리즘은 교착상태를 사후에 탐지하는 방법**이고, **자원 순서 부여는 회피(Avoidance) 기법**이며, **타조 알고리즘은 사이클을 반드시 검출하고 회복시키는 정밀 알고리즘**이다.

### 문제 13 [유사 기출: 2022년 2회 4과목 — 가상 메모리 · 워킹셋 · 스래싱 정형]
**가상 메모리(Virtual Memory) 이론**에 관한 설명 중 **가장 옳지 않은 것**은?

① **가상 메모리** — 프로세스 전체를 메모리에 올리지 않고, 필요한 페이지만 **요구 페이징(Demand Paging)** 으로 적재한다. 물리 메모리보다 큰 프로세스 실행이 가능하고 다중 프로그래밍 정도를 높일 수 있다.
② **워킹셋(Working Set) 모델** — 프로세스가 일정 시간창(Δ) 내에 참조한 페이지 집합. 워킹셋 크기의 합이 물리 프레임 수를 초과하지 않도록 유지하면 스래싱을 예방할 수 있다.
③ **스래싱(Thrashing)** — 다중 프로그래밍 정도가 지나치게 높아 페이지 부재율이 급증하고 CPU 활용률이 오히려 급락하는 현상. 대응 = **워킹셋 모델**·**페이지 부재 빈도(PFF) 알고리즘**·**적정 다중 프로그래밍 조정**·**메모리 증설**.
④ **가상 메모리는 물리 메모리와 동일 크기 이하만 표현 가능**하며, **워킹셋은 시간창과 무관하게 프로세스의 총 페이지 수**를 의미하고, **스래싱은 다중 프로그래밍 정도를 오히려 더 늘리면** 해결된다.

### 문제 14 [유사 기출: 2024년 2회 4과목 — 디스크 스케줄링(SSTF · SCAN · C-SCAN) 정형]
**디스크 스케줄링 알고리즘**에 관한 설명 중 **가장 옳지 않은 것**은?

① **FCFS** — 요청 도착 순서대로 처리. 공정하나 탐색 거리(Seek Time)가 길어질 수 있음. **SSTF(Shortest Seek Time First)** — 현재 헤드 위치에서 **가장 가까운** 요청 우선. **기아(Starvation)** 발생 가능.
② **SCAN(엘리베이터)** — 헤드를 한 방향으로 이동하며 요청을 처리하다가 끝에 도달하면 방향을 바꾸어 반대편 요청을 처리. 양쪽 끝 요청도 공정하게 처리되지만 방금 지나온 위치의 요청은 오래 기다린다.
③ **C-SCAN(Circular SCAN)** — SCAN의 변형으로 한 방향으로만 서비스하고 끝에 도달하면 **되돌아오지 않고 시작 위치로 즉시 점프**해 다시 같은 방향으로 서비스. 대기 시간 편차가 SCAN보다 균등.
④ **SSTF는 기아가 절대 발생하지 않고 항상 공정**하며, **SCAN은 원형(Circular)으로 되돌아오지 않고**, **C-SCAN은 양방향으로 왕복 이동**한다. 세 알고리즘 모두 FCFS보다 항상 탐색 시간이 길다.

### 문제 15 [유사 기출: 2021년 2회 4과목 — 파일 시스템 · 할당 방식 · i-node 정형]
**파일 시스템의 파일 할당 방식**에 관한 설명 중 **가장 옳지 않은 것**은?

① **연속 할당(Contiguous)** — 파일을 연속된 블록에 저장. 순차·임의 접근 모두 빠르나 **외부 단편화**·**파일 크기 확장 어려움**. FAT의 원형이 아닌 오래된 방식.
② **연결 할당(Linked)** — 파일 블록을 연결리스트로 연결. 외부 단편화 없음·확장 자유. **순차 접근 O(n)**·임의 접근 비효율·포인터 손상 시 파일 전체 파괴. **FAT(File Allocation Table)** 은 포인터를 별도 테이블에 모아둔 개선형.
③ **인덱스 할당(Indexed) · i-node(UNIX)** — 파일마다 인덱스 블록(i-node)을 두어 데이터 블록 주소를 모아둔다. **직접 블록 · 단일 · 이중 · 삼중 간접 블록**으로 대용량 파일 지원. 임의 접근 우수·ext2/ext3/ext4의 표준.
④ **연속 할당은 외부 단편화가 전혀 없고**, **연결 할당은 임의 접근이 O(1)** 로 매우 빠르며, **i-node는 대용량 파일을 절대 지원할 수 없어** 최대 파일 크기가 항상 단일 블록 크기로 제한된다.

### 문제 2
**정답 ④**. PCB는 **커널이 관리하는 자료구조**로 프로세스마다 하나씩 존재하며, CPU 캐시가 아니라 커널 메모리(주로 커널 힙)에 저장된다. 문맥 교환은 필연적으로 **오버헤드**를 발생시킨다(수 마이크로초).
- ✅ 정답 근거: 문맥 교환 중에는 유용한 사용자 작업이 수행되지 않아 100% 오버헤드. 그래서 스케줄러는 이를 최소화하려 함.
- ⚠️ 함정: "PCB가 CPU 캐시에 저장", "문맥 교환이 오버헤드가 아님", "처리량 극대화" 는 완전한 오답. PCB는 커널 자료구조.
- 🔑 암기: **PCB = 커널 자료구조** · **문맥 교환 = 순수 오버헤드**. 스레드 간 교환은 프로세스 간보다 저렴하지만 0이 아님.

### 문제 3
**정답 ④**. 선점/비선점 분류가 완전히 뒤바뀜. **비선점 = FCFS·SJF·HRRN·비선점 우선순위** / **선점 = RR·SRT·선점 우선순위·MLQ/MLFQ**. HRRN 공식은 **(대기+서비스)/서비스** — 대기가 길어질수록 응답률이 커져 우선순위 상승(에이징).
- ✅ 정답 근거: RR은 타이머로 강제 회수하므로 선점 · FCFS는 자발적 반납까지 기다리므로 비선점.
- ⚠️ 함정: "RR·SRT가 비선점", "FCFS·SJF가 선점", "HRRN 공식 뒤집기" 는 4중 함정. 정보처리기사 최빈출 트랩.
- 🔑 암기: **선점 4대장 = RR · SRT · 선점우선 · MLFQ** / **비선점 4대장 = FCFS · SJF · HRRN · 비선점우선**.

### 문제 4
**정답 ②**. **SJF(비선점)** = t=0 동시 도착·버스트 오름차순 처리. 순서: **P1(3) → P2(5) → P4(6) → P3(8)**.
- P1 대기 = 0 (t=0에 시작)
- P2 대기 = 3 (P1이 3만큼 실행 후 시작)
- P4 대기 = 3+5 = 8
- P3 대기 = 3+5+6 = 14
- 평균 = (0+3+8+14)/4 = **25/4 = 6.25**

- ✅ 정답 근거(공식): SJF 평균 대기 = Σ(각 프로세스 앞의 총 서비스 시간) / N. 동시 도착 시 짧은 것부터.
- ⚠️ 함정: 계산 착오로 P1(3)을 놓치고 P4(6) 먼저 실행하거나, FCFS 순서 그대로(P1→P2→P3→P4)로 계산하면 오답. SJF는 도착 순서 무시·**버스트 오름차순**.
- 🔑 암기: **SJF 순서 = 버스트 오름차순** · 대기 = 시작 - 도착 · 반환(Turnaround) = 대기 + 버스트.

### 문제 5
**정답 ④**. RR은 대표적인 **선점 스케줄링**. 시간 할당량 만료 시 타이머 인터럽트로 CPU를 강제 회수해 준비 큐 끝으로 보낸다. 문맥 교환은 필수적으로 발생하며, 할당량이 작을수록 문맥 교환 오버헤드가 커진다.
- ✅ 정답 근거: RR의 정체성 = 시간 할당량 기반 강제 선점. 이것이 대화형 응답성의 근거.
- ⚠️ 함정: "RR = 비선점", "자발적 반납만 기다림", "문맥 교환 없음" 은 RR의 정의 자체를 뒤집는 오답.
- 🔑 암기: **RR = 시분할 = 강제 선점 = 타이머 인터럽트 = 문맥 교환 필수**.

### 문제 6
**정답 ④**. 페이징=고정·내부 단편화, 세그멘테이션=가변·외부 단편화가 정확. ④는 두 특성을 뒤바꿈. TLB는 **페이징의 주소 변환 캐시**로 특히 페이징에서 필수.
- ✅ 정답 근거: 페이징의 마지막 페이지에 프로세스 크기가 안 맞으면 자투리(내부 단편화). 세그먼트를 가변 크기로 배치하다 보면 사이 공간이 남음(외부 단편화).
- ⚠️ 함정: "페이징=가변·외부", "세그=고정·내부", "TLB=세그 전용" 은 3중 함정. 매년 회차 필출.
- 🔑 암기: **페이징 = 고정 = 내부** / **세그멘테이션 = 가변 = 외부** / **TLB = 페이징 캐시**.

### 문제 7
**정답 ③**. 프레임=3·FIFO 손추적:
- 참조 7 → 부재(F1) [7]
- 0 → 부재(F2) [7,0]
- 1 → 부재(F3) [7,0,1]
- 2 → 부재(F4·7 교체) [0,1,2]
- 0 → 적중
- 3 → 부재(F5·0 교체) [1,2,3]
- 0 → 부재(F6·1 교체) [2,3,0]
- 4 → 부재(F7·2 교체) [3,0,4]
- 2 → 부재(F8·3 교체) [0,4,2]
- 3 → 부재(F9·0 교체) [4,2,3]
- 0 → 부재(F10·4 교체) [2,3,0]
- 3 → 적중
- 2 → 적중
- 1 → 부재(F11·2 교체) [3,0,1]
- 2 → 부재(F12·3 교체) [0,1,2]
- 0 → 적중
- 1 → 적중
- 7 → 부재(F13·0 교체) [1,2,7]
- 0 → 부재(F14·1 교체) [2,7,0]
- 1 → 부재(F15·2 교체) [7,0,1]

- ✅ 정답 근거: FIFO는 큐로 관리. 가장 먼저 들어온 페이지가 먼저 교체 대상.

### 문제 8
**정답 ③**. 프레임=3·LRU 손추적:
- 1 → 부재(F1) [1]
- 2 → 부재(F2) [1,2]
- 3 → 부재(F3) [1,2,3]
- 4 → 부재(F4·1 교체·1이 가장 오래됨) [2,3,4]
- 1 → 부재(F5·2 교체) [3,4,1]
- 2 → 부재(F6·3 교체) [4,1,2]
- 5 → 부재(F7·4 교체) [1,2,5]
- 1 → 적중 (1이 최근으로 이동) [2,5,1]
- 2 → 적중 [5,1,2]
- 3 → 부재(F8·5 교체) [1,2,3]
- 4 → 부재(F9·1 교체) [2,3,4]
- 5 → 부재(F10·2 교체) [3,4,5]

**총 10회** = ③.

- ✅ 정답 근거: LRU는 참조 시간이 가장 오래된 페이지를 교체. 참조 시 최근 사용으로 이동.
- ⚠️ 함정: 적중된 페이지가 갱신되는 걸 잊고 오래된 상태로 두면 오답. 참조 = 갱신 필수.
- 🔑 암기: **LRU = 최근 참조 최상단** · **Belady 이상현상 없음**(스택 알고리즘).

### 문제 9
**정답 ④**. **Belady 이상현상 = FIFO에서만 발생**. LRU·Optimal은 **스택 알고리즘(Stack Algorithm)** 이라 프레임 수를 늘리면 반드시 부재율이 감소하거나 유지된다. ④는 Belady 정리를 정반대로 왜곡.
- ✅ 정답 근거: 스택 알고리즘의 성질 = 프레임 k개일 때의 페이지 집합 ⊆ 프레임 k+1개일 때의 페이지 집합. 이 성질을 만족하면 이상현상 없음.
- ⚠️ 함정: "Belady는 LRU에서 발생", "FIFO는 안전", "프레임 증가 = 항상 부재 감소" 3중 함정.
- 🔑 암기: **Belady 이상현상 = FIFO 전용** / **스택 알고리즘(LRU·Optimal·LFU) = 안전**.

### 문제 10
**정답 ④**. LFU = **참조 횟수** / Clock = **참조 비트** / MRU = 최근 참조 우선 교체. ④는 세 알고리즘 모두를 뒤바꿈. Clock은 참조 비트로 근사 LRU, MRU는 특수 워크로드용.
- ✅ 정답 근거: Clock 알고리즘의 핵심 = 참조 비트 리셋 + 다음 페이지 검사 반복. Linux 커널의 실제 페이지 교체는 Clock 계열.
- ⚠️ 함정: "LFU=시간순", "Clock=크기", "MRU 항상 우수" 는 특성 완전 왜곡.
- 🔑 암기: **LFU = 카운트** / **Clock = 참조 비트 시계** / **MRU = 대용량 순차 스캔 특수**.

### 문제 11
**정답 ④**. 교착상태 **4대 필요조건 = 상호배제 · 점유대기 · 비선점 · 환형대기**. 4개 **모두 동시 성립**해야 교착상태 가능. ④는 조건을 잘못 나열(선점·순차대기)하고 "하나만 성립해도 발생" 은 정반대 서술.
- ✅ 정답 근거: 4개 중 하나만 무너뜨려도 교착상태는 발생하지 않음 = 예방(Prevention)의 원리.
- ⚠️ 함정: "선점이 필요조건", "비선점 무관", "하나만 성립해도" 는 3중 함정.
- 🔑 암기: **상·점·비·환** = 상호배제·점유대기·비선점·환형대기. **모두 동시 성립 시**에만 교착 가능.

### 문제 12
**정답 ④**. 은행원 알고리즘은 **회피(Avoidance)** · 자원 순서 부여는 **예방(Prevention)** · 타조 알고리즘은 **무시(Ignore)**. ④는 3가지 전략을 모두 뒤바꿈.
- ✅ 정답 근거: 회피 = 사전에 안전 상태 검사(은행원). 예방 = 4조건 중 하나를 사전 차단. 탐지·회복 = 사후 발견 후 대응. 무시 = 무시.
- ⚠️ 함정: "은행원=탐지", "자원순서=회피", "타조=탐지·회복" 3중 함정.
- 🔑 암기: **예방 = 사전 차단** / **회피(은행원) = 사전 검사** / **탐지·회복 = 사후 대응** / **무시(타조) = 실용주의**.

### 문제 13
**정답 ④**. 가상 메모리는 **디스크의 스왑 공간까지 활용해 물리 메모리보다 큰 주소공간**을 제공. 워킹셋 = **특정 시간창 Δ 내 참조한 페이지 집합**. 스래싱은 다중 프로그래밍 정도를 **낮추어** 해결.
- ✅ 정답 근거: 가상 메모리의 존재 이유 = 물리보다 큰 논리 주소. 스래싱은 CPU 활용률 급락 → 스케줄러가 오히려 다중 프로그램 증가시키는 악순환(정정 필요).
- ⚠️ 함정: "물리와 동일 이하", "워킹셋 = 총 페이지", "스래싱은 다중 프로그램 증가로 해결" 은 3중 정반대.
- 🔑 암기: **가상 메모리 = 물리 초월** · **워킹셋 = 시간창 Δ 참조 집합** · **스래싱 해결 = 다중 프로그램 정도 감소 + 워킹셋 유지**.

### 문제 14
**정답 ④**. SSTF는 특정 트랙 근처 요청이 몰리면 **원거리 요청 기아 발생**. SCAN은 양방향 왕복. C-SCAN은 한 방향 서비스 후 **되돌아가서** 같은 방향 반복. 세 알고리즘 모두 FCFS보다 항상 우수한 건 아님(워크로드 의존).
- ✅ 정답 근거: SSTF의 기아 = 트랙 5~10에 요청 폭주 시 트랙 100의 요청 무기한 대기. SCAN이 이를 완화. C-SCAN이 대기 편차를 더 줄임.
- ⚠️ 함정: "SSTF 기아 없음", "SCAN 원형", "C-SCAN 양방향" 3중 함정.
- 🔑 암기: **SSTF = 최단 · 기아 O** / **SCAN = 엘리베이터 왕복** / **C-SCAN = 원형 단방향**.

### 문제 15
**정답 ④**. 연속 할당은 **외부 단편화 발생**. 연결 할당은 임의 접근 **O(n)** (연결리스트 순회). i-node는 **삼중 간접 블록**으로 대용량 지원(ext4 최대 파일 16TB). ④는 세 방식 모두 완전 왜곡.
- ✅ 정답 근거: i-node의 다중 간접 블록 = 직접 12개 + 단일 간접(블록 포인터 하나) + 이중(포인터의 포인터) + 삼중(3중 참조) 로 4단계 확장.
- ⚠️ 함정: "연속=단편화 없음", "연결=임의 O(1)", "i-node 대용량 불가" 는 3중 왜곡.
- 🔑 암기: **연속(빠름·외부단편화)** / **연결(FAT·순차만 유리)** / **인덱스/i-node(직접+간접 3단·대용량)**.
