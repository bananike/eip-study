## 🎯 오늘의 기출 8문제

### 문제 1 [빈출 유형: 스키마 3단계 — 개념 스키마 정의]
조직 전체의 관점에서 데이터베이스의 전체적인 논리적 구조를 정의하며, 모든 응용 프로그램과 사용자가 필요로 하는 데이터를 통합한 조직 전체의 데이터베이스 명세에 해당하는 스키마는?
① 외부 스키마(External Schema)
② 개념 스키마(Conceptual Schema)
③ 내부 스키마(Internal Schema)
④ 서브 스키마(Sub Schema)

### 문제 2 [빈출 유형: 데이터 독립성]
데이터베이스의 내부 스키마(물리적 저장 구조)가 변경되더라도 개념 스키마나 응용 프로그램에 영향을 주지 않는 특성을 무엇이라 하는가?
① 논리적 데이터 독립성(Logical Data Independence)
② 물리적 데이터 독립성(Physical Data Independence)
③ 구조적 데이터 독립성(Structural Data Independence)
④ 내용적 데이터 독립성(Content Data Independence)

### 문제 3 [빈출 유형: DBMS 필수 기능 3종]
DBMS의 필수 기능 중, 데이터의 정확성과 안전성을 유지하기 위해 무결성·보안·권한 검사·병행 제어 등을 담당하는 기능은?
① 정의 기능(Definition Facility)
② 조작 기능(Manipulation Facility)
③ 제어 기능(Control Facility)
④ 회복 기능(Recovery Facility)

### 문제 4 [빈출 유형: E-R 다이어그램 표기법]
E-R(Entity-Relationship) 다이어그램의 기호 표기로 옳지 않은 것은?
① 개체(Entity) — 사각형(□)
② 속성(Attribute) — 타원(○)
③ 관계(Relationship) — 마름모(◇)
④ 개체와 속성의 연결 — 점선(┄)

### 문제 5 [빈출 유형: 순수 관계 연산자 vs 일반 집합 연산자]
다음 중 관계대수(Relational Algebra)의 순수 관계 연산자(Pure Relational Operator)에 해당하지 않는 것은?
① Select(σ)
② Project(π)
③ Join(⋈)
④ 교차곱(Cartesian Product, ×)

### 문제 6 [빈출 유형: 관계대수 vs 관계해석]
관계대수(Relational Algebra)와 관계해석(Relational Calculus)에 대한 설명으로 옳지 않은 것은?
① 관계대수는 원하는 결과를 얻기 위한 절차를 명시하는 절차적(procedural) 언어이다.
② 관계해석은 원하는 결과만 명시하는 비절차적(non-procedural) 언어이다.
③ 관계해석은 수학의 술어 해석(Predicate Calculus)에 기반을 둔다.
④ 관계대수와 관계해석은 표현 능력이 서로 달라 상호 변환이 불가능하다.

### 문제 7 [빈출 유형: SELECT(σ) vs PROJECT(π) 구분]
관계대수에서 릴레이션을 대상으로 주어진 조건(Predicate)을 만족하는 튜플(행)들의 부분 집합을 추출하는 수평적(horizontal) 연산자는?
① Select(σ)
② Project(π)
③ Join(⋈)
④ Division(÷)

### 문제 8 [빈출 유형: 관계 데이터 모델 용어]
릴레이션(Relation)에서 속성(Attribute)의 개수를 의미하는 용어는?
① 차수(Degree)
② 카디널리티(Cardinality)
③ 도메인(Domain)
④ 튜플(Tuple)

## 🔍 오답 역추적 개념 발췌

- 스키마 3단계: 외부(사용자/응용 뷰) / 개념(조직 전체 통합) / 내부(물리 저장)
- 외부 스키마 = 서브 스키마(Sub Schema), 다수 존재 가능
- 개념 스키마 = DB당 하나, DBA가 관리
- 사상(Mapping): 외부/개념 사상(논리적), 개념/내부 사상(물리적)
- 논리적 독립성 ↔ 개념 스키마 변경 / 물리적 독립성 ↔ 내부 스키마 변경
- DBMS 필수 3기능: 정의(DDL) / 조작(DML) / 제어(DCL·무결성·보안·병행)
- E-R 기호: 개체=□, 속성=○(타원), 관계=◇, 연결=실선, 다중값속성=이중타원, 기본키속성=밑줄
- 관계대수 순수 연산 4종: σ(Select)·π(Project)·⋈(Join)·÷(Division)
- 관계대수 일반 집합 연산 4종: ∪(합집합)·∩(교집합)·−(차집합)·×(교차곱)
- σ(Select)=행(수평) / π(Project)=열(수직)
- 관계대수=절차적(How), 관계해석=비절차적(What), 표현력 동등
- 관계해석: 튜플 관계해석 / 도메인 관계해석
- 관계 용어: 차수(Degree)=속성 수, 카디널리티(Cardinality)=튜플 수, 도메인=속성이 가질 수 있는 값의 집합

## 💡 상세 해설 (자기 설명과 대조)

### 문제 1 — 정답 ②
- **개념 스키마(Conceptual Schema)**: 데이터베이스 전체의 논리적 구조를 조직 전체 관점에서 정의. 모든 응용·사용자의 데이터를 통합한 명세이며, DB당 단 하나만 존재하고 DBA가 관리한다.
- ①: 외부 스키마(External Schema) = 개별 사용자/응용 프로그램이 보는 부분적 뷰(서브 스키마). 여러 개 존재.
- ③: 내부 스키마(Internal Schema) = 물리적 저장 장치 관점의 구조(인덱스, 레코드 배치 등). DB당 하나.
- ④: 서브 스키마는 외부 스키마의 다른 이름. 개념 스키마가 아님.

### 문제 2 — 정답 ②
- **물리적 데이터 독립성**: 내부 스키마(저장 구조·접근 경로)가 바뀌어도 개념 스키마와 응용 프로그램이 영향을 받지 않는 성질. 개념/내부 사상(Mapping)이 변화를 흡수.
- ①: 논리적 데이터 독립성 = 개념 스키마가 변경되어도 외부 스키마/응용 프로그램에 영향 없음. (변경 주체가 다름)
- ③, ④: 표준 용어가 아닌 함정 보기.

### 문제 3 — 정답 ③
- **제어 기능(Control Facility)**: 데이터의 정확성·안전성 유지를 위한 무결성 제약, 보안·권한 검사, 병행 제어(동시성)를 담당. 대응 언어는 DCL(GRANT/REVOKE/COMMIT 등).
- ①: 정의 기능 = 데이터의 형식·구조·제약을 정의(DDL: CREATE/ALTER/DROP).
- ②: 조작 기능 = 데이터의 검색·삽입·삭제·갱신(DML: SELECT/INSERT/UPDATE/DELETE).
- ④: 회복(Recovery)은 제어 기능의 일부로 다루며, DBMS '필수 3기능'은 정의·조작·제어로 분류.

### 문제 4 — 정답 ④
- **개체와 속성(또는 개체와 관계)의 연결은 실선(선)으로 표시**한다. "점선"이라 한 ④가 오답.
- ①: 개체(Entity) = 사각형(□). 정답.
- ②: 속성(Attribute) = 타원(○). 정답. (다중값 속성은 이중 타원, 유도 속성은 점선 타원)
- ③: 관계(Relationship) = 마름모(◇). 정답.

### 문제 5 — 정답 ④
- **교차곱(Cartesian Product, ×)은 일반 집합 연산자**이다. 순수 관계 연산자는 Select(σ)·Project(π)·Join(⋈)·Division(÷)의 4종.
- ①②③: 모두 순수 관계 연산자에 해당.
- 참고: 일반 집합 연산 4종 = 합집합(∪)·교집합(∩)·차집합(−)·교차곱(×).

### 문제 6 — 정답 ④
- **관계대수와 관계해석은 표현 능력(expressive power)이 동등**하며, E.F. Codd가 상호 변환 가능함을 증명했다(관계적 완전성). "상호 변환 불가능"이라 한 ④가 오답.
- ①: 관계대수는 절차적(How — 연산 순서를 명시). 정답.
- ②: 관계해석은 비절차적(What — 결과 조건만 명시). 정답.
- ③: 관계해석은 술어 해석(Predicate Calculus) 기반, 튜플 관계해석/도메인 관계해석으로 나뉜다. 정답.

### 문제 7 — 정답 ①
- **Select(σ)**: 조건(Predicate)을 만족하는 튜플(행)의 부분 집합을 추출하는 수평적(horizontal) 연산. 표기 σ<조건>(R).
- ②: Project(π) = 지정한 속성(열)만 추출하는 수직적(vertical) 연산.
- ③: Join(⋈) = 공통 속성을 기준으로 두 릴레이션을 결합.
- ④: Division(÷) = R이 S의 모든 튜플과 관련된 R의 튜플을 추출("~모두에 대해").

### 문제 8 — 정답 ①
- **차수(Degree)** = 릴레이션의 속성(Attribute, 열) 개수.
- ②: 카디널리티(Cardinality) = 튜플(Tuple, 행) 개수.
- ③: 도메인(Domain) = 한 속성이 가질 수 있는 원자값(atomic value)들의 집합.
- ④: 튜플(Tuple) = 릴레이션의 한 행(레코드).

### 관계대수 연산자 정리 (사후 검증용)
- 순수 관계 연산(4): σ Select(행 선택), π Project(열 선택), ⋈ Join(결합), ÷ Division(나눗셈)
- 일반 집합 연산(4): ∪ 합집합, ∩ 교집합, − 차집합, × 교차곱(Cartesian Product)
- Join 종류: Theta Join, Equi Join, Natural Join, Outer Join(Left/Right/Full), Semi Join

### 관계해석 (사후 검증용)
- 튜플 관계해석(Tuple Relational Calculus): 변수가 튜플 단위
- 도메인 관계해석(Domain Relational Calculus): 변수가 도메인(속성값) 단위
- 정량자: 전칭(∀, for all), 존재(∃, there exists)
- 관계대수와 표현력 동등(관계적 완전성, E.F. Codd)

### DBMS 정의·역사 (사후 검증용)
- DBMS 정의: 응용 프로그램과 데이터 사이를 중재하여 DB를 공용으로 사용·관리하는 소프트웨어
- 데이터베이스 특징: 실시간 접근성, 계속적 변화, 동시 공유, 내용에 의한 참조
- 데이터 언어: DDL(정의)·DML(조작)·DCL(제어)

### E-R 모델 (사후 검증용)
- 1976년 P. Chen이 제안
- 기호: 개체(□), 속성(○), 관계(◇), 연결(실선), 다중값 속성(이중 타원), 유도 속성(점선 타원), 기본키 속성(밑줄)
- 관계 대응(Cardinality): 1:1, 1:N, N:M
