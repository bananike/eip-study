## 🎯 오늘의 기출 8문제

### 문제 1 [출처: 빈출 유형 (다회 출제)]
다음 설명에 해당하는 소프트웨어 아키텍처 패턴은?

> 시스템을 서브시스템 셋(데이터 처리, 사용자 화면, 제어 로직)으로 분리하여 상호 영향을 최소화한다. 화면을 담당하는 부분과 비즈니스 로직을 담당하는 부분이 분리되어 유지보수가 용이하다.

① 파이프-필터(Pipe-Filter) 패턴
② MVC(Model-View-Controller) 패턴
③ 브로커(Broker) 패턴
④ 마스터-슬레이브(Master-Slave) 패턴

### 문제 2 [출처: 빈출 유형 (다회 출제)]
파이프-필터(Pipe-Filter) 아키텍처 패턴에 대한 설명으로 옳지 않은 것은?

① 데이터 스트림을 필터를 거쳐 순차적으로 변환·처리한다.
② 필터 컴포넌트는 재사용성이 높고 새로운 필터로 교체·추가가 쉽다.
③ 필터 간 데이터 이동에서 데이터 변환에 따른 오버헤드가 발생할 수 있다.
④ 중앙 저장소(공유 데이터)를 두고 여러 컴포넌트가 동시에 접근하는 방식이다.

### 문제 3 [출처: 빈출 유형 (다회 출제)]
소프트웨어 아키텍처 패턴에 해당하지 않는 것은?

① 계층화(Layered) 패턴
② 클라이언트-서버(Client-Server) 패턴
③ 팩토리 메서드(Factory Method) 패턴
④ 이벤트-버스(Event-Bus) 패턴

### 문제 4 [출처: 빈출 유형 (다회 출제)]
사용자 인터페이스(UI) 설계의 기본 원칙에 대한 설명으로 옳지 않은 것은?

① 직관성 — 누구나 쉽게 이해하고 사용할 수 있어야 한다.
② 유효성 — 사용자의 목적을 정확하고 완벽하게 달성할 수 있어야 한다.
③ 학습성 — 누구나 쉽게 배우고 익힐 수 있어야 한다.
④ 유연성 — 정보의 중요도에 따라 화면을 단순·간결하게 구성해야 한다.

### 문제 5 [출처: 빈출 유형 (다회 출제)]
다음 중 사용자 인터페이스(UI)의 유형에 대한 설명으로 옳은 것은?

① CLI(Command Line Interface)는 그래픽 아이콘을 마우스로 조작하는 방식이다.
② GUI(Graphical User Interface)는 명령어를 텍스트로 입력하는 방식이다.
③ NUI(Natural User Interface)는 사용자의 말·행동(터치·제스처·음성) 등 자연스러운 동작으로 조작하는 방식이다.
④ OUI(Organic User Interface)는 키보드만으로 명령을 입력하는 방식이다.

### 문제 6 [출처: 빈출 유형 (다회 출제)]
EAI(Enterprise Application Integration) 구축 유형 중 다음 설명에 해당하는 것은?

> 애플리케이션을 1:1로 직접 연결하는 가장 기본적인 통합 방식으로, 솔루션 없이 구축 가능하나 연결 대상이 늘어나면 유지보수·재사용이 어렵다.

① Hub & Spoke
② Message Bus(ESB 방식)
③ Point-to-Point
④ Hybrid

### 문제 7 [출처: 빈출 유형 (다회 출제)]
ESB(Enterprise Service Bus)에 대한 설명으로 가장 옳은 것은?

① 애플리케이션 간을 1:1로 직접 연결하여 결합도를 높이는 방식이다.
② 서비스 중심의 느슨한 결합(loosely coupled) 방식으로 메시지 기반 연계·통합을 지원한다.
③ 하나의 허브 시스템에 모든 연결을 집중시키는 중앙 집중형 물리 장비이다.
④ 데이터베이스와 응용 프로그램을 직접 연결하는 DB 접속 전용 미들웨어이다.

### 문제 8 [출처: 빈출 유형 (다회 출제)]
미들웨어(Middleware) 종류에 대한 설명으로 옳지 않은 것은?

① MOM(Message Oriented Middleware) — 메시지 기반의 비동기형 메시지 전달을 지원한다.
② RPC(Remote Procedure Call) — 원격 프로시저(함수)를 로컬 호출처럼 사용하게 한다.
③ TP-Monitor(Transaction Processing Monitor) — 트랜잭션을 처리·감시하며 안정적 운영을 보장한다.
④ WAS(Web Application Server) — 분산 객체 간 통신을 중계하는 객체 지향 미들웨어이다.

## 💡 상세 해설 (자기 설명과 대조)

### 문제 1
정답: ②
- 제공 해설: 데이터(Model)·화면(View)·제어(Controller) 셋으로 분리하여 상호 영향을 줄이고 유지보수성을 높이는 것은 MVC 패턴이다.
- 오답 이유: ① 파이프-필터는 데이터 스트림을 필터로 변환하는 구조 / ③ 브로커는 분산 컴포넌트 간 요청을 중개 / ④ 마스터-슬레이브는 작업을 분배하여 슬레이브가 처리.

### 문제 2
정답: ④
- 제공 해설: 파이프-필터는 데이터가 파이프를 따라 필터를 순차 통과하며 변환되는 구조이다. "중앙 공유 저장소에 여러 컴포넌트가 동시 접근"하는 것은 **블랙보드(Blackboard)** 패턴의 설명으로, 옳지 않다.
- 오답 이유: ①②③ 모두 파이프-필터의 올바른 특징(순차 변환, 필터 재사용·교체 용이, 변환 오버헤드).

### 문제 3
정답: ③
- 제공 해설: 팩토리 메서드는 GoF **디자인 패턴(생성)** 으로 아키텍처 패턴이 아니다.
- 오답 이유: ① 계층화 ② 클라이언트-서버 ④ 이벤트-버스는 모두 대표적 아키텍처 패턴.

### 문제 4
정답: ④
- 제공 해설: 유연성(Flexibility)은 "사용자의 요구사항을 최대한 수용하고 오류를 최소화한다"는 원칙이다. "정보 중요도에 따라 단순·간결 구성"은 유연성의 정의가 아니므로 옳지 않다.
- 오답 이유: ①②③ 직관성·유효성·학습성의 정의는 모두 올바름.

### 문제 5
정답: ③
- 제공 해설: NUI는 터치·제스처·음성 등 자연스러운 동작으로 조작하는 인터페이스로 옳다.
- 오답 이유: ① CLI는 텍스트 명령어 입력(아이콘·마우스 ✗) / ② GUI는 그래픽 아이콘·마우스 조작(텍스트 명령 ✗) / ④ OUI는 모든 사물에 입출력이 통합된 유기적 인터페이스(키보드 전용 ✗).

### 문제 6
정답: ③
- 제공 해설: 1:1 직접 연결로 솔루션 없이 구축 가능하나 연결 대상 증가 시 유지보수·재사용이 어려운 방식은 Point-to-Point이다.
- 오답 이유: ① Hub&Spoke는 중앙 허브 집중형 / ② Message Bus는 미들웨어(버스) 경유 / ④ Hybrid는 Hub&Spoke + Message Bus 혼합.

### 문제 7
정답: ②
- 제공 해설: ESB는 서비스 중심의 느슨한 결합으로 메시지 기반 연계·통합을 지원하는 아키텍처(미들웨어 버스)이다.
- 오답 이유: ① 1:1 직접 연결은 Point-to-Point(결합도 ↑) / ③ 중앙 집중 물리 허브는 Hub&Spoke 설명 / ④ DB 접속 전용 미들웨어는 ESB가 아님.

### 문제 8
정답: ④
