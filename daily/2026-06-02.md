### 문제 4 [출처: 2022년 1회 정보처리기사 필기 — 유사 기출]
클라우드 컴퓨팅 서비스 모델에 대한 설명으로 옳지 않은 것은?
① IaaS(Infrastructure as a Service)는 서버, 스토리지, 네트워크 등 하드웨어 인프라를 가상화하여 제공한다.
② PaaS(Platform as a Service)는 응용 프로그램을 개발·실행할 수 있는 플랫폼(OS, 미들웨어, 런타임)을 제공한다.
③ SaaS(Software as a Service)는 사용자가 OS와 미들웨어를 직접 관리하며 응용 소프트웨어를 설치한다.
④ DaaS(Desktop as a Service)는 사용자 데스크톱 환경을 클라우드에서 가상으로 제공한다.

## 🔍 오답 역추적 개념 발췌

오늘 문제 풀이 시 헷갈릴 가능성이 높은 키워드(설명 없음, 정답표·해설 참조).
- BaaS(Blockchain as a Service) ↔ SaaS / PaaS / IaaS / DaaS 구별
- SDDC(Software Defined Data Center) — 데이터센터 자원 가상화
- OTT(Over The Top) — 인터넷 기반 콘텐츠 전송 서비스
- Wi-SUN — IEEE 802.15.4g 기반 저전력 장거리 IoT 통신(스마트미터링)
- Mashup ↔ MapReduce ↔ Memristor ↔ MVNO 함정 어휘
- Hadoop 생태계: HDFS / MapReduce / YARN / Hive / Pig / Sqoop / Tajo / Spark / Oozie / ZooKeeper
- Sqoop = SQL ↔ Hadoop 양방향 데이터 전송 도구
- Tajo = 하둡 기반 대용량 데이터 웨어하우스(국내 개발)
- Hive = HiveQL(SQL 유사) → MapReduce 변환 쿼리 엔진
- Pig = 데이터 흐름 언어(Pig Latin)
- SaaS는 사용자가 OS/미들웨어를 관리하지 "않는다" (제공자가 관리)
- 디지털 트윈 ↔ 메타버스 구별: 디지털 트윈 = 현실 자산의 시뮬레이션 / 메타버스 = 가상세계 사회·경제 활동
- 엣지 컴퓨팅 ↔ 포그(Fog) 컴퓨팅: 엣지는 단말, 포그는 단말~클라우드 사이 계층
- 그리드 컴퓨팅 — 분산된 컴퓨터 자원을 가상의 단일 슈퍼컴퓨터로 통합
- MapReduce ↔ MongoDB ↔ Memcached ↔ Mesos 약어 함정
- 비콘(Beacon) — BLE 기반 근거리 무선(50m 내외)
- N-Screen — 끊김 없는 멀티 디바이스 콘텐츠 이용
- Qubit — 양자 중첩(0과 1 동시 표현)
- Wi-SUN ≠ 차량 통신, 차량 통신은 V2X(C-V2X/DSRC)

## 💡 상세 해설 (자기 설명과 대조)

### 문제 1
정답: ②
- 제공 해설: BaaS(Blockchain as a Service)는 블록체인 개발 환경을 클라우드 서비스로 제공하는 모델. 노드 추가·제거가 용이하고, 인프라를 추상화해 블록체인 응용 개발에 집중할 수 있다.
- 오답 이유:
  - ①: OTT(Over The Top) — 인터넷 기반 콘텐츠(영상·음성) 전송 서비스. 블록체인과 무관.
  - ③: SDDC(Software Defined Data Center) — 데이터센터 자원(서버/스토리지/네트워크) 전체를 소프트웨어로 가상화·관리.
  - ④: Wi-SUN — IEEE 802.15.4g 기반 저전력 장거리 IoT 통신(주로 스마트미터). 블록체인과 무관.
- 함정: 보기 4개 모두 IT 신기술 빈출 약어. 키워드 "블록체인 + 클라우드 + 응용 개발 추상화" = BaaS로 즉시 매칭.

### 문제 2
정답: ①
- 제공 해설: 매시업(Mashup)은 웹에서 제공되는 정보/서비스를 결합해 새로운 서비스를 만드는 기술. 대표 예시로 지도 API + 부동산 데이터 결합.
- 오답 이유:
  - ②: Memristor — 메모리+레지스터 합성어. 전류 흐름 이력을 저장하는 차세대 비휘발성 소자.
  - ③: MVNO(Mobile Virtual Network Operator) — 가상 이동통신망 사업자(알뜰폰).
  - ④: MapReduce — 대용량 데이터 분산 병렬 처리 모델(문제 7 정답).
- 함정: M으로 시작하는 약어 함정 보기 다발. 약어 풀네임 암기 필수.

### 문제 3
정답: ①
- 제공 해설: Sqoop(에스큐쿱)은 SQL과 Hadoop의 합성어. RDBMS ↔ HDFS 간 대용량 데이터를 효율적으로 import/export 하기 위해 설계된 도구.
- 오답 이유:
  - ②: Tajo — 하둡 기반 대용량 데이터 웨어하우스 시스템(국내 Apache 프로젝트). 분석 쿼리용.
  - ③: Hive — HiveQL(SQL 유사)로 작성한 쿼리를 MapReduce 작업으로 변환해 실행.
  - ④: Pig — Pig Latin이라는 데이터 흐름 언어로 MapReduce 작업을 작성하는 도구.
- 함정: Hive와 Sqoop 모두 RDBMS 친화적 도구로 혼동 빈번. "데이터 전송(import/export)" 키워드 = Sqoop.

### 문제 4
정답: ③
- 제공 해설: SaaS는 응용 소프트웨어까지 제공자가 모두 관리·제공한다. 사용자는 OS·미들웨어·앱을 직접 설치/관리하지 "않는다". 사용자는 단지 웹·앱으로 접속해 사용.
- 오답 이유:
  - ①: IaaS 설명 정확. 서버·스토리지·네트워크 등 인프라 가상화 제공.
  - ②: PaaS 설명 정확. OS·미들웨어·런타임 포함 개발/실행 플랫폼 제공.
  - ④: DaaS 설명 정확. 가상 데스크톱 인프라(VDI)를 클라우드로 제공.
- 함정: SaaS = "Software 다 제공" → 사용자가 관리할 것이 가장 적음. 보기 ③은 PaaS 설명을 SaaS에 끼워 넣은 함정.

### 문제 5
정답: ①
- 제공 해설: 디지털 트윈(Digital Twin)은 현실의 물리 자산을 디지털 공간에 동일하게 구현해 실시간 데이터로 시뮬레이션·예측·최적화를 수행. 2002년 Michael Grieves가 개념 제안, GE가 항공기 엔진 모니터링에 도입.
- 오답 이유:
  - ②: 메타버스 — 가상세계에서 사회·경제·문화 활동이 이루어지는 3D 가상공간. "현실 자산의 디지털 복제"가 아니라 "독립된 가상세계".
  - ③: IoT(사물인터넷) — 사물에 센서/통신을 부착해 데이터를 주고받는 네트워크. 디지털 트윈의 데이터 수집 기반이지만 동일 개념은 아님.
  - ④: AR(증강현실) — 현실 영상에 가상 정보를 겹쳐 보여주는 기술. 시뮬레이션·예측 기능 아님.
- 함정: "쌍둥이" 키워드 = Digital Twin. 메타버스와의 변별: 실제 자산 ↔ 가상 자산 매핑 여부.

### 문제 6
정답: ②
- 제공 해설: 엣지 컴퓨팅(Edge Computing)은 데이터 발생 지점 근처(단말, 게이트웨이)에서 처리해 지연을 줄이고 대역폭 부담을 낮춘다. 자율주행·스마트팩토리·IoT 실시간 응답에 필수.
- 오답 이유:
  - ①: 그리드 컴퓨팅 — 분산된 다수의 컴퓨터 자원을 가상의 단일 슈퍼컴퓨터처럼 통합. 발생 지점 처리와는 다름.
  - ③: 유틸리티 컴퓨팅 — 컴퓨팅 자원을 전기·수도처럼 사용량 기반 종량제로 제공(과금 모델 개념).
  - ④: 클러스터 컴퓨팅 — 동일 네트워크의 컴퓨터를 묶어 하나의 시스템처럼 운용(고가용성·고성능).
- 함정: 엣지 ↔ 포그 변별 — 엣지(단말 가장 가까이) vs 포그(단말과 클라우드 사이 계층). 문제 키워드 "발생 지점 근처(단말)" = 엣지.

### 문제 7
정답: ①
- 제공 해설: MapReduce는 구글이 2004년 발표한 대용량 데이터 분산 병렬 처리 모델. Map(키-값 쌍 생성) → Shuffle → Reduce(집계) 단계로 작업을 나눠 수행. Apache Hadoop이 이를 오픈소스로 구현.
- 오답 이유:
  - ②: Memcached — 분산 메모리 객체 캐싱 시스템(웹 가속용 인메모리 캐시).
  - ③: MongoDB — NoSQL 문서(Document) 지향 DB.
  - ④: Mesos — Apache Mesos. 데이터센터 자원 관리·스케줄러 프레임워크.
- 함정: M으로 시작하는 분산/빅데이터 약어 함정. "Map + Reduce 단계" 키워드 = MapReduce.

### 문제 8
정답: ④
- 제공 해설: Wi-SUN은 IEEE 802.15.4g 기반 저전력 장거리(LPWA) 무선 IoT 통신. 스마트 미터링·스마트 시티 인프라용. 차량 간 통신(V2V/V2X)과는 완전히 다른 기술. 차량 통신은 DSRC, C-V2X(LTE/5G 기반) 표준.
- 오답 이유:
  - ①: 비콘 설명 정확. BLE(Bluetooth Low Energy) 기반 근거리 무선으로 위치/정보 전달.
  - ②: N-Screen 설명 정확. 멀티 디바이스 끊김 없는 콘텐츠 이용.
  - ③: Qubit 설명 정확. 양자 중첩으로 0과 1을 동시 표현하는 양자 정보 최소 단위.
- 함정: Wi-SUN의 "SUN"을 자동차/태양 등으로 오인. 실제는 Smart Utility Network. "차량 통신 = V2X" 별도 암기.

### 핵심 개념
- **클라우드 서비스 모델 책임 범위:**
  - IaaS: 사용자가 OS·미들웨어·앱 관리 / 제공자는 하드웨어·가상화 관리
  - PaaS: 사용자가 앱·데이터 관리 / 제공자는 OS·미들웨어·런타임까지 관리
  - SaaS: 사용자는 사용만 / 제공자가 앱까지 모두 관리
  - BaaS: 블록체인 인프라까지 제공자가 추상화
  - DaaS: 가상 데스크톱 자체를 클라우드로 제공
- **컴퓨팅 처리 위치 스펙트럼 (단말 → 클라우드):**
  - 엣지(Edge): 단말 가장 근접
  - 포그(Fog): 단말~클라우드 사이 게이트웨이/로컬 서버
  - 클라우드: 원격 데이터센터 중앙 집중
- **Hadoop 생태계 데이터 흐름:**
  - HDFS(저장) → MapReduce/Spark(처리) → Hive/Pig/Tajo(쿼리) → Sqoop(외부 RDBMS 연계)
  - YARN(자원 관리) / ZooKeeper(분산 조정) / Oozie(워크플로우)
- **약어 함정 그룹 (M으로 시작):**
  - Mashup(웹 결합) / MapReduce(분산 처리) / Memristor(소자) / MVNO(통신) / Memcached(캐시) / Mesos(자원관리) / MongoDB(NoSQL DB)
- **현실 ↔ 가상 융합 스펙트럼:**
  - VR: 완전 가상 / AR: 현실+가상정보 / MR: 가상+현실 상호작용 / XR: VR+AR+MR 포괄
  - Digital Twin: 현실 자산을 디지털로 복제(산업) / Metaverse: 가상세계 자체(사회)
