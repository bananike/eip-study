### 문제 3 [유사 기출: 2024년 2회 1과목 — SOLID + 응집도/결합도 정형]
객체지향 설계 원칙과 **모듈 응집도(Cohesion)/결합도(Coupling)** 에 관한 설명 중 **가장 옳지 않은 것**은?

① SOLID 5원칙 = **S**RP(단일 책임) · **O**CP(개방-폐쇄) · **L**SP(리스코프 치환) · **I**SP(인터페이스 분리) · **D**IP(의존관계 역전).
② 응집도는 **높을수록** 좋으며 강도는 **기능적(Functional) > 순차적 > 통신적 > 절차적 > 시간적 > 논리적 > 우연적(Coincidental)** 순으로 강함.
③ 결합도는 **낮을수록** 좋으며 강도는 **자료(Data) < 스탬프 < 제어 < 외부 < 공통 < 내용(Content)** 순으로 약함(자료가 가장 낮고 내용이 가장 높음).
④ **응집도는 낮을수록** 좋고 **결합도는 높을수록** 좋다. 응집도 최상은 우연적(Coincidental)이며 결합도 최상은 내용(Content) 결합이다.

### 문제 6 [유사 기출: 2022년 3회 2과목 — 블랙박스 vs 화이트박스 테스트 정형]
소프트웨어 테스트 기법 중 **블랙박스 테스트(Black-box Testing)** 와 **화이트박스 테스트(White-box Testing)** 에 관한 설명 중 **가장 옳지 않은 것**은?

① 블랙박스는 **명세(요구사항) 기반** 테스트로 내부 구조를 보지 않으며 **동등 분할, 경계값 분석, 결정 테이블, 상태 전이, 원인-결과 그래프, 오류 추정** 등이 대표 기법이다.
② 화이트박스는 **구조(코드) 기반** 테스트로 내부 흐름을 검사하며 **문장/분기/조건/조건-결정/다중 조건/기본 경로(McCabe) 커버리지, 루프 테스트, 데이터 흐름 테스트** 등이 대표 기법이다.
③ **McCabe 순환 복잡도(V(G))** = **간선(E) − 노드(N) + 2** (연결된 강한 컴포넌트 수를 P=1로 가정한 단일 진입·단일 종료 그래프 기준).
④ **경계값 분석(Boundary Value Analysis)**, **동등 분할(Equivalence Partitioning)**, **원인-결과 그래프(Cause-Effect Graph)** 는 **모두 화이트박스 테스트 기법**이며 코드 내부 분기 커버리지를 측정한다.

### 문제 7 [유사 기출: 2024년 1회 3과목 — DELETE vs TRUNCATE vs DROP 3차 비교 정형]
관계형 데이터베이스의 **DELETE / TRUNCATE / DROP** 문에 관한 설명 중 **가장 옳지 않은 것**은?

① `DELETE FROM 테이블 WHERE 조건;` — **DML**, 행 단위 로그 기록, 트랜잭션 롤백 가능, 조건 지정 가능, 테이블 구조/인덱스/트리거 **유지**.
② `TRUNCATE TABLE 테이블;` — **DDL(자동 커밋)**, 최소 로그, 롤백 **일반적으로 불가**, 조건 지정 **불가**(모든 행 삭제), 테이블 구조 **유지**, DELETE보다 **빠름**.
③ `DROP TABLE 테이블;` — **DDL**, **테이블 구조·데이터·인덱스·트리거 전부 삭제**, 롤백 불가(오라클 10g+ FLASHBACK 예외).
④ **DELETE는 DDL이며 롤백 불가**이고, **TRUNCATE는 DML이며 조건(WHERE)** 을 지정할 수 있다. **DROP은 테이블 구조는 유지하고 데이터만 삭제**한다.

### 문제 8 [유사 기출: 2022년 2회 3과목 — 조인 5종 결과 카디널리티 정형]
다음 두 테이블에 대한 조인 결과의 **행 수(카디널리티)** 로 **가장 옳지 않은 것**은? (테이블 A = 5행, 테이블 B = 3행, 매칭되는 공통 키 값은 A의 2행 ↔ B의 2행 = 총 2쌍 매칭)

① `A INNER JOIN B ON A.key = B.key` — 매칭된 행만 반환 → **2행**.
② `A LEFT OUTER JOIN B ON A.key = B.key` — A의 모든 행(매칭 없으면 B 컬럼 NULL) → **5행**.
③ `A FULL OUTER JOIN B ON A.key = B.key` — A 5 + B 매칭 안 된 1 = **6행**(A의 매칭 2 + A의 비매칭 3 + B의 비매칭 1).
④ `A CROSS JOIN B` — 데카르트 곱(m × n) → **8행**이며, `RIGHT OUTER JOIN`은 항상 `LEFT OUTER JOIN`과 결과가 동일하다.

### 문제 9 [유사 기출: 2023년 3회 3과목 — CAP 정리 + NoSQL 4유형 정형]
분산 데이터베이스의 **CAP 정리(Brewer, 2000)** 와 **NoSQL 4유형**에 관한 설명 중 **가장 옳지 않은 것**은?

① **CAP 정리** = **일관성(Consistency)·가용성(Availability)·분할 내성(Partition Tolerance)** 중 **최대 2가지만** 동시에 보장 가능. 네트워크 분할이 발생하면 C와 A 중 하나를 포기해야 한다.
② **RDBMS**는 일반적으로 **CA(일관성+가용성)** 성향, **HBase·MongoDB**는 **CP**, **Cassandra·DynamoDB·CouchDB**는 **AP** 성향으로 분류된다.
③ **NoSQL 4유형** = **Key-Value**(Redis·DynamoDB·Riak) / **Document**(MongoDB·CouchDB) / **Column-Family**(Cassandra·HBase·Bigtable) / **Graph**(Neo4j·JanusGraph). **BASE 특성**(Basically Available·Soft state·Eventually consistent)을 따른다.
④ CAP 정리에서는 **세 가지(C·A·P)를 모두 동시에** 보장할 수 있으며, NoSQL의 **BASE**는 RDBMS의 **ACID**와 동일한 **강한 일관성**을 제공한다. **Graph DB**는 **키-값** 저장 구조로 조인이 없는 단순 조회에 최적화되어 있다.

### 문제 12 [유사 기출: 2024년 2회 4과목 — 서브넷 마스크 + IPv6 정형]
네트워크 주소 및 서브넷에 관한 설명 중 **가장 옳지 않은 것**은?

① IPv4 주소 `192.168.10.0/26`의 서브넷 마스크는 `255.255.255.192`이며 서브넷당 **총 64개 주소** 중 **네트워크·브로드캐스트 주소를 제외한 62개**를 호스트에 할당할 수 있다.
② IPv6 주소는 **128비트**이며 **16비트씩 8그룹**을 콜론(`:`)으로 구분한다. 연속된 `0000` 그룹은 **`::` (더블 콜론)** 로 한 번만 축약 가능하다.
③ 라우팅 프로토콜 중 **RIP**는 **거리 벡터(Distance Vector)·홉 수 최대 15·IGP**, **OSPF**는 **링크 상태(Link State)·다익스트라·Area·IGP**, **BGP**는 **경로 벡터(Path Vector)·AS 간·EGP**의 대표 프로토콜이다.
④ IPv6 주소 `2001:0db8:0000:0000:0000:ff00:0042:8329`는 `2001:db8::ff00:42:8329` **와 동일**하지만, `::`는 **한 주소 안에서 두 번 이상 사용 가능**하며 IPv6 주소 공간은 총 **2⁶⁴** 개이다.

### 문제 13 [유사 기출: 2023년 3회 5과목 — 시큐어 코딩 7대 카테고리 + OWASP Top 10 2021 정형]

### 문제 14 [유사 기출: 2022년 2회 5과목 — 정보보호 관리체계 정형]
정보보호 관리체계 및 인증에 관한 설명 중 **가장 옳지 않은 것**은?

③ **Common Criteria(CC·ISO/IEC 15408)** 는 IT 제품·시스템의 보안 평가 국제 표준이며 **EAL(Evaluation Assurance Level) 1~7단계**로 신뢰 수준을 부여한다(**EAL7 = 최고**).
④ **ISMS-P는 국제 인증**이며 **PDCA 사이클을 사용하지 않는다**. **CC EAL 등급은 1~5단계**로 구성되어 있으며 **EAL1이 최고 등급**이다.

### 문제 3
- **정답 ④** — 응집도는 **높을수록** 좋고 결합도는 **낮을수록** 좋음. 완전 반대 서술.
- **응집도 7단계 (약 → 강)**: **우연적 < 논리적 < 시간적 < 절차적 < 통신적 < 순차적 < 기능적**.
  - **우-논-시-절-통-순-기** 두문자.
- **결합도 6단계 (약 → 강)**: **자료 < 스탬프 < 제어 < 외부 < 공통 < 내용**.
  - **자-스-제-외-공-내** 두문자.
- **SOLID 5원칙**:
  - **S**RP (Single Responsibility) — 하나의 클래스는 하나의 책임만.
  - **O**CP (Open-Closed) — 확장에는 열려, 수정에는 닫혀.
  - **L**SP (Liskov Substitution) — 자식은 부모를 대체 가능해야.
  - **I**SP (Interface Segregation) — 인터페이스는 클라이언트별로 분리.
  - **D**IP (Dependency Inversion) — 추상화에 의존, 구체에 의존 X.
- **오답 함정**: 응집도/결합도의 좋고 나쁨을 정면 반전하는 초빈출 트랩.
- **실전 암기**: **응집도 高 + 결합도 低 = 좋은 설계**. 응집도 최상 = 기능적, 결합도 최상(가장 나쁜 것) = 내용.

### 문제 4
- **정답 ④** — 합병 정렬은 평균/최악 모두 **O(n log n)** 이며 **추가 메모리 O(n) 필요**(제자리 정렬 아님).
- **정렬 알고리즘 시간복잡도 표**:

| 알고리즘 | 최선 | 평균 | 최악 | 공간 | 안정성 |
|---------|------|------|------|------|-------|
| 버블(Bubble) | O(n) | O(n²) | O(n²) | O(1) | ✅ |
| 선택(Selection) | O(n²) | O(n²) | O(n²) | O(1) | ❌ |
| 삽입(Insertion) | O(n) | O(n²) | O(n²) | O(1) | ✅ |
| 셸(Shell) | O(n) | O(n^1.5) | O(n²) | O(1) | ❌ |
| 퀵(Quick) | O(n log n) | O(n log n) | **O(n²)** | O(log n) | ❌ |
| 합병(Merge) | O(n log n) | O(n log n) | **O(n log n)** | **O(n)** | ✅ |
| 힙(Heap) | O(n log n) | O(n log n) | **O(n log n)** | O(1) | ❌ |
| 기수(Radix) | O(dn) | O(dn) | O(dn) | O(n+k) | ✅ |

- **오답 함정**: 합병 정렬을 O(n²)·제자리로 왜곡. 실제로는 **분할정복 O(n log n) 안정 정렬**이며 **보조 배열 O(n) 필수**.
- **실전 암기**: **평균 O(n log n) 3인방 = 퀵·힙·합병**. **퀵만 최악 O(n²)**. **안정 정렬 = 버블·삽입·합병·기수**. **제자리(In-place) = 버블·선택·삽입·셸·힙·퀵**(합병은 제자리 아님).

### 문제 6
- **정답 ④** — **경계값 분석·동등 분할·원인-결과 그래프는 모두 블랙박스 테스트 기법**. 화이트박스 아님.
- **테스트 기법 분류**:

| 블랙박스(명세 기반) | 화이트박스(구조 기반) |
|------------------|--------------------|
| 동등 분할(Equivalence Partitioning) | 문장 커버리지(Statement) |
| 경계값 분석(Boundary Value Analysis) | 분기 커버리지(Branch/Decision) |
| 결정 테이블(Decision Table) | 조건 커버리지(Condition) |
| 상태 전이(State Transition) | 조건-결정 커버리지(Condition/Decision) |
| 원인-결과 그래프(Cause-Effect Graph) | 다중 조건 커버리지(Multiple Condition) |
| 오류 추정(Error Guessing) | 기본 경로(Basis Path·McCabe) |
| 페어와이즈(Pairwise) | 루프 테스트, 데이터 흐름 |

- **McCabe 순환 복잡도**: **V(G) = E − N + 2**(단일 진입/종료), 또는 **판단 노드 수 + 1** = **폐구간(닫힌 영역) 수**.
- **오답 함정**: ④의 세 기법은 명세를 기반으로 입력 조합·경계를 다루는 **블랙박스**이며 코드 내부 흐름을 검사하지 않음.
- **실전 암기**: **블랙박스 = 명세·입출력 조합**(내부 안 봄), **화이트박스 = 코드·경로·커버리지**(내부 봄). **경계값·동등분할·결정테이블·상태전이·원인결과·오류추정 = 블랙박스 6대장**.

### 문제 7
- **정답 ④** — **DELETE는 DML(롤백 가능)**, **TRUNCATE는 DDL(조건 X)**, **DROP은 구조까지 완전 삭제**.
- **3차 비교표**:

| 항목 | DELETE | TRUNCATE | DROP |
|------|--------|----------|------|
| 분류 | **DML** | **DDL** | **DDL** |
| WHERE 조건 | ✅ 가능 | ❌ 불가 | ❌ 불가 |
| 롤백 | ✅ 가능 | ❌ 일반적으로 불가 | ❌ 불가 |
| 로그 | 행 단위(대량) | 최소 | 최소 |
| 테이블 구조 | **유지** | **유지** | **삭제** |
| 인덱스/트리거 | 유지 | 유지 | **삭제** |
| 자동 커밋 | ❌ | ✅ | ✅ |
| 속도 | 느림 | 빠름 | 매우 빠름 |
| 트리거 실행 | ✅ | ❌ | ❌ |

- **오답 함정**: DELETE↔TRUNCATE의 DDL/DML 분류를 정면 반전, DROP이 구조를 유지한다고 왜곡. 3중 부정 함정.
- **실전 암기**: **DELETE는 데이터만·롤백O·느림·트리거O / TRUNCATE는 데이터만·롤백X·빠름·트리거X / DROP은 구조까지·롤백X**.

### 문제 8
- **정답 ④** — **CROSS JOIN 결과 = 5 × 3 = 15행**(8행 아님), **RIGHT OUTER JOIN ≠ LEFT OUTER JOIN**(테이블 순서 반대).
- **조인 5종 카디널리티**:

| 조인 | 결과 행 수 (A 5행 · B 3행 · 매칭 2쌍) |
|------|-------------------------------------|
| INNER JOIN | 매칭 = **2행** |
| LEFT OUTER JOIN | A 전부 (매칭 2 + 비매칭 3) = **5행** |
| RIGHT OUTER JOIN | B 전부 (매칭 2 + 비매칭 1) = **3행** |
| FULL OUTER JOIN | A 5 + B의 비매칭 1 = **6행** |
| CROSS JOIN | 데카르트 곱 m × n = 5 × 3 = **15행** |
| SELF JOIN | 같은 테이블 조인 (별칭 필요) |

- **오답 함정**: CROSS JOIN을 A+B로 오해(8행), LEFT/RIGHT를 대칭이라고 오해(테이블 순서에 따라 다름). RIGHT는 B가 기준이므로 A/B 뒤바꾸면 LEFT와 동일 결과.
- **실전 암기**: **CROSS = m×n**(더하기 아님), **LEFT = 왼쪽 전부 + 매칭된 오른쪽**, **FULL = 양쪽 전부 + 매칭 병합**, **RIGHT = 오른쪽 전부 + 매칭된 왼쪽**.

### 문제 9
- **정답 ④** — **CAP는 3가지 동시 보장 불가**(2가지만), **BASE는 약한 일관성**(강한 일관성 아님), **Graph DB는 노드/엣지 관계 저장**(키-값 저장 아님).
- **CAP 정리 3속성**:
  - **C** Consistency — 모든 노드가 같은 데이터.
  - **A** Availability — 모든 요청에 응답 가능.
  - **P** Partition Tolerance — 네트워크 분할에도 동작.
  - **네트워크 분할이 발생하면 C 또는 A 중 하나만 선택 가능**.
- **NoSQL 4유형**:

| 유형 | 대표 제품 | 특징 |
|------|----------|------|
| Key-Value | Redis, DynamoDB, Riak | 가장 단순, 캐싱·세션 |
| Document | MongoDB, CouchDB | JSON/BSON, 반정형 |
| Column-Family | Cassandra, HBase, Bigtable | 컬럼 그룹, 대용량 분산 |
| Graph | Neo4j, JanusGraph, ArangoDB | 노드·엣지·관계, 소셜 네트워크 |

- **BASE**: **B**asically Available(기본적 가용), **S**oft state(비영구 상태·최종 일관성까지 변화), **E**ventually consistent(최종 일관성).
- **오답 함정**: CAP를 "3속성 모두 보장"으로 왜곡(불가능), BASE를 ACID 수준의 강한 일관성으로 오해, Graph DB를 Key-Value 저장으로 왜곡.
- **실전 암기**: **CAP = 2/3만 가능**, **RDBMS ≈ CA(P 없음)**, **NoSQL = AP 또는 CP**, **BASE ↔ ACID**(대비), **Graph DB = 관계 저장**.

### 문제 10
- **정답 ③ 7.75**.
- **비선점 SJF 손추적**:
  - t=0: P1만 도착 → **P1 실행**(0~8, Burst 8).
  - t=8 시점 대기열 = P2(도착1·Burst4), P3(도착2·Burst9), P4(도착3·Burst5) → Burst 최소 **P2 선택**.
  - **P2 실행**(8~12, Burst 4).
  - t=12 대기열 = P3(Burst9), P4(Burst5) → Burst 최소 **P4 선택**.
  - **P4 실행**(12~17, Burst 5).
  - t=17 → **P3 실행**(17~26, Burst 9).
- **각 프로세스 대기시간 = 시작 시각 − 도착 시각**:

| 프로세스 | 도착 | 시작 | 완료 | 대기시간 |
|----------|------|------|------|----------|
| P1 | 0 | 0 | 8 | 0 − 0 = **0** |
| P2 | 1 | 8 | 12 | 8 − 1 = **7** |
| P4 | 3 | 12 | 17 | 12 − 3 = **9** |
| P3 | 2 | 17 | 26 | 17 − 2 = **15** |

- 합계 = 0 + 7 + 9 + 15 = **31**. 평균 = 31 / 4 = **7.75** → **정답 ③**.
- **오답 함정**: ① SJF의 우선순위를 도착순으로 오해 / ② FCFS로 계산 시 결과(P1:0·P2:7·P3:11·P4:17→합35→평균 6.5과 8.75 사이 혼동)와 혼동 / ④ 완료 시각 평균으로 오해.
- **실전 암기**: **SJF는 Burst 최소 우선**(비선점 = 도착해도 뺏지 않음), **대기시간 = 시작시각 − 도착시각**, **반환시간 = 완료시각 − 도착시각 = 대기시간 + Burst**.

### 문제 12
- **정답 ④** — **`::`는 IPv6 주소 안에서 오직 한 번만 사용 가능**(모호성 방지), **IPv6 총 주소 = 2¹²⁸** (2⁶⁴ 아님).
- **IPv6 주소 축약 규칙**:
  1. 각 그룹의 **앞자리 0 생략**: `0db8` → `db8`.
  2. **연속된 0 그룹**을 `::` 로 축약 (한 주소당 **1번만** 가능).
  3. `2001:0db8:0000:0000:0000:ff00:0042:8329` = `2001:db8::ff00:42:8329` (연속 0 그룹 3개를 `::` 로).
- **IPv4/IPv6 비교**:

| 항목 | IPv4 | IPv6 |
|------|------|------|
| 비트 수 | 32 | 128 |
| 표기 | 8비트×4 (10진·`.`) | 16비트×8 (16진·`:`) |
| 총 주소 | 약 43억(2³²) | 약 3.4×10³⁸(2¹²⁸) |
| 헤더 | 가변(20~60B) | 고정 40B |
| 브로드캐스트 | ✅ | ❌ (멀티캐스트로 대체) |

- **서브넷 마스크 /26 계산**: 26비트가 네트워크 → 6비트가 호스트 → 2⁶ = 64 주소, 사용 가능 = 64-2 = 62 (네트워크 주소·브로드캐스트 주소 제외).
- **라우팅 프로토콜**:

| 프로토콜 | 유형 | 지표 | 범위 |
|---------|------|------|------|
| RIP | 거리 벡터 | 홉 수(최대 15) | IGP |
| OSPF | 링크 상태 | Cost(대역폭) | IGP |
| EIGRP | 하이브리드 | 복합 지표 | IGP (시스코) |
| BGP | 경로 벡터 | AS-Path 등 | EGP (AS 간) |

- **오답 함정**: `::` 두 번 사용 시 축약 위치를 특정할 수 없어 규칙 위반. IPv6 주소 공간을 2⁶⁴로 축소 오해.
- **실전 암기**: **IPv6 = 128비트·16진 8그룹·`::`는 한 번만**. **RIP=거리, OSPF=링크, BGP=AS**.
