① 스택은 LIFO(Last In First Out) 구조이며, 함수 호출의 복귀주소 관리에 사용된다.
② 큐는 FIFO(First In First Out) 구조이며, 운영체제의 작업 스케줄링에 사용된다.
③ 스택의 입력 동작은 push, 출력 동작은 pop이라 부른다.
④ 큐는 한쪽 끝(rear)에서만 삽입과 삭제가 모두 일어나는 자료구조이다.

## 🔍 오답 역추적 개념 발췌
- **스택 응용**: 중위→후위 변환, 후위식 계산, 함수호출 복귀주소, 깊이우선탐색(DFS), 괄호 검사.
- **큐 응용**: 너비우선탐색(BFS), 작업/프린터 스케줄링, 원형큐 (front=(front+1)%N, rear=(rear+1)%N).
- **덱(Deque, Double-ended Queue)**: 양쪽 끝에서 삽입·삭제 모두 가능. 스택+큐의 일반화.
- **트리 순회 4종**:
  - 전위(Preorder): 루트 → 좌 → 우
  - 중위(Inorder): 좌 → 루트 → 우
  - 후위(Postorder): 좌 → 우 → 루트
  - 레벨(Level-order, BFS): 큐 사용, 깊이 순
- **이진탐색트리(BST)**: 좌<루트<우. 균형 시 평균 O(log n), 편향 시 최악 O(n).
- **힙(Heap)**: 완전이진트리 + 부모-자식 대소관계. 최대힙(부모≥자식), 최소힙(부모≤자식). 삽입·삭제 O(log n).
- **그래프 표현**:
  - 인접행렬: 공간 O(V²), 간선 존재여부 O(1)
  - 인접리스트: 공간 O(V+E), 인접정점 순회 O(deg(v))
- **선형 vs 비선형**: 선형(배열·연결리스트·스택·큐·덱), 비선형(트리·그래프).

## 💡 상세 해설

### 문제 1
정답: ①
- 중위 `A + B * C - D / E`는 연산자 우선순위(`*`,`/` > `+`,`-`)와 좌결합을 적용해 `(A + (B * C)) - (D / E)`로 묶인다.
- 후위 변환: `B C *` → 묶어서 `A B C * +` → `D E /` 추가 → 마지막 `-` 연결 → **`A B C * + D E / -`**.
- 오답 이유:
  - ② `A B + C *`: `+`를 먼저 계산해 `(A+B)*C`가 되어 우선순위 위반.
  - ③ `A B C + *`: `B+C`를 먼저 계산 — 우선순위 위반.
  - ④ 후위가 아닌 중위 그대로 — 변환 안 됨.

### 문제 2
정답: ③
- 스택으로 추적:
  1. `5` push → [5]
  2. `4` push → [5, 4]
  3. `+` → pop 4, pop 5 → 5+4=9 → push → [9]
  4. `2` push → [9, 2]
  5. `*` → pop 2, pop 9 → 9*2=18 → push → [18]
  6. `6` push → [18, 6]
  7. `-` → pop 6, pop 18 → 18-6=**12** → push → [12]
- 오답 이유: ①②④는 연산 순서 오류 (특히 후위는 피연산자가 먼저, 연산자가 나중).

### 문제 3
정답: ②
- 중위 순회 = 좌 → 루트 → 우. 재귀적으로 적용:
  - A의 좌서브트리 순회: B의 좌(D) → B → B의 우(E) = **D B E**
  - 루트 A
  - A의 우서브트리 순회: C의 좌(없음) → C → C의 우(F) = **C F**
- 결과: **D B E A C F**
- 오답 이유:
  - ① A B D E C F: 전위(루트 먼저)와 혼동.
  - ③ D B E A F C: C와 F의 순서가 뒤바뀜 — C의 우자식이 F이므로 C가 먼저, F가 나중.
  - ④ D E B F C A: 후위(루트 마지막)와 혼동.

### 문제 4
정답: ④
- ④가 옳지 않음: 큐는 **rear에서 삽입, front에서 삭제**가 일어나는 양쪽 끝 구조. 한쪽 끝에서만 모두 일어나는 것은 스택.
- 오답 이유 (다른 보기는 모두 옳음):
  - ① 스택 LIFO + 함수호출 복귀주소(activation record) 관리 정상.
  - ② 큐 FIFO + OS 작업 큐 정상.
  - ③ push/pop 용어 정상.

### 문제 5
정답: ③
- 인접 행렬: n×n 2차원 배열로 표현 → 공간 복잡도 **O(n²)**. 무방향이면 대칭이지만 빅오는 동일.
- 오답 이유:
  - ① O(n): 정점 수만큼만 저장 — 인접리스트의 정점 배열 부분.
  - ② O(n+e): 인접리스트 전체 공간.
  - ④ O(log n): 자료구조 표현 공간으로 비현실적.

### 문제 6
정답: ②
- BST는 균형 잡힌 경우 트리 높이 ≈ log n → 평균 **O(log n)**.
- 한쪽으로 편향(skewed)된 경우(예: 정렬된 순서로 삽입) 연결리스트와 동일 → 최악 **O(n)**.
- 오답 이유:
  - ① 최악도 O(log n): 균형 보장 자료구조(AVL/RB-Tree)에 해당.
  - ③ 평균 O(n): BST는 평균은 log n.
  - ④ 평균 O(1): 해시테이블에 해당.

### 문제 7
정답: ②
- 원형 큐는 배열을 원형처럼 사용 → rear가 배열 끝(N-1)에 도달하면 0으로 wrap-around.
- 공식: **`rear = (rear + 1) % N`** (front 갱신도 동일 형태 `(front+1) % N`).
- 오답 이유:
  - ① 단순 증가: 일반 선형 큐. 끝에 도달 시 더 못 넣음.
  - ③ rear-1: 역방향 갱신 — 의미 없음.
  - ④ (rear+N) % (N-1): 모듈로 분모 잘못. N으로 나눠야 0~N-1 인덱스.

### 문제 8
정답: ②
- 선형(linear): 데이터가 일렬 → 스택, 큐, 덱, 배열, 연결리스트.
- 비선형(non-linear): 계층/망 구조 → **트리, 그래프**.
- ㄷ(트리), ㄹ(그래프) = 비선형 → 정답 ②.
- 오답 이유:
  - ① ㄱ,ㄴ: 스택·큐 = 선형.
  - ③ ㄱ,ㄴ,ㅁ: 모두 선형.
  - ④ ㅁ(연결리스트): 선형. 트리·그래프와 묶으면 안 됨.

## 📚 역추적 참조 (사후 검증용)

### 핵심 용어
| 용어 | 정의 |
|---|---|
| Stack (스택) | LIFO 자료구조. push/pop. top 포인터 1개. |
| Queue (큐) | FIFO 자료구조. enqueue(rear)/dequeue(front). |
| Deque (덱) | 양쪽 끝 삽입·삭제 가능 (Double-ended Queue). |
| Circular Queue (원형큐) | 배열 wrap-around. front, rear 모두 `% N`. |
| Tree (트리) | 계층적 비선형. 노드/간선/루트/리프. |
| BST (이진탐색트리) | 좌<루트<우. 탐색 평균 O(log n), 최악 O(n). |
| Heap (힙) | 완전이진트리 + 부모-자식 우선순위. O(log n) 삽입/삭제. |
| Graph (그래프) | 정점+간선. 방향/무방향, 가중/비가중. |
| Adjacency Matrix | 인접행렬. 공간 O(V²), 간선조회 O(1). |
| Adjacency List | 인접리스트. 공간 O(V+E), 간선조회 O(deg). |
| Preorder (전위) | 루트 → 좌 → 우 (DFS, 트리 복제·식의 전위표기). |
| Inorder (중위) | 좌 → 루트 → 우 (BST에서 정렬된 순서 출력). |
| Postorder (후위) | 좌 → 우 → 루트 (후위표기·트리 삭제). |
| Level-order (레벨) | 루트부터 깊이 순 (BFS, 큐 이용). |
| Postfix (후위표기) | 연산자가 피연산자 뒤. 스택으로 계산 (괄호 불필요). |

### 핵심 개념
- **중위→후위 변환 손계산 팁**: 연산자별 우선순위에 따라 괄호 묶기 → 각 괄호를 후위로 변환 → 합치기. `(A+(B*C))-(D/E)` → `A B C * + D E / -`.
- **후위 계산**: 왼쪽에서 토큰 읽으며 피연산자는 스택 push, 연산자는 상위 2개 pop 후 계산해 push. 마지막 스택 top이 결과.
- **트리 순회 빈출 함정**: 중위/전위/후위/레벨 결과를 헷갈리게 4개 보기로 줌. 루트의 위치(맨 앞=전위, 중간=중위, 맨 뒤=후위)로 빠르게 식별.
- **BST 시간복잡도 함정**: "평균 O(log n)"만 답하면 함정. **반드시 "최악 O(n)" 같이** 기억. 균형보장(AVL/RB)이 아니면 편향 가능.
- **원형큐 공식 빈출**: `(index+1) % N`. 분모는 반드시 N, N-1이나 N+1로 함정 보기 자주 등장.
- **선형/비선형 분류 함정**: 연결리스트는 "연결" 단어 때문에 비선형으로 착각하기 쉬움 → **연결리스트는 선형**.
- **인접행렬 vs 인접리스트**: 정점이 많고 간선이 적은(희소) 그래프 → 인접리스트 유리. 간선 조회 빈번 + 조밀 그래프 → 인접행렬 유리.
