## 🎯 오늘의 기출 8문제

### 문제 1 [유사 기출: 정보처리기사 필기 — 정렬 알고리즘 반복 출제 정형]
빅오(Big-O) 표기법에 대한 설명으로 옳지 않은 것은?

① 입력 크기 n이 충분히 커질 때 알고리즘 수행 시간의 증가율 상한(upper bound)을 나타낸다.
② O(1)은 입력 크기와 무관하게 일정한 시간이 걸리는 상수 시간(constant time)을 의미한다.
③ 증가율이 큰 순서로 나열하면 O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) 순이다.
④ O(2n² + 3n + 5)는 빅오 표기로 O(2n²)으로 표기하는 것이 가장 적절하다.

### 문제 2 [유사 기출: 정보처리기사 필기 — 정렬 알고리즘 반복 출제 정형]
다음 정렬 알고리즘 중 평균 시간복잡도가 O(n log n)이 아닌 것은?

① 퀵 정렬(Quick Sort)
② 합병 정렬(Merge Sort)
③ 힙 정렬(Heap Sort)
④ 선택 정렬(Selection Sort)

### 문제 3 [유사 기출: 정보처리기사 필기 — 정렬 알고리즘 반복 출제 정형]
다음 배열을 버블 정렬(Bubble Sort)로 오름차순 정렬할 때, 1회전(1 pass) 수행 후의 결과로 옳은 것은? (인접한 두 원소를 비교하여 큰 값을 뒤로 보내는 방식)

> 초기 배열: 6, 3, 5, 1, 4

① 3, 5, 1, 4, 6
② 3, 6, 5, 1, 4
③ 1, 3, 5, 6, 4
④ 1, 3, 4, 5, 6

### 문제 4 [유사 기출: 정보처리기사 필기 — 정렬 알고리즘 반복 출제 정형]
다음 배열을 삽입 정렬(Insertion Sort)로 오름차순 정렬할 때, 1회전(1 pass, 두 번째 원소를 정렬된 부분에 삽입) 수행 후의 결과로 옳은 것은?

> 초기 배열: 8, 5, 6, 2, 4

① 5, 8, 6, 2, 4
② 5, 6, 8, 2, 4
③ 2, 5, 6, 8, 4
④ 8, 5, 6, 2, 4

---

### 문제 5 [유사 기출: 정보처리기사 필기 — 정렬 알고리즘 반복 출제 정형]
퀵 정렬(Quick Sort)에 대한 설명으로 옳지 않은 것은?

① 피벗(pivot)을 기준으로 작은 값과 큰 값을 분할(partition)하여 재귀적으로 정렬하는 분할 정복(divide and conquer) 방식이다.
② 평균 시간복잡도는 O(n log n)이다.
③ 피벗 선택이 한쪽으로 치우쳐 분할이 불균형해지면 최악의 경우 O(n²)이 된다.
④ 추가 메모리를 입력 크기에 비례(O(n))하여 항상 사용하는 정렬이다.

---

### 문제 6 [유사 기출: 정보처리기사 필기 — 정렬 알고리즘 반복 출제 정형]
합병 정렬(Merge Sort)에 대한 설명으로 옳은 것은?

① 리스트를 절반으로 분할한 뒤 각각을 정렬하고 다시 병합하는 분할 정복 방식이다.
② 최악의 경우 시간복잡도가 O(n²)이다.
③ 추가 메모리 공간이 전혀 필요 없는 제자리(in-place) 정렬이다.
④ 같은 값의 원소 순서가 정렬 후 뒤바뀌는 불안정(unstable) 정렬이다.

---

### 문제 7 [유사 기출: 정보처리기사 필기 — 정렬 알고리즘 반복 출제 정형]
다음 배열을 선택 정렬(Selection Sort)로 오름차순 정렬할 때, 1회전(1 pass, 전체에서 최솟값을 찾아 맨 앞과 교환) 수행 후의 결과로 옳은 것은?

> 초기 배열: 7, 4, 9, 2, 5

① 2, 4, 9, 7, 5
② 4, 7, 9, 2, 5
③ 2, 4, 5, 7, 9
④ 4, 7, 2, 9, 5

---

### 문제 8 [유사 기출: 정보처리기사 필기 — 정렬 알고리즘 반복 출제 정형]
정렬 알고리즘에 대한 설명으로 옳지 않은 것은?

① 힙 정렬(Heap Sort)은 완전 이진 트리 형태의 힙(heap) 자료구조를 이용하며 시간복잡도는 O(n log n)이다.
② 버블 정렬과 삽입 정렬은 같은 값의 상대적 순서를 유지하는 안정 정렬(stable sort)이다.
③ 쉘 정렬(Shell Sort)은 일정 간격(gap)으로 떨어진 원소끼리 부분 정렬을 수행하여 삽입 정렬을 개선한 방식이다.
④ 선택 정렬은 입력이 거의 정렬된 경우에도 비교 횟수가 크게 줄어 평균 O(n)에 수렴한다.

## 🔍 오답 역추적 개념 발췌

오늘 문제에서 헷갈리기 쉬운 키워드만 발췌. 정의·설명은 사후 검증용 섹션 참조.

- 빅오(Big-O) 표기 = 점근적 상한(upper bound), 최고차항만 남기고 계수·상수 제거
- 증가율 순서: O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) < O(n!)
- O(n log n) 정렬: 퀵(평균)·합병·힙
- O(n²) 정렬: 버블·선택·삽입(최악)
- 버블 정렬 1회전 = 인접 비교, 최댓값이 맨 뒤로 확정
- 선택 정렬 1회전 = 전체 최솟값을 찾아 맨 앞과 교환
- 삽입 정렬 1회전 = 두 번째 원소를 정렬된 앞부분에 끼워넣기
- 퀵 정렬 = 피벗(pivot) 기준 분할 정복, 최악 O(n²)
- 합병 정렬 = 분할 후 병합, 추가 메모리 O(n) 필요, 안정 정렬
- 안정 정렬(stable): 버블·삽입·합병 / 불안정(unstable): 선택·퀵·힙
- 제자리 정렬(in-place) vs 추가 메모리 필요 정렬

## 💡 상세 해설 (자기 설명과 대조)

### 문제 1
정답: ④ (옳지 않은 것)
- 제공 해설: 빅오(Big-O)는 최고차항만 남기고 계수와 하위 항을 모두 제거한다. O(2n² + 3n + 5)는 O(n²)으로 표기해야 한다. 계수 2를 그대로 남긴 O(2n²)은 빅오 표기 규칙 위반이므로 ④가 틀린 설명.
- 오답 이유 (옳은 보기들):
  - ① 빅오는 입력 n이 커질 때 수행 시간 증가율의 점근적 상한(upper bound) — 옳음.
  - ② O(1)은 입력 크기와 무관한 상수 시간(constant time) — 옳음.
  - ③ 증가율 순서 O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) — 옳음.

### 문제 2
정답: ④ 선택 정렬(Selection Sort)
- 제공 해설: 선택 정렬은 매 회전마다 남은 구간 전체를 훑어 최솟값을 찾으므로 비교 횟수가 n(n-1)/2로 고정되어 평균·최악 모두 O(n²)이다. 퀵(평균)·합병·힙은 O(n log n).
- 오답 이유:
  - ① 퀵 정렬: 평균 O(n log n) (최악만 O(n²)).
  - ② 합병 정렬: 평균·최악 모두 O(n log n).
  - ③ 힙 정렬: 평균·최악 모두 O(n log n).

### 문제 3
정답: ① 3, 5, 1, 4, 6
- 제공 해설: 버블 정렬 1회전은 왼쪽부터 인접한 두 원소를 비교해 큰 값을 뒤로 보낸다. 초기 [6,3,5,1,4]에서:
  - (6,3) → 교환 → [3,6,5,1,4]
  - (6,5) → 교환 → [3,5,6,1,4]
  - (6,1) → 교환 → [3,5,1,6,4]
  - (6,4) → 교환 → [3,5,1,4,6]
  결과 [3,5,1,4,6]. 가장 큰 값 6이 맨 뒤로 확정됨.
- 오답 이유:
  - ② 3,6,5,1,4: 첫 비교만 한 중간 상태(1회전 미완료).
  - ③ 1,3,5,6,4: 최솟값을 앞으로 보내는 방식(버블 오름차순 1회전 동작 아님).
  - ④ 1,3,4,5,6: 완전 정렬 결과(1회전이 아닌 전체 정렬 종료 상태).

### 문제 4
정답: ① 5, 8, 6, 2, 4
- 제공 해설: 삽입 정렬은 두 번째 원소부터 시작해 앞의 정렬된 부분에 끼워넣는다. 초기 [8,5,6,2,4]에서 1회전은 두 번째 원소 5를 처리: 앞의 8과 비교해 5 < 8이므로 8을 뒤로 밀고 5를 맨 앞에 삽입 → [5,8,6,2,4]. 앞 두 칸만 정렬된 상태.
- 오답 이유:
  - ② 5,6,8,2,4: 2회전까지 진행한 결과(6까지 삽입).
  - ③ 2,5,6,8,4: 여러 회전 진행한 중간 상태.
  - ④ 8,5,6,2,4: 초기 배열 그대로(1회전 미수행).

### 문제 5
정답: ④ (옳지 않은 것)
- 제공 해설: 퀵 정렬은 피벗 기준으로 제자리(in-place) 분할을 수행하므로 별도의 큰 배열을 항상 O(n)만큼 쓰지 않는다. 추가 공간은 재귀 호출 스택 정도로 평균 O(log n)이다. "추가 메모리를 입력 크기에 비례(O(n))하여 항상 사용"은 합병 정렬의 특징이므로 ④가 틀림.
- 오답 이유 (옳은 보기들):
  - ① 피벗 기준 분할 후 재귀 정렬하는 분할 정복 방식 — 옳음.
  - ② 평균 시간복잡도 O(n log n) — 옳음.
  - ③ 분할이 한쪽으로 치우치면 최악 O(n²) — 옳음.

### 문제 6
정답: ① 분할 정복 방식이다
- 제공 해설: 합병 정렬은 리스트를 절반씩 재귀 분할 후 각 부분을 정렬해 다시 병합(merge)하는 분할 정복 방식. 평균·최악 모두 O(n log n)이며, 병합용 임시 배열로 O(n) 추가 메모리를 쓰고, 같은 값의 순서를 유지하는 안정 정렬이다.
- 오답 이유:
  - ② 최악 O(n²): 틀림. 합병은 최악도 O(n log n).
  - ③ 제자리 정렬: 틀림. 병합용 추가 배열 O(n)이 필요.
  - ④ 불안정 정렬: 틀림. 합병 정렬은 대표적 안정(stable) 정렬.

### 문제 7
정답: ① 2, 4, 9, 7, 5
- 제공 해설: 선택 정렬 1회전은 전체에서 최솟값을 찾아 맨 앞 원소와 교환한다. 초기 [7,4,9,2,5]의 최솟값은 2(인덱스 3). 맨 앞 7과 2를 교환 → [2,4,9,7,5]. 맨 앞 칸에 최솟값 2가 확정됨.
- 오답 이유:
  - ② 4,7,9,2,5: 인접 두 원소만 교환한 형태(선택 정렬 동작 아님).
  - ③ 2,4,5,7,9: 완전 정렬 결과(1회전이 아님).
  - ④ 4,7,2,9,5: 임의 교환, 최솟값을 맨 앞에 두지 못함.

### 문제 8
정답: ④ (옳지 않은 것)
- 제공 해설: 선택 정렬은 입력이 이미 정렬되어 있어도 매 회전 남은 구간 전체를 비교해 최솟값을 찾으므로 비교 횟수가 n(n-1)/2로 고정된다. 따라서 입력 상태와 무관하게 항상 O(n²)이며 O(n)에 수렴하지 않는다. ④가 틀림. (입력이 거의 정렬된 경우 O(n)에 가까워지는 것은 삽입 정렬·버블 정렬(교환 없으면 조기 종료)이다.)
- 오답 이유 (옳은 보기들):
  - ① 힙 정렬은 완전 이진 트리 기반 힙을 이용, O(n log n) — 옳음.
  - ② 버블·삽입 정렬은 안정 정렬(stable) — 옳음.
  - ③ 쉘 정렬은 간격(gap) 단위 부분 정렬로 삽입 정렬을 개선 — 옳음.

## 📚 역추적 참조 (사후 검증용 — 학습 자료 아님)

### 핵심 용어
| 용어 | 정의 |
|---|---|
| 빅오 표기법(Big-O notation) | 입력 n이 커질 때 알고리즘 수행 시간의 증가율 상한(점근적 upper bound). 최고차항만 남기고 계수·상수 제거. |
| 분할 정복(Divide and Conquer) | 문제를 작은 하위 문제로 분할 → 각각 해결 → 결과 결합. 퀵·합병 정렬의 기반. |
| 피벗(Pivot) | 퀵 정렬에서 분할 기준이 되는 원소. 이보다 작은 값은 왼쪽, 큰 값은 오른쪽으로 배치. |
| 안정 정렬(Stable Sort) | 같은 값(key)을 가진 원소들의 정렬 전 상대 순서가 정렬 후에도 유지되는 정렬. |
| 제자리 정렬(In-place Sort) | 입력 크기에 비례하는 추가 배열 없이(상수~로그 수준 추가 공간) 정렬. |
| 힙(Heap) | 완전 이진 트리(complete binary tree) 형태로 부모-자식 간 대소 관계가 유지되는 자료구조. |
| 회전/패스(Pass) | 정렬 알고리즘이 데이터 전체(또는 남은 구간)를 한 번 훑는 단위. |

### 핵심 개념 (사후 검증용)

**빅오 증가율 순서 (작은 것 → 큰 것)**
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)

**빅오 표기 규칙**
- 최고차항만 남긴다: O(2n² + 3n + 5) → O(n²)
- 계수 제거: O(3n) → O(n)
- 상수 항 제거: O(n + 100) → O(n)

**정렬 알고리즘 시간복잡도 / 안정성 표**
| 정렬 | 평균 | 최악 | 안정성 | 추가 메모리 |
|---|---|---|---|---|
| 버블(Bubble) | O(n²) | O(n²) | 안정 | O(1) |
| 선택(Selection) | O(n²) | O(n²) | 불안정 | O(1) |
| 삽입(Insertion) | O(n²) | O(n²) | 안정 | O(1) |
| 퀵(Quick) | O(n log n) | O(n²) | 불안정 | O(log n) 평균 |
| 합병(Merge) | O(n log n) | O(n log n) | 안정 | O(n) |
| 힙(Heap) | O(n log n) | O(n log n) | 불안정 | O(1) |
| 쉘(Shell) | O(n^1.5) 내외 | O(n²) | 불안정 | O(1) |

**정렬 1회전(1 pass) 동작 정리**
- 버블 정렬: 인접한 두 원소를 비교·교환하며 끝까지 이동 → 1회전마다 가장 큰 값이 맨 뒤로 확정.
- 선택 정렬: 남은 구간 전체에서 최솟값을 찾아 맨 앞과 교환 → 1회전마다 가장 작은 값이 맨 앞으로 확정.
- 삽입 정렬: 다음 원소를 정렬된 앞부분의 알맞은 위치에 끼워넣음 → 1회전마다 앞쪽 정렬 구간이 한 칸 확장.

**퀵 정렬 vs 합병 정렬 (분할 정복 비교)**
| 항목 | 퀵 정렬 | 합병 정렬 |
|---|---|---|
| 분할 기준 | 피벗(pivot) 값 기준 | 길이를 절반으로 분할 |
| 핵심 작업 | 분할(partition) 시점 | 병합(merge) 시점 |
| 평균 | O(n log n) | O(n log n) |
| 최악 | O(n²) (불균형 분할) | O(n log n) |
| 추가 메모리 | O(log n) 평균(재귀 스택) | O(n) (병합 임시 배열) |
| 안정성 | 불안정 | 안정 |

**기타**
- 힙 정렬: 배열을 최대 힙(max-heap)으로 구성 후 루트(최댓값)를 맨 뒤로 빼는 과정을 반복. 평균·최악 모두 O(n log n), 제자리 정렬.
- 쉘 정렬: 간격(gap)을 두고 떨어진 원소끼리 삽입 정렬 → 간격을 줄여가며 반복. 삽입 정렬의 개선판.
- 안정 정렬: 버블, 삽입, 합병 / 불안정 정렬: 선택, 퀵, 힙, 쉘.
