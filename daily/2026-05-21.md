## 🎯 오늘의 기출 8문제

### 문제 1 [복원/유사 기출: OSI 7계층 빈출 — 네트워크 계층의 기능]
OSI 7계층 모델에서 데이터 패킷의 경로(라우팅)를 결정하고 논리 주소(IP 주소)를 부여하는 계층은?
① 데이터링크 계층(Data Link Layer)
② 네트워크 계층(Network Layer)
③ 전송 계층(Transport Layer)
④ 세션 계층(Session Layer)

### 문제 2 [복원/유사 기출: OSI 계층 ↔ 장비 매칭]
다음 중 OSI 7계층과 동작하는 네트워크 장비의 매칭이 옳지 않은 것은?
① 1계층(물리) — 리피터(Repeater), 허브(Hub)
② 2계층(데이터링크) — 브리지(Bridge), 스위치(Switch)
③ 3계층(네트워크) — 라우터(Router), L3 스위치
④ 4계층(전송) — 게이트웨이(Gateway), 리피터

### 문제 3 [복원/유사 기출: TCP vs UDP 비교 — 정보처리기사 단골]
TCP(Transmission Control Protocol)와 UDP(User Datagram Protocol)에 관한 설명으로 옳지 않은 것은?
① TCP는 연결지향형(connection-oriented), UDP는 비연결형(connectionless) 프로토콜이다.
② TCP는 흐름 제어와 혼잡 제어를 수행하지만 UDP는 수행하지 않는다.
③ TCP 헤더 크기는 기본 20바이트, UDP 헤더 크기는 8바이트이다.
④ UDP는 신뢰성 있는 데이터 전송을 보장하며 패킷 순서를 보장한다.

### 문제 4 [복원/유사 기출: IPv6 특징]
IPv6에 대한 설명으로 옳지 않은 것은?
① 주소 길이는 128비트이며 16비트씩 8개의 필드를 콜론(:)으로 구분한다.
② 헤더 길이가 가변적이어서 옵션 처리가 매우 빠르다.
③ 보안 기능(IPsec)이 기본으로 포함되어 있다.
④ 유니캐스트(Unicast), 멀티캐스트(Multicast), 애니캐스트(Anycast)의 3가지 주소 유형을 지원한다.

### 문제 5 [복원/유사 기출: 서브넷 마스크와 호스트 수]
IPv4 주소 192.168.10.0/26 네트워크에서 사용할 수 있는 호스트의 최대 수는?
① 30
② 62
③ 126
④ 254

### 문제 6 [복원/유사 기출: 라우팅 프로토콜 분류]
다음 중 IGP(Interior Gateway Protocol) 내부 라우팅 프로토콜에 해당하지 않는 것은?
① RIP(Routing Information Protocol)
② OSPF(Open Shortest Path First)
③ IS-IS(Intermediate System to Intermediate System)
④ BGP(Border Gateway Protocol)

### 문제 7 [복원/유사 기출: 응용 계층 프로토콜 포트 번호]
응용 계층 프로토콜과 기본 포트 번호의 매칭이 옳지 않은 것은?
① HTTP — 80
② HTTPS — 443
③ SMTP — 25
④ DNS — 53/TCP만 사용

### 문제 8 [복원/유사 기출: ARP / RARP / ICMP — 단골 키워드 구분]
IP 주소(논리 주소)를 알고 있을 때 해당 호스트의 MAC 주소(물리 주소)를 알아내기 위해 사용되는 프로토콜은?
① ARP(Address Resolution Protocol)
② RARP(Reverse ARP)
③ ICMP(Internet Control Message Protocol)
④ IGMP(Internet Group Management Protocol)

## 🔍 오답 역추적 개념 발췌

- OSI 7계층 PDU 단위: 비트 → 프레임 → 패킷 → 세그먼트 → 데이터
- 2계층 장비 vs 3계층 장비 구분점
- 게이트웨이 동작 계층 (보기에 자주 등장하는 함정)
- TCP 3-way handshake / 4-way termination
- TCP 헤더 필드 (Sequence, Ack, Window, Flag)
- IPv4 → IPv6 전환 기술 3종: Dual Stack / Tunneling / Translation
- 서브넷팅 — /N 의 호스트 수 = 2^(32-N) - 2
- 거리벡터(distance-vector) vs 링크상태(link-state)
- AS(Autonomous System) 내부/외부 라우팅
- 잘 헷갈리는 포트: DNS 53, SMTP 25, POP3 110, IMAP 143, FTP 20/21, Telnet 23, SSH 22, SNMP 161/162
- ARP ↔ RARP 방향성, ICMP ↔ IGMP 차이

## 💡 상세 해설 (자기 설명과 대조)

### 문제 1 — 정답 ②
- **네트워크 계층(3계층)**: 라우팅 경로 결정, 논리 주소(IP) 부여, 패킷 단위 전달. 대표 프로토콜 IP, ICMP, IGMP, ARP, RARP. 대표 장비 라우터, L3 스위치.
- ①: 데이터링크 계층은 인접 노드 간 프레임 전송, MAC 주소(물리 주소) 사용. 라우팅 아님.
- ③: 전송 계층은 종단 간(end-to-end) 신뢰성 있는 데이터 전달, TCP/UDP. 경로 결정과 무관.
- ④: 세션 계층은 통신 세션의 설정·유지·종료 관리.

### 문제 2 — 정답 ④
- **게이트웨이(Gateway)는 OSI 모든 계층(주로 7계층 응용 계층까지) 변환을 수행하는 장비**이며, 리피터는 1계층(물리) 신호 증폭 장비이다. 4계층 장비로 묶인 것이 오답.
- 4계층 대표 장비는 L4 스위치(포트 기반 부하분산).
- ①, ②, ③: 매칭 정확.

### 문제 3 — 정답 ④
- **UDP는 비신뢰·비연결 프로토콜**로 데이터그램 단위 전송, 패킷 순서·도달 보장 없음, 오류 시 재전송 없음. 신뢰성을 보장한다고 한 ④가 오답.
- ①: 정확. TCP=연결지향, UDP=비연결.
- ②: 정확. TCP만 흐름/혼잡 제어.
- ③: 정확. TCP 기본 헤더 20바이트(옵션 없으면), UDP 헤더 8바이트(고정).

### 문제 4 — 정답 ②
- **IPv6 기본 헤더는 40바이트 고정**이다(IPv4는 20~60바이트 가변). 옵션은 확장 헤더(Extension Header)로 분리되어 라우터 처리 속도가 빨라진다. "가변적"이라고 한 ②가 오답.
- ①: 128비트 = 16비트 × 8필드, 콜론 hex 표기. 정답.
- ③: IPsec은 IPv6에서 필수(IPv4는 선택). 정답.
- ④: 브로드캐스트는 폐지되고 멀티캐스트로 대체. 유니/멀티/애니의 3종. 정답.

### 문제 5 — 정답 ②
- /26 → 네트워크 비트 26개, 호스트 비트 32-26 = 6
- 사용 가능 호스트 수 = 2^6 - 2 = 64 - 2 = **62** (네트워크 주소와 브로드캐스트 주소 제외)
- ①: /27의 호스트 수(2^5-2=30)
- ③: /25의 호스트 수(2^7-2=126)
- ④: /24의 호스트 수(2^8-2=254)

### 문제 6 — 정답 ④
- **BGP는 AS(Autonomous System) 간 라우팅을 담당하는 EGP**(외부 게이트웨이 프로토콜). 경로 벡터(Path-Vector) 알고리즘.
- ①: RIP — 거리벡터, 홉 수 최대 15. IGP.
- ②: OSPF — 링크상태, 다익스트라(Dijkstra). IGP.
- ③: IS-IS — 링크상태, OSI 기반. IGP.

### 문제 7 — 정답 ④
- **DNS는 기본 UDP/53을 사용**하며, 메시지 크기가 512바이트를 초과하거나 영역 전송(Zone Transfer)에는 TCP/53을 사용한다. "TCP만"이라고 한 ④가 오답.
- ①, ②, ③: 모두 표준 포트와 정확히 일치.

### 문제 8 — 정답 ①
- **ARP(Address Resolution Protocol)**: IP 주소(논리) → MAC 주소(물리) 변환.
- ②: RARP — MAC 주소를 알 때 IP 주소를 알아냄 (반대 방향).
- ③: ICMP — 네트워크 진단/오류 메시지(Ping, Traceroute).
- ④: IGMP — 멀티캐스트 그룹 관리.

## 📚 역추적 참조 (사후 검증용 — 학습 자료 아님)

### 핵심 용어
| 용어 | 정의 |
|---|---|
| OSI 7계층 | 국제표준화기구(ISO)가 정의한 통신 시스템 표준 모델 |
| TCP/IP 4계층 | 실제 인터넷에서 사용되는 통신 프로토콜 스택 |
| PDU | Protocol Data Unit, 각 계층에서 다루는 데이터 단위 |
| MAC 주소 | Media Access Control, 48비트 물리 주소 |
| IP 주소 | Internet Protocol, 논리 주소 (v4: 32비트, v6: 128비트) |
| AS | Autonomous System, 단일 관리 정책 하의 네트워크 집합 |
| IGP | Interior Gateway Protocol, AS 내부 라우팅 |
| EGP | Exterior Gateway Protocol, AS 간 라우팅 |
| 거리벡터 | 인접 라우터와 거리(홉) 정보 교환 — RIP |
| 링크상태 | 전체 네트워크 토폴로지 정보 공유 — OSPF, IS-IS |

### TCP/UDP 헤더 핵심 필드 (사후 검증용)
- TCP 20B 고정 필드: Source/Dest Port (각 16b), Sequence Number (32b), Ack Number (32b), Header Length, Flag(URG/ACK/PSH/RST/SYN/FIN), Window Size, Checksum, Urgent Pointer
- UDP 8B 필드: Source Port, Dest Port, Length, Checksum

### IPv4 사설 IP 대역 (사후 검증용)
- 10.0.0.0/8 (Class A)
- 172.16.0.0/12 (Class B)
- 192.168.0.0/16 (Class C)

### IPv4 → IPv6 전환 기술 3종 (사후 검증용)
1. **듀얼 스택(Dual Stack)**: 호스트가 v4/v6 동시 지원
2. **터널링(Tunneling)**: v6 패킷을 v4 패킷으로 캡슐화 — 6to4, ISATAP, Teredo
3. **주소 변환(Translation)**: NAT-PT 등으로 v6↔v4 변환

### 3-way handshake / 4-way termination (사후 검증용)
- 연결: SYN → SYN+ACK → ACK
- 종료: FIN → ACK → FIN → ACK
