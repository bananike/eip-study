## 🔍 오답 역추적 개념 발췌
- **2차원 배열 기본 등가식**: `a[i][j] == *(a[i] + j) == *(*(a+i) + j) == (*(a+i))[j]`. `a[i]`는 i행의 시작 주소(= `int*` 형) — 1차원 포인터로 받아 1차원 배열처럼 다룰 수 있다.
- **포인터 산술 단위**: `p + 1`은 **바이트가 아니라 sizeof(*p) 단위**로 증가. `int *p`이면 4바이트씩, `struct Student *p`이면 sizeof(struct Student) 단위.
- **논리 연산자 단축 평가(Short-Circuit)**:
  - `A && B`: A가 **거짓이면 B 평가 안 함**
  - `A || B`: A가 **참이면 B 평가 안 함**
  - 부수효과(`b++`)가 포함된 식에서 평가 누락이 시험 단골 함정.
- **문자열·널문자(`\0`)**:
  - C 문자열은 끝에 `\0`이 반드시 있어야 함. `strlen`은 `\0` **앞까지 세고**, `\0`은 포함 안 함.
  - `strcat(dst, src)`: dst의 `\0` 위치에 src를 이어 붙이고, 새 `\0` 추가. dst의 버퍼 크기가 충분해야 함.
  - `strcpy(dst, src)`: src를 `\0` 포함해 dst로 복사.
- **재귀(Recursion) 추적 요령**: 종료조건(base case) 먼저 명시, 호출 트리를 그려 leaf부터 거꾸로 계산. 같은 인자의 호출이 반복되면 메모이제이션 대상(DP).
- **비트 시프트 의미**:
  - `x << n`: x를 **2ⁿ 배 곱한 효과** (좌측으로 n비트 이동, 빈 자리는 0)
  - `x >> n`: x를 **2ⁿ 으로 나눈 효과(정수 나눗셈)** (부호 있는 음수의 `>>`는 산술/논리 시프트가 구현 정의)
- **기억 클래스(Storage Class)**:
  - `auto` (지역 기본): 함수 진입 시 생성, 종료 시 소멸, 초기화 없으면 쓰레기값
  - `static` (지역에 사용 시): 프로그램 종료까지 **수명 유지**, 단 가시성은 정의된 블록 내, **0으로 자동 초기화**
  - `extern`: 다른 파일에 정의된 전역 변수의 **선언**만 (선언과 정의 구분)
  - `register`: 자주 쓰이는 변수를 **레지스터에 배치 권장** (실제 배치는 컴파일러 결정), `&` 연산자 불가
- **구조체 멤버 접근 우선순위**: `.`/`->`는 단항 연산자보다 우선. `(p+i)->score` ≠ `p+i->score` (후자는 `p + (i->score)`로 해석되어 에러).
- **`char *arr[]`(문자열 포인터 배열) vs `char arr[][]`(2차원 문자 배열)**:
  - 전자: arr[i]는 문자열 리터럴의 시작 주소(읽기 전용), 길이 가변
  - 후자: arr[i]는 고정 길이 문자 배열(수정 가능), 메모리 연속 할당

## 💡 상세 해설

### 문제 1
**정답: ③ 11**

- `int a[3][3]`은 3×3 2차원 배열. 행 단위 메모리 배치: `{1,2,3}{4,5,6}{7,8,9}`
- `a[1]`은 **두 번째 행의 시작 주소**(= `&a[1][0]`)이며 타입은 `int*`. 따라서 `p = a[1]`은 행 `{4, 5, 6}`의 첫 원소를 가리킨다.
- `*(p + 1)` = `p[1]` = a[1][1] = **5**
- `*(p + 2)` = `p[2]` = a[1][2] = **6**
- 합: 5 + 6 = **11**

**오답 분석**
- ① 9 = a[1][0]+a[1][1]=4+5 (인덱스 0,1 더한 오류)
- ② 10 = a[1][0]+a[1][2]=4+6 (인덱스 0,2 더한 오류)
- ④ 15 = a[1] 전체 합(4+5+6) (전 원소 합산 오류)

### 문제 2
**정답: ① 0 1 0**

- `c = a && b++`에서 `&&`는 **단축 평가(short-circuit)**.
- `a`가 **0(거짓)** 이므로 `&&`는 결과를 0으로 확정하고 우측 `b++`는 **평가하지 않는다**.
- 따라서 `b`의 후위 증가는 **발생하지 않음** → b는 1 유지.
- c = 0 (0 && _ = 0)
- 출력: `0 1 0`

**오답 분석**
- ② 0 2 0 — b++가 실행됐다고 잘못 본 경우 (단축평가 무시)
- ③ 0 2 1 — 추가로 c=1로 잘못 본 경우
- ④ 0 1 1 — c가 1로 계산되는 식이 없음

> 핵심: **`A && B`에서 A가 거짓이면 B는 평가 자체를 하지 않으므로 부수효과(b++)도 실행되지 않는다.**

### 문제 3
**정답: ③ 13**

- `char s[20] = "Hello"` — s에 `H e l l o \0`이 저장됨(6바이트 사용, 19바이트 여유).
- `strcat(s, ", World!")` — s의 `\0` 위치부터 `, World!`를 이어 붙인 뒤 새 `\0`을 추가.
- 결과: `Hello, World!` + `\0`
- `strlen(s)`는 **`\0`을 제외한 문자 수**를 반환.
- 문자 카운트: `H`(1) `e`(2) `l`(3) `l`(4) `o`(5) `,`(6) ` `(7) `W`(8) `o`(9) `r`(10) `l`(11) `d`(12) `!`(13) → **13**

**오답 분석**
- ① 5 — strcat 호출을 누락하고 초기 "Hello"만 센 경우
- ② 12 — 공백 또는 `!`를 빠뜨린 경우
- ④ 14 — `\0`까지 포함해 센 경우 (strlen은 `\0` 제외)

### 문제 4
**정답: ② 8**

- 피보나치 수열: `f(0)=0, f(1)=1, f(n)=f(n-1)+f(n-2)`
- 호출 트리 추적:
  - f(2)=f(1)+f(0)=1+0=**1**
  - f(3)=f(2)+f(1)=1+1=**2**
  - f(4)=f(3)+f(2)=2+1=**3**
  - f(5)=f(4)+f(3)=3+2=**5**
  - f(6)=f(5)+f(4)=5+3=**8**

**오답 분석**
- ① 5 = f(5) (한 단계 덜 호출)
- ③ 13 = f(7) (한 단계 더)
- ④ 21 = f(8) (두 단계 더)

> 함정: `n < 2` 종료조건에서 `f(0)=0, f(1)=1`을 명확히 식별해야 함.

### 문제 5
**정답: ① 3 24**

- a = 12 = 이진 `1100`
- `a >> 2`: 우측으로 2비트 시프트 → `0011` = **3** (12 ÷ 2² = 3)
- `a << 1`: 좌측으로 1비트 시프트 → `11000` = **24** (12 × 2¹ = 24)
- 출력: `3 24`

**오답 분석**
- ② 6 24 — `>>1`로 잘못 계산 (12>>1=6)
- ③ 3 12 — `<<1`을 적용 안 함
- ④ 6 6 — 둘 다 ÷2로 잘못 본 경우

> 핵심 공식: `x << n = x × 2ⁿ`, `x >> n = x ÷ 2ⁿ`(정수 나눗셈, 양수 가정)

### 문제 6
**정답: ④ 10**

- `static int x = 0;` — **함수가 처음 호출될 때 단 한 번만 초기화**, 함수 호출이 끝나도 값이 **메모리에 유지**됨.
- `auto`(기본 지역)와 달리 호출마다 0으로 리셋되지 않는다.
- 호출별 반환값 추적:
  - 1차 호출: x: 0→1, return 1
  - 2차: x: 1→2, return 2
  - 3차: x: 2→3, return 3
  - 4차: x: 3→4, return 4
- sum = 1 + 2 + 3 + 4 = **10**

**오답 분석**
- ① 1 — 마지막 반환값만 본 경우
- ② 4 — static을 잊고 매번 0으로 초기화로 본 경우(1+1+1+1)
- ③ 6 — 1+2+3 만 더한 경우(루프 횟수 오인)

> 핵심: static 지역 변수는 **수명=프로그램 종료까지, 범위=정의된 블록**, **0 자동 초기화**.

| 부수효과(Side Effect) | 식 평가 중 변수 값을 바꾸는 효과(`++`, `--`, 대입, 함수 호출 등) |

### 핵심 개념
- **2차원 배열 핵심 등가식**: `a[i][j] == *(a[i] + j) == *(*(a + i) + j) == (*(a + i))[j]`. 시험에서 4가지 표기가 같은 의미임을 묻는 패턴이 반복된다. `a[i]`가 1차원 포인터 형태임을 잡으면 `int *p = a[1]`도 자연스럽게 이해된다.
- **포인터 산술 단위 함정**: `p + 1`은 1바이트가 아니다. 항상 `sizeof(*p)` 단위. `int*`이면 4바이트, 구조체 포인터면 sizeof(struct)만큼. 이걸 잘못 잡으면 모든 포인터 추적이 어긋난다.
- **단축 평가의 부수효과 무시**: `0 && b++`, `1 || b++`에서 `b++`은 **실행되지 않는다**. 시험에서 `&&`/`||` 좌변이 단순 상수/0/1이면 단축 평가 함정을 의심해야 한다.
- **문자열·널문자 길이 산정**:
  - `char s[20] = "Hello";` → 메모리에는 6바이트 사용(`H,e,l,l,o,\0`), 배열 크기는 20.
  - `strlen("Hello")` = 5 (널 제외)
  - `sizeof("Hello")` = 6 (널 포함, 컴파일 타임 상수)
  - 배열 크기 sizeof는 배열 전체 바이트 — `sizeof(s)` = 20(위 선언 기준).
- **재귀 호출 추적 핵심**: ① 종료조건 명확화 → ② leaf(base case)부터 거꾸로 채워 올리기 → ③ 동일 인자가 반복되면 메모이제이션 가능(DP의 진입점).
- **시프트의 직관**: 이진수를 그대로 좌/우로 미는 것. `<<`는 곱, `>>`는 나눗셈(양수). 부호 있는 음수의 `>>`는 컴파일러마다 산술/논리 시프트가 갈리므로 시험에서는 보통 양수만 다룬다.
- **static의 두 가지 의미 구분**:
  - **함수 내부의 static 지역 변수**: 수명 연장 + 0 자동 초기화 + 한 번만 초기화
  - **파일 범위의 static 전역 변수/함수**: 외부 연결(linkage) 차단(=해당 파일 내에서만 사용 가능)
  - 시험에서는 보통 전자(지역 static)를 묻지만 의미가 다르다는 점을 알아야 함.
- **구조체 멤버 접근 우선순위 함정**: `*`보다 `.`/`->`/`[]`가 우선순위 높음. 그래서 `*p.x`는 `*(p.x)`로 해석. `(p + i)->score`에서 괄호를 빼면 `p + (i->score)`가 되어 컴파일 에러. 시험 보기에 일부러 괄호 위치를 바꾼 함정이 자주 나온다.
- **`char *arr[]` vs `char arr[][]` 결정적 차이**:
  - `char *arr[3] = {"apple", ...}`: 각 원소는 문자열 리터럴의 **시작 주소(포인터)**. 메모리상 각 문자열은 따로 떨어져 있음. arr[i] 자체는 수정 가능하나 가리키는 문자열 리터럴 내용 수정은 UB.
  - `char arr[3][7] = {"apple", ...}`: 각 행이 7바이트 고정 폭으로 **연속 할당**. 내용 수정 가능. 길이가 7을 넘으면 컴파일 에러.
