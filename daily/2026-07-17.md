### 문제 3 [유사 기출: 2022년 1회 2과목 — 스택 vs 큐 vs 덱 정형 비교]
다음 자료구조에 대한 설명 중 **가장 옳지 않은 것**은?

① **스택(Stack)** = 한쪽 끝(top)에서만 삽입·삭제가 일어나는 **후입선출(LIFO, Last In First Out)** 구조. 함수 호출 스택·수식 계산·괄호 검사·DFS·undo 기능의 기반.
② **큐(Queue)** = 삽입은 rear·삭제는 front에서 일어나는 **선입선출(FIFO, First In First Out)** 구조. 프로세스 스케줄링(FCFS)·프린터 스풀·BFS·메시지 큐의 기반.
③ **덱(Deque, Double-Ended Queue)** = **양쪽 끝**에서 삽입·삭제가 모두 가능한 구조. 입력 제한 덱(scroll)·출력 제한 덱(shelf) 변형 존재. 파이썬 `collections.deque`, Java `ArrayDeque` 가 대표 구현.
④ **스택은 FIFO 구조로 프린터 스풀에서 사용**되며, **큐는 LIFO 구조로 함수 호출·undo 기능에서 사용**되고, **덱은 한쪽 끝에서만 삽입·삭제가 이루어져 스택과 완전 동일하다**.

### 문제 4 [유사 기출: 2023년 2회 2과목 — 연결리스트(Linked List) 정형 패턴]
**연결리스트(Linked List)** 에 관한 설명 중 **가장 옳지 않은 것**은?

① **단일 연결리스트(Singly Linked List)** — 각 노드가 **데이터 + 다음 노드 포인터(next) 1개**를 가지는 구조. 순방향 순회만 가능하고, 특정 노드 삭제 시 이전 노드의 next를 갱신하려면 **처음부터 재탐색**이 필요.
② **이중 연결리스트(Doubly Linked List)** — 각 노드가 **데이터 + prev + next 2개 포인터**를 가지는 구조. 양방향 순회·특정 노드 삭제가 O(1)에 가능(해당 노드 포인터를 이미 알고 있을 때). 대신 저장공간이 더 소요된다.
③ **원형 연결리스트(Circular Linked List)** — **마지막 노드의 next가 head를 가리키는** 구조. 순환 반복이 자연스럽고, tail 포인터 하나만으로 head·tail을 모두 O(1)에 접근 가능. 조세퍼스(Josephus) 문제·라운드 로빈 스케줄링에 적합.
④ **연결리스트는 배열보다 임의 접근(random access)이 빠르며 O(1)에 인덱스 접근이 가능**하고, **삽입·삭제는 배열보다 항상 느리며 O(n²) 이 소요**된다.

### 문제 5 [유사 기출: 2021년 3회 2과목 — 이진트리(Binary Tree) 정형 분류]
**이진트리(Binary Tree)의 종류**에 관한 설명 중 **가장 옳지 않은 것**은?

① **포화 이진트리(Full/Perfect Binary Tree)** — 모든 레벨의 노드가 완전히 채워진 트리. 높이가 h일 때 **노드 수 = 2^(h+1) - 1**(루트 레벨을 0으로 볼 때). 각 레벨 i의 노드 수는 2^i.
② **완전 이진트리(Complete Binary Tree)** — 마지막 레벨을 **왼쪽부터 순서대로** 채운 이진트리. 배열 표현이 자연스럽고(인덱스 i의 자식 = 2i, 2i+1), **힙(Heap)** 이 대표 예. 마지막 레벨 외에는 모든 레벨이 완전히 채워져 있다.
③ **편향(사향) 이진트리(Skewed Binary Tree)** — 모든 노드가 **왼쪽(또는 오른쪽) 자식만**을 가져 사실상 연결리스트 모양이 되는 트리. 높이가 노드 수 n에 근접해 탐색이 O(n)까지 늘어난다.
④ **포화 이진트리는 마지막 레벨을 왼쪽부터 채운 트리**이며, **완전 이진트리는 모든 레벨이 완전히 채워진 트리**이고, **높이 h 포화 이진트리의 노드 수는 h + 1개** 이며 편향 이진트리는 O(log n)에 탐색이 가능하다.

### 문제 8 [유사 기출: 2023년 1회 2과목 — 이진 탐색 트리(BST) 정형 정의]
**이진 탐색 트리(BST, Binary Search Tree)** 에 관한 설명 중 **가장 옳지 않은 것**은?

① **정의** — 각 노드에 대해 **좌 서브트리의 모든 키 < 노드 키 < 우 서브트리의 모든 키** 조건을 만족하는 이진트리. 중위 순회 결과가 **오름차순 정렬**이 되는 것이 특징.
② **탐색·삽입·삭제 평균 시간복잡도** — **O(log n)**. 다만 트리가 편향되면 **최악 O(n)** 까지 늘어난다(정렬된 데이터를 순서대로 삽입하는 경우가 대표적).
③ **균형 유지 장치** — 최악 O(n)을 방지하기 위해 **AVL 트리 · Red-Black 트리 · Splay 트리 · B-트리** 등의 자가 균형(self-balancing) 변형을 사용. Java `TreeMap`·`TreeSet` 은 Red-Black 트리 기반이다.
④ **BST는 좌 서브트리의 키가 항상 노드 키보다 크고 우 서브트리의 키가 항상 노드 키보다 작다**. **중위 순회 결과는 내림차순 정렬**이 되며, **탐색의 최악 시간복잡도는 항상 O(1)** 이다.

### 문제 9 [유사 기출: 2021년 1회 2과목 — 힙(Heap) 정형 개념]
**힙(Heap) 자료구조**에 관한 설명 중 **가장 옳지 않은 것**은?

① **정의** — **완전 이진트리** 형태이며, **부모-자식 간 대소 관계**만 만족하고 형제 간 순서는 정의되지 않는다. **최대 힙(Max-Heap)** = 부모 ≥ 자식, **최소 힙(Min-Heap)** = 부모 ≤ 자식.
② **연산 시간복잡도** — 삽입·삭제(루트) 모두 **O(log n)**. 완전 이진트리이므로 배열로 표현하면 인덱스 i의 부모 = i/2, 자식 = 2i, 2i+1(1-based 기준). 루트(최댓값 또는 최솟값) 접근은 O(1).
③ **활용** — **우선순위 큐(Priority Queue)** 의 기본 구현, **힙 정렬(Heap Sort)** O(n log n), **다익스트라·프림 알고리즘**의 최소 거리 노드 선택, **Top-K 문제** 등에 사용.
④ **힙은 이진 탐색 트리와 완전히 동일한 자료구조**이며, **중위 순회하면 정렬된 결과가 얻어지고**, **최대 힙에서 루트를 삭제한 후에도 항상 O(1)에 다음 최댓값을 얻을 수 있다**.

### 문제 10 [유사 기출: 2022년 2회 2과목 — AVL/Red-Black 균형 트리 정형 비교]
**자가 균형 이진 탐색 트리(Self-Balancing BST)** 에 관한 설명 중 **가장 옳지 않은 것**은?

① **AVL 트리** — 모든 노드에서 **좌·우 서브트리의 높이 차(Balance Factor)** 가 **-1, 0, +1 중 하나**여야 하는 조건을 유지. 삽입·삭제 후 조건 위반 시 **LL·RR·LR·RL 회전**으로 재균형. 탐색이 매우 빠른 반면 삽입·삭제 시 회전 비용이 크다.
② **Red-Black 트리** — 각 노드를 **빨강/검정**으로 색칠하고 5가지 속성(루트=검정 · NIL=검정 · 빨강-빨강 금지 · 모든 리프 경로의 검정 노드 수 동일 · 루트 검정)을 유지. AVL보다 균형 조건이 느슨해 삽입·삭제 재조정 비용이 낮다. Java `TreeMap`·`HashMap` 버킷 내부·Linux 커널 스케줄러 등에 사용.
③ **B-트리(B-Tree)** — 각 노드가 다수의 키·자식을 가지는 다진(m-ary) 균형 탐색 트리. 노드 크기를 디스크 블록에 맞춰 **디스크 I/O를 최소화**하며, DBMS 인덱스(B+트리)·파일 시스템(NTFS·ext4 등)의 표준 구조.
④ **AVL 트리는 균형 조건이 없어 편향 트리와 동일**하며, **Red-Black 트리는 삽입·삭제 시 회전이 절대 발생하지 않는다**. **B-트리는 단일 이진 노드 구조로 디스크 I/O가 많은 것이 단점이다**.

### 문제 12 [유사 기출: 2023년 3회 2과목 — 그래프 표현 정형 비교]
그래프의 **인접 행렬(Adjacency Matrix) vs 인접 리스트(Adjacency List) 표현**에 관한 설명 중 **가장 옳지 않은 것**은?

① **인접 행렬** — |V|×|V| 크기의 2차원 배열. `A[i][j] = 1` 이면 i→j 간선 존재. 공간복잡도 **O(V²)**. **두 정점 간 간선 존재 여부 확인 O(1)** 로 매우 빠름. 밀집(Dense) 그래프에 유리.
② **인접 리스트** — 각 정점에 대해 인접한 정점들의 연결리스트를 유지. 공간복잡도 **O(V + E)**. **한 정점의 모든 이웃 순회 O(deg(v))** 로 효율적. 희소(Sparse) 그래프에 유리하며, DFS·BFS 등 순회 알고리즘의 표준.
③ **간선 존재 확인 시간** — 인접 행렬은 **O(1)**, 인접 리스트는 **O(deg(v))**. 반대로 **모든 간선 순회**는 인접 행렬 **O(V²)** vs 인접 리스트 **O(V + E)** 로 인접 리스트가 유리.
④ **인접 행렬은 공간복잡도가 O(V + E)로 인접 리스트보다 항상 작고**, **간선 존재 확인이 O(deg(v))로 느리며**, **밀집 그래프에는 인접 리스트가 유리하고 희소 그래프에는 인접 행렬이 유리하다**.

### 문제 13 [유사 기출: 2021년 2회 2과목 — BFS vs DFS 정형 비교]
그래프의 **너비 우선 탐색(BFS) · 깊이 우선 탐색(DFS)** 에 관한 설명 중 **가장 옳지 않은 것**은?

① **BFS(Breadth-First Search)** — **큐(Queue)** 를 사용해 시작 정점에서 가까운(레벨이 낮은) 정점부터 차례로 방문. 최단 경로(unweighted) 탐색·레벨별 처리·웹 크롤링에 사용. 시간복잡도 O(V + E).
② **DFS(Depth-First Search)** — **스택(Stack)** 또는 **재귀 호출**을 사용해 한 방향으로 가능한 깊이까지 내려간 뒤 되돌아온다. 사이클 검출·위상 정렬·강한 연결 요소(SCC)·미로 탐색에 사용. 시간복잡도 O(V + E).
③ **탐색 순서 예시** — 트리 뿌리에서 시작할 때 BFS = 레벨 순회(1,2,3, / 4,5,6,7, …), DFS(전위) = 좌측 깊이 우선(1,2,4,5,3,6,7 …). 방문 여부(visited) 배열로 재방문을 방지하는 것은 두 알고리즘 공통.
④ **BFS는 스택을 사용하고 DFS는 큐를 사용**하며, **BFS의 시간복잡도는 O(V²), DFS는 O(V!)** 이다. **BFS는 사이클 검출·위상 정렬에 사용되고, DFS는 최단 경로 탐색에 사용된다**.

### 문제 14 [유사 기출: 2022년 1회 2과목 — 트리 용어 정형 함정]
다음 트리 용어의 정의로 **가장 옳지 않은 것**은?

① **차수(Degree)** — 한 **노드가 가진 자식(서브트리)의 수**. **트리의 차수** = 트리 내 모든 노드의 차수 중 **최댓값**. 이진트리의 차수는 최대 2.
② **단말 노드(Leaf, Terminal Node)** — 자식이 없는 노드(차수 = 0). **비단말(내부) 노드** = 차수 ≥ 1인 노드.
③ **깊이(Depth) · 높이(Height)** — 깊이 = 루트에서 특정 노드까지의 **경로 길이(간선 수)**. 트리의 높이 = 루트에서 가장 깊은 리프까지의 깊이(단말 노드 최대 깊이). 루트의 깊이는 0으로 관례.
④ **차수는 트리의 총 노드 수와 항상 동일**하며, **단말 노드는 자식이 반드시 2개 이상인 노드**를 의미하고, **깊이는 항상 트리의 총 노드 수와 같으며 루트의 깊이는 반드시 1로 정의된다**.

## 💡 상세 해설

### 문제 1
- **핵심**: 중위 → 후위 변환은 **연산자 우선순위 + 좌결합**을 따라 피연산자를 그대로, 연산자를 뒤로 미룬다. `*`·`/` 는 `+`·`-` 보다 우선순위가 높다.
- **손추적**: `A + B * C - D / E`
  - `B * C` 먼저 → `B C *`
  - `A + (B C *)` → `A B C * +`
  - `D / E` → `D E /`
  - `(A B C * +) - (D E /)` → **`A B C * + D E / -`**
- **정답 ② 함정**: ①은 `A + B` 를 먼저 묶어버림(`A B +` 로 시작하면 `+`·`*` 우선순위 무시), ③은 `*`·`+` 순서를 뒤바꿈, ④는 `A`를 그대로 남겨 후위 표기가 아닌 부분 중위. **후위 표기법은 피연산자 순서는 그대로, 연산자만 뒤로 미룬다**는 원칙 재확인.
- **암기 팁**: 후위 변환 = "**우선순위 높은 연산자부터 괄호 → 괄호 닫는 순간 그 연산자를 뒤로 밀기**". Shunting-yard 알고리즘(Dijkstra)이 표준 방법.

### 문제 2
- **핵심**: 원형큐에서 **한 칸을 비워두는 관례**를 쓰면 공백/포화 조건을 명확히 구분할 수 있다. **최대 저장 = N-1**, **포화 = (rear+1) % N == front**, **공백 = rear == front**.
- **정답 ③ 함정**: ①은 크기 공식·포화 조건 모두 오답, ②는 최대가 N이라 하지만 그러면 공백과 포화가 모두 rear==front 로 구분 불가, ④는 포화 조건을 공백 조건과 뒤바꾼 트랩.
- **암기 팁**: **"공백 = rear == front / 포화 = (rear+1) % N == front / 저장 최대 = N-1"** 3연 세트 암기. 배열 크기를 그대로 다 쓰려면 별도 카운터(count) 변수를 두어야 한다는 트레이드오프도 시험 포인트.

### 문제 3
- **핵심**: 스택 = **LIFO(함수 호출·undo)** / 큐 = **FIFO(프린터 스풀·BFS)** / 덱 = **양쪽 끝 모두 삽입·삭제 가능**.
- **정답 ④ 함정**: 스택-큐를 뒤바꾸고 덱을 스택과 동일하다고 주장하는 3중 뒤바꾸기 함정. **스택이 FIFO 라는 서술은 즉시 오답 신호**.
- **암기 팁**: **"L**IFO = **L**ast Stack, **F**IFO = **F**ront Queue"로 첫 글자 매칭. 덱은 "**Deque = Doubly-Ended Queue**"로 어원 자체가 양쪽 끝을 명시.

### 문제 4
- **핵심**: 연결리스트는 **임의 접근(random access)이 불가**하고 O(n)에 순회. 배열은 O(1) 임의 접근 가능하지만 삽입·삭제는 O(n)(이동 비용). **연결리스트의 강점은 삽입·삭제, 배열의 강점은 임의 접근**이라는 상보 관계.
- **정답 ④ 함정**: "연결리스트가 배열보다 임의 접근이 빠르고 O(1)에 인덱스 접근" 이라는 완전 반대 서술. 배열이 O(1), 연결리스트가 O(n)이 정답. "삽입·삭제가 O(n²)" 도 완전 오답으로 O(n)(재탐색 포함) 또는 O(1)(포인터 이미 있을 때).
- **실무·시험 포인트**: **배열 vs 연결리스트 = 임의접근/삽입·삭제 상보 관계 · 이중 연결리스트는 양방향 순회 · 원형 연결리스트는 라운드 로빈**의 3세트 즉시 매칭. Java `ArrayList` vs `LinkedList` 차이도 이 원리 그대로.

### 문제 5
- **핵심**: 포화(Perfect) = **모든 레벨 완전 채움**, 완전(Complete) = **마지막 레벨을 왼쪽부터 순차적으로 채움**. 편향(Skewed)은 한쪽 자식만 가져 O(n) 탐색.
- **정답 ④ 함정**: 포화 ↔ 완전 정의를 뒤바꾸고, 노드 수 공식(2^(h+1)-1)을 (h+1)로 오답 처리한 3중 트랩. **편향 이진트리가 O(log n)** 도 완전 오답으로 사실은 O(n).
- **암기 팁**: **"포화 = Perfect(빈틈 없음, 모든 레벨 다 채움) / 완전 = Complete(왼쪽부터 순차 채움, 힙의 구조) / 편향 = Skewed(한쪽만, 사실상 연결리스트)"** 3분 매칭. 노드 수 공식 2^(h+1)-1 은 등비수열 합 공식과 동일.

### 문제 6
- **핵심**: **중위 = 좌 → 루트 → 우** 재귀 순회. 손추적으로 각 서브트리를 좌-루트-우 순으로 방문.
- **손추적**:
  - A의 좌서브트리(B) 먼저 → B의 좌(D) 방문 → **D** → B 루트 → **B** → B의 우서브트리(E) → E의 좌(G) → **G** → E 루트 → **E** → (E는 우 없음)
  - A 루트 → **A**
  - A의 우서브트리(C) → (C는 좌 없음) → C 루트 → **C** → C의 우(F) → **F**
  - 결과: **D - B - G - E - A - C - F**
- **정답 ③ 함정**: ①은 전위(루트-좌-우), ②는 G와 E 순서 뒤바꿈(E의 좌서브트리가 G인데 E 먼저 방문 = 전위/후위 혼동), ④는 D와 G 처리 순서 뒤엉킴.
- **암기 팁**: **재귀 함수 프레임**을 머릿속에 그리며 "좌서브트리 완전 종료 → 루트 방문 → 우서브트리 완전 종료"를 반복. 순회는 반드시 손추적 연습.

### 문제 7
- **핵심**: **전위 = 루트-좌-우 / 중위 = 좌-루트-우 / 후위 = 좌-우-루트**. 루트의 위치가 순회 이름과 일치(**前**=앞, **中**=중간, **後**=뒤).
- **정답 ② 함정**: ①은 전위와 중위 뒤바꿈, ③은 좌·우 순서 자체 뒤엉킴, ④는 좌·우 방향 반전(우 먼저) 트랩.
- **암기 팁**: **"루트 위치가 순회 이름이다"** — 전(前)위=루트 앞·중(中)위=루트 중간·후(後)위=루트 뒤. 좌→우 순서는 세 방식 모두 동일(예외: 역순회 = 우→좌 는 별개).

### 문제 8
- **핵심**: BST = **좌 < 노드 < 우** · 중위 순회 시 **오름차순 정렬**. 최악은 편향 트리에서 O(n).
- **정답 ④ 함정**: BST 조건(좌 < 노드 < 우)을 완전 뒤바꾸고, 중위 순회 결과를 내림차순으로 오답 처리, 최악 시간복잡도를 O(1)로 잘못 서술한 3중 트랩.
- **암기 팁**: **"BST + 중위 = 오름차순 정렬"** 은 시험 필수 암기. Red-Black·AVL 은 최악을 O(log n)으로 보장하기 위한 자가 균형 장치. Java `TreeMap`·`TreeSet` = Red-Black, Windows 커널 프로세스 스케줄러도 Red-Black 사용.

### 문제 9
- **핵심**: 힙 = **완전 이진트리 + 부모-자식 대소 조건만 유지**(형제 순서 무관). 삽입·삭제 O(log n), 루트 접근 O(1). **BST와는 완전히 다른 구조**.
- **정답 ④ 함정**: "힙 = BST 동일 · 중위 순회 시 정렬 · 삭제 후 다음 최댓값 O(1)" 세 서술 모두 오답. 힙 중위 순회는 정렬 결과가 아니며, 루트 삭제 후 재구조화(sift-down)로 O(log n) 필요.
- **암기 팁**: **"힙은 BST가 아니다"** 는 즉시 함정 신호. 힙은 **부분 순서(partial order)** 만 유지, BST는 **완전 순서(total order)** 유지. 우선순위 큐 = 힙 표준 구현이라는 매칭도 필수.

### 문제 10
- **핵심**: AVL = **엄격 균형(BF ∈ {-1,0,+1})** · 회전 잦음 · 탐색 빠름 / Red-Black = **느슨 균형(색 규칙 5개)** · 회전 적음 · Java `TreeMap` / B-트리 = **다진 균형** · DBMS 인덱스 표준.
- **정답 ④ 함정**: AVL을 편향 트리로, Red-Black을 회전 없는 것으로, B-트리를 단일 이진으로 서술한 3중 오답.
- **암기 팁**: **"AVL = 엄격/회전多/탐색빠름 · RB = 느슨/회전少/삽입빠름 · B-tree = 다진/디스크I/O최소/DB인덱스"** 3분 매칭. Java `HashMap` 버킷은 노드 8개 이상이면 Red-Black 트리로 전환됨(Java 8+).

### 문제 11
- **핵심**: **무방향 최대 간선 = n(n-1)/2 / 방향 최대 간선 = n(n-1)**. 무방향은 (u,v)와 (v,u)가 동일 간선이므로 2로 나눔.
- **정답 ② 함정**: ①은 두 공식을 완전히 뒤바꿈, ③·④는 완전 오답 공식.
- **암기 팁**: **"완전 그래프 K_n 의 간선 수 = n(n-1)/2"** 는 조합 C(n,2) 와 동일. 자기 자신 간선(self-loop)은 계산에서 제외. 예: n=4 → 무방향 6개, 방향 12개.

### 문제 12
- **핵심**: 인접 행렬 = **O(V²) 공간 · O(1) 간선 확인 · 밀집 그래프 유리** / 인접 리스트 = **O(V+E) 공간 · O(deg) 간선 확인 · 희소 그래프 유리 · BFS/DFS 표준**.
- **정답 ④ 함정**: 두 표현의 특징을 완전히 뒤바꾸고, 밀집·희소 그래프 선호 방향도 반대로 서술한 4중 트랩.
- **암기 팁**: **"밀집(Dense) = 행렬 / 희소(Sparse) = 리스트"** 즉시 매칭. 실무 그래프 라이브러리(NetworkX·JGraphT)는 대부분 인접 리스트 기본. 페이스북 소셜 그래프(수십억 정점) = 반드시 인접 리스트.

### 문제 13
- **핵심**: **BFS = 큐(FIFO) · 최단 경로 · 레벨 순회 / DFS = 스택 또는 재귀 · 사이클 검출 · 위상 정렬**. 둘 다 O(V+E).
- **정답 ④ 함정**: BFS ↔ DFS 자료구조 뒤바꾸기(스택/큐), 시간복잡도 오답 (O(V²)·O(V!)), 응용 뒤바꾸기(사이클 검출 vs 최단 경로) 의 3중 트랩.
- **암기 팁**: **"BFS = Breadth = 큐 = 최단 경로 · DFS = Depth = 스택 = 위상/사이클"** 매칭. BFS의 첫 방문이 시작점으로부터의 최단 경로임을 이용해 unweighted graph shortest path가 결정됨(가중치 있는 그래프는 Dijkstra 사용).

### 문제 14
- **핵심**: **차수 = 자식 수** · 트리의 차수 = 노드 차수 중 **최댓값** · **단말 노드 = 차수 0** · **깊이 = 루트에서 특정 노드까지 간선 수 · 루트 깊이 = 0**.
- **정답 ④ 함정**: 모든 정의를 완전 오답으로 뒤바꾼 4중 함정. **차수 = 총 노드 수** 는 즉시 오답 신호. **단말 = 자식 ≥ 2** 도 완전 반대(자식 = 0이 정답). **루트 깊이 = 1** 도 오답(관례상 0).
- **암기 팁**: **"차수 = 자식 수(최댓값이 트리 차수) · 리프 = 자식 없음 · 깊이 = 루트에서 간선 수(루트=0)"** 3세트 즉시 매칭. 노드 수(size) 와 깊이(depth) · 높이(height) 는 별개 개념임을 반드시 구분.

### 문제 15
- **핵심**: 후위 표기식 계산은 **스택 push/pop**으로 진행. 피연산자는 push, 연산자를 만나면 top 2개 pop 해서 연산 후 결과 push.
- **손추적**: `5 3 2 * + 8 4 / -`
  - `5` push → 스택: [5]
  - `3` push → [5, 3]
  - `2` push → [5, 3, 2]
  - `*` → pop 2,3 → 3*2=6 push → [5, 6]
  - `+` → pop 6,5 → 5+6=11 push → [11]
  - `8` push → [11, 8]
  - `4` push → [11, 8, 4]
  - `/` → pop 4,8 → 8/4=2 push → [11, 2]
  - `-` → pop 2,11 → 11-2=**9** → 최종 [9]
- **정답 ③ 함정**: ①(3)은 뺄셈 방향을 뒤집어 8/4-... 를 먼저 처리하는 오답, ②(6)은 `3*2=6` 단계만 계산하고 이후 진행을 놓친 오답, ④(11)은 `5+3*2=11` 까지만 계산하고 `-8/4` 를 놓친 오답. 후위 계산은 반드시 끝까지 스택 진행.
- **암기 팁**: 후위 계산은 반드시 손으로 스택을 그리며 검산. **연산자 만나면 top 2개 pop, 뒤에서 나온 것이 뒷 피연산자**(예: `-` 만나면 top이 뺄셈의 뺄 값). Shunting-yard의 역순.
