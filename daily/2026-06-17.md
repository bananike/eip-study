## 🔍 오답 역추적 개념 발췌
- **응집도(Cohesion)** — 모듈 **내부** 구성요소들이 얼마나 밀접하게 관련되어 있는가. **높을수록 좋다.** 약→강 두음 "우·논·시·절·통·순·기".
- **결합도(Coupling)** — 모듈 **간** 상호 의존성의 정도. **낮을수록 좋다.** 약→강 두음 "자·스·제·외·공·내".
- 두 척도의 황금률: **"높은 응집도 + 낮은 결합도" = 좋은 모듈화** — 변경 파급 최소·재사용성 최대.
- **모듈화 효과 4종** — 유지보수성↑ / 재사용성↑ / 복잡도↓ / 변경 영향도↓.
- **응집도 함정 7항** ─ "우연=무관계", "논리=비슷한 종류 분기(switch case)", "시간=동시에", "절차=정해진 순서", "통신=같은 입출력 자료", "순차=출력이 다음 입력", "기능=단일 기능".
- **결합도 함정 6항** ─ "자료=값(파라미터)", "스탬프=배열·구조체(레퍼런스)", "제어=플래그·논리 제어 신호", "외부=외부 디바이스·프로토콜·라이브러리", "공통=전역변수(Global)", "내용=내부 코드 직접 참조(jump in)".

### 소프트웨어 아키텍처 패턴 8종 1줄 요약
- **레이어드(Layered)** — 계층 분리(프레젠테이션/비즈니스/데이터). OSI 7계층·TCP/IP 4계층이 예.
- **클라이언트-서버(Client-Server)** — 요청-응답 모델. 서버는 자원 제공자.
- **MVC** — Model(데이터·로직) / View(표현) / Controller(입력 처리·중계). 웹 프레임워크 표준.
- **파이프-필터(Pipe-Filter)** — 데이터 스트림을 필터들이 단계적으로 변환. Unix Shell `|`, 컴파일러 파이프라인.
- **브로커(Broker)** — 분산 컴포넌트 사이 호출 중개. CORBA·메시지 브로커.
- **블랙보드(Blackboard)** — 공유 저장소(블랙보드)에 모든 컴포넌트가 접근. AI 추론·음성인식.
- **마스터-슬레이브(Master-Slave)** — 마스터가 작업 분할→슬레이브가 병렬 수행→결과 합산.
- **이벤트버스(Event-Bus / Pub-Sub)** — 발행자가 이벤트 발행→구독자가 수신. 비동기·느슨한 결합.

### Kruchten 4+1 뷰 모델
| 뷰 | 관심사 | 다이어그램 예 |
|---|---|---|
| **논리(Logical)** | 기능·객체 구조 | 클래스 / 객체 다이어그램 |
| **프로세스(Process)** | 동시성·동기화·성능 | 활동 / 시퀀스 다이어그램 |
| **개발(Development / 구현)** | 모듈·패키지·소스 구성 | 컴포넌트 다이어그램 |
| **물리(Physical / Deployment)** | 하드웨어 배치·노드·네트워크 | 배치(Deployment) 다이어그램 |
| **+1 시나리오(Scenarios / Use Case)** | 위 4뷰 통합·검증 | **유스케이스 다이어그램** |

→ **보안 / 정보 / 데이터** 뷰는 4+1에 **없음**(함정).

### 아키텍처 평가 방법 4종
| 방법 | 핵심 | 특징 |
|---|---|---|
| **SAAM** | 시나리오 기반 | 최초의 정형 평가법(수정용이성·기능 중심) |
| **ATAM** | 품질속성 + **트레이드오프** | 다중 품질속성 절충점 식별, SAAM 확장 |
| **CBAM** | 비용 vs 편익(경제성) | ATAM 결과를 비용편익 관점에서 보완 |
| **ARID** | 부분(중간) 설계 평가 | 설계 초기에 일부분만 검토 |

## 💡 상세 해설

### 문제 1
**정답 ④ 기능적 응집도(Functional Cohesion)**
- 응집도 7단계 약→강: **우연적 < 논리적 < 시간적 < 절차적 < 통신적 < 순차적 < 기능적**.
- **기능적 응집도** = 모듈이 **단 하나의 명확한 기능**만 수행 → 응집도 최강·이상적.
- 보기 ①②③ 모두 기능적보다 약함. ① 시간적(동일 시각에 처리되는 활동 묶음)은 중간 이하, ② 절차적(순서대로 수행)은 ③ 통신적(같은 입출력 자료 공유)보다 약함.
- **두음 "우·논·시·절·통·순·기"** — 마지막 "**기**(기능)"가 답.

### 문제 2
**정답 ③ 자료 결합도(Data Coupling)**
- 결합도 6단계 약→강: **자료 < 스탬프 < 제어 < 외부 < 공통 < 내용**.
- **자료 결합도** = 모듈 간 **값(파라미터)만** 주고받음 → 결합도 최약·이상적.
- ① 내용 결합도는 가장 강함(최악), ② 공통(전역변수 공유)·④ 제어(플래그 전달) 모두 자료보다 강함.
- **두음 "자·스·제·외·공·내"** — 첫 "**자**(자료)"가 답.

### 문제 3
**정답 ② 시간적 응집도(Temporal Cohesion)**
- **시간적 응집도** = 동일한 **시점/시간대**에 처리되는 활동들을 한 모듈로 묶음 — 대표 예: **초기화(init) 모듈**, **종료(shutdown) 모듈**.
- ① 논리적 = "비슷한 종류"의 작업을 분기(switch). ③ 절차적 = "정해진 **순서**". ④ 통신적 = "같은 **입출력 데이터**" 공유 — 시간이 아닌 데이터 기준.
- 함정 키워드 **"같은 시간 / 동일 시점"** → 시간적.

### 문제 4
**정답 ① 내용 결합도(Content Coupling)**
- **내용 결합도** = 한 모듈이 **다른 모듈의 내부 코드·지역변수·논리 흐름을 직접 참조/변경** — 결합도 최강(최악). goto·branch into 등 사용.
- ② 공통 = **전역(Global) 자료** 공유. ③ 외부 = 외부 디바이스·프로토콜 공유. ④ 제어 = **플래그·제어 신호** 전달.
- 함정 키워드 **"내부 코드 직접 참조 / 다른 모듈의 지역변수"** → 내용 결합도.

### 문제 5
**정답 ④ Connector**
- **MVC 3구성요소** = **Model**(데이터·비즈니스 로직) / **View**(사용자에게 표현·UI) / **Controller**(입력 처리·Model-View 중계).
- **Connector**는 컴포넌트 기반 아키텍처(CBA)의 용어로 컴포넌트 간 연결 추상화 — MVC 구성요소가 **아님**.
- 함정: "MVC + Adapter / Connector / Mediator" 등 외부 패턴 용어가 보기로 자주 끼어듦.

### 문제 6
**정답 ④ (블랙보드 패턴의 설명)**
- **파이프-필터** = 필터(컴포넌트)들이 데이터 스트림을 **단계별로 변환·처리**, 필터 간 데이터는 **파이프**로 흘림. 대표 예: **Unix Shell `cmd1 | cmd2 | cmd3`**, 컴파일러(렉서→파서→코드젠).
- ④의 "**공유 저장소(Blackboard)에 두고 모든 컴포넌트가 동시 접근**"은 **블랙보드 패턴**의 정의 — AI 추론·음성인식이 예. 파이프-필터와 정반대 데이터 공유 방식.
- ①②③ 모두 파이프-필터의 핵심 특징.

### 문제 7
**정답 ③ 보안(Security) 뷰**
- **Kruchten 4+1 뷰** = **논리(Logical)** + **프로세스(Process)** + **개발(Development / 구현)** + **물리(Physical / Deployment)** + **시나리오(Scenarios / Use Case)**.
- **보안·정보·데이터 뷰**는 4+1에 포함되지 **않음** (함정 빈출).
- 암기 두음 "**논·프·개·물 + 시**" — 4개 본 뷰 + 1개 통합용 시나리오.

### 문제 8
**정답 ② ATAM (Architecture Tradeoff Analysis Method)**
- **ATAM** = 여러 **품질속성**(성능·가용성·보안·수정용이성 등) 사이의 **트레이드오프 지점**을 식별·평가 — SAAM의 확장판.
- ① **SAAM** = 최초의 시나리오 기반 평가법(주로 수정용이성). ③ **CBAM** = **비용 편익**(경제성) 관점, ATAM 보완. ④ **ARID** = 부분 설계의 능동적 검토.
- 함정 키워드 **"트레이드오프"** → ATAM 즉시 호출.
