## 🎯 오늘의 기출문제 — 2021년 정처기 필기 20문제 (5과목 골골루 · 회독 2)

### 문제 1 [유사 기출: 2021년 1회 1과목 — 객체지향 SOLID 5원칙 정의 매칭]
다음 설명에 해당하는 **객체지향 설계 원칙(SOLID)** 으로 가장 적절한 것은?

> 클래스는 **확장에는 열려 있고, 변경에는 닫혀** 있어야 한다. 즉, 기존 코드를 **수정하지 않고** 새로운 동작을 **추가**할 수 있도록 설계해야 한다.

① **SRP(Single Responsibility Principle)** — 단일 책임 원칙. 클래스는 단 하나의 변경 이유만 가져야 한다.
② **OCP(Open-Closed Principle)** — 개방-폐쇄 원칙. 확장 개방·수정 폐쇄.
③ **LSP(Liskov Substitution Principle)** — 리스코프 치환 원칙. 자식 객체는 부모 객체로 대체 가능해야 한다.
④ **DIP(Dependency Inversion Principle)** — 의존 역전 원칙. 상위 모듈은 하위 모듈이 아닌 추상화에 의존.

### 문제 2 [유사 기출: 2021년 2회 1과목 — 응집도(Cohesion) 강도 순서 정형 패턴]
다음 중 응집도(Cohesion)의 강도가 **가장 강한 것**은?

① **논리적 응집도(Logical Cohesion)** — 유사한 성격·동일 카테고리의 기능이 한 모듈에 모임.
② **시간적 응집도(Temporal Cohesion)** — 특정 시점에 함께 실행되는 기능(초기화·종료 루틴).
③ **순차적 응집도(Sequential Cohesion)** — 한 요소의 출력이 다음 요소의 입력으로 사용.
④ **기능적 응집도(Functional Cohesion)** — 모듈 내 모든 요소가 단일 기능 수행에 기여(가장 이상적).

### 문제 3 [유사 기출: 2021년 3회 1과목 — 럼바우(Rumbaugh) 객체지향 분석 기법 정형 패턴]
다음 중 **럼바우(Rumbaugh)의 객체지향 분석(OMT) 3가지 모델링**에 해당하지 **않는** 것은?

① **객체 모델링(Object Modeling)** — 시스템의 정적 구조를 객체·클래스·속성·관계로 표현(객체 다이어그램).
② **동적 모델링(Dynamic Modeling)** — 상태도(State Diagram)를 활용해 시간 흐름에 따른 객체의 상태 변화·이벤트 표현.
③ **기능 모델링(Functional Modeling)** — 자료 흐름도(DFD)를 활용해 자료 흐름·변환 처리 표현.
④ **물리 모델링(Physical Modeling)** — 하드웨어·네트워크·배치 등 시스템의 물리적 구성 요소를 표현(럼바우 OMT 단계에 없음).

### 문제 4 [유사 기출: 2021년 1회·3회 1과목 — Singleton 디자인 패턴 식별 정형 패턴]
다음 설명에 해당하는 **GoF 디자인 패턴**으로 가장 적절한 것은?

> 어떤 클래스의 인스턴스가 **오직 하나만 생성**되도록 보장하고, 그 인스턴스에 접근할 수 있는 **전역 접근점(global access point)** 을 제공한다. 데이터베이스 연결 풀·로깅 객체·설정 관리자 등에 주로 사용된다.

① **Factory Method** — 객체 생성을 서브클래스에 위임하여 어떤 클래스의 인스턴스를 만들지 결정.
② **Singleton** — 인스턴스 유일 보장·전역 접근.
③ **Prototype** — 원본 객체를 복제(clone)하여 새 객체 생성.
④ **Observer** — 한 객체의 상태 변화를 의존 객체들에게 자동 통지(행위 패턴).

### 문제 7 [유사 기출: 2021년 1회 2과목 — 이진 탐색(Binary Search) 시간복잡도 정형 패턴]

① O(1) — 상수 시간. 해시 테이블 평균 탐색.
② O(log n) — 매 단계마다 탐색 범위가 절반으로 축소.

### 문제 9 [유사 기출: 2021년 1회 3과목 — 키 종류 5개 정의 매칭 정형 패턴]
다음 설명에 해당하는 **관계형 데이터베이스의 키 종류**로 가장 적절한 것은?

> 한 릴레이션에서 **기본키로 선정되지 못한 나머지 후보키들**을 의미한다. 후보키이면서도 기본키 역할을 수행하지 않는 키.

① **슈퍼키(Super Key)** — 유일성은 있으나 최소성을 만족하지 않을 수 있는 속성 집합.
② **대체키(Alternate Key)** — 후보키 중 기본키로 선정되지 않은 나머지 후보키.
③ **외래키(Foreign Key)** — 다른 릴레이션의 기본키를 참조하는 속성.
④ **복합키(Composite Key)** — 두 개 이상 속성을 결합해 만든 키.

### 문제 10 [유사 기출: 2021년 2회 3과목 — 이상 현상(Anomaly) 종류 정형 패턴]
다음 중 정규화되지 않은 릴레이션에서 발생하는 **이상 현상(Anomaly)** 에 해당하지 **않는** 것은?

① **삽입 이상(Insertion Anomaly)** — 원하지 않는 데이터를 함께 삽입해야 하거나, 일부 속성 값이 없어 삽입이 불가한 현상.
② **삭제 이상(Deletion Anomaly)** — 한 튜플을 삭제할 때 의도치 않은 다른 정보까지 함께 삭제되는 현상.
③ **갱신 이상(Update Anomaly)** — 중복된 데이터 중 일부만 갱신되어 데이터 불일치가 발생하는 현상.
④ **조회 이상(Retrieval Anomaly)** — 데이터 조회 시 결과가 누락되는 현상(정규화 이상 3종에 포함되지 않는 용어).

### 문제 11 [유사 기출: 2021년 3회 3과목 — 인덱스(B-Tree) 특성 정형 패턴]
**관계형 데이터베이스의 인덱스(Index)** 에 대한 설명으로 옳지 **않은** 것은?

① 인덱스는 **검색(SELECT) 성능을 향상**시키지만, **삽입·삭제·갱신 성능을 저하**시킬 수 있다.
② 대부분의 DBMS는 **B-Tree(또는 B+Tree)** 구조의 인덱스를 기본적으로 지원한다.
③ **기본키(Primary Key)** 에는 일반적으로 **자동으로 인덱스가 생성**된다(클러스터드 인덱스).
④ 인덱스를 많이 만들수록 **모든 종류의 쿼리 성능이 항상 향상**되며, **저장 공간 비용도 발생하지 않는다**.

### 문제 12 [유사 기출: 2021년 1회·3회 3과목 — 관계 대수 연산자 정형 패턴]
다음 중 **관계 대수(Relational Algebra) 연산자**의 기호와 기능 매칭으로 옳지 **않은** 것은?

① **σ(SELECT)** — 조건을 만족하는 **튜플(행)** 을 선택. (예: σ_age>20(STUDENT))
② **π(PROJECT)** — 지정한 **속성(열)** 만 추출. (예: π_name(STUDENT))
③ **⋈(JOIN)** — 두 릴레이션을 공통 속성으로 결합.
④ **÷(DIVISION)** — 두 릴레이션의 **모든 튜플**을 합집합으로 결합하는 연산(합집합 ∪ 의 기능과 혼동한 잘못된 매칭).

### 문제 13 [유사 기출: 2021년 2회 4과목 — C 문자열 길이(strlen) 정형 패턴]

#include <stdio.h>

④ 6 5  (strlen에 NULL 포함, sizeof에 미포함)

### 문제 14 [유사 기출: 2021년 1회·3회 4과목 — Java 인터페이스 vs 추상 클래스 정형 패턴]

### 문제 16 [유사 기출: 2021년 1회·2회 4과목 — 프로세스 상태 전이 정형 패턴]
다음 중 **프로세스 상태 전이(State Transition)** 에 대한 설명으로 옳지 **않은** 것은?

① **준비(Ready) → 실행(Running)** — 디스패처(Dispatcher)에 의해 CPU를 할당받음(Dispatch).
② **실행(Running) → 대기(Waiting/Blocked)** — I/O 요청, 이벤트 대기 등으로 CPU를 자진 반납.
③ **대기(Waiting) → 준비(Ready)** — I/O 완료 등 대기 이벤트 발생으로 CPU 할당 가능 상태로 복귀(Wake up).
④ **대기(Waiting) → 실행(Running)** — I/O가 완료된 프로세스는 **준비 상태를 거치지 않고 즉시 CPU를 점유**한다.

### 문제 2
**정답 ④** — 응집도 **7단계 약→강**: **우연적 → 논리적 → 시간적 → 절차적 → 통신적 → 순차적 → 기능적**. (두문자: "**우·논·시·절·통·순·기**")
- **기능적 응집도**는 모듈 내 모든 요소가 **단일 기능 수행**에 기여 → 가장 이상적(가장 강함).

- 결합도 6단계 약→강: **자료 → 스탬프 → 제어 → 외부 → 공통 → 내용**(자·스·제·외·공·내).

### 문제 16
**정답 ④** — **대기(Waiting/Blocked) → 실행(Running) 직접 전이 불가**. I/O 완료 시 항상 **대기 → 준비(Ready)** 로 먼저 가서 디스패처가 CPU를 할당할 때까지 기다림.

- 함정 단골: "대기 → 실행", "준비 → 종료" 둘 다 직접 전이 불가.

### 문제 17
**정답 ④** — **스크럼 공식 3역할** = **Product Owner / Scrum Master / Development Team** (스크럼 가이드 2017 이전). 2020 개정에서는 "Developers"로 통합 명명이지만 핵심 3역할 구조는 유지.
- "프로젝트 관리자(PM)" 역할은 전통적 폭포수·PMBOK 용어 — Scrum에는 없음.
- Scrum 4미팅: 스프린트 계획 / 일일 스크럼(15분) / 스프린트 리뷰 / 회고. Scrum 3산출물: 제품 백로그 / 스프린트 백로그 / 증분(Increment).

### 문제 18
**정답 ④** — **HTTP/2** 는 응용 계층 웹 프로토콜로, TCP·고대역 환경 — IoT 저전력·근거리 무선과 무관.
- IoT 표준 무선: **ZigBee**(802.15.4) / **BLE**(블루투스) / **LoRa·NB-IoT**(LPWAN) / **Z-Wave**(스마트홈) / **Thread**(메쉬).
- IoT 응용 프로토콜: **MQTT**(Pub/Sub, 경량) / **CoAP**(REST UDP) — HTTP/2 아님.

### 문제 19
**정답 ②** — SQL Injection 1순위 대응 = **Prepared Statement(파라미터화 쿼리)**. 입력값은 데이터로만 처리되어 SQL 구문으로 해석되지 않음.
- 보조 방어: 입력값 검증(화이트리스트), 이스케이프 처리, 최소 권한 원칙(DBA 권한 부여 X), 에러 메시지에 SQL 구문 노출 금지.
- ①③④는 모두 취약점을 키우는 잘못된 방법 — 함정.

### 문제 20
**정답 ④** — **RAID 6 = 이중(Dual) 패리티 사용 → 디스크 2개까지 동시 고장 복구 가능**. ④는 RAID 5와 동일하게 단일 패리티라 한 점이 오류.
- RAID 0: 스트라이핑(성능 ↑, 내결함성 X) / RAID 1: 미러링(가용성 ↑, 용량 효율 50%) / RAID 5: 단일 분산 패리티(N+1, 1개 고장 복구) / RAID 6: 이중 분산 패리티(N+2, 2개 고장 복구) / RAID 10(1+0): 미러링 + 스트라이핑(고가용+고성능).
- 함정: RAID 5는 최소 디스크 3개, RAID 6는 최소 4개 필요.

## 📊 자주 틀리는 함정 5가지 (2021년 회독 2/4 기준)
1. **OCP("확장 개방·수정 폐쇄")** ↔ **DIP("추상화 의존")** 정의 뒤바꿈 — 문제 1·함정 보기.
2. **응집도 강약 순서**: 시간적·논리적은 약, 순차적·기능적은 강. 보기에 "절차적(중간) + 통신적(중상)" 끼워넣어 헷갈리게 함.
3. **트리 순회 순서 정의**: **전위 = 루·좌·우 / 중위 = 좌·루·우 / 후위 = 좌·우·루**. 보기에서 1글자만 바꿔 함정.
4. **상향식 = 드라이버 / 하향식 = 스텁**. 한 글자만 바꿔 출제 — 절대 기억.
5. **RAID 5(단일 패리티 / 1개 복구)** vs **RAID 6(이중 패리티 / 2개 복구)** — RAID 6는 항상 "이중" 키워드와 함께 출제됨.
