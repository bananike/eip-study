### 문제 1 [유사 기출: 2021년 2회 5과목 — 정보보안 3원칙 CIA·AAA 정형]
정보보안의 **3대 원칙(CIA Triad)** 과 **AAA(3A)** 개념에 관한 설명 중 **가장 옳지 않은 것**은?

① **Confidentiality(기밀성)** — 인가된 사용자에게만 정보 접근을 허용. 대응 통제 = **암호화·접근통제·라벨링**. 위협 = 도청·스니핑·정보 유출.
② **Integrity(무결성)** — 정보가 인가 없이 변경되지 않도록 보장. 대응 통제 = **해시(MD5·SHA)·전자서명·MAC·버전관리**. 위협 = 변조·재전송·부인.
③ **Availability(가용성)** — 인가된 사용자가 필요할 때 정보·서비스에 접근 가능해야 함. 대응 통제 = **이중화·백업·부하분산·재해복구·DDoS 방어**. 위협 = DoS/DDoS·시스템 장애.
④ **AAA**는 **Anonymity(익명성)·Anti-repudiation(부인방지)·Archiving(보관)** 의 약자이며 CIA와 **완전히 동일한 3원칙**이다. Confidentiality가 실제로는 **가용성**을 뜻하고 Availability는 기밀성을 뜻한다.

### 문제 2 [유사 기출: 2022년 1회 5과목 — 대칭키 vs 비대칭키 정형]
**대칭키(Symmetric) 암호** 와 **비대칭키(Asymmetric/공개키) 암호** 의 비교 설명 중 **가장 옳지 않은 것**은?

① **대칭키** — 암호화·복호화에 **동일한 비밀키(공유키)** 를 사용. **속도가 빠르고** 대용량 데이터 암호화에 적합하나 **키 분배·관리가 어렵고** N명 사용 시 **N(N-1)/2개** 키 필요.
② **비대칭키** — **공개키(Public Key)로 암호화** → **개인키(Private Key)로 복호화**(기밀성 목적). 또는 **개인키로 서명** → **공개키로 검증**(전자서명 목적). N명 사용 시 **2N개** 키만 필요.
③ **하이브리드 방식** — 대용량 데이터는 **대칭키로 암호화**, 그 대칭키(세션키)를 다시 **수신자의 공개키로 암호화**하여 전송(SSL/TLS·PGP·S/MIME이 채택).
④ 비대칭키는 **동일한 비밀키를 양측이 공유**하며 대칭키보다 **훨씬 빠르다**. 대칭키는 **공개키·개인키 두 쌍**을 사용하며 전자서명 기능을 기본으로 제공한다.

### 문제 3 [유사 기출: 2023년 2회 5과목 — 대칭키 알고리즘(DES/AES/SEED/ARIA/IDEA) 정형]
대표적인 **대칭키 블록 암호 알고리즘** 에 관한 설명 중 **가장 옳지 않은 것**은?

① **DES(Data Encryption Standard)** — 1977년 미국 표준. **64비트 블록·56비트 키**·16라운드 Feistel 구조. **키 길이가 짧아 무차별 대입에 취약**하여 현재는 폐기. **3DES(Triple DES)** = DES를 3회 적용(112 또는 168비트 실효 키).
② **AES(Advanced Encryption Standard)** — 2001년 미국 NIST 채택(Rijndael). **128비트 블록·키 128/192/256비트** 선택. **SPN(Substitution-Permutation Network)** 구조. DES 후속 사실상 세계 표준.
③ **SEED** — **KISA(한국인터넷진흥원) 개발**(1999년)·**128비트 블록·128비트 키**·16라운드 Feistel. **국내 표준**·전자상거래·인터넷뱅킹. **ARIA** — **국가정보원 주도 개발**(2004년)·**128비트 블록·키 128/192/256비트**·SPN 구조·**국가 표준**.
④ AES는 **국내 KISA가 개발한 국가 표준**이고, SEED와 ARIA는 **미국 NIST가 채택한 국제 표준**이다. DES의 블록 크기는 **256비트**이며 IDEA는 **한국 표준 알고리즘**이다.

### 문제 4 [유사 기출: 2024년 1회 5과목 — 비대칭키(RSA/ECC/DH) 정형]
대표적인 **비대칭키(공개키) 암호 알고리즘** 에 관한 설명 중 **가장 옳지 않은 것**은?

① **RSA(Rivest-Shamir-Adleman)** — 1978년 발표. **큰 소수 두 개의 곱을 소인수분해하기 어렵다**는 수학적 난제 기반. 암호화·전자서명 모두 가능. 안전을 위해 현재 **2048비트 이상** 권장.
② **ECC(Elliptic Curve Cryptography·타원곡선 암호)** — 타원곡선 위의 이산대수 문제(ECDLP) 기반. **같은 안전 강도에서 RSA보다 키 길이가 훨씬 짧음**(RSA 2048 ≈ ECC 224). 모바일·IoT 등 자원 제약 환경 유리.
③ **Diffie-Hellman(DH) 키 교환** — 1976년. **공개된 채널에서 안전하게 대칭키(공유 세션키)를 교환**하는 프로토콜. **키 교환 전용**이며 그 자체로는 암호화·전자서명 기능 없음(중간자 공격 방어에는 인증 결합 필요).
④ RSA는 **타원곡선 이산대수 기반**이고 ECC는 **소인수분해 기반**이다. DH는 **직접 데이터 암호화·전자서명 표준 알고리즘**으로 널리 쓰이며 별도의 대칭키 없이 통신 전 과정을 담당한다.

### 문제 5 [유사 기출: 2020년 4회 5과목 — 해시 함수 · HMAC 정형]
**해시(Hash) 함수** 와 **HMAC** 에 관한 설명 중 **가장 옳지 않은 것**은?

① 해시 함수의 성질 — **① 일방향성(One-way, 프리이미지 저항)** · **② 두 번째 프리이미지 저항** · **③ 충돌 저항성(Collision Resistance)** · **④ 고정 길이 출력** · **⑤ 눈사태 효과(Avalanche Effect)**. 무결성 검증·전자서명·비밀번호 저장에 사용.
② **MD5**(128비트 출력, 1992)·**SHA-1**(160비트, 1995) — 충돌 취약점 발견·현재는 안전하지 않음. **SHA-2 계열**(SHA-224/256/384/512)·**SHA-3**(Keccak, 2015 표준) — 현재 안전한 표준.
③ **HMAC(Hash-based MAC)** — 해시 함수 + **비밀 키**를 결합해 만드는 메시지 인증 코드(RFC 2104). 무결성 + **인증(송신자 확인)** 을 동시에 제공. TLS·IPSec에서 사용.
④ 해시 함수는 **양방향(가역)** 이어서 원문 복원이 가능하며, **충돌은 반드시 없어야 함이 아니라 없어도 그만**이다. **SHA-256의 출력은 512비트**이고 **MD5는 현재 가장 안전한 해시 알고리즘**으로 권장된다.

### 문제 6 [유사 기출: 2023년 1회 5과목 — 전자서명·PKI·X.509 정형]
**전자서명(Digital Signature)** 과 **PKI(공개키 기반구조)** 에 관한 설명 중 **가장 옳지 않은 것**은?

① **전자서명 절차** — 송신자가 **① 메시지의 해시값을 계산 → ② 자신의 개인키로 해시값을 암호화(=서명)** 하여 첨부. 수신자는 **③ 송신자의 공개키로 서명 복호화(해시값 획득) → ④ 수신 메시지 해시와 비교** 하여 무결성·부인방지·인증 확인.
② **PKI(Public Key Infrastructure)** 구성 — **CA(Certificate Authority·인증기관)·RA(Registration Authority·등록기관)·저장소(Repository)·CRL(인증서 폐기 목록)** ·OCSP(실시간 인증서 상태 확인 프로토콜).
③ **X.509** — **ITU-T가 정한 공개키 인증서 표준 형식**. 필드 = 버전·일련번호·서명 알고리즘·발급자(CA)·유효기간·소유자·소유자 공개키·CA 전자서명. **SSL/TLS·S/MIME·IPSec** 인증서에 사용.
④ 전자서명은 **송신자의 공개키로 서명·수신자의 개인키로 검증**하며(기밀성 암호화와 동일한 키 방향), CA는 **개별 사용자의 개인키를 임의로 발급·보관**하는 기관이다. X.509는 **비밀키(공유키) 기반의 대칭 인증서 표준**이다.

### 문제 7 [유사 기출: 2022년 3회 5과목 — 접근제어 모델(DAC/MAC/RBAC/ABAC) 정형]
**접근제어(Access Control) 모델** 4종에 관한 설명 중 **가장 옳지 않은 것**은?

① **DAC(Discretionary Access Control·임의적 접근통제)** — **자원의 소유자(사용자)** 가 다른 사용자의 접근 권한을 **재량으로 부여**. UNIX/Linux 파일 권한(rwx), Windows NTFS ACL이 대표. 유연하지만 **트로이 목마 취약**.
② **MAC(Mandatory Access Control·강제적 접근통제)** — **관리자(시스템)** 가 정한 **보안 라벨(비밀·기밀·극비 등)** 에 따라 강제. 사용자 재량 X. **군사·정부·SELinux** 에서 사용. **높은 보안·낮은 유연성**.
③ **RBAC(Role-Based Access Control·역할 기반 접근통제)** — **역할(직무)에 권한을 부여** 하고, 사용자에게 역할을 할당. 대규모 조직·기업 환경에 적합. **ABAC(Attribute-Based Access Control)** = 주체·객체·환경 **속성(속성 = 부서·시각·위치·기기 등)** 을 조합한 정책 기반 접근통제로 더 세밀함.
④ **DAC는 관리자가 라벨로 강제**하고 **MAC는 소유자 재량**으로 부여하며, **RBAC는 개별 사용자별로 일일이 권한을 부여** 하는 것이 원칙이라 대규모 조직에는 부적합하다. **ABAC는 역할 하나에만 의존**하는 가장 단순한 모델이다.

### 문제 8 [유사 기출: 2021년 3회 5과목 — 보안 모델(Bell-LaPadula/Biba/Clark-Wilson/Chinese Wall) 정형]
**주요 보안 모델**에 관한 설명 중 **가장 옳지 않은 것**은?

① **Bell-LaPadula(BLP·1973)** — **기밀성(Confidentiality)** 을 강조한 최초의 수학적 보안 모델. **No Read Up(상위 등급을 아래로 못 읽음)** + **No Write Down(하위 등급으로 못 씀·기록)** 규칙. 군사·정부 시스템 기반.
② **Biba(1977)** — **무결성(Integrity)** 을 강조. BLP를 뒤집은 형태. **No Read Down(하위 등급을 위로 못 읽음)** + **No Write Up(상위 등급으로 못 씀)**. 데이터 오염 방지 목적.
③ **Clark-Wilson(1987)** — **상용(commercial) 환경 무결성 모델**. **잘 정의된 트랜잭션·직무분리(Separation of Duty)** 원칙. **Chinese Wall(Brewer-Nash·1989)** — 이해상충 방지·**과거 접근 이력에 따라 현재 접근이 제한**(금융·법률 컨설팅 산업).
④ Bell-LaPadula는 **무결성 모델**로 **No Read Down·No Write Up**을 규정하고, Biba는 **기밀성 모델**로 **No Read Up·No Write Down**을 규정한다. Clark-Wilson은 **가용성** 전용 모델이고 Chinese Wall은 **국가 기밀 등급 라벨링** 전용 모델이다.

### 문제 9 [유사 기출: 2024년 2회 5과목 — 방화벽 세대·IDS/IPS·NAC 정형]
**방화벽(Firewall) 세대** 와 **IDS/IPS·NAC** 에 관한 설명 중 **가장 옳지 않은 것**은?

① **1세대 패킷 필터링(Packet Filtering) 방화벽** — 네트워크·전송 계층 헤더(IP/포트/프로토콜)만 검사. 빠르지만 상태를 모름. **2세대 상태 검사(Stateful Inspection·SPI) 방화벽** — 연결 상태 테이블 유지, 정상 세션의 응답 트래픽 자동 허용(Check Point 등).
② **3세대 애플리케이션 게이트웨이(Application Proxy) 방화벽** — 응용 계층까지 프록시로 검사(HTTP·FTP·SMTP). 세밀하나 성능 저하. **4세대(NGFW·Next-Generation Firewall)** — DPI(심층 패킷 검사)·IPS 통합·앱 식별·SSL 검사·위협 인텔리전스 결합.
③ **IDS(Intrusion Detection System·침입탐지)** — 침입을 **탐지만** 수행(수동·경보). **IPS(Intrusion Prevention System·침입방지)** — **탐지 + 실시간 차단**(능동·인라인). 탐지 기법 = **시그니처(오용) 탐지 · 이상(anomaly) 탐지**. **NAC(Network Access Control)** — 접속 단말의 보안 상태(패치·백신) 검사 후 네트워크 접근 제어.
④ **1세대 방화벽이 응용 계층 프록시**이고 **3세대가 패킷 필터링**이다. **IDS가 실시간 차단**하고 **IPS는 탐지만** 수행하며, **NAC는 방화벽의 다른 이름**으로 아무 정책도 강제하지 않는다.

### 문제 10 [유사 기출: 2020년 3회 5과목 — VPN · SSL/TLS · IPSec 정형]
**VPN(Virtual Private Network)** 과 **SSL/TLS · IPSec** 에 관한 설명 중 **가장 옳지 않은 것**은?

① **VPN 유형** — **사이트-투-사이트 VPN**(지사-본사 간 게이트웨이 대 게이트웨이) · **원격접속 VPN**(개별 사용자→회사). VPN 프로토콜 = **PPTP·L2TP·IPSec·SSL VPN·OpenVPN·WireGuard**.
② **SSL/TLS(Transport Layer Security)** — 응용 계층과 전송 계층 사이에서 동작. 핸드셰이크로 **대칭키(세션키)를 교환**하고 이후 **대칭키로 데이터를 암호화**. TLS 1.2(RFC 5246)·TLS 1.3(RFC 8446·2018) 현재 표준. HTTPS의 기반.
③ **IPSec(Internet Protocol Security)** — 네트워크(IP) 계층에서 동작. **AH(Authentication Header)** = **무결성·인증만**(암호화 X)·**ESP(Encapsulating Security Payload)** = **암호화 + 무결성 + 인증**. **전송(Transport) 모드**(원 IP 헤더 유지·호스트 간)·**터널(Tunnel) 모드**(전체 패킷 캡슐화·게이트웨이 간).
④ IPSec의 **AH가 암호화**를 담당하고 **ESP는 인증만** 담당한다. TLS는 **응용 계층이 아니라 물리 계층에서 동작**하며 대칭키를 사용하지 않고 오직 공개키만으로 전 데이터를 암호화한다.

### 문제 11 [유사 기출: 2023년 3회 5과목 — Kerberos · SSO · OAuth · SAML 정형]
**인증 프로토콜(Kerberos · SSO · OAuth 2.0 · SAML)** 에 관한 설명 중 **가장 옳지 않은 것**은?

① **Kerberos(RFC 4120)** — MIT 개발·**대칭키 기반**·**티켓(Ticket)** 인증. 구성 = **AS(Authentication Server)·TGS(Ticket Granting Server)·KDC(Key Distribution Center)**. **시간 동기화(TGT 유효시간) 필요**·재전송 공격 방지 위해 타임스탬프 사용.
② **SSO(Single Sign-On)** — **한 번의 인증**으로 여러 서비스를 이용. 구현 방식 = **Delegation·Propagation(쿠키·토큰 전달)**. 편의성·중앙 감사 장점, **단일 실패점(SPOF) 위험**.
③ **OAuth 2.0(RFC 6749)** — **인가(Authorization)** 프로토콜. 사용자 자원 접근 권한을 **액세스 토큰(Access Token)** 으로 위임(비밀번호 노출 X). **SAML(Security Assertion Markup Language)** — **XML 기반** 인증·인가 표현 표준. 주로 **엔터프라이즈 SSO·B2B 페더레이션**에 사용.
④ Kerberos는 **공개키(비대칭키) 기반**이며 티켓 개념이 없다. OAuth는 **사용자 인증 자체를 표준화한 프로토콜**로 자원 위임과는 무관하고, SAML은 **바이너리 프로토콜**로 XML을 사용하지 않는다.

### 문제 12 [유사 기출: 2022년 2회 5과목 — DoS/DDoS 유형 정형]
**DoS(서비스 거부)·DDoS(분산 서비스 거부) 공격** 유형에 관한 설명 중 **가장 옳지 않은 것**은?

① **SYN Flood** — TCP 3-way handshake의 **SYN 요청만 대량 전송·ACK는 안 보냄** → 서버의 **연결 대기 큐(백로그) 고갈**. 대응 = **SYN Cookie·백로그 확장**.
② **Smurf** — 공격자가 **피해자 IP로 위조**한 ICMP Echo Request를 브로드캐스트 주소로 전송 → 네트워크 전체가 **피해자에게 응답 폭탄**. **Ping of Death** — 규격(65535B) 초과 크기의 **비정상 ICMP 패킷** 을 조립·수신 측 버퍼 오버플로 유발.
③ **Land Attack** — 출발지 IP·포트를 **목적지와 동일하게 위조**한 패킷 전송 → 시스템 자기 자신에게 응답 무한 반복. **Teardrop** — **IP 조각(Fragment)의 오프셋을 조작**한 겹침 조각 전송 → 재조립 시 크래시. **DDoS** = 좀비 PC(봇넷)를 이용한 분산 DoS.
④ **SYN Flood는 ICMP 대량 전송 공격**이고, **Smurf는 TCP SYN 남발 공격**이며, **Ping of Death는 정상 크기의 ICMP 패킷을 소량 전송**하는 공격이다. **DDoS는 단일 공격자가 단일 IP로 수행**하며 봇넷과 무관하다.

### 문제 13 [유사 기출: 2021년 1회 5과목 — 스푸핑·스니핑·MITM·재전송 정형]
**네트워크 위협 유형(스푸핑·스니핑·MITM·재전송)** 에 관한 설명 중 **가장 옳지 않은 것**은?

① **스니핑(Sniffing)** — 네트워크 트래픽을 **몰래 도청·수집**(수동적 공격). **평문 프로토콜**(Telnet·FTP·HTTP)에서 계정·데이터 노출. 대응 = **암호화(SSH·SSL/TLS·IPSec)·스위치 환경**.
② **ARP 스푸핑(2계층)** — 위조된 ARP 응답으로 **MAC 주소 매핑을 오염**시켜 트래픽을 공격자로 우회. **IP 스푸핑** — 출발지 IP 위조로 인증 우회·역추적 회피. **DNS 스푸핑(캐시 포이즈닝)** — DNS 응답을 위조하여 사용자를 **위장 사이트로 유도**.
③ **MITM(Man-In-The-Middle·중간자 공격)** — 통신 양측 사이에 개입해 트래픽 가로채고 위·변조. **재전송(Replay) 공격** — 정상 인증·거래 패킷을 **캡처 후 재전송**하여 인증 우회·중복 거래 유도. 대응 = **타임스탬프·논스(Nonce)·순차번호**.
④ **ARP 스푸핑은 응용 계층(URL) 조작 공격**이고 **DNS 스푸핑은 2계층(MAC) 조작 공격**이다. **재전송 공격은 새로운 패킷을 창조**하여 보내는 것으로 캡처와 무관하고, **MITM은 통신 한쪽 끝에만 개입**하여 상대측은 결코 볼 수 없다.

### 문제 14 [유사 기출: 2024년 3회 5과목 — 웹 취약점(OWASP Top 10·SQLi/XSS/CSRF) 정형]
**웹 애플리케이션 보안 취약점(OWASP Top 10·SQLi/XSS/CSRF/SSRF)** 에 관한 설명 중 **가장 옳지 않은 것**은?

① **SQL Injection(SQLi)** — 사용자 입력에 포함된 SQL 조각이 **원본 쿼리에 합쳐져** 실행되는 공격. `' OR '1'='1` 같은 payload로 인증 우회·데이터 유출·삭제. 대응 = **매개변수화 쿼리(PreparedStatement)·입력 검증·최소 권한**.
② **XSS(Cross-Site Scripting)** — 공격자가 삽입한 **스크립트가 피해자 브라우저(클라이언트)** 에서 실행. **Stored(저장형)·Reflected(반사형)·DOM 기반** 3종. 대응 = **출력 인코딩·CSP(Content Security Policy)·HttpOnly 쿠키**.
③ **CSRF(Cross-Site Request Forgery)** — **이미 인증된 사용자의 세션을 도용**해 사용자가 의도하지 않은 요청을 서버에 전송시키는 공격. 대응 = **CSRF 토큰·SameSite 쿠키·Referer 검사**. **SSRF(Server-Side Request Forgery)** — **서버가 대신 요청**을 보내도록 유도, 내부망 스캔·메타데이터 API 접근.
④ **SQL Injection은 클라이언트(브라우저)에서 실행**되는 공격이고 **XSS는 서버 DB에서 실행**되는 공격이다. **CSRF는 인증되지 않은 사용자를 대상으로 하는 인증 자체 우회 공격**이며 CSRF 토큰으로는 결코 방어할 수 없다.

### 문제 15 [유사 기출: 2023년 2회 5과목 — ISMS-P · ISO 27001 · CC 평가 · 개인정보보호법 정형]
**정보보호 관리체계·평가 기준**(ISMS-P·ISO 27001·CC·개인정보보호법)에 관한 설명 중 **가장 옳지 않은 것**은?

① **ISMS-P(정보보호 및 개인정보보호 관리체계·국내)** — 과학기술정보통신부·개인정보보호위원회 공동 고시. 기존 **ISMS + PIMS(개인정보관리체계)** 통합(2018년). 인증기관 = **KISA·금융보안원 등**. 정보보호 + **개인정보 처리 단계별 요구사항** 포함.
② **ISO/IEC 27001** — 국제표준화기구 정보보안경영시스템(ISMS) **국제 표준**. **PDCA(Plan-Do-Check-Act)** 순환 관리·**Annex A** 통제항목. 2022년 개정판 통제 카테고리 4개(조직·인적·물리·기술).
③ **CC(Common Criteria·공통평가기준) — ISO/IEC 15408** — IT 제품·시스템의 보안성 평가 국제 표준. **EAL(Evaluation Assurance Level) 1~7** 등급으로 평가. EAL이 높을수록 엄격.
④ **ISMS-P는 미국 NIST가 개발**한 국제 표준이고 **ISO 27001은 국내 KISA 고시**로 개인정보보호를 다룬다. **CC는 EAL 0~10** 등급으로 평가하며, **개인정보보호법은 오직 공공기관에만 적용**되고 민간 사업자에는 적용되지 않는다.

## 💡 상세 해설

### 문제 1
- **핵심 한 줄** — CIA = **Confidentiality(기밀성)·Integrity(무결성)·Availability(가용성)** 3원칙. AAA = **Authentication(인증)·Authorization(인가)·Accounting(계정관리·감사)** — CIA와 완전히 다른 개념(혼동 함정).
- **오답 함정 키워드(④)** — **AAA를 "Anonymity·Anti-repudiation·Archiving"으로 왜곡** + **CIA 항목 정의를 서로 뒤바꿈**(Confidentiality가 가용성이 아니라 기밀성) → 근본 왜곡. ①②③은 CIA 3원칙과 각 위협·대응 통제 표준 정의.
- **실전 암기** — **CIA + AAA = 정보보안 6대 축**. 부인방지(Non-repudiation) 는 CIA에 추가된 **4번째 원칙**으로 언급되기도 함(디지털 서명으로 달성). **CIA를 "기밀·무결·가용"** · **AAA를 "인증·인가·감사"** 로 짝지어 암기.

### 문제 2
- **핵심 한 줄** — 대칭키 = **동일 키·빠름·키 분배 문제** / 비대칭키 = **공개키/개인키·느림·PKI 필요·전자서명 지원**. 실무는 **하이브리드(TLS·PGP)** — 데이터는 대칭키·세션키는 비대칭키.
- **오답 함정 키워드(④)** — **대칭키와 비대칭키의 정의를 완전히 뒤바꿈**(비대칭키가 "동일 비밀키 공유"이고 대칭키가 "공개키·개인키 쌍"이라고 서술) → 명백한 오류. ①②③은 표준 비교표.
- **실전 암기** — **대칭키 = 빠름(F)·공유(S)·많은 키(N²/2)** · **비대칭키 = 느림(S)·개인·공개 쌍(P)·적은 키(2N)**. RSA/ECC 등 비대칭키가 대칭키보다 **약 100~1000배 느림** → 그래서 하이브리드.

### 문제 3
- **핵심 한 줄** — **DES(미국·64블록·56키·폐기)** · **AES(미국·128블록·128/192/256키·현재 표준)** · **SEED(KISA·128블록·128키)** · **ARIA(국정원 주도·128블록·128/192/256키)** · **IDEA(스위스·64블록·128키)**.
- **오답 함정 키워드(④)** — **국적을 뒤바꿈**(AES가 KISA·SEED/ARIA가 NIST) + DES 블록 크기 왜곡(64→256) + IDEA 국적 왜곡(스위스→한국) → 전면적 오류. ①②③은 표준 스펙.
- **실전 암기** — **국내 국산 대칭키 = SEED(KISA)·ARIA(국정원)·HIGHT(경량)·LEA(경량)** · **국제 대칭키 = DES·3DES·AES·IDEA·RC4(스트림)·Blowfish·Twofish**. **AES 블록 = 128비트 고정, 키만 128/192/256 선택**. ARIA는 **국가정보원·국가보안기술연구소** 개발(2004 국가표준·2010 국제 표준).

### 문제 4
- **핵심 한 줄** — **RSA(소인수분해)** · **ECC(타원곡선 이산대수·짧은 키)** · **Diffie-Hellman(키 교환 전용·이산대수 기반)** · **ElGamal(이산대수·암호화·서명)** · **DSA(전자서명 전용)**.
- **오답 함정 키워드(④)** — **RSA와 ECC의 수학적 기반을 뒤바꿈** + **DH를 "직접 암호화·전자서명" 알고리즘으로 왜곡**(DH는 오직 키 교환) → 핵심 개념 오류. ①②③은 각 알고리즘 표준 정의.
- **실전 암기** — **비대칭키 6대장 = RSA·ECC·DH·ElGamal·DSA·Rabin**. **ECC = 짧은 키·모바일/IoT 유리**(RSA 3072 ≈ ECC 256). **DH는 인증 결합 없이는 MITM 취약** → 실무는 **ECDHE(Ephemeral)**. 순수 DH는 서명·암호화 불가, 키 교환만.

### 문제 5
- **핵심 한 줄** — 해시 = **일방향·고정길이·충돌 저항성** 필수. **MD5·SHA-1 폐기**·**SHA-2/SHA-3 안전**. **HMAC = 해시 + 비밀키**로 무결성 + 인증 동시.
- **오답 함정 키워드(④)** — **해시를 양방향/가역으로 왜곡** + **충돌 필수 요건 부정** + **SHA-256 출력을 512비트로 왜곡**(SHA-256은 이름 그대로 256비트) + **MD5를 안전 알고리즘으로 왜곡**(2004 충돌 발견 이후 폐기) → 전면 오류. ①②③은 해시 성질·계열·HMAC 표준 정의.
- **실전 암기** — **해시 출력 크기 = 알고리즘 이름의 숫자**(SHA-256 → 256비트·SHA-512 → 512비트·MD5 → 128비트·SHA-1 → 160비트). **MD5·SHA-1 = 폐기** · **SHA-2(224/256/384/512)·SHA-3(Keccak) = 안전**. HMAC-SHA256이 TLS·JWT 표준.

### 문제 6
- **핵심 한 줄** — **전자서명 = 송신자 개인키로 서명 → 수신자가 송신자 공개키로 검증**(암호화의 반대 방향). PKI = **CA·RA·저장소·CRL·OCSP**. X.509 = **공개키 인증서 표준 형식**(ITU-T).
- **오답 함정 키워드(④)** — **전자서명 키 방향을 반대로 왜곡**(공개키로 서명·개인키로 검증 → 실제는 개인키 서명·공개키 검증) + **CA가 개인키 임의 발급·보관**이라고 왜곡(CA는 오직 공개키 인증서만 발급, 개인키는 소유자만 보관) + **X.509를 대칭키 기반**으로 왜곡 → 근본 오류. ①②③은 표준 절차.
- **실전 암기** — **암호화 = 수신자 공개키(암) → 수신자 개인키(복)** · **전자서명 = 송신자 개인키(서명) → 송신자 공개키(검증)** — 키 방향 반대. **PKI 5요소 = CA·RA·저장소·CRL·OCSP**. X.509 v3(1996) 확장 필드 포함 현재 표준.

### 문제 7
- **핵심 한 줄** — **DAC(소유자 재량)** · **MAC(관리자 강제·라벨·군사)** · **RBAC(역할)** · **ABAC(속성)** — 유연성은 DAC > RBAC > MAC 순, 보안은 반대.
- **오답 함정 키워드(④)** — **DAC/MAC의 정의를 완전히 뒤바꿈**(DAC를 관리자 강제·MAC를 소유자 재량) + **RBAC를 개별 사용자 권한 부여로 왜곡**(RBAC의 핵심은 역할 매개) + **ABAC를 역할 하나에만 의존**하는 단순 모델로 왜곡(ABAC는 다속성 정책 조합) → 정의 완전 왜곡. ①②③은 표준 4모델.
- **실전 암기** — **DAC = 재량(Discretionary)** · **MAC = 강제(Mandatory)** · **RBAC = 역할(Role)** · **ABAC = 속성(Attribute)**. UNIX rwx = DAC · SELinux/군사 = MAC · 기업 SAP/기업정보시스템 = RBAC · 세밀한 정책 엔진 = ABAC.

### 문제 8
- **핵심 한 줄** — **Bell-LaPadula(기밀성·No Read Up + No Write Down)** vs **Biba(무결성·No Read Down + No Write Up)** — 두 모델은 규칙이 서로 뒤집힘. **Clark-Wilson(상용 무결성·직무분리)** · **Chinese Wall(이해상충 방지)**.
- **오답 함정 키워드(④)** — **BLP와 Biba의 목표와 규칙을 완전히 뒤바꿈** + **Clark-Wilson을 가용성 전용으로 왜곡** + **Chinese Wall을 라벨링 전용으로 왜곡** → 근본 오류. ①②③은 각 모델의 저자·연도·규칙·용도 표준 서술.
- **실전 암기** — **BLP = 기밀성 → 위 못 읽고(NRU) 아래 못 씀(NWD)** — "높은 곳으로 흘리지 마" · **Biba = 무결성 → 아래 못 읽고(NRD) 위 못 씀(NWU)** — "낮은 곳에서 오염시키지 마" · **Clark-Wilson = 상용·트랜잭션·직무분리** · **Chinese Wall = 컨설팅 산업·과거 접근 이력 기반**.

### 문제 9
- **핵심 한 줄** — 방화벽 = **1(패킷 필터링) → 2(상태 검사) → 3(응용 프록시) → 4(NGFW/UTM·DPI+IPS 통합)**. **IDS = 탐지·수동**·**IPS = 탐지+차단·인라인**·**NAC = 접속 단말 검증**.
- **오답 함정 키워드(④)** — **방화벽 세대 순서를 뒤집음**(1세대가 응용 프록시·3세대가 패킷 필터링) + **IDS·IPS 기능 뒤바꿈** + **NAC를 방화벽의 다른 이름**으로 왜곡(NAC는 별도 개념: 접속 시 단말 보안 상태 검사) → 명백한 오류. ①②③은 표준 계보.
- **실전 암기** — **방화벽 세대 = 1(패킷·L3/L4) → 2(상태·세션) → 3(응용·프록시) → 4(NGFW·DPI/IPS/앱 식별)**. **IDS는 탐지(Detection·수동 경보)**·**IPS는 방지(Prevention·인라인 차단)**. **NAC = Network Access Control** — 접속 이전 단말 보안 상태 평가.

### 문제 10
- **핵심 한 줄** — **VPN**(암호화 터널) 프로토콜 = **PPTP(구식)·L2TP(터널링만·IPSec 결합)·IPSec(L3)·SSL/TLS VPN(L7)·WireGuard(경량)**. **IPSec AH = 인증·무결성만** · **IPSec ESP = 암호화+인증+무결성**.
- **오답 함정 키워드(④)** — **AH와 ESP 기능을 뒤바꿈**(AH가 암호화·ESP가 인증만 → 정반대) + **TLS를 물리 계층에서 동작·공개키만 사용**으로 왜곡(TLS는 응용/전송 사이에서 동작·핸드셰이크 후 대칭키로 데이터 암호화) → 근본 오류. ①②③은 표준 정의.
- **실전 암기** — **AH = 인증만 (Authentication Header)** · **ESP = 암호화+인증 (Encapsulating Security Payload)** — 이름 그대로. **IPSec 모드 = 전송(호스트 간·원 IP 헤더 유지)·터널(게이트웨이 간·전체 캡슐화)**. TLS = 세션키(대칭) 교환 후 대량 데이터는 대칭키로 암호화(성능).

### 문제 11
- **핵심 한 줄** — **Kerberos(대칭키·티켓·KDC/AS/TGS·타임스탬프)** · **OAuth 2.0(인가·액세스 토큰)** · **SAML(XML·엔터프라이즈 SSO)** · **OIDC(OAuth 위 인증 계층)**.
- **오답 함정 키워드(④)** — **Kerberos를 공개키·티켓 없음**으로 왜곡(실제 대칭키·티켓 중심) + **OAuth를 인증 프로토콜**로 왜곡(OAuth는 인가·자원 위임·인증은 OIDC 추가층) + **SAML을 바이너리 프로토콜**로 왜곡(SAML은 XML 기반) → 근본 오류. ①②③은 각 프로토콜 표준 정의.
- **실전 암기** — **Kerberos = 대칭키·티켓·시간 동기화** · **OAuth = 인가(권한 위임)·비밀번호 노출 X·토큰** · **SAML = XML·엔터프라이즈 SSO·B2B** · **OIDC = OAuth 2.0 + ID Token(JWT)로 사용자 인증 표준화**. **SSO는 SAML·OAuth+OIDC·Kerberos** 등으로 구현.

### 문제 12
- **핵심 한 줄** — **SYN Flood(TCP)·Smurf(ICMP 브로드캐스트 증폭)·Ping of Death(비정상 크기 ICMP)·Land(자기 IP·자기 포트)·Teardrop(조각 오프셋 조작)**. **DDoS = 봇넷 이용 분산**.
- **오답 함정 키워드(④)** — **SYN Flood와 Smurf 유형을 뒤바꿈**(SYN Flood는 TCP·Smurf는 ICMP) + **Ping of Death를 정상 크기 소량 전송**으로 왜곡(실제는 비정상 초과 크기) + **DDoS를 단일 공격자·단일 IP**로 왜곡(정의상 분산) → 명백한 오류. ①②③은 각 공격 원리 표준.
- **실전 암기** — **SYN Flood → 백로그 고갈** · **Smurf → ICMP 증폭 반사** · **Ping of Death → 크기 초과 조립** · **Land → 자기 IP·자기 포트** · **Teardrop → 조각 오버랩** · **UDP Flood → UDP 대량** · **HTTP GET/POST Flood → 애플리케이션 레이어**. DDoS = **봇넷(Zombie PC 다수)** 이용.

### 문제 13
- **핵심 한 줄** — **스니핑(수동 도청)** · **스푸핑(위조 · ARP=L2·IP=L3·DNS=응용)** · **MITM(양측 사이 개입)** · **재전송(캡처 후 재사용)**.
- **오답 함정 키워드(④)** — **ARP 스푸핑을 응용 계층 URL 조작**으로 왜곡(ARP는 2계층·MAC 매핑) + **DNS 스푸핑을 2계층 MAC 조작**으로 왜곡(DNS는 응용 계층 이름 해석 조작) + **재전송을 새 패킷 창조**로 왜곡(핵심은 캡처된 정상 패킷 재사용) + **MITM을 통신 한쪽에만 개입**으로 왜곡(양측 사이 개입이 정의) → 정의 완전 왜곡. ①②③은 표준 정의.
- **실전 암기** — **스푸핑 계층 = ARP(L2·MAC)·IP(L3·주소)·DNS(응용·이름)·이메일(응용·From)** · **스니핑 대응 = 암호화**(TLS·SSH·IPSec) · **재전송 대응 = 타임스탬프·논스·순차번호** · **MITM 대응 = 상호 인증·인증서 검증·HSTS·인증서 고정**.

### 문제 14
- **핵심 한 줄** — **SQLi(서버 DB)·XSS(클라이언트 브라우저·스크립트 실행)·CSRF(인증된 사용자 세션 도용)·SSRF(서버가 대신 요청 수행)**. OWASP Top 10 = 웹 10대 위험.
- **오답 함정 키워드(④)** — **SQLi와 XSS 실행 위치를 뒤바꿈**(SQLi=서버 DB·XSS=클라이언트 브라우저) + **CSRF를 인증 자체 우회 공격**으로 왜곡(CSRF의 정의는 **이미 인증된** 사용자 세션 도용) + **CSRF 토큰이 무효**로 왜곡(CSRF 토큰은 표준 방어책) → 근본 오류. ①②③은 각 취약점 표준 정의.
- **실전 암기** — **SQLi 방어 = 매개변수화 쿼리(PreparedStatement)·최소 권한·화이트리스트 검증** · **XSS 방어 = 출력 인코딩·CSP·HttpOnly·X-XSS-Protection** · **CSRF 방어 = 토큰·SameSite=Lax/Strict·Referer 검사·재인증** · **SSRF 방어 = URL 화이트리스트·내부망 접근 차단·메타데이터 IP 차단**.

### 문제 15
- **핵심 한 줄** — **ISMS-P(국내·정보보호+개인정보 통합·KISA)** · **ISO 27001(국제·PDCA·Annex A)** · **CC(ISO 15408·EAL 1~7·제품 평가)** · **개인정보보호법(공공·민간 모두 적용·정보주체 권리)**.
- **오답 함정 키워드(④)** — **ISMS-P를 NIST 개발 국제 표준**으로 왜곡(실제 국내 과기부·개인정보위 고시·KISA 인증) + **ISO 27001을 국내 KISA 고시**로 왜곡(실제 ISO 국제 표준) + **CC EAL 등급을 0~10**으로 왜곡(실제 1~7) + **개인정보보호법을 공공만 적용**으로 왜곡(민간 사업자도 적용) → 근본 오류. ①②③은 각 체계 표준 정의.
- **실전 암기** — **국내 = ISMS-P(정보보호+개인정보)·개인정보보호법·정보통신망법** · **국제 = ISO 27001(관리체계)·ISO 27002(통제)·ISO 27017/27018(클라우드)** · **제품 평가 = CC(ISO 15408·EAL 1~7)**. EAL이 높을수록(1→7) 엄격, 실용은 대개 EAL 3~4.

## 📊 오늘의 최다 함정 5

1. **전자서명 키 방향** — **개인키 서명·공개키 검증**(암호화의 반대). 함정: 공개키 서명·개인키 검증(오답 다수 트랩).
2. **BLP vs Biba** — BLP는 기밀성(NRU + NWD)·Biba는 무결성(NRD + NWU) — 규칙 그대로 뒤집힘.
3. **IPSec AH vs ESP** — AH = 인증만(암호화 X), ESP = 암호화+인증. 이름 그대로 기억.
4. **XSS vs SQLi 실행 위치** — XSS는 클라이언트(브라우저), SQLi는 서버(DB). 뒤바꾸기 함정 반복 출제.
5. **국산 vs 국제 대칭키** — SEED·ARIA(국내), AES·DES·IDEA(국제). 국적 뒤바꾸기 = 4번 트랩 단골.
