### 문제 3 [출처: 2020년 4회 필기]

### 문제 6 [출처: 2020년 3회 필기]
다음 중 인덱스(INDEX)에 대한 설명으로 옳지 않은 것은?
① 인덱스의 추가·변경·삭제 명령어는 각각 CREATE, MODIFY, DROP이다.
② 인덱스는 검색 속도를 빠르게 하기 위한 보조 자료구조이다.
③ B-트리 인덱스는 가장 일반적으로 사용되는 인덱스 형태이다.
④ 기본키(PRIMARY KEY)와 UNIQUE 키에는 자동으로 인덱스가 생성된다.

### 문제 7 [출처: 2021년 2회 필기]
다음 SQL문을 실행한 결과로 옳은 것은?

[STUDENT 테이블]
| 학번 | 이름 | 학년 | 학과   |
|------|------|------|--------|
| 1001 | 홍길동 | 1   | 컴퓨터  |
| 1002 | 임꺽정 | 2   | 컴퓨터  |
| 1003 | 장길산 | 2   | 전자    |
| 1004 | 강감찬 | 3   | 컴퓨터  |
| 1005 | 이순신 | 3   | 전자    |

```sql
SELECT 학과, COUNT(*) AS 학과별인원
FROM STUDENT
GROUP BY 학과
HAVING COUNT(*) >= 3;
```
① (컴퓨터, 3)
② (컴퓨터, 3), (전자, 2)
③ (전자, 2)
④ 결과 없음

### 문제 8 [출처: 2022년 2회 필기]
SQL의 JOIN에 대한 설명으로 옳지 않은 것은?
① INNER JOIN은 조인 조건을 만족하는 행만 반환한다.
② LEFT OUTER JOIN은 왼쪽 테이블의 모든 행을 결과에 포함한다.
③ CROSS JOIN은 두 테이블의 카티션 곱(Cartesian Product)을 반환한다.
④ NATURAL JOIN은 두 테이블에 동일한 이름의 컬럼이 없어도 모든 컬럼을 기준으로 조인한다.

## 💡 상세 해설 (자기 설명과 대조)

### 문제 1 — 정답 ④ ALTER
- DML: SELECT, INSERT, UPDATE, DELETE
- ALTER는 테이블 구조를 변경하는 DDL 명령어.

### 문제 2 — 정답 ④ GRANT
- ①②③ SELECT/UPDATE/INSERT는 DML.
- GRANT는 권한 부여 명령으로 DCL에 속함(REVOKE, COMMIT, ROLLBACK과 같은 분류).

### 문제 3 — 정답 ④ "DROP은 DCL에 해당한다"
- DROP은 DDL(Data Definition Language) — 객체(테이블/뷰/인덱스)를 제거.
- CASCADE: 참조하는 객체까지 연쇄 제거 / RESTRICT: 참조하는 객체가 있으면 제거 거부.

### 문제 4 — 정답 ② GRANT
- 권한 부여 문법: `GRANT 권한리스트 ON 객체 TO 사용자 [WITH GRANT OPTION];`
- WITH GRANT OPTION: 부여받은 사용자가 다른 사용자에게 재부여 가능.
- 회수: `REVOKE 권한 ON 객체 FROM 사용자 [CASCADE];`

### 문제 5 — 정답 ③ "뷰는 자체 인덱스를 가질 수 있다"
- 뷰는 자체적인 인덱스를 가질 수 없다(기반 테이블의 인덱스만 사용).
- ① 뷰는 물리적으로 존재하지 않는 가상 테이블 (정의만 데이터 사전에 저장).
- ② 특정 컬럼/행만 노출시켜 보안 도구로 활용 가능.
- ④ 뷰의 정의는 변경(ALTER) 불가 → 변경하려면 DROP 후 다시 CREATE.

### 문제 6 — 정답 ① "추가·변경·삭제 = CREATE, MODIFY, DROP"
- 인덱스에는 "변경(ALTER/MODIFY)" 명령어가 없다 → CREATE / DROP 두 가지만.
- ②③④ 모두 옳음. B-트리는 가장 보편적인 인덱스 구조, PK·UNIQUE는 자동 인덱스 생성.

### 문제 7 — 정답 ① (컴퓨터, 3)
- 학과별 인원: 컴퓨터=3, 전자=2.
- HAVING COUNT(*) >= 3 → 컴퓨터 그룹만 통과.
- WHERE는 그룹 형성 전 행 필터, HAVING은 그룹 형성 후 그룹 필터.

### 문제 8 — 정답 ④ "NATURAL JOIN은 동일한 이름의 컬럼이 없어도 모든 컬럼을 기준으로 조인"
- NATURAL JOIN은 두 테이블에 **동일 이름·동일 도메인** 컬럼이 있어야 그 컬럼을 기준으로 자동 조인.
- 동일 이름 컬럼이 없으면 CROSS JOIN과 동일한 결과(카티션 곱)가 됨 → "모든 컬럼 기준 조인"이 아님.
- ①②③ 정의 정확.

## 📚 역추적 참조 (사후 검증용 — 학습 자료 아님)

### 핵심 용어
| 용어 | 정의 |
|---|---|
| DDL | 데이터 정의어. 데이터베이스 객체의 구조 정의. CREATE / ALTER / DROP / TRUNCATE / RENAME |
| DML | 데이터 조작어. 데이터의 검색·삽입·갱신·삭제. SELECT / INSERT / UPDATE / DELETE |
| DCL | 데이터 제어어. 권한 부여·회수 및 트랜잭션 제어. GRANT / REVOKE / COMMIT / ROLLBACK |
| TCL | 트랜잭션 제어어. COMMIT / ROLLBACK / SAVEPOINT (DCL과 통합 분류되기도 함) |
| CASCADE | DROP/REVOKE 시 종속 객체·권한까지 연쇄 삭제 |
| RESTRICT | DROP/REVOKE 시 종속 객체·권한 있으면 작업 거부 |
| WITH GRANT OPTION | 부여받은 사용자가 권한을 또 다른 사용자에게 부여할 수 있는 옵션 |
| VIEW | 하나 이상의 기본 테이블에서 유도된 가상 테이블 |
| INDEX | 데이터 검색 속도를 향상시키기 위한 보조 자료구조 |
| HAVING | GROUP BY로 형성된 그룹에 대한 조건 |

### 핵심 개념 (사후 검증용)
- DDL: CREATE / ALTER / DROP / TRUNCATE / RENAME
- DML: SELECT / INSERT / UPDATE / DELETE
- DCL: GRANT / REVOKE / COMMIT / ROLLBACK / SAVEPOINT
- VIEW 특징: 가상 / 자체 인덱스 없음 / ALTER 불가 / 삽입·갱신·삭제에 제약(집계 함수·DISTINCT·GROUP BY 포함 시 갱신 불가)
- INDEX: CREATE INDEX / DROP INDEX만 존재(변경 명령 없음). PK·UNIQUE는 자동 생성.
- GROUP BY: 컬럼 기준 그룹화 / HAVING: 그룹 결과 필터 / WHERE: 행 필터
- JOIN: INNER, LEFT OUTER, RIGHT OUTER, FULL OUTER, CROSS, NATURAL, SELF
- 서브쿼리: 단일행(`=`, `<`, `>`) / 다중행(`IN`, `ANY`, `ALL`, `EXISTS`)
