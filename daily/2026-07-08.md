### 문제 1 [유사 기출: 2024년 1회 1과목 — 소프트웨어 생명주기(SDLC) 4모델 비교 정형 패턴]
소프트웨어 생명주기(SDLC) 대표 모델에 관한 설명 중 **가장 옳지 않은 것**은?

① **폭포수(Waterfall) 모델** — 요구분석→설계→구현→테스트→유지보수를 순차적으로 진행하며, 각 단계 산출물이 다음 단계 입력이 된다.
② **프로토타입(Prototype) 모델** — 사용자 요구사항을 명확히 하기 위해 시제품을 반복 제작·평가하며, 최종 시스템 개발 전 사용자 피드백을 수렴한다.
③ **나선형(Spiral) 모델** — 계획→위험 분석→개발→고객 평가의 4단계를 반복하는 점진적 모델로, **위험 분석(Risk Analysis)** 단계가 핵심이다.
④ **애자일(Agile) 모델** — 요구사항의 변경을 억제하고, 개발 시작 전 모든 문서를 완벽히 확정한 후 순차적으로 진행하는 대규모 사전 설계 방식이다.

**MDA(Model Driven Architecture)**에 관한 설명으로 **가장 옳은 것**은?

① 특정 플랫폼(예: Java EE, .NET)에 의존적인 코드를 직접 작성하는 절차 지향 개발 방법론이다.
② **플랫폼 독립적 모델(PIM)**을 **플랫폼 종속적 모델(PSM)**로 변환한 뒤, 자동 변환으로 코드를 생성하는 OMG(Object Management Group) 표준 아키텍처이다.
③ 데이터베이스 스키마와 무결성 제약을 정의하는 데이터 모델링 전용 표기법이다.
④ 사용자 인터페이스의 색상·폰트·레이아웃을 지정하는 스타일시트 아키텍처이다.

### 문제 3 [유사 기출: 2024년 3회 1과목 — 소프트웨어 3R(재공학·재사용·역공학) 정형 패턴]
소프트웨어 3R(3R: Reverse Engineering·Re-engineering·Reuse)에 관한 설명 중 **가장 옳지 않은 것**은?

① **역공학(Reverse Engineering)** — 실행 파일이나 소스 코드로부터 설계 문서·요구사항을 재추출하여 시스템 이해도를 높이는 활동이다.
② **재공학(Re-engineering)** — 기존 시스템의 인터페이스를 유지하면서 내부 구조·품질을 개선하여 유지보수성을 높인다.
③ **재사용(Reuse)** — 기존에 개발된 컴포넌트·모듈·라이브러리를 새로운 시스템 개발에 활용하여 개발 생산성을 높인다.
④ **역공학(Reverse Engineering)** — 오로지 신규 시스템 개발만 가능하며, 기존 시스템 분석·문서화에는 사용될 수 없다.

### 문제 4 [유사 기출: 2024년 1회 1과목 — UI/UX 설계 4대 원칙 정형 패턴]
사용자 인터페이스(UI) 설계의 4대 기본 원칙에 관한 설명 중 **가장 옳지 않은 것**은?

① **직관성(Intuitiveness)** — 누구나 쉽게 이해하고 사용할 수 있도록 설계되어야 한다.
② **유효성(Efficiency)** — 사용자 목적을 정확하고 완벽하게 달성할 수 있어야 한다.
③ **학습성(Learnability)** — 초보자도 쉽게 배우고 사용할 수 있도록 설계되어야 한다.
④ **유연성(Flexibility)** — 사용자의 요구사항을 무시하고 개발자 중심의 고정된 인터페이스만 제공해야 한다.

### 문제 5 [유사 기출: 2024년 2회 2과목 — 스택(Stack) LIFO 응용 3대 사례 정형 패턴]

② 중위 표기식(Infix)을 **후위 표기식(Postfix)으로 변환**하고 계산하는 처리.
③ 수식·프로그램 소스에서 **여는 괄호와 닫는 괄호의 짝(Matching Parentheses)을 검사**하는 처리.

### 문제 6 [유사 기출: 2024년 3회 2과목 — 힙 정렬(Heap Sort) 트레이싱 정형 패턴]
**힙 정렬(Heap Sort)**에 관한 설명으로 **가장 옳지 않은 것**은?

① 완전 이진 트리(Complete Binary Tree) 형태의 힙(Heap) 자료구조를 이용한 정렬 알고리즘이다.
② 최대 힙(Max Heap)을 구성하면 루트에 최댓값이 오며, 이를 배열 끝과 교환하고 힙 크기를 하나 줄여 재구성하는 과정을 반복한다.
③ 평균·최선·최악 시간 복잡도가 모두 **O(n log n)**이며, 추가 공간 없이 제자리(in-place) 정렬이 가능하다.
④ 리스트를 순차 탐색하여 최솟값을 맨 앞에 놓는 과정을 반복하는 방식이며, 이진 트리 구조를 사용하지 않는다.

### 문제 8 [유사 기출: 2024년 2회 2과목 — 형상 관리(Configuration Management) 4단계 정형 패턴]
**소프트웨어 형상 관리(SCM: Software Configuration Management)**의 4대 활동에 관한 설명 중 **가장 옳지 않은 것**은?

① **형상 식별(Configuration Identification)** — 형상 항목(SCI)을 정의하고 이름·번호·버전으로 관리 대상을 명확히 한다.
② **형상 통제(Configuration Control)** — 변경 요청(CR)을 접수·심의하여 승인된 변경만 반영하고 형상 통제 위원회(CCB)가 심사한다.
③ **형상 감사(Configuration Audit)** — 변경이 요구사항·계획대로 정확히 이행되었는지 확인·검증한다.
④ **형상 기록(Configuration Status Accounting)** — 형상 항목의 변경 이력·상태를 기록하지 않고, 오직 최종 산출물만 관리한다.

### 문제 9 [유사 기출: 2024년 3회 3과목 — Peterson(피터슨) 표기법 vs Chen 표기법 정형 패턴]
ERD(Entity-Relationship Diagram) 표기법에 관한 설명 중 **가장 옳지 않은 것**은?

① **Chen 표기법** — 개체는 사각형, 관계는 마름모, 속성은 타원으로 표기하는 최초의 ERD 표기법이다.
② **Peterson(Crow's Foot, IE) 표기법** — 개체는 사각형, 관계는 실선, 카디널리티는 까마귀 발(Crow's Foot)·원·수직선 기호로 표기한다.
③ **Peterson 표기법**은 실무·CASE 도구에서 널리 사용되며, 관계의 카디널리티(1:1, 1:N, M:N)와 옵션성(필수/선택)을 시각적으로 표기한다.
④ **Peterson 표기법**은 관계를 반드시 마름모(Diamond)로 표기해야 하며, 까마귀 발 기호는 사용할 수 없다.

### 문제 10 [유사 기출: 2024년 1회 3과목 — 3NF vs BCNF 이행 종속 제거 정형 패턴]
**정규화(Normalization)**의 3NF와 BCNF에 관한 설명으로 **가장 옳은 것**은?

① 3NF는 부분 함수 종속(Partial Dependency)을 제거한 형태이며, BCNF는 다치 종속(Multi-Valued Dependency)을 제거한 형태이다.
② 3NF는 이행 함수 종속(Transitive Dependency)을 제거한 형태이며, BCNF는 3NF에서 **모든 결정자(Determinant)가 후보키(Candidate Key)**인 상태로 강화한 형태이다.
③ BCNF는 3NF의 약한 형태이며, BCNF를 만족하면 3NF는 만족하지 않을 수 있다.
④ 3NF는 원자값(Atomic Value)만 허용하는 1NF의 다른 이름이다.

### 문제 11 [유사 기출: 2024년 2회 3과목 — GROUP BY + HAVING 절 정형 패턴]
다음 SQL 문의 실행 결과 조건으로 **가장 옳은 것**은?

```sql
SELECT 부서, COUNT(*) AS 인원수
FROM 사원
WHERE 급여 >= 3000000
GROUP BY 부서
HAVING COUNT(*) >= 5
ORDER BY 인원수 DESC;
```

① 급여 300만 원 이상인 모든 사원의 목록을 부서별로 출력한다.
② **급여 300만 원 이상인 사원을 부서별로 집계**한 뒤, **인원수가 5명 이상인 부서**만 인원수 내림차순으로 출력한다.
③ 급여 300만 원 이상 조건은 무시되며, 모든 부서의 전체 인원수를 계산한다.
④ HAVING과 WHERE는 동일한 절이므로 두 조건이 중복되어 오류가 발생한다.

### 문제 12 [유사 기출: 2024년 3회 3과목 — 옵티마이저(Optimizer) RBO vs CBO 정형 패턴]
**질의 최적화(Query Optimization)**의 옵티마이저(Optimizer)에 관한 설명 중 **가장 옳지 않은 것**은?

① **RBO(Rule-Based Optimizer, 규칙 기반)** — 미리 정해진 규칙(우선순위)에 따라 실행 계획을 결정하며, 통계 정보와 무관하다.
② **CBO(Cost-Based Optimizer, 비용 기반)** — 테이블·인덱스의 통계 정보를 활용하여 예상 비용이 가장 낮은 실행 계획을 선택한다.
③ 현대 상용 RDBMS(Oracle 11g 이후, MySQL, PostgreSQL 등)는 **대부분 CBO**를 기본 옵티마이저로 사용한다.
④ **RBO는 통계 정보를 실시간으로 반영**하여 데이터 분포 변화에 즉시 대응하는 최신 옵티마이저이다.

### 문제 13 [유사 기출: 2024년 1회 4과목 — C 문자열 함수 4종 정형 패턴]
다음 C 프로그램의 실행 결과로 가장 옳은 것은?

```c
#include <stdio.h>
#include <string.h>
int main() {
    char a[20] = "Hello";
    char b[] = "World";
    strcat(a, b);
    printf("%d %s %d\n", strlen(a), a, strcmp("abc","abd"));
    return 0;
}
```

① `10 HelloWorld -1` — strcat으로 "HelloWorld" 결합(길이 10), strcmp는 "abc"<"abd"이므로 음수 반환
② `5 Hello 0` — strcat 미실행
③ `10 World -1` — 원본이 덮어써짐
④ `15 HelloWorld 1` — 길이 계산 오류

### 문제 14 [유사 기출: 2024년 2회 4과목 — Java 인터페이스(interface) 다중 상속 정형 패턴]
Java의 **인터페이스(interface)**에 관한 설명으로 **가장 옳지 않은 것**은?

① 인터페이스는 **다중 상속(Multiple Inheritance)**을 지원하며, 클래스는 `implements` 키워드로 여러 인터페이스를 동시에 구현할 수 있다.
② 인터페이스 내의 모든 필드는 기본적으로 **public static final**(상수)로 처리된다.
③ Java 8부터 인터페이스에 **default 메서드**와 **static 메서드** 정의가 허용되어, 구현체 없이도 메서드 본체를 가질 수 있다.
④ 인터페이스는 오직 **단일 상속(Single Inheritance)만 허용**되며, 클래스는 하나의 인터페이스만 구현할 수 있다.

### 문제 15 [유사 기출: 2024년 3회 4과목 — Python 데코레이터(@decorator) 정형 패턴]
다음 Python 프로그램의 실행 결과로 가장 옳은 것은?

```python
def deco(func):
    def wrapper(x):
        return func(x) * 2
    return wrapper

@deco
def square(x):
    return x * x

print(square(3))
```

① `18` — square(3) = 9를 데코레이터가 감싸 wrapper(3) = 9 * 2 = 18
② `9` — 데코레이터가 무시됨
③ `6` — 3 * 2만 계산
④ `TypeError` — 데코레이터 문법 오류

### 문제 16 [유사 기출: 2024년 1회 4과목 — TCP 3-way 핸드셰이크·4-way 종료 정형 패턴]
**TCP(Transmission Control Protocol)** 연결 설정 및 종료 과정에 관한 설명으로 **가장 옳지 않은 것**은?

① **3-way 핸드셰이크**: 클라이언트 → 서버 **SYN** 전송 → 서버 → 클라이언트 **SYN+ACK** 응답 → 클라이언트 → 서버 **ACK** 전송으로 연결 설정.
② **4-way 종료(Termination)**: **FIN → ACK → FIN → ACK**의 4단계로 양방향 종료를 처리한다.
③ TCP는 연결 지향(Connection-Oriented) 프로토콜로, 신뢰성(Reliable) 있는 데이터 전송을 보장한다.
④ TCP는 **비연결형(Connectionless)** 프로토콜로, UDP처럼 핸드셰이크 없이 즉시 데이터를 전송한다.

### 문제 17 [유사 기출: 2024년 2회 5과목 — 소프트웨어 개발 프레임워크(Spring·전자정부) 정형 패턴]
**소프트웨어 개발 프레임워크**에 관한 설명 중 **가장 옳지 않은 것**은?

① **Spring 프레임워크** — Java 기반 오픈소스 경량(Lightweight) 프레임워크로, IoC(제어의 역전)·DI(의존성 주입)·AOP(관점 지향 프로그래밍)를 핵심 기능으로 제공한다.
② **전자정부 표준 프레임워크(eGovFrame)** — 공공기관 SW 개발 표준화를 위해 한국 정부가 오픈소스 기반으로 개발한 Java EE 프레임워크로, Spring 기반이다.
③ **Struts 프레임워크** — Apache 재단의 Java 웹 프레임워크로, MVC 패턴을 지원한다.
④ **Spring 프레임워크**는 오직 **상용(유료) 라이선스**로만 배포되며, 소스 코드는 공개되지 않는다.

### 문제 18 [유사 기출: 2024년 3회 5과목 — 빅데이터 처리(Hadoop·HDFS·MapReduce) 정형 패턴]
**하둡(Hadoop) 생태계**의 핵심 구성 요소에 관한 설명 중 **가장 옳지 않은 것**은?

① **HDFS(Hadoop Distributed File System)** — 대용량 파일을 블록(기본 128MB) 단위로 분할하여 여러 노드에 분산·복제 저장하는 분산 파일 시스템이다.
② **MapReduce** — 데이터를 병렬로 처리하기 위한 프로그래밍 모델로, **Map(분할·매핑) → Shuffle → Reduce(집계·통합)** 단계로 구성된다.
③ **YARN(Yet Another Resource Negotiator)** — 하둡 2.x부터 도입된 자원 관리(Resource Management) 계층으로, 작업 스케줄링을 담당한다.
④ **HDFS**는 데이터를 **단 하나의 노드에만 저장**하여 저장 공간을 절약하며, 노드 장애 시 데이터는 영구 손실된다.

### 문제 19 [유사 기출: 2024년 1회 5과목 — 접근 통제 모델 3종(DAC·MAC·RBAC) 정형 패턴]
**접근 통제(Access Control) 모델**에 관한 설명 중 **가장 옳지 않은 것**은?

① **DAC(Discretionary Access Control, 임의적 접근 통제)** — 객체의 소유자가 접근 권한을 직접 부여·회수하며, UNIX 파일 시스템의 `chmod`가 대표 사례이다.
② **MAC(Mandatory Access Control, 강제적 접근 통제)** — 시스템(관리자)이 정한 보안 등급(Top Secret·Secret·Confidential·Unclassified)에 따라 강제 적용되며, 군사·정부 시스템에서 사용한다.
③ **RBAC(Role-Based Access Control, 역할 기반 접근 통제)** — 사용자에게 직접 권한을 부여하지 않고 **역할(Role)에 권한을 부여한 뒤 사용자를 역할에 배정**하여 관리 편의성을 높인다.
④ **DAC**은 소유자와 무관하게 시스템이 강제로 권한을 결정하며, 소유자는 어떠한 권한도 조정할 수 없다.

### 문제 20 [유사 기출: 2024년 2회 5과목 — APT(Advanced Persistent Threat) 공격 단계 정형 패턴]
**APT(Advanced Persistent Threat, 지능형 지속 공격)**에 관한 설명 중 **가장 옳지 않은 것**은?

① 특정 대상(기업·정부·기간산업)을 **장기간 은밀히 침투**하여 정보 수집·자산 탈취를 목적으로 하는 표적형 공격이다.
② 대표 공격 단계 = **정찰(Reconnaissance) → 침투(Initial Compromise) → 지속 확보(Persistence)·권한 상승(Escalation) → 내부 이동(Lateral Movement) → 목적 달성(Exfiltration)**.
③ 스피어 피싱(Spear Phishing)·워터링 홀(Watering Hole)·제로데이(Zero-day) 취약점 등을 조합하여 사용한다.
④ APT는 오로지 **1~2시간 이내 단기 공격**만 수행하며, 침투 후 즉시 랜섬웨어를 실행해 흔적을 남기는 방식이다.

- 함정 키워드: strlen = **null 미포함 길이** / strcmp = **음수(s1<s2)·0(같음)·양수(s1>s2)**

### 문제 14
**정답 ④** — Java 인터페이스는 **다중 상속 지원**(클래스 다중 상속은 불가). `class Foo implements A, B, C { ... }` 형태로 여러 인터페이스 동시 구현 가능. "인터페이스 단일 상속만 허용" 완전 오답. 인터페이스 = 상수(public static final) + 추상 메서드 + Java 8+ default/static 메서드.
- 함정 키워드: 인터페이스 = **다중 상속 O**(클래스는 단일 상속만) / Java 8+ default 메서드

### 문제 15
**정답 ①** — 데코레이터 `@deco`는 `square = deco(square)`와 동일. wrapper(x) = square(x) * 2. square(3) = 9, wrapper 반환값 = 9 * 2 = **18**. 데코레이터 = 함수를 인자로 받아 새 함수를 반환하는 고차 함수(High-Order Function).
- 함정 키워드: `@deco` = **deco(square)** / 데코레이터 = **함수 감싸기(wrapping)**

### 문제 16
**정답 ④** — TCP는 **연결 지향(Connection-Oriented)·신뢰성(Reliable) 프로토콜**. "비연결형·UDP처럼 즉시 전송"은 UDP 특성. TCP 3-way = SYN → SYN+ACK → ACK, 4-way 종료 = FIN → ACK → FIN → ACK(양방향 종료 필요해 4단계).
- 함정 키워드: TCP = **연결 지향·신뢰성** / UDP = **비연결형·비신뢰성·핸드셰이크 X**

### 문제 17
**정답 ④** — Spring 프레임워크는 **Apache License 2.0 기반 오픈소스**로, 소스 코드가 공개되고 무료로 사용 가능. "상용 유료·소스 비공개" 완전 오답. Spring = Java 진영 사실상 표준 웹 프레임워크(IoC·DI·AOP·Spring Boot·Spring MVC·Spring Data).
- 함정 키워드: Spring = **오픈소스(Apache 2.0)·무료·공개** / 전자정부 프레임워크 = **Spring 기반 한국 표준**

### 문제 18
**정답 ④** — HDFS는 **기본 3개(default replication factor = 3) 복제본을 여러 노드에 분산 저장**하여 노드 장애 시에도 데이터 손실 방지. "단 하나의 노드에만 저장·장애 시 영구 손실"은 완전 오답. HDFS = NameNode(메타데이터) + DataNode(실제 블록) + 블록 128MB 기본.
- 함정 키워드: HDFS = **분산·복제(3개)·내결함성** / 블록 = **128MB 기본**

### 문제 19
**정답 ④** — DAC(임의적 접근 통제)의 정확한 의미 = **객체의 소유자(Owner)가 재량으로 권한을 부여·회수**. UNIX `chmod`, Windows NTFS ACL이 대표 사례. "소유자와 무관하게 시스템이 강제"는 **MAC(강제적 접근 통제)** 정반대 함정. DAC = 유연 / MAC = 강제 / RBAC = 역할 기반.
- 함정 키워드: DAC = **소유자 재량** / MAC = **시스템 강제·보안 등급** / RBAC = **역할 기반·관리 편의**

### 문제 20
**정답 ④** — APT의 핵심 특성 = **장기간(수개월~수년) 은밀·지속·표적형**. "1~2시간 단기 공격"은 APT의 정반대 특성. APT 단계 = 정찰(Reconnaissance) → 침투(Initial Compromise) → C2 통신 → 지속(Persistence)·권한 상승(Escalation) → 내부 이동(Lateral Movement) → 목적 달성(Exfiltration)·흔적 제거. 대표 사례 = 스턱스넷·오퍼레이션 오로라·SolarWinds.
- 함정 키워드: APT = **장기·은밀·지속·표적**(단기 랜섬웨어 정반대) / 5단계 = **정찰→침투→지속→이동→탈취**

## 📊 자주 틀리는 함정 5가지 (2024년 회독 1/4 진입 · 신규 영역 위주)

1. **애자일 vs 폭포수 방향성 뒤바꾸기** — 애자일 = **변화 수용·짧은 반복·문서보다 동작 SW**. 폭포수 = **BDUF·순차·문서 완결**. "애자일이 사전 문서를 완벽히 확정한다"는 항상 오답 유도 함정. Agile Manifesto 4대 가치·12원칙 암기 필수.
2. **Chen 표기법 vs Peterson 표기법 기호 혼동** — Chen = **마름모(관계)·사각형(개체)·타원(속성)** 3요소. Peterson(Crow's Foot, IE) = **까마귀 발(카디널리티)·원(0)·수직선(1)·실선**. "Peterson이 마름모를 사용한다"는 완전 오답. 최근 실무 CASE 도구(ERwin, PowerDesigner)는 Peterson 표기법이 주류.
3. **3NF vs BCNF 강도 뒤바꾸기** — 정규화 강도 = 1NF < 2NF < 3NF < **BCNF** < 4NF < 5NF. BCNF는 3NF보다 **강한(엄격한) 조건**이며, BCNF ⊆ 3NF(BCNF 만족 → 3NF 자동 만족). 역방향은 성립 안 함. "BCNF가 3NF의 약한 형태"는 정반대 함정.
4. **DAC vs MAC 통제 주체 뒤바꾸기** — **DAC = 소유자(Owner) 재량**(UNIX chmod), **MAC = 시스템(관리자) 강제**(보안 등급). "DAC이 시스템 강제 통제"는 MAC과 완전히 뒤바꾼 오답. RBAC은 **역할(Role) 기반**으로 사용자 다수·역할 소수 조직에서 관리 편의성 극대화.
5. **APT vs 일반 사이버 공격 시간축 뒤바꾸기** — APT의 P(Persistent) = **장기·지속·은밀**. "1~2시간 단기·즉시 랜섬웨어 실행"은 일반 랜섬웨어·DDoS 공격 특성이지 APT 아님. APT 5단계 = **정찰→침투→지속·권한상승→내부이동→목적달성** 순서 암기.
