## 🎯 오늘의 기출문제 — 2022년 정처기 필기 20문제 (5과목 골골루 · 회독 1/4)

### 문제 1 [유사 기출: 2022년 1회 1과목 — MSA(Microservices Architecture) 정형 패턴]
다음 중 **마이크로서비스 아키텍처(Microservices Architecture, MSA)** 의 특징으로 옳지 **않은** 것은?

① 각 서비스가 **독립적으로 배포·확장**될 수 있으며, 서비스마다 **다른 언어·DB**를 선택할 수 있다(폴리글랏).
② 서비스 간 통신은 주로 **REST API·gRPC·메시지 큐(Kafka·RabbitMQ)** 등 경량 프로토콜을 사용한다.
③ 하나의 서비스 장애가 전체 시스템으로 전파되지 않도록 **회로 차단기(Circuit Breaker)** ·**격벽(Bulkhead)** 패턴을 활용한다.
④ 모든 서비스가 **단일 데이터베이스를 공유**하여 **강한 일관성(Strong Consistency)** 과 **트랜잭션 ACID**를 전체 범위로 보장한다.

### 문제 2 [유사 기출: 2022년 2회 1과목 — SOA(Service-Oriented Architecture) 정형 패턴]
다음 중 **서비스 지향 아키텍처(Service-Oriented Architecture, SOA)** 의 설명으로 옳지 **않은** 것은?

① 비즈니스 기능을 **재사용 가능한 서비스 단위**로 분리하여 느슨하게 결합(Loosely Coupled)한다.
② 서비스 간 통신은 주로 **ESB(Enterprise Service Bus)** 라는 중앙 메시지 버스를 통해 이루어진다.
③ 표준 프로토콜인 **SOAP/WSDL/UDDI** 기반으로 서비스를 정의·검색·호출한다.
④ MSA보다 **서비스 단위가 더 작고**, 서비스마다 **독립적인 DB**를 강제하여 운영체제까지 격리한다.

### 문제 3 [유사 기출: 2022년 3회 1과목 — Singleton 디자인 패턴 정형 패턴]
다음 중 **Singleton(싱글톤) 패턴**에 대한 설명으로 옳지 **않은** 것은?

① **생성(Creational) 패턴**의 하나로, 클래스의 인스턴스가 **단 하나**만 생성되도록 제한한다.
② 전역 접근점(Global Access Point)을 제공해 **어디서나 동일 인스턴스**를 얻을 수 있다.
③ 멀티스레드 환경에서는 **이중 검사 잠금(Double-Checked Locking)** ·**Holder** 기법으로 동시성 문제를 처리한다.
④ **상속·다형성**을 적극 활용해 서브클래스 인스턴스를 **여러 개** 동시에 보유하는 패턴이다.

### 문제 4 [유사 기출: 2022년 1회 1과목 — SOLID 원칙(OCP) 정형 패턴]
다음 중 객체지향 설계 원칙 **SOLID** 의 **OCP(Open-Closed Principle, 개방-폐쇄 원칙)** 에 대한 설명으로 가장 옳은 것은?

① 클래스는 **단 하나의 책임**만 가져야 하며 변경 사유가 하나여야 한다.
② 소프트웨어 구성요소는 **확장에는 열려(Open)** 있고 **변경에는 닫혀(Closed)** 있어야 한다.
③ 자식 클래스는 부모 클래스를 **완전히 대체**할 수 있어야 한다.
④ 클라이언트는 자신이 사용하지 않는 메서드에 **의존하지 않아야** 한다.

### 문제 5 [유사 기출: 2022년 2회 2과목 — 스택(Stack) 자료구조 정형 패턴]
다음 중 **스택(Stack)** 에 대한 설명으로 옳지 **않은** 것은?

① **LIFO(Last In First Out, 후입선출)** 구조이며 한쪽 끝(top)에서만 삽입·삭제가 일어난다.
② 주요 연산은 **push(삽입)·pop(삭제)·peek(top 조회)·isEmpty** 이다.
③ 함수 호출의 **복귀 주소 저장**, 수식의 **후위 표기 변환·계산**, **DFS(깊이 우선 탐색)** 구현에 사용된다.
④ 데이터가 들어온 **순서대로** 처리되므로 **FIFO(First In First Out)** 방식이며, 프린터 작업 대기열에 사용된다.

### 문제 6 [유사 기출: 2022년 3회 2과목 — 화이트박스 테스트 커버리지 정형 패턴]
다음 중 **화이트박스 테스트(White-box Test)** 의 **커버리지(Coverage)** 종류에 대한 설명으로 옳지 **않은** 것은?

① **구문(Statement) 커버리지** — 프로그램의 **모든 명령문**이 적어도 한 번 이상 실행되도록 테스트 케이스를 설계한다.
② **분기(Branch / Decision) 커버리지** — 모든 **분기(if·switch 등)** 의 **참·거짓 결과**가 한 번 이상 실행되도록 한다.
③ **조건(Condition) 커버리지** — 분기의 **각 개별 조건식**이 참·거짓을 모두 가지도록 테스트한다.
④ **경계값 분석(Boundary Value Analysis)** — 입력값의 경계 부근을 집중적으로 테스트하는 **화이트박스 기법**이다.

### 문제 7 [유사 기출: 2022년 1회 2과목 — DevOps / CI-CD 파이프라인 정형 패턴]
다음 중 **CI/CD(Continuous Integration/Continuous Delivery·Deployment) 파이프라인**에 대한 설명으로 옳지 **않은** 것은?

① **CI(지속적 통합)** — 개발자가 작성한 코드를 **자주(매일·매시간)** 메인 브랜치에 통합하고 **자동 빌드·자동 테스트**를 수행한다.
② **CD(지속적 전달)** — CI 산출물을 **스테이징/운영 환경**까지 자동 배포 가능한 상태로 만들고, 운영 배포는 사람이 승인한다.
③ **CD(지속적 배포)** — 테스트 통과 시 **운영 환경까지 자동 배포**까지 수행한다(사람 승인 불필요).
④ CI/CD는 **수동 빌드·수동 회귀 테스트·수동 배포**를 원칙으로 하며, 자동화는 안티패턴으로 권장되지 않는다.

### 문제 8 [유사 기출: 2022년 2회 2과목 — 인터페이스 데이터 형식 JSON 정형 패턴]
다음 중 **JSON(JavaScript Object Notation)** 에 대한 설명으로 옳지 **않은** 것은?

① **키-값(key-value)** 쌍과 **배열**로 구성된 **경량(lightweight) 텍스트 기반 데이터 교환 형식**이다.
② JavaScript 객체 리터럴에서 파생되었으나 **언어 독립적**이며 대부분의 언어가 파싱 라이브러리를 제공한다.
③ XML 대비 **태그가 없어 문서 크기가 작고**, 사람이 읽기 쉬우며 REST API 표준 포맷으로 널리 사용된다.
④ JSON은 **DTD·XML Schema**처럼 **태그 기반 마크업** 으로 데이터를 정의하며 **주석(/* */, //)** 을 표준으로 지원한다.

### 문제 9 [유사 기출: 2022년 1회 3과목 — NoSQL 4종 분류 정형 패턴]
다음 중 **NoSQL** 데이터베이스의 **유형(Type)** 과 대표 제품 매칭으로 옳지 **않은** 것은?

① **Key-Value Store** — Redis, Amazon DynamoDB, Riak. 단순 키 기반 조회에 최적.
② **Document Store** — MongoDB, CouchDB. JSON·BSON 문서 단위로 저장.
③ **Column-Family Store** — HBase, Cassandra. 컬럼 단위 분산 저장. 대규모 분석에 강함.
④ **Graph Store** — Oracle DB, MySQL, PostgreSQL. **관계형(RDBMS)** 의 표준 SQL 기반 트랜잭션 처리에 특화.

다음 중 **SQL DML(Data Manipulation Language)** 문에 해당하지 **않는** 것은?

① **SELECT** — 데이터 조회.
② **INSERT** — 데이터 삽입.
③ **UPDATE** — 데이터 수정.
④ **CREATE** — 테이블·뷰 등 객체 생성(DDL).

### 문제 11 [유사 기출: 2022년 3회 3과목 — 정규화 1NF·2NF·3NF 단계 정형 패턴]
다음 중 **관계 데이터베이스 정규화(Normalization)** 의 단계에 대한 설명으로 옳지 **않은** 것은?

① **1NF(제1정규형)** — 모든 속성이 **원자값(Atomic Value)** 을 가져야 한다(반복 그룹 제거).
② **2NF(제2정규형)** — 1NF를 만족하면서 **기본키에 부분 함수 종속(Partial Dependency)** 을 제거한다.
③ **3NF(제3정규형)** — 2NF를 만족하면서 **이행 함수 종속(Transitive Dependency)** 을 제거한다.
④ **4NF(제4정규형)** — 1NF의 전 단계로, **다치 종속(Multi-valued Dependency)** 만을 허용하기 위해 적용한다.

### 문제 12 [유사 기출: 2022년 1회 3과목 — 트랜잭션 ACID 정형 패턴]
다음 중 **트랜잭션(Transaction)** 의 **ACID** 속성에 대한 설명으로 옳지 **않은** 것은?

① **원자성(Atomicity)** — 트랜잭션의 모든 연산이 **모두 반영(Commit)** 되거나 **모두 취소(Rollback)** 되어야 한다.
② **일관성(Consistency)** — 트랜잭션 실행 전후에 데이터베이스의 **무결성 제약**이 유지되어야 한다.
③ **고립성(Isolation)** — 동시에 실행되는 트랜잭션들이 **서로 영향을 주지 않도록** 격리되어야 한다.
④ **지속성(Durability)** — 트랜잭션이 **commit되기 전**의 임시 상태도 시스템 장애 시 영구 저장되어야 한다.

### 문제 14 [유사 기출: 2022년 3회 4과목 — Java 접근 제어자 정형 패턴]
다음 중 Java의 **접근 제어자(Access Modifier)** 에 대한 설명으로 옳지 **않은** 것은?

① **public** — **모든 클래스**에서 접근 가능. 클래스·메서드·필드 모두 사용 가능.
② **protected** — **같은 패키지** + **다른 패키지의 자식 클래스**에서 접근 가능.
③ **default(package-private)** — 접근 제어자를 **명시하지 않은** 경우. **같은 패키지** 내에서만 접근 가능.
④ **private** — **같은 패키지 + 같은 모듈** 내 모든 클래스에서 접근 가능하며, **자식 클래스에서도 상속**된다.

### 문제 16 [유사 기출: 2022년 2회 4과목 — TCP vs UDP 비교 정형 패턴]
다음 중 **TCP(Transmission Control Protocol)** 와 **UDP(User Datagram Protocol)** 의 비교 설명으로 옳지 **않은** 것은?

① **TCP는 연결 지향(Connection-Oriented)**, **3-way handshake(SYN·SYN-ACK·ACK)** 로 연결을 수립한다.
② **TCP는 신뢰성** 있는 전송을 보장하며 **흐름 제어·혼잡 제어·재전송** 메커니즘을 제공한다.
③ **UDP는 비연결형(Connectionless)** 이며 헤더가 작고 **빠르지만**, 데이터 순서·재전송을 보장하지 않는다.
④ **UDP는 TCP보다 헤더가 크고**, **3-way handshake** 와 **재전송 제어**를 모두 지원해 더 신뢰성 있는 전송을 보장한다.

### 문제 17 [유사 기출: 2022년 3회 5과목 — Docker 컨테이너 vs VM 정형 패턴]
다음 중 **Docker 컨테이너(Container)** 와 **가상머신(Virtual Machine, VM)** 의 비교 설명으로 옳지 **않은** 것은?

① 컨테이너는 호스트 OS의 **커널을 공유**하므로 **부팅이 빠르고** 자원 사용량이 적다.
② VM은 **하이퍼바이저(Hypervisor·Type 1·Type 2)** 위에 **각자 게스트 OS**를 가지므로 격리도가 높다.
③ 컨테이너는 **이미지(Image) → 컨테이너(Container)** 모델로 실행되며, **Dockerfile**로 빌드 정의를 코드화할 수 있다.
④ 컨테이너는 **VM보다 격리 수준이 강하고** 항상 **각자의 커널을 별도로 부팅**해 실행되므로 VM보다 무거운 구조다.

### 문제 18 [유사 기출: 2022년 1회 5과목 — OWASP Top 10 정형 패턴]
다음 중 **OWASP Top 10(2021)** 에 포함된 **웹 애플리케이션 주요 보안 위험**에 해당하지 **않는** 것은?

① **A01:2021 Broken Access Control** — 권한 검사 누락·우회로 인한 권한 상승.
② **A03:2021 Injection** — SQL Injection·OS Command Injection·LDAP Injection 등.
③ **A07:2021 Identification and Authentication Failures** — 약한 비밀번호·세션 고정·자격 증명 노출.
④ **시스템 콘솔의 화면 캡처(Screenshot)** — 사용자가 PC 화면을 캡처해 SNS에 공유하는 행위.

### 문제 19 [유사 기출: 2022년 2회 5과목 — Agile/Scrum 방법론 정형 패턴]
다음 중 **스크럼(Scrum)** 의 핵심 요소에 대한 설명으로 옳지 **않은** 것은?

① **스프린트(Sprint)** — 일반적으로 **1~4주**의 짧은 개발 주기로, 종료 시 작동 가능한 증분을 산출한다.
② **데일리 스크럼(Daily Scrum)** — **매일 15분** 이내로 진행되는 개발자 회의(어제·오늘·장애물 공유).
③ **제품 백로그(Product Backlog)** — 제품 오너(Product Owner)가 관리하는 **우선순위가 매겨진 요구사항 목록**이다.
④ 스크럼은 **폭포수(Waterfall) 모델**의 한 종류로, **요구분석→설계→구현→테스트→유지보수** 단계를 한 번만 순차적으로 진행한다.

### 문제 20 [유사 기출: 2022년 3회 5과목 — 대칭키 vs 비대칭키 암호 정형 패턴]
다음 중 **대칭키(Symmetric Key)** 와 **비대칭키(Asymmetric Key·공개키)** 암호화 알고리즘에 대한 설명으로 옳지 **않은** 것은?

① **대칭키 알고리즘** — 암호화·복호화에 **같은 키**를 사용. 예) **DES, 3DES, AES, SEED, ARIA, IDEA, RC4**.
② **비대칭키 알고리즘** — **공개키(public)** 로 암호화, **개인키(private)** 로 복호화(혹은 그 반대). 예) **RSA, ECC, ElGamal, DSA**.
③ 대칭키는 **속도가 빠르지만** 키 교환·관리가 어렵고, 비대칭키는 **속도가 느리지만** 키 교환·전자서명에 강하다.
④ **RSA·ECC**는 **대칭키 알고리즘**의 대표 사례이며, **AES**는 **비대칭키(공개키) 알고리즘**의 대표 사례다.

## 💡 상세 해설

### 문제 1
**정답 ④** — MSA는 **서비스마다 독립적인 DB(Database per Service)** 를 가지므로 **단일 DB 공유·전체 ACID 트랜잭션 불가**. 분산 트랜잭션은 **사가(Saga) 패턴·이벤트 소싱·결과적 일관성(Eventual Consistency)** 으로 해결.
- 함정: MSA = 마이크로서비스, 서비스 단위 작음·독립 배포·폴리글랏 OK / 모놀리식 = 단일 코드베이스·공유 DB.

### 문제 2
**정답 ④** — SOA는 MSA보다 **서비스 단위가 크고**(Coarse-grained), 일반적으로 **공유 DB·공유 모델**을 허용한다. MSA는 SOA의 진화형으로 **더 작은 단위(Fine-grained)** ·**서비스별 독립 DB**를 강제.
- 함정: ESB ⊂ SOA (중앙 메시지 버스) vs MSA = API Gateway + 비동기 메시징(중앙 없음).

### 문제 3
**정답 ④** — Singleton은 **인스턴스를 단 하나로 제한**하는 패턴이지 다중 인스턴스 보유 패턴이 아님. 상속·다형성을 적극 활용해 여러 인스턴스를 만들면 Singleton 정의를 위반.
- 함정: GoF 23 중 **생성 패턴 5종** = Singleton·Factory Method·Abstract Factory·Builder·Prototype.

### 문제 4
**정답 ② (OCP)** — ① SRP, ② OCP, ③ LSP, ④ ISP. 마지막 DIP(Dependency Inversion)는 "고수준 모듈은 저수준 모듈에 의존하지 않고 추상화에 의존".
- 함정: SOLID = **S**RP·**O**CP·**L**SP·**I**SP·**D**IP. OCP는 인터페이스·추상클래스·전략(Strategy) 패턴으로 구현.

### 문제 5
**정답 ④** — 스택은 **LIFO(후입선출)**. **FIFO(선입선출)** 은 **큐(Queue)** 의 특징. 프린터 작업 대기열·BFS·CPU 스케줄링은 큐 사용.
- 함정: 스택 응용 = 함수 호출·재귀·후위표기·DFS·괄호 검사·undo 기능 / 큐 응용 = 프린터·BFS·메시지 큐.

### 문제 6
**정답 ④** — **경계값 분석(BVA)** 은 **블랙박스 테스트** 기법(동치분할과 함께). 화이트박스 커버리지는 **구문·분기·조건·다중조건·경로** 등 내부 구조 기반.
- 함정: 블랙박스 = 동치분할·경계값·결정테이블·원인-결과 그래프·페어와이즈 / 화이트박스 = 기초경로·구문·분기·조건·MC/DC.

### 문제 7
**정답 ④** — CI/CD의 **본질은 자동화**(자동 빌드·자동 테스트·자동 배포). 수동 회귀 테스트·수동 배포는 CI/CD의 안티패턴.
- 함정: CI = 통합·빌드·테스트 자동화 / CD = Continuous Delivery(스테이징까지) or Continuous Deployment(운영까지). 도구: Jenkins·GitHub Actions·GitLab CI·CircleCI·ArgoCD(GitOps).

### 문제 8
**정답 ④** — JSON은 **태그 없는** key-value/배열 구조. 표준 JSON은 **주석 미지원**(JSONC·JSON5 등 확장 포맷에서만 지원). 태그 기반은 XML.
- 함정: JSON 자료형 = string·number·boolean·null·object{}·array[]. XML = 마크업·DTD·XML Schema 지원, YAML = 들여쓰기 기반.

### 문제 9
**정답 ④** — **Graph Store**의 대표는 **Neo4j, JanusGraph, Amazon Neptune, OrientDB**. Oracle·MySQL·PostgreSQL은 **RDBMS**.
- 함정: NoSQL 4종 분류 + 대표 제품 = Key-Value(Redis·DynamoDB) / Document(MongoDB·CouchDB) / Column-Family(Cassandra·HBase) / Graph(Neo4j).

### 문제 10
**정답 ④** — **CREATE / ALTER / DROP / TRUNCATE / RENAME** 은 **DDL(Data Definition Language)**. DML은 **SELECT·INSERT·UPDATE·DELETE·MERGE**.
- 함정: SQL 3분류 = DDL(정의)·DML(조작)·DCL(제어=GRANT·REVOKE·COMMIT·ROLLBACK·SAVEPOINT). 일부 교재는 TCL(트랜잭션)을 DCL과 분리.

### 문제 11
**정답 ④** — **4NF는 3NF·BCNF 이후 단계**로, **다치 종속(MVD)을 제거**한다. "1NF의 전 단계", "다치 종속만을 허용"은 정의를 정반대로 서술한 함정.
- 함정: 정규화 순서 = 1NF(원자값) → 2NF(부분종속 제거) → 3NF(이행종속 제거) → BCNF(결정자=후보키) → 4NF(다치종속 제거) → 5NF(조인종속 제거).

### 문제 12
**정답 ④** — **지속성(Durability)** 은 **commit이 완료된** 트랜잭션의 결과가 시스템 장애에도 영구 저장됨을 보장. **commit 이전**의 임시 상태가 아닌, **commit 이후의 결과**만 보장한다.
- 함정: ACID = **A**tomicity·**C**onsistency·**I**solation·**D**urability. 로그 기반 회복(WAL·REDO·UNDO)으로 구현.

### 문제 13
**정답 ③ (30)** — `p = a`이므로 `p`는 `a[0]` 주소. `*(p + 2) = a[2] = 30`. C 포인터 산술에서 `p + n`은 **n개 요소만큼 이동**(int 4바이트라면 8바이트).
- 함정: `*(p+2)` ≡ `p[2]` ≡ `a[2]`. `p++` 도 한 요소 단위로 이동.

### 문제 14
**정답 ④** — **private** 는 **같은 클래스 내부**에서만 접근 가능. 자식 클래스에서도 상속되지만 **직접 접근은 불가**(getter/setter로 접근).
- 함정: 접근 범위 좁은 순 = **private < default(package-private) < protected < public**. protected는 같은 패키지 + 다른 패키지의 자식.

### 문제 15
**정답 ② (20 0 4)** — `d.get('b')` = 20(`d['b'] = 20`으로 갱신), `d.get('e', 0)` = 0(기본값 반환), `len(d)` = 4(`'a','b','c','d'`).
- 함정: `dict.get(key, default)` — 키 없을 때 default 반환(KeyError 회피). 키만 있으면 None 반환.

### 문제 16
**정답 ④** — **UDP는 TCP보다 헤더가 작고(8바이트)** 단순. **3-way handshake·재전송·흐름제어**는 모두 **TCP**의 기능. UDP는 비연결형·신뢰성 없음·빠름.
- 함정: TCP 헤더 = 20바이트 기본 / UDP 헤더 = 8바이트. UDP 활용 = DNS·DHCP·SNMP·VoIP·스트리밍·게임.

### 문제 17
**정답 ④** — 컨테이너는 **호스트 OS의 커널을 공유**하므로 **VM보다 가볍고 부팅 빠름**. **VM은 게스트 OS 부팅 필요**해서 무겁다. "컨테이너가 각자 커널 별도 부팅"은 정확히 반대.
- 함정: 컨테이너 격리 = namespace + cgroups. VM 격리 = 하이퍼바이저 + 게스트 OS. 격리 강도는 VM > 컨테이너.

### 문제 18
**정답 ④** — **OWASP Top 10(2021)** 은 웹 애플리케이션 보안 위험 10가지: A01 Broken Access Control, A02 Cryptographic Failures, A03 Injection, A04 Insecure Design, A05 Security Misconfiguration, A06 Vulnerable and Outdated Components, A07 Identification and Authentication Failures, A08 Software and Data Integrity Failures, A09 Security Logging and Monitoring Failures, A10 Server-Side Request Forgery(SSRF).
- 함정: PC 화면캡처는 **사용자 단말 행위**이며 OWASP 웹 애플리케이션 보안 범주가 아님.

### 문제 19
**정답 ④** — Scrum은 **Agile(애자일) 방법론**의 한 종류. 폭포수와 반대로 **반복적·점진적(iterative & incremental)** 으로 짧은 스프린트를 반복.
- 함정: Scrum 3 Role = PO(Product Owner)·SM(Scrum Master)·Dev Team. 4 Ceremony = Sprint Planning·Daily Scrum·Sprint Review·Retrospective. 3 Artifact = Product Backlog·Sprint Backlog·Increment.

### 문제 20
**정답 ④** — **RSA·ECC**는 **비대칭키(공개키)** 알고리즘, **AES**는 **대칭키** 알고리즘이다. 보기 ④는 정확히 반대로 서술.
- 함정: 대칭키 키개수 = n(n-1)/2, 비대칭키 키개수 = 2n. AES는 미국 NIST 표준 대칭키(128/192/256비트). RSA는 큰 소수 인수분해 어려움, ECC는 타원곡선 이산로그 어려움.

## 📊 자주 틀리는 함정 5가지 (2022년 회독 1/4 진입)

1. **MSA ≠ SOA** — MSA는 SOA의 진화형으로 **더 작은 단위·서비스별 독립 DB**. SOA는 ESB 중심·공유 가능. 시험은 "MSA = 공유 DB"를 함정으로 자주 출제.
2. **OCP의 핵심 = "확장 열림·변경 닫힘"** — SRP("책임 하나")와 혼동 빈출. **SOLID 5개 정의를 1줄씩 외울 것**.
3. **CREATE는 DDL** — SELECT/INSERT/UPDATE/DELETE만 DML. CREATE/ALTER/DROP/TRUNCATE는 DDL. GRANT/REVOKE는 DCL.
4. **Durability는 commit 이후만 보장** — "commit 전 임시 상태"가 보기에 들어가면 함정. ACID 4개를 commit 시점 기준으로 다시 점검.
5. **Scrum은 Agile, Waterfall 아님** — 스프린트·반복·점진. Waterfall은 한 번 순차. "Scrum은 폭포수 한 종류"는 빈출 함정.
