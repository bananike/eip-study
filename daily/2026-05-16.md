## 🔑 핵심 용어 (10개)
| 용어 | 한 줄 정의 | 핵심 키워드 |
|---|---|---|
| 아키텍처 패턴 (Architecture Pattern) | 소프트웨어 전체의 골격을 잡는 고수준 구조 | 시스템 단위, 재사용 |
| 레이어드 (Layered) | 시스템을 계층으로 나눠 상→하 단방향 호출 | OSI 7계층, 계층 분리 |
| 클라이언트-서버 (Client-Server) | 서비스 요청자(C)와 제공자(S)로 분리 | 다대일 요청/응답 |
| 파이프-필터 (Pipe-Filter) | 데이터 스트림을 필터들이 순차 변환 | UNIX 셸, 컴파일러 |
| MVC (Model-View-Controller) | 데이터·표현·제어 3분할 | 웹 프레임워크 |
| 마스터-슬레이브 (Master-Slave) | 마스터가 작업 분할, 슬레이브가 병렬 처리 후 통합 | 장애 허용, 부하 분산 |
| 브로커 (Broker) | 클라이언트-서비스 간 중개자 패턴 | CORBA, ORB |
| UI (User Interface) | 사용자와 시스템이 정보를 주고받는 매개체 | CLI/GUI/NUI/VUI/OUI |
| UI 4원칙 | 직관성·유효성·학습성·유연성 | "직유학유" |
| 미들웨어 (Middleware) | 응용 SW와 OS 사이를 잇는 중간 SW | DB·RPC·MOM·TP모니터·ORB·WAS |

## 📖 핵심 개념 정리

### 1. 아키텍처 패턴 9종 (필수 암기)

| # | 패턴 | 특징 | 예시 |
|---|---|---|---|
| 1 | **레이어드(Layered)** | N개 계층, 인접 계층끼리만 통신, 상→하 단방향 | OSI 7계층, JSP 3티어 |
| 2 | **클라이언트-서버(C/S)** | 1개 서버 + 다수 클라이언트, 요청/응답 | 웹 브라우저 |
| 3 | **파이프-필터(Pipe-Filter)** | 데이터 스트림을 필터(처리 단위)가 순차 처리, 파이프(채널)로 전달, 무상태 | UNIX 셸, 컴파일러 |
| 4 | **브로커(Broker)** | 분산 컴포넌트들의 중개자, 요청/위치투명성 제공 | CORBA, 메시지 큐 |
| 5 | **MVC** | Model(데이터)·View(표현)·Controller(제어) 3분할 | Spring, Django |
| 6 | **마스터-슬레이브(Master-Slave)** | 마스터가 작업 분할 → 슬레이브가 병렬 수행 → 마스터가 통합 | DB 복제, 장애 허용 시스템 |
| 7 | **피어 투 피어(Peer-to-Peer)** | 노드들이 클라이언트이자 서버 역할 동시 수행 | P2P 파일 공유 |
| 8 | **이벤트 버스(Event-Bus)** | 발행(Publisher)/구독(Subscriber) 모델, 중앙 버스 | 알림 시스템 |
| 9 | **블랙보드(Blackboard)** | 결정적 해법 없는 문제 → 공유 저장소(블랙보드)에 모든 지식원이 접근 | 음성 인식, AI |

**암기법 (두문자): "레클파브엠 마피이블"** (레이어드-클라이언트서버-파이프필터-브로커-MVC-마스터슬레이브-피어투피어-이벤트버스-블랙보드)

⚠ **주의**: 아키텍처 패턴 vs 디자인 패턴(Day 4)
- 아키텍처 패턴 = **시스템 전체** 골격 (거시적, top-down)
- 디자인 패턴 = **클래스·객체** 수준 협력 (미시적, GoF 23)

### 2. UI/UX 핵심

#### (1) UI 종류 5가지 (CLI → OUI 진화)
| 종류 | 풀네임 | 입력 방식 |
|---|---|---|
| **CLI** | Command Line Interface | 텍스트 명령 (cmd, bash) |
| **GUI** | Graphical UI | 아이콘·마우스 (Windows) |
| **NUI** | Natural UI | 사용자의 자연스러운 행동(터치, 제스처) |
| **VUI** | Voice UI | 음성 (Siri, Bixby) |
| **OUI** | Organic UI | 모든 사물 = 입출력 장치 (IoT) |

#### (2) UI 설계 **4대 원칙** ★★★ (매년 출제)
| 원칙 | 의미 |
|---|---|
| **직관성** | 누구나 쉽게 이해·사용 |
| **유효성** | 사용자의 목적을 정확·완벽하게 달성 |
| **학습성** | 누구나 쉽게 배우고 익힘 |
| **유연성** | 사용자 요구를 최대한 수용, 실수 최소화 |

→ 두문자 **"직유학유"**

#### (3) UI 설계 **지침**(원칙 외 보조)
사용자 중심, 일관성, 단순성, 결과 예측 가능, **가시성**(주요 기능 노출), **표준화**(한번 학습하면 재사용), **접근성**(다양한 계층 사용), 명확성, 오류 발생 해결, 심미성

#### (4) UI 설계 도구 4종 (순서: 와이어 → 목업 → 스토리보드 → 프로토타입)
| 도구 | 정의 | 특징 |
|---|---|---|
| **와이어프레임 (Wireframe)** | 페이지에 콘텐츠 영역만 잡은 골격 | 흑백, 화면 단위 |
| **목업 (Mockup)** | 와이어프레임 + 시각적 디자인 | 정적, 실물 흡사 |
| **스토리보드 (Storyboard)** | 정책·프로세스·디스크립션 포함 최종 설계서 | 디자이너·개발자 의사소통 |
| **프로토타입 (Prototype)** | 인터랙션 동작하는 **동적** 모형 | 사용성 테스트 가능 |

암기: "**와목스프**" (지하철 1호선 정류장처럼 순서)

#### (5) UI vs UX
- **UI**: 화면·아이콘 같은 **시각·구조적 요소**
- **UX**: 제품을 사용하며 느끼는 **총체적 경험**(접근성·심미성·시간성·환경성 포함)

### 3. 인터페이스 & 미들웨어

#### (1) 미들웨어 6종 ★★★
| 종류 | 풀네임 | 핵심 역할 |
|---|---|---|
| **DB 미들웨어** | Database Middleware | 다양한 DB를 표준화된 방식으로 연결 (ODBC 등) |
| **RPC** | Remote Procedure Call | **원격** 프로시저를 **로컬처럼** 호출 (동기) |
| **MOM** | Message-Oriented Middleware | **비동기** 메시지 전달, 느리지만 안정적 |
| **TP-Monitor** | Transaction Processing Monitor | 분산 트랜잭션 처리·감시 (항공·철도 예약) |
| **ORB** | Object Request Broker | 객체 간 통신 중개 (CORBA 표준) |
| **WAS** | Web Application Server | **동적** 콘텐츠 처리 (Tomcat, JBoss) |

암기: "**DB-RPC-MOM-TP-ORB-WAS**"

⚠ **빈출 함정**: **Web Server는 미들웨어가 아니다** (정적 콘텐츠만 처리).

#### (2) 인터페이스 시스템 구성
**송신 시스템** → **중계 서버**(옵션) → **수신 시스템**

**인터페이스 명세서 항목**: 인터페이스 ID, 송수신 시스템 정보, 송수신 데이터, 데이터 크기(평균/최대), 최대 처리 횟수

#### (3) 연계 모듈 구현 방식 (EAI vs ESB)
| 항목 | EAI | ESB |
|---|---|---|
| 풀네임 | Enterprise Application Integration | Enterprise Service Bus |
| 목적 | 기업 내 애플리케이션 통합 | 서비스 중심 아키텍처(SOA) 구현 |
| 결합도 | 비교적 강함 | 느슨한 결합 |
| 방식 | 4가지 — Point-to-Point, Hub & Spoke, Message Bus, Hybrid | 메시지 버스 + 표준화된 서비스 인터페이스 |

#### (4) 데이터 포맷
**JSON**(JavaScript Object Notation, 경량 키-값), **XML**(Extensible Markup Language, 태그 기반), **YAML**, **AJAX**

#### (5) 인터페이스 구현 검증 도구 5종
**xUnit**(언어별 단위테스트, JUnit/CppUnit/NUnit), **STAF**(분산 환경, 데몬 기반), **FitNesse**(웹 기반 협업), **NTAF**(STAF+FitNesse, 네이버), **Selenium**(웹 브라우저 자동화), **watir**(루비 기반 웹 테스트)

암기: "**x-STAF-Fit-NTAF-Sel-watir**"

## 💡 상세 해설

### 문제 1 해설 — 정답 ②

- ① 노드·간선 = 그래프(블랙보드/이벤트 버스 등). ③ 계층 모델 = 레이어드. ④ Model/View/Controller 3분할 = MVC.

### 문제 2 해설 — 정답 ②
미들웨어 6종: DB, RPC, MOM, TP-Monitor, ORB, **WAS**. **Web Server**(Apache, Nginx)는 정적 콘텐츠만 처리하므로 미들웨어가 **아님**. 자주 나오는 함정.

### 문제 3 해설 — 정답 ③
UI 4원칙 = **직관성·유효성·학습성·유연성** ("직유학유"). **무결성**(Integrity)은 보안/DB 개념이지 UI 원칙이 아님.

### 문제 4 해설 — 정답 ④
- CLI: 명령줄 텍스트
- GUI: 그래픽
- NUI: 자연스러운 동작(터치/제스처)
- **VUI**: Voice = 음성 → 정답
- OUI: 모든 사물

### 문제 5 해설 — 정답 ②
"비동기식", "메시지 저장", "이기종 분산 데이터 동기화" → **MOM**(Message-Oriented Middleware).

### 문제 6 해설 — 정답 ④

### 문제 7 해설 — 정답 ③
