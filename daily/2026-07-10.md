## 🎯 오늘의 기출문제 — 2024년 정처기 필기 20문제 (5과목 균등 · 회독 3/4)

### 문제 1 [유사 기출: 2024년 1회 1과목 — 럼바우(Rumbaugh) OMT 3모델 정형 패턴]
럼바우(Rumbaugh)의 객체지향 분석(OMT: Object Modeling Technique) 3대 모델링에 관한 설명 중 **가장 옳지 않은 것**은?

① **객체(Object) 모델링** — 시스템 내 정적 구조를 표현하며, 객체(Class)·속성(Attribute)·연산(Operation)·관계(Association·Aggregation·Generalization)를 클래스 다이어그램으로 나타낸다.
② **동적(Dynamic) 모델링** — 시간 흐름에 따른 시스템 상태 변화를 표현하며, 상태(State)·이벤트(Event)·전이(Transition)를 상태 다이어그램(State Diagram)으로 나타낸다.
③ **기능(Functional) 모델링** — 데이터 흐름과 처리를 표현하며, DFD(Data Flow Diagram)·프로세스·데이터 저장소·외부 개체를 나타낸다.
④ **관계(Relational) 모델링** — 데이터베이스 스키마와 외래키를 표현하는 럼바우 OMT의 네 번째 정식 모델링 활동이다.

### 문제 2 [유사 기출: 2024년 2회 1과목 — GoF 생성 패턴 5종 정형 패턴]
**GoF(Gang of Four) 생성(Creational) 디자인 패턴**에 관한 설명 중 **가장 옳지 않은 것**은?

① **Singleton(싱글턴)** — 클래스의 인스턴스가 시스템 내 오직 하나만 생성되도록 보장하며, 전역 접근점을 제공한다.
② **Factory Method(팩토리 메서드)** — 객체 생성을 서브클래스에 위임하여, 부모 클래스는 생성 인터페이스만 정의하고 실제 생성 로직은 자식 클래스가 구현하도록 한다.
③ **Builder(빌더)** — 복잡한 객체의 생성 과정을 단계별로 분리하여, 동일한 생성 절차에서 서로 다른 표현(Representation)을 만들 수 있게 한다.
④ **Decorator(데코레이터)** — 객체 생성을 담당하는 GoF 생성 패턴의 대표적 사례로, 여러 인스턴스를 미리 생성해 두었다가 요청 시 배포하는 객체 풀 패턴이다.

### 문제 3 [유사 기출: 2024년 3회 1과목 — 응집도(Cohesion) 7단계 정형 패턴]
**모듈의 응집도(Cohesion)**에 관한 설명 중 **가장 옳지 않은 것**은?

① 응집도는 모듈 내부 구성 요소들이 서로 얼마나 밀접하게 관련되어 있는지를 나타내며, **응집도는 강할수록 좋은 설계**로 평가된다.
② 응집도 7단계 강도 순서(약 → 강)는 **우연적(Coincidental) → 논리적(Logical) → 시간적(Temporal) → 절차적(Procedural) → 통신적(Communicational) → 순차적(Sequential) → 기능적(Functional)**이다.
③ **기능적 응집도(Functional Cohesion)** — 모듈 내 모든 구성 요소가 하나의 목적을 위해 협력하는 가장 이상적인 형태이다.
④ **우연적 응집도(Coincidental Cohesion)** — 모듈 내 요소들이 하나의 명확한 기능을 수행하도록 정교하게 설계된 **가장 이상적이고 권장되는 응집도**이다.

### 문제 4 [유사 기출: 2024년 1회 1과목 — 결합도(Coupling) 7단계 정형 패턴]
**모듈의 결합도(Coupling)**에 관한 설명 중 **가장 옳지 않은 것**은?

① 결합도는 서로 다른 모듈 간의 상호 의존 정도를 의미하며, **결합도는 약할수록(낮을수록) 좋은 설계**로 평가된다.
② 결합도 7단계 강도 순서(약 → 강)는 **자료(Data) → 스탬프(Stamp) → 제어(Control) → 외부(External) → 공통(Common) → 내용(Content)**이며, 자료 결합도가 가장 약하고 내용 결합도가 가장 강하다.
③ **자료 결합도(Data Coupling)** — 모듈 간에 단순 매개변수(값)만 주고받는 가장 이상적인 결합 형태이다.
④ **내용 결합도(Content Coupling)** — 결합도가 가장 낮고 이상적인 형태로, 유지보수성과 재사용성을 극대화한다.

### 문제 7 [유사 기출: 2024년 1회 2과목 — 블랙박스 테스트 5종 정형 패턴]
**블랙박스 테스트(Black-box Testing) 기법**에 관한 설명 중 **가장 옳지 않은 것**은?

① **동등 분할(Equivalence Partitioning)** — 입력 값의 도메인을 유효(Valid)·무효(Invalid) 등가 클래스로 분할하여 각 클래스에서 대표 값 하나씩 테스트한다.
② **경곗값 분석(Boundary Value Analysis)** — 입력 범위의 경계(최솟값·최댓값·경계 근처)에서 오류가 자주 발생한다는 경험에 근거해 경곗값을 집중 테스트한다.
③ **원인-결과 그래프(Cause-Effect Graphing)** — 입력 조건(원인)과 출력 결과 간의 논리적 관계를 그래프로 표현하여 테스트 케이스를 도출한다.
④ **기초 경로 검사(Basis Path Testing)** — 프로그램의 순환 복잡도(V(G) = E - N + 2)를 기반으로 독립적인 실행 경로를 도출하는 **블랙박스** 대표 기법이다.

### 문제 8 [유사 기출: 2024년 2회 2과목 — 통합 테스트 4종 정형 패턴]
**통합 테스트(Integration Testing) 전략**에 관한 설명 중 **가장 옳지 않은 것**은?

① **하향식(Top-down) 통합** — 최상위 모듈부터 하위로 통합하며, 아직 개발되지 않은 하위 모듈은 **스텁(Stub)**으로 대체한다.
② **상향식(Bottom-up) 통합** — 최하위 모듈부터 상위로 통합하며, 상위 모듈 역할을 대신하는 **드라이버(Driver)**로 하위 모듈을 호출·테스트한다.
③ **샌드위치(Sandwich) 통합** — 하향식과 상향식을 병행하며, 중간 계층에서 만나는 혼합 방식으로 대규모 시스템에 유리하다.
④ **빅뱅(Big Bang) 통합** — 모든 모듈을 개별 단위 테스트 없이 **한꺼번에 통합·테스트**하는 방식으로, 결함 원인 추적이 매우 쉽고 대규모 시스템에서 가장 권장된다.

### 문제 9 [유사 기출: 2024년 3회 3과목 — 함수 종속(FD) 3종 정형 패턴]
**함수 종속(Functional Dependency)** 3종에 관한 설명 중 **가장 옳지 않은 것**은?

① **완전 함수 종속(Full FD)** — 속성 Y가 복합키의 **일부가 아닌 전체 X**에 대해서만 종속되는 관계로, 어떤 부분집합에도 종속되지 않는다.
② **부분 함수 종속(Partial FD)** — 속성 Y가 복합키의 **일부 속성**에만 종속되는 관계로, **2NF 위반의 원인**이며 2NF 정규화 시 제거된다.
③ **이행 함수 종속(Transitive FD)** — X→Y이고 Y→Z이면 X→Z가 성립하는 간접 종속으로, **3NF 위반의 원인**이며 3NF 정규화 시 제거된다.
④ **부분 함수 종속(Partial FD)** — 후보키가 단일 속성일 때에만 발생하는 종속으로, 단일 속성 기본키를 사용하면 **자동으로 완전 함수 종속**이 되지 못한다.

### 문제 10 [유사 기출: 2024년 1회 3과목 — 2단계 로킹(2PL) + S/X-Lock 정형 패턴]
데이터베이스 트랜잭션의 **2단계 로킹 프로토콜(2PL: Two-Phase Locking)** 및 로킹에 관한 설명 중 **가장 옳지 않은 것**은?

① **2PL은 확장 단계(Growing Phase)**에서만 락을 획득하고 **축소 단계(Shrinking Phase)**에서만 락을 해제하며, 한번 락을 해제한 후에는 새로운 락을 획득할 수 없다.
② **공유 락(S-Lock, Shared Lock)** — 여러 트랜잭션이 동시에 획득 가능(S-S 호환)하며, 읽기 전용 연산에 사용된다.
③ **배타 락(X-Lock, Exclusive Lock)** — 한 트랜잭션만 획득 가능하며, S-X·X-S·X-X 모두 호환되지 않아(대기) 쓰기 연산 시 사용된다.
④ **2PL은 직렬 가능성(Serializability)을 보장하지 못하며**, S-Lock과 X-Lock의 호환성도 반대로 정의되어 모든 락이 동시에 획득 가능하다.

### 문제 11 [유사 기출: 2024년 2회 3과목 — 데이터 웨어하우스(DW) + OLAP 정형 패턴]
**데이터 웨어하우스(Data Warehouse)** 및 **OLAP(Online Analytical Processing)**에 관한 설명 중 **가장 옳지 않은 것**은?

① 데이터 웨어하우스는 의사결정 지원을 위해 **주제 지향적(Subject-Oriented)·통합적(Integrated)·시간 변형적(Time-Variant)·비휘발성(Non-Volatile)** 4가지 특성을 가진 데이터의 집합이다(Inmon 정의).
② **OLAP(온라인 분석 처리)** — 다차원(Multi-dimensional) 큐브 구조로 데이터를 분석하며, 슬라이싱(Slicing)·다이싱(Dicing)·드릴다운(Drill-Down)·롤업(Roll-Up)·피벗(Pivot) 등의 연산을 지원한다.
③ **데이터 마트(Data Mart)** — 특정 부서·주제 영역에 특화된 소규모 데이터 웨어하우스로, 전사 DW에서 추출·구축된다.
④ 데이터 웨어하우스는 **일별 OLTP(온라인 트랜잭션 처리) 요구를 실시간 초당 수천 건 처리**하는 것이 주 목적이며, 분석·의사결정 지원과는 무관하다.

### 문제 12 [유사 기출: 2024년 3회 3과목 — NoSQL 4유형 + CAP 정리 정형 패턴]
**NoSQL 데이터베이스 4가지 유형** 및 **CAP 정리**에 관한 설명 중 **가장 옳지 않은 것**은?

① **Key-Value Store** — 단순 키-값 쌍으로 데이터를 저장하며, Redis·DynamoDB·Riak가 대표 사례이다.
② **Document Store** — JSON·BSON·XML 등 문서 형식으로 저장하며, MongoDB·CouchDB가 대표 사례이다. **Column Family Store** — 열(Column) 지향 저장 구조로, Cassandra·HBase가 대표 사례이다.
③ **Graph Database** — 노드(정점)·에지(간선)로 관계를 표현하며, 소셜 네트워크·추천 시스템에 유리하다. Neo4j·JanusGraph가 대표 사례이다.
④ **CAP 정리** — Consistency(일관성)·Availability(가용성)·Partition Tolerance(분할 내성) 3가지를 **모두 동시에 만족 가능**하다는 것이 CAP 정리의 결론이다.

### 문제 13 [유사 기출: 2024년 1회 4과목 — C 포인터·배열 정형 패턴]

### 문제 16 [유사 기출: 2024년 1회 4과목 — OSI 7계층 vs TCP/IP 4계층 정형 패턴]
**OSI 7계층**과 **TCP/IP 4계층**의 대응 관계에 관한 설명 중 **가장 옳지 않은 것**은?

① OSI 7계층은 물리(Physical)·데이터링크(Data Link)·네트워크(Network)·전송(Transport)·세션(Session)·표현(Presentation)·응용(Application) 순으로 구성된다.
② TCP/IP 4계층은 네트워크 접근(Network Access)·인터넷(Internet)·전송(Transport)·응용(Application) 순으로 구성된다.
③ **OSI 전송 계층(4계층)**은 TCP/IP의 **전송 계층**과 대응되며, TCP·UDP가 이 계층의 대표 프로토콜이다.
④ **OSI 응용(7)·표현(6)·세션(5) 3개 계층**은 TCP/IP에서 **인터넷(Internet) 계층 하나**로 합쳐지며, IP·ICMP가 이 계층의 대표 프로토콜이다.

### 문제 17 [유사 기출: 2024년 2회 5과목 — CMMi 5단계 vs SPICE 6단계 정형 패턴]
**CMMi(Capability Maturity Model Integration)** 및 **SPICE(ISO/IEC 15504)** 모델에 관한 설명 중 **가장 옳지 않은 것**은?

① CMMi 5단계 = **초기(Initial) → 관리(Managed) → 정의(Defined) → 정량적 관리(Quantitatively Managed) → 최적화(Optimizing)** 순으로 성숙도를 평가한다.
② SPICE 6단계(레벨 0 포함) = **불완전(Incomplete) → 수행(Performed) → 관리(Managed) → 확립(Established) → 예측(Predictable) → 최적화(Optimizing)**로 프로세스 능력을 평가한다.
③ CMMi는 조직 전체의 성숙도를 5단계로 평가하며, SPICE는 프로세스별 능력을 0~5단계(총 6단계)로 평가하여 **개별 프로세스 진단에 유리**하다.
④ CMMi 최상위 5단계는 "**직관과 경험**에 의존한 개선"을 의미하며, SPICE 최상위 5단계는 CMMi보다 낮은 수준의 성숙도를 나타낸다.

### 문제 18 [유사 기출: 2024년 3회 5과목 — LOC·COCOMO 비용 산정 정형 패턴]
**소프트웨어 비용 산정 기법**에 관한 설명 중 **가장 옳지 않은 것**은?

① **LOC(Line of Code) 기법** — 낙관치(a)·기대치(m)·비관치(b)를 활용하여 **예측 LOC = (a + 4m + b) / 6** 공식으로 규모를 산정하는 하향식 기법이다.
② **COCOMO(Constructive Cost Model)** — Boehm이 제안한 원시 프로그램 크기(KDSI)에 의한 비용 산정 방식으로, 조직형(Organic, 5만 라인 이하)·반분리형(Semi-Detached, 30만 라인 이하)·내장형(Embedded, 30만 라인 초과) 3가지 유형으로 분류한다.
③ **기능 점수(FP: Function Point)** — Albrecht가 제안한 방식으로 사용자 관점의 외부입력(EI)·외부출력(EO)·외부조회(EQ)·내부논리파일(ILF)·외부인터페이스파일(EIF) 5개 기능 요소로 규모를 산정한다.
④ **COCOMO의 3가지 유형** — 조직형·반분리형·**애자일형(Agile)** 3가지로 분류되며, 애자일형은 소규모 팀·짧은 반복 개발에 특화된 COCOMO 표준 유형이다.

### 문제 19 [유사 기출: 2024년 1회 5과목 — IPSec vs SSL/TLS 정형 패턴]
**IPSec(IP Security)** 및 **SSL/TLS(Secure Sockets Layer / Transport Layer Security)**에 관한 설명 중 **가장 옳지 않은 것**은?

① **IPSec** — OSI **네트워크 계층(L3)** 에서 동작하며, AH(Authentication Header, 인증·무결성)·ESP(Encapsulating Security Payload, 기밀성·무결성·인증)·IKE(Internet Key Exchange, 키 교환) 프로토콜로 구성된다.
② **IPSec**은 전송 모드(Transport Mode, 페이로드만 암호화)와 터널 모드(Tunnel Mode, 전체 IP 패킷 암호화) 2가지 동작 모드를 지원하며, VPN 구축의 표준으로 사용된다.
③ **SSL/TLS** — OSI **전송·세션 계층(L4~L7)** 에서 동작하며, HTTPS·SMTPS·IMAPS 등 응용 계층 프로토콜에 보안을 제공한다. TLS 1.3(RFC 8446)이 현재 표준이다.
④ **IPSec은 응용 계층(L7)에서 동작**하며, SSL/TLS는 **네트워크 계층(L3)**에서 동작하는 완전히 반대의 위치·구조를 가진다.

### 문제 20 [유사 기출: 2024년 2회 5과목 — 사회공학 공격(피싱·스미싱·파밍) 정형 패턴]
**사회공학(Social Engineering) 공격**에 관한 설명 중 **가장 옳지 않은 것**은?

① **피싱(Phishing)** — 정상 사이트로 위장한 가짜 사이트나 이메일로 사용자를 유인하여 개인정보·금융정보를 탈취하는 공격이다.
② **스미싱(Smishing)** — SMS + Phishing의 합성어로, **문자메시지(SMS)**를 통해 악성 URL·앱을 유포하여 개인정보를 탈취한다.
③ **파밍(Pharming)** — 사용자 PC의 hosts 파일 변조 또는 DNS 서버 변조를 통해 정상 URL을 입력해도 위조 사이트로 접속하도록 하는 공격이다.
④ **파밍(Pharming)** — 사용자가 정확한 URL을 입력했을 때 **반드시 정상 사이트로만 접속**되며, DNS 변조·hosts 파일 변조와는 무관한 물리적 회선 절단 공격이다.

**정답 ④** — **2PL은 직렬 가능성(Serializability)을 보장**하며, 락 호환성 매트릭스는 **S-S만 호환(○)** · **S-X·X-S·X-X 모두 비호환(×)**. 확장 단계(Growing)에서 락 획득만·축소 단계(Shrinking)에서 락 해제만 하며, 한번 해제 후에는 새 락 획득 불가(그래서 "2단계"). **함정 키워드**: 2PL이 직렬가능성 미보장·모든 락 동시 획득 가능이라는 정반대 함정.

### 문제 11
**정답 ④** — 데이터 웨어하우스는 **의사결정 지원(DSS)·분석·리포팅**이 주 목적이며, OLTP(실시간 트랜잭션 처리, 초당 수천 건)와는 **역할이 정반대**. Inmon 4특성 = 주제 지향·통합·시간 변형·비휘발성(비휘발성 = 갱신 없이 축적). OLAP 연산 5종 = 슬라이싱·다이싱·드릴다운·롤업·피벗. **함정 키워드**: DW를 OLTP 목적으로 뒤바꾸는 완전 오답.

### 문제 12
**정답 ④** — **CAP 정리**는 분산 시스템에서 Consistency·Availability·Partition Tolerance **3가지 중 최대 2개까지만 동시 만족 가능**하다는 것이 결론(Brewer's Theorem). 실무에서 네트워크 분할은 필연적이므로 P는 필수 선택 → CP(MongoDB·HBase) 또는 AP(Cassandra·DynamoDB) 중 선택. **함정 키워드**: CAP 3가지 모두 동시 만족 가능이라는 완전 오답.

### 문제 13
**정답 ③** — C에서 배열 이름 arr은 배열 첫 요소의 주소. `p = arr`이므로 `p+2`는 arr[2]의 주소 → `*(p+2) = arr[2] = 30`. `*(arr+3) = arr[3] = 40`. `p[4] = arr[4] = 50`. 결과 **`30 40 50`**. **함정 키워드**: 포인터 산술이 바이트 단위가 아니라 요소 단위(sizeof(int)만큼 이동).

### 문제 14
**정답 ③** — Java에서 **필드는 정적 바인딩(선언 타입 기준)·메서드는 동적 바인딩(실제 인스턴스 타입 기준·오버라이딩)**. `d.hello()` 호출 → Dog의 hello() 실행: 우선 자신 name="Dog" 출력 → `super.hello()` = Animal의 hello() 실행: `name`은 Animal 클래스의 name="Animal" 참조(super는 부모 필드 접근) → **`Hi, Dog / Hi, Animal`**. **함정 키워드**: super.hello() 내부의 name이 Dog의 name을 참조한다는 오답(super는 부모 필드).

### 문제 15
**정답 ③** — Python 리스트 컴프리헨션 `[x*x for x in nums if x%2==0]`: nums=[1,2,3,4,5,6]에서 짝수(x%2==0)만 필터링 → [2,4,6] → 각각 제곱 → **[4, 16, 36]**. 문법 = `[표현식 for 요소 in 반복가능객체 if 조건]`. **함정 키워드**: 필터 조건 없이 전체 요소 제곱([1,4,9,16,25,36])·필터만 하고 제곱 안 함([2,4,6]).

### 문제 16
**정답 ④** — **OSI 응용(7)·표현(6)·세션(5) 3개 계층 → TCP/IP 응용(Application) 계층 1개로 통합**(인터넷 계층 아님). OSI 네트워크(3) → TCP/IP 인터넷(IP·ICMP·ARP), OSI 물리(1)+데이터링크(2) → TCP/IP 네트워크 접근(Network Access, MAC·이더넷·PPP). 완전한 대응표 암기 필수. **함정 키워드**: OSI 상위 3계층이 TCP/IP 인터넷 계층으로 합쳐진다는 완전 오답.

### 문제 17
**정답 ④** — CMMi 최상위 5단계(Optimizing)는 **정량적 데이터에 기반한 지속적 개선**을 의미하며(직관 아님), CMMi와 SPICE는 각각 5단계·6단계로 성숙도 수준을 표현하되 **동등 매핑 가능**(SPICE 5 = CMMi 5). CMMi = 조직 전체 성숙도·SPICE = 프로세스별 능력 평가에 유리. **함정 키워드**: CMMi 5단계가 "직관·경험 의존"이라는 정반대 함정(초기 단계 1의 특성과 뒤바꾸기).

### 문제 18
**정답 ④** — **COCOMO의 3가지 유형은 조직형(Organic)·반분리형(Semi-Detached)·내장형(Embedded)**만 정식 분류(암기: 5만·30만 라인 임계). "애자일형"은 COCOMO 표준 유형이 아니며 오답. LOC 공식 = **(낙관치 + 4×기대치 + 비관치)/6**. FP 5요소 = EI·EO·EQ·ILF·EIF. **함정 키워드**: COCOMO 유형에 애자일·CBD·객체지향 등 신규 방법론 이름 끼워넣는 함정.

### 문제 19
**정답 ④** — **IPSec = 네트워크 계층(L3, IP 위)** · **SSL/TLS = 전송 계층(L4) 위, 응용 계층(L7) 아래**(약칭 L4~L7). 완전 뒤바꾸기 오답. IPSec 3구성 = AH(인증)·ESP(암호화+인증)·IKE(키교환), 모드 2가지 = Transport(페이로드만)·Tunnel(패킷 전체·VPN 표준). SSL/TLS = HTTPS·SMTPS 계열, TLS 1.3(RFC 8446) 최신. **함정 키워드**: IPSec을 L7·SSL을 L3로 뒤바꾸는 완전 오답.

### 문제 20
**정답 ④** — **파밍(Pharming)**은 사용자 PC의 **hosts 파일 변조** 또는 **DNS 서버 변조(DNS 캐시 포이즈닝)**로 정상 URL 입력에도 위조 사이트로 유도하는 대표적 사회공학 공격. "정확한 URL 입력 = 반드시 정상 접속"이 안 되는 것이 파밍의 위험성. 물리적 회선 절단은 사회공학이 아니라 물리적 공격. 피싱 = 이메일/사이트 유인, 스미싱 = SMS 유인, 파밍 = DNS/hosts 변조. **함정 키워드**: 파밍이 "정확한 URL은 무조건 안전"이라는 정반대 함정.

## 📊 자주 틀리는 함정 5가지 (2024년 회독 3/4 · 신규 영역 위주)

1. **응집도·결합도 방향 뒤바꾸기** — 응집도는 **강할수록 좋음**(우연 최악, 기능 최고), 결합도는 **약할수록 좋음**(내용 최악, 자료 최고). "약할수록 나쁨" 뒤바꾸기 초빈출.
2. **BST 순회 3종** — 전위(NLR: 루트→좌→우), 중위(LNR: 좌→루트→우 = **정렬 결과**), 후위(LRN: 좌→우→루트). 중위=오름차순은 BST 핵심 성질.
3. **GoF 패턴 분류** — 생성 5(Factory Method·Abstract Factory·Builder·Prototype·Singleton)·구조 7(Adapter·Bridge·Composite·Decorator·Facade·Flyweight·Proxy)·행위 11. Decorator·Adapter를 생성으로 오분류 초빈출.
4. **CAP 정리** — 3가지 중 최대 **2가지만 동시 만족 가능**(모두 만족 불가). P(분할 내성) 필수 선택이므로 실무는 CP vs AP 선택.
5. **IPSec vs SSL/TLS 계층** — IPSec **L3(네트워크)** · SSL/TLS **L4~L7(전송~응용 사이)**. 완전 뒤바꾸기 함정 초빈출.
