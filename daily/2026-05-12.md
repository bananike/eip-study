### 문제 2 [유사 기출: 정보처리기사 필기 — 자료구조 반복 출제 정형]
큐(Queue)에 대한 설명으로 옳은 것은?

③ 먼저 삽입된 데이터가 먼저 삭제되는 선입선출(FIFO, First In First Out) 구조이다.

① n − 1

### 문제 7 [유사 기출: 정보처리기사 필기 — 원형 큐/연결 리스트 반복 출제 정형]
선형 리스트(linear list, 배열 기반)와 비교한 연결 리스트(linked list)의 특징으로 옳지 않은 것은?

① 노드의 삽입·삭제 시 자료의 이동(shift) 없이 포인터(링크) 변경만으로 처리할 수 있다.
② 데이터를 저장하기 위해 별도의 링크(포인터) 영역이 추가로 필요하다.
③ 임의의 원소에 대한 직접 접근(random access)이 배열보다 빠르다.
④ 메모리 내에서 노드들이 물리적으로 떨어진 위치에 저장될 수 있다.

### 문제 8 [유사 기출: 정보처리기사 필기 — 그래프 표현(인접 행렬/리스트) 정형 패턴]
그래프(Graph)의 표현 방법에 대한 설명으로 옳지 않은 것은?

① 인접 행렬(adjacency matrix)은 정점이 n개일 때 n × n 크기의 2차원 배열로 표현한다.
② 인접 행렬은 두 정점이 연결되었는지 O(1) 시간에 확인할 수 있다.
③ 인접 리스트(adjacency list)는 간선의 수가 적은 희소 그래프(sparse graph)에서 메모리 측면에 유리하다.
④ 무방향 그래프의 인접 행렬은 항상 비대칭(asymmetric)이다.

## 🔍 오답 역추적 개념 발췌

오늘 문제에서 헷갈리기 쉬운 키워드만 발췌. 정의·설명은 사후 검증용 섹션 참조.

- 스택(Stack, LIFO) vs 큐(Queue, FIFO) vs 덱(Deque, 양방향)
- front(앞, 삭제) / rear(뒤, 삽입) — 큐 용어 헷갈림 주의
- 원형 큐(circular queue) / 선형 큐의 단점(rear 포화)
- 중위(infix) → 후위(postfix) 변환: 연산자 우선순위(* / 가 + − 보다 높음)
- 트리 용어: 차수(degree) / 단말 노드(leaf, 잎) / 깊이(depth) / 레벨(level) / 루트(root)
- 트리 전체 차수 = 임의 노드가 가진 자식 수의 최댓값
- 이진 트리 순회: 전위(preorder, 루트→좌→우) / 중위(inorder, 좌→루트→우) / 후위(postorder, 좌→우→루트)
- 무방향 그래프 최대 간선 수 = n(n−1)/2, 방향 그래프 = n(n−1)
- 인접 행렬(adjacency matrix) vs 인접 리스트(adjacency list) — 밀집/희소 그래프
- 연결 리스트는 직접 접근(random access) 불리, 순차 접근만 가능

## 💡 상세 해설 (자기 설명과 대조)

### 문제 1
정답: ② 스택(Stack)
- 제공 해설: 한쪽 끝(top)에서만 삽입(push)·삭제(pop)가 일어나고, 가장 나중에 넣은 것이 먼저 나오는 후입선출(LIFO) 구조 = 스택. 함수 호출(콜 스택), 후위표기 계산, 괄호 검사 등에 활용.
- 오답 이유:
  - ① 큐(Queue): 선입선출(FIFO) 구조. 한쪽(rear)에서 삽입, 반대쪽(front)에서 삭제.
  - ③ 덱(Deque, Double Ended Queue): 양쪽 끝에서 삽입·삭제 모두 가능. "한쪽 끝에서만"이라는 설명과 불일치.
  - ④ 그래프(Graph): 정점과 간선의 집합인 비선형 자료구조. 입출력 순서 개념이 아님.

### 문제 2
정답: ③ 선입선출(FIFO) 구조이다
- 제공 해설: 큐는 먼저 들어온 데이터가 먼저 나가는 선입선출(FIFO) 구조. 줄 서기(대기열), 버퍼, 너비 우선 탐색(BFS) 등에 사용.
- 오답 이유:
  - ① 후입선출(LIFO)은 스택의 특징. 큐는 FIFO.
  - ② front와 rear가 뒤바뀜. 삽입은 rear(뒤), 삭제는 front(앞)에서 일어난다.
  - ④ 양쪽 끝에서 삽입·삭제가 가능한 것은 덱(Deque). 큐는 한쪽에서 삽입, 반대쪽에서 삭제.

### 문제 3
정답: ① A B C * +
- 제공 해설: 중위 `A + B * C`에서 `*`가 `+`보다 우선순위가 높다. 우선순위 괄호를 명시하면 `A + (B * C)`. 후위 변환 시 연산자는 피연산자 뒤로 이동하므로 먼저 `B C *` (= B*C)를 만들고, 그 결과와 A를 `+`로 묶어 `A B C * +`.
- 오답 이유:
  - ② A B + C *: `(A+B)*C`를 후위로 표기한 것. 우선순위가 다름.
  - ③ A B C + *: `A*(B+C)`에 해당. 원식과 불일치.
  - ④ A B * C +: `(A*B)+C`에 해당. 원식과 불일치.

### 문제 4
정답: ② 단말 노드(잎 노드)의 개수는 4개이다
- 제공 해설: 단말 노드는 자식이 없는 노드. 트리에서 자식이 없는 노드는 E, F, C, G로 4개. (A는 자식 B·C·D, B는 자식 E·F, D는 자식 G를 가짐)
- 오답 이유:
  - ① 트리의 차수는 임의 노드가 가진 자식 수의 최댓값. A가 자식 3개(B,C,D)를 가지므로 트리 차수는 3이다(2 아님).
  - ③ 노드 A는 자식이 B, C, D로 3개이므로 A의 차수는 3이다(2 아님).
  - ④ 깊이(depth/높이)는 루트(레벨1 또는 0)에서 가장 깊은 단말까지의 레벨 수. A(루트)–B–E 경로로 3레벨(또는 높이 2). 루트를 레벨1로 셀 때 깊이는 3이며, "2"는 일반적 정의와 불일치. 가장 명확히 옳은 보기는 ②.

### 문제 5
정답: ③ D E B F C A
- 제공 해설: 후위 순회(postorder)는 "왼쪽 서브트리 → 오른쪽 서브트리 → 루트" 순. 루트 A의 왼쪽 서브트리(B,D,E)를 먼저 후위로 처리: B의 좌(D) → B의 우(E) → B = `D E B`. 그다음 오른쪽 서브트리(C,F): C의 좌(F) → (C의 우 없음) → C = `F C`. 마지막에 루트 A. 종합 = `D E B F C A`.
- 오답 이유:
  - ① A B D E C F: 후위가 아님(루트 A가 맨 앞). 순회 순서 불일치.
  - ② D B E A F C: 중위 순회(inorder, 좌→루트→우)의 결과 형태. 후위가 아님.
  - ④ A B C D E F: 레벨 순서(BFS) 나열에 가까움. 후위 순회 결과가 아님.

### 문제 6
정답: ③ n(n − 1) / 2
- 제공 해설: 무방향 완전 그래프는 서로 다른 두 정점을 하나의 간선으로 모두 연결한다. 정점 n개에서 두 개를 고르는 조합 수 = nC2 = n(n−1)/2가 최대 간선 수.
- 오답 이유:
  - ① n − 1: 트리(또는 신장 트리)에서 정점 n개일 때의 간선 수. 완전 그래프가 아님.
  - ② n(n − 1): 방향(directed) 완전 그래프의 최대 간선 수. 양방향을 각각 세므로 무방향의 2배.
  - ④ n²: 자기 자신으로의 간선(self-loop)까지 포함한 과대 계산. 단순 그래프에서는 성립하지 않음.

### 문제 7
정답: ③ (옳지 않은 것)
- 제공 해설: 연결 리스트는 원하는 원소까지 head부터 링크를 따라 순차 접근(sequential access)해야 한다. 따라서 임의 원소의 직접 접근은 배열보다 느리다(배열은 인덱스로 O(1) 접근). ③의 진술이 거짓.
- 오답 이유 (옳은 보기들):
  - ① 삽입·삭제 시 자료 이동 없이 링크만 변경 — 옳음(연결 리스트의 장점).
  - ② 링크(포인터) 저장을 위한 추가 메모리 필요 — 옳음(단점).
  - ④ 노드가 물리적으로 떨어진 위치에 저장 가능 — 옳음(동적 할당 특성).

### 문제 8
정답: ④ (옳지 않은 것)
- 제공 해설: 무방향 그래프에서 정점 i–j가 연결되면 j–i도 연결된 것이므로 인접 행렬은 주대각선을 기준으로 대칭(symmetric)이다. "항상 비대칭"이라는 ④는 거짓.
- 오답 이유 (옳은 보기들):
  - ① 인접 행렬은 n × n 2차원 배열로 표현 — 옳음.
  - ② 행렬은 두 정점의 연결 여부를 배열 인덱스 접근으로 O(1)에 확인 — 옳음.
  - ③ 인접 리스트는 실제 존재하는 간선만 저장하므로 희소 그래프에서 메모리 효율이 좋음 — 옳음.

## 📚 역추적 참조 (사후 검증용 — 학습 자료 아님)

### 핵심 용어
| 용어 | 정의 |
|---|---|
| 스택(Stack) | 한쪽 끝(top)에서만 삽입·삭제. 후입선출(LIFO). |
| 큐(Queue) | rear에서 삽입, front에서 삭제. 선입선출(FIFO). |
| 덱(Deque, Double Ended Queue) | 양쪽 끝 모두에서 삽입·삭제 가능. |
| 원형 큐(Circular Queue) | 선형 큐의 rear 포화 문제를 해결하기 위해 배열을 원형으로 연결한 큐. |
| 후위 표기법(Postfix/Postorder notation) | 연산자를 피연산자 뒤에 두는 표기. 스택으로 계산. |
| 차수(Degree, 노드) | 한 노드가 가진 자식 노드의 수. |
| 차수(Degree, 트리) | 트리 내 모든 노드의 차수 중 최댓값. |
| 단말 노드(Leaf, 잎 노드) | 자식이 없는 노드. |
| 깊이/높이(Depth/Height) | 루트에서 가장 깊은 단말 노드까지의 레벨 수. |
| 전위 순회(Preorder) | 루트 → 왼쪽 → 오른쪽. |
| 중위 순회(Inorder) | 왼쪽 → 루트 → 오른쪽. |
| 후위 순회(Postorder) | 왼쪽 → 오른쪽 → 루트. |
| 인접 행렬(Adjacency Matrix) | n×n 2차원 배열로 간선 표현. 밀집 그래프 유리. |
| 인접 리스트(Adjacency List) | 각 정점의 인접 정점을 연결 리스트로 표현. 희소 그래프 유리. |

### 핵심 개념 (사후 검증용)

**선형 자료구조 입출력 방식**
- 스택(Stack): LIFO — push/pop, top 한 곳. 콜 스택, 후위표기 계산, 괄호 검사, DFS.
- 큐(Queue): FIFO — rear 삽입(enqueue), front 삭제(dequeue). 버퍼, BFS, 작업 대기열.
- 덱(Deque): 양쪽 끝 삽입·삭제. 스택+큐 일반화.
- 원형 큐: 선형 큐가 rear가 끝에 닿으면 앞이 비어도 못 쓰는 문제를 해결.

**중위 → 후위 변환 우선순위**
- 연산자 우선순위: 괄호 > `*` `/` > `+` `−`
- `A + B * C` → `A + (B*C)` → `A B C * +`
- 변환은 스택을 이용(연산자 스택)하거나 우선순위 괄호 묶기로 손계산.

**문제 4 트리 분석 (자식 수 기준)**
- A 자식: B, C, D (차수 3) / B 자식: E, F (차수 2) / D 자식: G (차수 1)
- 단말 노드: E, F, C, G (4개)
- 트리의 차수 = 최댓값 3 (A의 차수)
- 깊이: A(1) – B(2) – E(3) → 3레벨

**이진 트리 순회 (문제 5 트리: A(좌B 우C), B(좌D 우E), C(좌F))**
- 전위(preorder): A B D E C F
- 중위(inorder): D B E A F C
- 후위(postorder): D E B F C A

**그래프 간선 수 공식 (정점 n, 완전 그래프)**
| 종류 | 최대 간선 수 |
|---|---|
| 무방향(undirected) | n(n−1)/2 |
| 방향(directed) | n(n−1) |
- 참고: 트리(연결, 사이클 없음)의 간선 수 = n − 1

**그래프 표현 비교**
| 항목 | 인접 행렬 | 인접 리스트 |
|---|---|---|
| 공간 | O(n²) | O(n + e) |
| 연결 확인 | O(1) | O(차수) |
| 유리한 그래프 | 밀집(dense) | 희소(sparse) |
| 무방향 대칭성 | 대칭(symmetric) | — |

**배열 vs 연결 리스트**
| 항목 | 배열(선형 리스트) | 연결 리스트 |
|---|---|---|
| 직접 접근 | O(1), 빠름 | O(n), 순차 접근만 |
| 삽입·삭제 | 자료 이동 필요(느림) | 링크만 변경(빠름) |
| 추가 메모리 | 불필요 | 링크(포인터) 필요 |
| 물리 저장 | 연속 | 비연속 가능 |
