## 🎯 오늘의 기출문제 — 2025년 정처기 필기 20문제 (5과목 균등 · 회독 3/4 신규 비중복)

### 문제 1 [유사 기출: 2025년 1회 1과목 — 서비스 메시(Service Mesh · Istio · Linkerd) 정형 패턴]
**서비스 메시(Service Mesh)**에 관한 설명 중 **가장 옳지 않은 것**은?

① **정의** — 마이크로서비스 간 통신을 처리하는 전용 인프라 계층으로, 트래픽 라우팅·관측성(Observability)·보안(mTLS)·복원성(재시도·서킷 브레이커)을 애플리케이션 코드 수정 없이 인프라 레벨에서 제공한다.
② **대표 구현** — **Istio**(Envoy 프록시 기반 · Control Plane = istiod)·**Linkerd**(Rust 기반 초경량)·**Consul Connect** 등이 있으며, 오픈소스 CNCF 프로젝트로 성장했다.
③ **아키텍처** — 각 마이크로서비스 파드에 **사이드카(Sidecar) 프록시**(Envoy 등)를 함께 배치하여 애플리케이션 트래픽을 모두 프록시가 가로채 처리하며, 컨트롤 플레인이 프록시 설정을 중앙 관리한다.
④ **서비스 메시는 애플리케이션 코드에 통신 로직(재시도·서킷 브레이커·mTLS)을 직접 삽입해야 동작**하며, **사이드카 프록시 없이 SDK만으로 구현되고**, **관측성·트래픽 제어는 서비스 메시의 관심사가 아니다**.

### 문제 2 [유사 기출: 2025년 2회 1과목 — 사이드카(Sidecar) 패턴 정형 패턴]
**사이드카(Sidecar) 패턴**에 관한 설명 중 **가장 옳지 않은 것**은?

① 주 애플리케이션 컨테이너 옆에 **보조 컨테이너(Sidecar)** 를 함께 배포하여 로깅·프록시·설정 동기화·보안 등 **횡단 관심사(Cross-cutting Concern)** 를 분리·전담시킨다.
② Kubernetes에서는 **동일 Pod 내 여러 컨테이너**가 네트워크·볼륨을 공유하므로 사이드카가 자연스럽게 구현된다(로컬호스트 통신·공유 볼륨 활용).
③ 대표 사례로 **Envoy 프록시**(서비스 메시)·**Fluentd/Fluent Bit**(로그 수집)·**Vault Agent**(시크릿 주입)·**init 컨테이너 후 실행되는 헬퍼**가 있으며, 언어·프레임워크에 독립적이다.
④ 사이드카 패턴은 **주 애플리케이션 소스에 횡단 관심사를 하드코딩**하는 방식이며, **별도 컨테이너 없이 하나의 프로세스로 모든 관심사를 통합 처리**하는 것이 원칙이다.

### 문제 3 [유사 기출: 2025년 3회 1과목 — 플랫폼 엔지니어링 · IDP · Backstage 정형 패턴]
**플랫폼 엔지니어링(Platform Engineering)·내부 개발자 플랫폼(IDP: Internal Developer Platform)·백스테이지(Backstage)**에 관한 설명 중 **가장 옳지 않은 것**은?

① **플랫폼 엔지니어링** — 개발팀의 인지 부하(Cognitive Load)를 줄이고 **셀프서비스(Self-Service)** 로 인프라·CI/CD·관측성·보안을 제공하기 위해 **내부 개발자 플랫폼(IDP)** 을 구축·운영하는 실천 분야이다.
② **골든 패스(Golden Path)** — 조직 표준을 반영한 **권장 개발·배포 경로**로, 신규 서비스 생성 시 템플릿(Boilerplate·Cookiecutter·Backstage Software Template)을 통해 몇 분 안에 CI/CD·관측성이 구성된 저장소를 만들 수 있다.
③ **Backstage** — Spotify가 만든 오픈소스 IDP 프레임워크로, 서비스 카탈로그(Catalog)·소프트웨어 템플릿(Templates)·TechDocs·플러그인 생태계를 통해 개발자 포털을 구축한다.
④ **플랫폼 엔지니어링은 각 개발팀이 인프라·CI/CD·관측성을 개별적으로 자체 구축·운영하는 것을 지향**하며, **골든 패스·셀프서비스·IDP는 안티패턴**으로 취급되고, **Backstage는 모니터링 전용 도구**이다.

### 문제 4 [유사 기출: 2025년 1회 1과목 — 서비스 메시 트래픽 제어(카나리·A/B) 정형 패턴]
**서비스 메시(Istio 등)를 활용한 트래픽 관리 기법**에 관한 설명 중 **가장 옳지 않은 것**은?

① **카나리 배포(Canary Deployment)** — 신버전에 소량 트래픽(예: 5%)을 우선 전달하고 지표(에러율·지연·비즈니스 KPI)를 검증한 뒤 점진적으로 비율을 확대(5%→25%→50%→100%)한다.
② **블루/그린 배포(Blue-Green Deployment)** — 구버전(Blue)과 신버전(Green)을 동시에 운영하다가 **라우팅을 한 번에 스위칭**하여 즉시 롤백이 가능하도록 한다.
③ **A/B 테스트 · 트래픽 미러링** — 사용자 세그먼트(헤더·쿠키·지역)별로 다른 버전을 노출하거나, 프로덕션 트래픽을 **테스트 환경에 미러링(Shadowing)** 하여 실제 부하로 검증한다.
④ **서비스 메시는 오직 100% 즉시 배포(Big-Bang Deploy)만 지원**하며, **카나리·블루/그린·미러링은 서비스 메시로는 구현할 수 없고 별도 로드 밸런서 하드웨어가 반드시 필요**하다.

### 문제 5 [유사 기출: 2025년 2회 2과목 — 병렬 정렬(Parallel Merge Sort · Sample Sort) 정형 패턴]
**병렬 정렬 알고리즘(Parallel Sorting)**에 관한 설명 중 **가장 옳지 않은 것**은?

① **병렬 병합 정렬(Parallel Merge Sort)** — 분할 정복 특성상 병렬화가 용이하며, 각 부분 배열을 서로 다른 스레드에서 독립 정렬한 후 병합 단계를 병렬로 수행한다.
② **Sample Sort** — 입력에서 표본(Sample)을 추출해 피벗을 여러 개 선택하고 각 파티션을 병렬 프로세서에 분배하여 정렬하는 **분산·병렬 환경 친화적 정렬**이다.
③ **비교 기반 정렬의 하한**은 **Ω(n log n)** 이며, 병렬화로 **wall-clock 시간**은 단축할 수 있으나 **총 작업량(work)** 은 이 하한 아래로 내려가지 않는다.
④ **병렬 정렬은 코어 수만 늘리면 시간 복잡도 하한 Ω(n log n) 자체를 O(1)까지 낮출 수 있어**, **입력 크기에 무관하게 상수 시간 정렬이 이론적으로 성립**한다.

### 문제 6 [유사 기출: 2025년 3회 2과목 — 근사 알고리즘 · 확률 알고리즘(Monte Carlo · Las Vegas) 정형 패턴]
**근사 알고리즘(Approximation Algorithm)·확률 알고리즘(Randomized Algorithm)**에 관한 설명 중 **가장 옳지 않은 것**은?

① **근사 알고리즘** — NP-Hard 문제(TSP·배낭·정점 커버 등)에 대해 **최적해 대신 근사 비율(Approximation Ratio) 보장**을 가진 다항 시간 해를 제공하며, 2-근사·PTAS·FPTAS 등이 대표적이다.
② **Monte Carlo 알고리즘** — 항상 다항 시간 내 결과를 반환하지만 **결과가 확률적으로 오답일 수 있다**(예: Miller-Rabin 소수 판정). 반복으로 오류 확률을 지수적으로 낮출 수 있다.
③ **Las Vegas 알고리즘** — 결과는 **항상 정답**이나 **실행 시간이 확률적으로 변동**된다(예: Randomized QuickSort). 기대 실행 시간이 성능 지표가 된다.
④ **근사 알고리즘은 NP-Hard 문제를 항상 다항 시간에 정확한 최적해로 해결**하고, **Monte Carlo는 항상 정답을 보장**하며, **Las Vegas는 오답을 확률적으로 반환**한다.

### 문제 7 [유사 기출: 2025년 1회 2과목 — 계약 기반 테스트(Consumer-Driven Contract · Pact) 정형 패턴]
**계약 기반 테스트(Contract Testing) · 소비자 주도 계약(CDC · Consumer-Driven Contract · Pact)**에 관한 설명 중 **가장 옳지 않은 것**은?

① **CDC 개념** — 소비자(Consumer)가 제공자(Provider) API에 기대하는 **인터페이스 계약(요청·응답 스키마)** 을 **계약 파일(Contract)** 로 명시하고, 양측이 이 계약을 만족하는지 검증한다.
② **Pact 등 도구** — Consumer 테스트 단계에서 **Pact 파일**(JSON 계약)을 생성해 브로커(Pact Broker)에 게시하고, Provider는 이를 다운로드해 자신의 API가 계약을 만족하는지 검증하는 파이프라인을 구성한다.
③ **강점** — **엔드투엔드(E2E) 통합 테스트 대비 빠르고 안정적**이며, 소비자·제공자를 **독립 배포**하면서도 API 호환성 회귀를 조기 탐지할 수 있다(MSA·독립 배포 파이프라인 전제).
④ **CDC 계약 테스트는 각 서비스 간 완전한 E2E 실 트래픽만으로 검증**하며, **계약 파일·Pact 브로커 없이 소비자·제공자가 직접 모든 종단 시나리오를 항상 함께 배포해야만** 유효하다.

### 문제 8 [유사 기출: 2025년 2회 2과목 — 카오스 엔지니어링(Chaos Engineering · Netflix Chaos Monkey) 정형 패턴]
**카오스 엔지니어링(Chaos Engineering)**에 관한 설명 중 **가장 옳지 않은 것**은?

① **정의** — 프로덕션(또는 유사 환경)에 **의도적 장애(네트워크 지연·인스턴스 종료·DNS 실패 등)** 를 주입하여 시스템의 **복원력(Resilience)** 에 대한 **가설을 실험**하는 실천 분야이다.
② **Netflix Chaos Monkey · Simian Army** — 무작위로 EC2 인스턴스를 종료시켜 팀이 **장애 내성 설계**를 상시 유지하도록 강제한다. Gremlin·Litmus·Chaos Mesh 등 후속 도구가 등장했다.
③ **원칙** — 정상 상태(Steady State) 정의 → 가설 수립 → 최소 폭발 반경(Blast Radius) 실험 → 결과 검증 → 폭발 반경 확장. **관측성(Observability)** 이 전제되어야 안전한 실행이 가능하다.
④ **카오스 엔지니어링은 프로덕션에 예고 없이 대규모 장애를 무차별 주입하여 최대한 많은 인스턴스를 파괴하는 것이 목표**이며, **가설·정상 상태·폭발 반경 통제·관측성은 원칙과 무관**하다.

### 문제 9 [유사 기출: 2025년 3회 3과목 — 벡터 DB(Vector Database · 임베딩 유사도) 정형 패턴]
**벡터 데이터베이스(Vector Database · Pinecone · Weaviate · Milvus · pgvector)**에 관한 설명 중 **가장 옳지 않은 것**은?

① **정의** — 임베딩(Embedding) 벡터를 저장·인덱싱·유사도 검색(Similarity Search)에 특화된 데이터베이스로, RAG(Retrieval-Augmented Generation)·의미 검색·추천 시스템의 핵심 컴포넌트이다.
② **유사도 척도** — **코사인 유사도(Cosine)** · **L2 거리(유클리드)** · **내적(Dot Product)** 이 대표적이며, 임베딩 모델과 정합해 선택한다.
③ **인덱스** — **ANN(Approximate Nearest Neighbor)** 알고리즘으로 **HNSW(Hierarchical Navigable Small World)** · **IVF(Inverted File Index)** · **PQ(Product Quantization)** 등을 사용해 대규모 벡터에서도 빠른 근사 최근접 이웃 검색을 제공한다.
④ **벡터 DB는 벡터가 아닌 정형 관계형 테이블 검색 전용**이며, **유사도 검색·ANN·임베딩과는 무관**하고, **RAG·추천 시스템에서는 사용할 수 없다**.

### 문제 10 [유사 기출: 2025년 1회 3과목 — 시계열 DB(TSDB · InfluxDB · TimescaleDB) 정형 패턴]
**시계열 데이터베이스(Time-Series Database · TSDB)**에 관한 설명 중 **가장 옳지 않은 것**은?

① 시간(Timestamp)이 1급 컬럼인 데이터에 특화된 DB로, **높은 쓰기 처리량·시간 범위 쿼리·다운샘플링(Downsampling)·리텐션 정책·연속 집계(Continuous Aggregate)** 를 지원한다.
② **대표 제품** — **InfluxDB**(전용 스토리지·Flux/InfluxQL)·**TimescaleDB**(PostgreSQL 확장·하이퍼테이블)·**Prometheus**(모니터링 특화·PromQL)·**QuestDB**·**VictoriaMetrics** 등이 있다.
③ 활용 영역 — **IoT 센서 데이터**·**애플리케이션·인프라 메트릭**·**금융 틱 데이터**·**로그·감사 이벤트** 등 시간 축이 지배적인 워크로드에 적합하다.
④ **TSDB는 시간 축이 없는 정적 데이터 저장 전용**이며, **연속 집계·다운샘플링·시간 범위 쿼리·높은 쓰기 처리량 최적화는 TSDB의 관심사가 아니다**.

### 문제 11 [유사 기출: 2025년 2회 3과목 — 그래프 DB · CDC · 데이터 메시 정형 패턴]
**그래프 DB(Neo4j · Cypher)·CDC(Change Data Capture · Debezium)·데이터 메시(Data Mesh)**에 관한 설명 중 **가장 옳지 않은 것**은?

① **그래프 DB** — 노드(Node)·엣지(Edge)·프로퍼티(Property) 모델로 관계(Relationship)를 1급 시민으로 취급하며, **Neo4j Cypher** · 표준 **GQL(ISO/IEC 39075)** 로 질의한다. 소셜·추천·사기 탐지에 강점.
② **CDC** — 소스 DB의 **트랜잭션 로그(binlog · WAL · redo)** 를 읽어 변경 이벤트를 스트림(Kafka 등)으로 전달한다. **Debezium**(Kafka Connect 커넥터)·**Oracle GoldenGate**·**AWS DMS** 가 대표적이며, 배치 ETL 대비 지연이 극히 낮다.
③ **데이터 메시(Data Mesh)** — Zhamak Dehghani가 제안한 4대 원칙 = **도메인 소유(Domain Ownership)** · **데이터를 프로덕트로(Data as a Product)** · **셀프서비스 데이터 인프라** · **연합 컴퓨테이셔널 거버넌스(Federated Computational Governance)**.
④ **그래프 DB는 관계 표현이 불가능하며 Cypher/GQL은 문법 오류**이고, **CDC는 반드시 전체 테이블 야간 스냅샷 배치로만 동작**하며, **데이터 메시는 단일 중앙 집중식 모놀리식 데이터 웨어하우스만 인정**한다.

### 문제 12 [유사 기출: 2025년 3회 3과목 — 벡터 DB vs 시계열 DB vs 그래프 DB 특성 비교 정형 패턴]
**다중 모델 데이터베이스(Multi-Model)·특수 목적 DB의 대응 매핑**으로 **가장 옳지 않은 것**은?

① 임베딩 유사도 검색(RAG·추천) → **벡터 DB**(Pinecone · pgvector · Milvus · HNSW/IVF/PQ 인덱스).
② 시간 축이 지배적인 IoT·메트릭·틱 데이터 → **시계열 DB**(InfluxDB · TimescaleDB · Prometheus · 다운샘플링·리텐션).
③ 관계·경로 탐색(6촌·최단 경로·커뮤니티) → **그래프 DB**(Neo4j Cypher · 표준 GQL · AllegroGraph).
④ 임베딩 유사도 검색은 **정통 RDBMS의 정수 기본키 인덱스로 대체**하고, 시계열 데이터는 **관계형 튜플의 자연 순서**로 충분하며, 그래프 관계 탐색은 **다중 JOIN이 그래프 DB보다 항상 더 빠르므로** 특수 DB는 불필요하다.

### 문제 14 [유사 기출: 2025년 2회 4과목 — Java 리액티브(Reactor · Mono · Flux · Backpressure) 정형 패턴]
**Java 리액티브 프로그래밍(Project Reactor · Mono · Flux · Spring WebFlux)**에 관한 설명 중 **가장 옳지 않은 것**은?

① **Mono<T>** 는 0 또는 1개의 요소를, **Flux<T>** 는 0..N개의 요소를 비동기적으로 방출하는 리액티브 스트림 발행자(Publisher)이며, 구독(subscribe) 전까지는 아무 일도 일어나지 않는다(Cold vs Hot 스트림 개념 존재).
② **연산자(Operator)** — `map`·`filter`·`flatMap`·`zip`·`merge`·`retry`·`onErrorResume`·`timeout` 등을 체이닝하여 비동기 파이프라인을 선언적으로 구성하며, **불변 스트림 변환**이 원칙이다.
③ **Backpressure(배압)** — 구독자(Consumer)가 처리 가능한 속도로 발행자(Publisher)에게 요청량을 신호(request(n))하여 **소비자 과부하를 방지**한다. `onBackpressureBuffer`·`onBackpressureDrop`·`onBackpressureLatest` 전략이 있다.
④ **Reactor의 Flux/Mono는 반드시 즉시 동기적으로 실행**되며, **subscribe 없이도 값이 자동으로 방출**되고, **Backpressure는 리액티브 스트림 명세와 무관한 개념**이다.

### 문제 16 [유사 기출: 2025년 1회 4과목 — HTTP/2 서버 푸시 폐기 · gRPC-Web · 최신 웹 프로토콜 정형 패턴]
**HTTP/2 서버 푸시 · gRPC-Web · 최신 웹 프로토콜 사실관계**에 관한 설명 중 **가장 옳지 않은 것**은?

① **HTTP/2 서버 푸시(Server Push)** — 초기에는 성능 최적화 기능으로 도입되었으나, **캐시 히트율·복잡도 문제로 브라우저에서 점진 폐기**되어 **Chrome 106(2022년)부터 기본 비활성/제거**되었고, **Early Hints(103)** 가 사실상 대안으로 자리 잡았다.
② **gRPC-Web** — 브라우저는 HTTP/2 트레일러·바이너리 프레임을 직접 다룰 수 없어, **Envoy 등 프록시가 gRPC ↔ gRPC-Web 프로토콜 변환**을 수행한다. 브라우저에서 gRPC 서비스를 호출할 때 사실상 표준적으로 사용된다.
③ **HTTP/3(QUIC)** — UDP 기반으로 **연결 확립 시 0-RTT/1-RTT**, **커넥션 마이그레이션**, **HoL(Head-of-Line) Blocking 완화** 등의 이점을 제공하며 주요 브라우저·CDN이 지원한다.
④ **HTTP/2 서버 푸시는 여전히 모든 브라우저에서 완전 지원되는 표준 최적화 기법**이고, **gRPC-Web은 별도 프록시 없이 브라우저에서 원시 gRPC를 직접 호출**하며, **HTTP/3는 TCP 기반**이라 QUIC과 무관하다.

### 문제 17 [유사 기출: 2025년 2회 5과목 — OWASP Top 10 2021 정형 패턴]
**OWASP Top 10 2021**의 카테고리 매핑으로 **가장 옳지 않은 것**은?

① **A01: Broken Access Control(접근 통제 실패)** — IDOR·권한 우회·수직/수평 권한 상승 등이 포함되며 2021년 개정판에서 **1위**로 승격되었다.
② **A02: Cryptographic Failures(암호화 실패)** — 이전 "Sensitive Data Exposure"의 확장 이름으로, 취약한 알고리즘·평문 저장·약한 키·부적절한 TLS 설정 등을 포괄한다.
③ **A03: Injection** — SQL·NoSQL·OS Command·LDAP 인젝션 및 **XSS도 2021년부터 Injection에 통합**되어 관리된다. 파라미터화 쿼리·컨텍스트 인코딩이 대표 대응책이다.
④ **OWASP Top 10 2021의 A10 항목은 "물리 보안 실패(Physical Security Failure)"** 이며, **SSRF(Server-Side Request Forgery)는 Top 10에 포함된 적이 없다**.

### 문제 18 [유사 기출: 2025년 3회 5과목 — OWASP API Security Top 10 정형 패턴]
**OWASP API Security Top 10**에 관한 설명 중 **가장 옳지 않은 것**은?

① **BOLA(Broken Object Level Authorization)** — REST 경로의 `id` 파라미터를 조작하여 **타 사용자의 리소스에 접근**하는 취약점으로, API Top 10에서 최상위권에 위치한다.
② **Broken Authentication** — 토큰 재사용·약한 JWT 서명 알고리즘(예: `alg: none`)·리프레시 토큰 회수 미비 등이 포함된다.
③ **Excessive Data Exposure / Broken Object Property Level Authorization** — API가 필요 이상의 필드(사용자 이메일·전화·해시된 비밀번호 등)를 응답 본문에 노출하는 문제로, DTO·응답 스키마 화이트리스트로 대응한다.
④ **API 보안에서는 BOLA·인증·데이터 노출은 관심사가 아니며**, **API에는 HTTPS만 적용하면 모든 취약점이 자동으로 해결**되어 별도 대응이 필요 없다.

### 문제 19 [유사 기출: 2025년 1회 5과목 — 컨테이너 CIS 벤치마크 · CSPM · CIEM 정형 패턴]
**컨테이너 CIS 벤치마크 · CSPM(Cloud Security Posture Management) · CIEM(Cloud Infrastructure Entitlement Management)**에 관한 설명 중 **가장 옳지 않은 것**은?

① **CIS 벤치마크** — Center for Internet Security가 발행하는 **보안 설정 표준**으로, **Docker CIS**·**Kubernetes CIS**·**클라우드 CIS** 등이 존재하며 **kube-bench**·**Docker Bench for Security** 등 자동 점검 도구가 있다.
② **CSPM** — **클라우드 리소스 설정 오류(퍼블릭 S3 버킷·공개 보안 그룹·평문 스토리지·MFA 미적용 등)** 를 지속 스캔·시정 권고하는 도구 카테고리로, Prisma Cloud·Wiz·Orca 등이 대표적이다.
③ **CIEM** — **클라우드 자격(권한·역할)** 을 관측·정리해 **과잉 권한(Over-Privileged Identities)** 을 **최소 권한(Least Privilege)** 으로 회수하는 도구 카테고리다. IAM 정책 최적화가 핵심.
④ **CIS 벤치마크는 보안 표준이 아닌 임의의 마케팅 자료**이며, **CSPM은 설정 오류 대신 애플리케이션 로직만 검사**하고, **CIEM은 권한 관리와 무관하며 오히려 권한을 최대화하는 도구**이다.

### 문제 20 [유사 기출: 2025년 2회 5과목 — 클라우드 보안 3계층(CSPM · CWPP · CIEM · CNAPP) 종합 정형 패턴]
**클라우드 네이티브 보안 도구 카테고리**에 관한 설명 중 **가장 옳지 않은 것**은?

① **CSPM(Cloud Security Posture Management)** — 클라우드 설정 오류 탐지·규정 준수(PCI-DSS·ISO 27001·CIS) 지속 점검.
② **CWPP(Cloud Workload Protection Platform)** — VM·컨테이너·서버리스 등 **워크로드 런타임 보호**(취약점 스캔·런타임 이상 탐지·EDR 확장).
③ **CNAPP(Cloud Native Application Protection Platform)** — CSPM + CWPP + CIEM + IaC 스캔 + 컨테이너 이미지 스캔을 **통합 제공**하는 차세대 카테고리로, 좌시프트(Shift Left)·우측 런타임을 함께 다룬다.
④ **CSPM·CWPP·CIEM·CNAPP는 모두 동일한 온프레미스 방화벽 어플라이언스**를 지칭하는 이름이며, **클라우드·컨테이너·IaC 보안은 이 도구들과 무관**하고, **워크로드 런타임 보호·설정 오류 탐지·과잉 권한 회수는 각기 별개의 관심사가 아니라 단일 스위치로 통합 처리**된다.

### 문제 13
**정답 ①** — **x86-64 GCC 기본 정렬** 규칙 적용.
- `struct A { char a; int b; char c; }`:
  - `char a` (1B, offset 0) → `int b` 정렬 4B 경계 → **3B padding** → `int b` (4B, offset 4~7) → `char c` (1B, offset 8) → 구조체 전체 정렬 4B 경계 → **3B padding** → **총 12B**.
- `struct B { char a; char c; int b; }`:
  - `char a` (1B, offset 0) → `char c` (1B, offset 1) → `int b` 정렬 4B 경계 → **2B padding** → `int b` (4B, offset 4~7) → **총 8B**.
- 따라서 출력 = **`12 8`** → **①**. 구조체 필드 재배치(정렬 큰 순 → 작은 순)로 패딩을 최소화하는 것이 실무 기법. **함정 키워드**: "필드 개수만 세서 6·6·9" 오답은 alignment/padding 규칙 무시.

### 문제 15
**정답 ①** — **Pydantic v1/v2 기본 동작**:
- `User(id="42", name="Alice")`: `"42"` 문자열을 **int로 강제 변환(coerce)** → `u1.id = 42`(int), `is_active` 기본값 `True`. 출력: `42 int True`.

## 📊 자주 틀리는 함정 5가지 (2025년 회독 3/4 · 서비스 메시·알고리즘·특수DB·최신언어·클라우드 보안 영역 위주)

1. **서비스 메시·사이드카·플랫폼 엔지니어링 · IDP · 트래픽 제어** — 서비스 메시 = **사이드카 프록시(Envoy) + 컨트롤 플레인**으로 애플리케이션 코드 수정 없이 인프라 레벨에서 통신·mTLS·관측성·복원성·카나리·블루/그린·미러링 제공. 플랫폼 엔지니어링 = **개발자 인지 부하 감소 · IDP · 셀프서비스 · Backstage 골든 패스**. "SDK로 코드 삽입·주 애플리케이션 하드코딩·각 팀 개별 구축·빅뱅 배포만·별도 하드웨어 필요"는 각 개념이 명시적으로 해결하려는 문제를 정면 부정하는 초빈출 함정.
2. **병렬 정렬 하한 · Monte Carlo vs Las Vegas · 근사 알고리즘 정의** — 비교 기반 정렬 총 작업량 하한 **Ω(n log n)** 은 병렬화로도 낮출 수 없다(낮출 수 있는 것은 span). **Monte Carlo = 시간 고정 · 결과 확률적 오답**, **Las Vegas = 결과 정답 · 시간 확률 변동**, 근사 알고리즘 = **근사 비율 보장(최적 X)**. "코어 수로 O(1)·Monte Carlo=항상 정답·Las Vegas=오답 반환·근사=항상 최적"은 세 개념 3중 뒤바꾸기 초빈출 함정.
3. **CDC 계약 테스트 · 카오스 엔지니어링 원칙** — CDC(Consumer-Driven Contract) = **E2E 없이 계약 파일(Pact)로 소비자·제공자 독립 검증**. 카오스 엔지니어링 = **가설 · 정상 상태 · 최소 폭발 반경 · 관측성 전제**. "E2E만·계약 없음·항상 함께 배포"와 "예고 없이 무차별 파괴·가설/관측성 무관"은 각각의 방법론 정반대.
4. **벡터 DB · 시계열 DB · 그래프 DB · CDC · 데이터 메시** — 벡터 DB = **임베딩 유사도 · ANN(HNSW/IVF/PQ) · RAG 핵심**. TSDB = **시간 축 특화 · 다운샘플링 · 연속 집계 · 리텐션**. 그래프 DB = **관계 1급 시민 · Cypher/GQL**. CDC = **트랜잭션 로그 저지연 스트리밍**(야간 배치 X). 데이터 메시 = **도메인 소유 · 데이터 프로덕트 · 연합 거버넌스**. "정형 테이블 전용·시간 축 없는 정적 데이터·관계 표현 불가·야간 배치만·중앙 집중 DW만 인정"은 각 특수 DB·데이터 아키텍처의 존재 이유 정반대.
5. **최신 언어·프로토콜·클라우드 보안(OWASP 2021 · API 보안 · CSPM/CIEM/CNAPP)** — HTTP/2 서버 푸시 **Chrome 106에서 제거** · gRPC-Web은 **Envoy 프록시 필요** · HTTP/3 = **UDP/QUIC**. Reactor Mono/Flux = **Cold·subscribe 필요·Backpressure는 리액티브 명세 핵심**. Pydantic = **문자열 → int 강제 변환**·검증 실패 시 `ValidationError`. OWASP 2021 **A10 = SSRF**. API 보안 = **BOLA·인증·데이터 노출**(HTTPS만으로 해결 X). CSPM = 설정 오류 · CWPP = 워크로드 런타임 · CIEM = 과잉 권한 회수 · CNAPP = 통합. "서버 푸시 완전 지원·gRPC-Web 프록시 불필요·HTTP/3=TCP·Reactor 즉시 동기·Pydantic 그대로 통과·A10=물리 보안·HTTPS 만능·CIEM=권한 최대화"는 각 최신 개념의 사실관계를 통째로 뒤바꾸는 초빈출 함정.
