## 🎯 오늘의 기출 8문제 (Day 25 2회독 — 분류·매칭·시나리오 각도)

### 문제 1 [유사 기출: 정처기 필기 2020년 4회·2022년 2회·2024년 2회 반복 — 버퍼오버플로우 종류 정형 패턴]
다음 중 **버퍼 오버플로우(Buffer Overflow) 공격의 종류**에 대한 설명으로 옳지 **않은** 것은?

① **스택 버퍼 오버플로우(Stack BOF)** — 함수의 **지역변수 버퍼** 크기를 초과하는 데이터를 입력해 **복귀 주소(Return Address)** 와 SFP를 덮어쓰고, 공격자가 지정한 주소로 제어 흐름을 변경하는 공격이다.
② **힙 버퍼 오버플로우(Heap BOF)** — `malloc`/`new`로 할당된 **힙 영역의 청크 메타데이터(이전·다음 포인터)** 를 덮어써 `free()` 호출 시 임의 쓰기를 유발하는 공격이다.
③ **포맷 스트링 공격(Format String Attack)** — `printf(user_input)`처럼 **포맷 지정자(`%n`, `%x`, `%s`)** 의 통제권을 외부 입력에 넘긴 코드를 악용해 메모리 누설 또는 임의 쓰기를 일으키는 공격이다.
④ **정수 오버플로우(Integer Overflow)** — **CPU 캐시 라인 경계**를 넘어선 산술 연산으로 **CPU 파이프라인이 중단**되는 하드웨어 결함이며, 버퍼 오버플로우와 무관한 별개의 공격이다.

### 문제 2 [유사 기출: 정처기 필기 2021년 2회·2023년 1회·2025년 1회 반복 — 메모리 공격 대응 기술 정형 패턴]
다음 중 **버퍼 오버플로우 공격에 대한 대응 기술**에 대한 설명으로 옳지 **않은** 것은?

① **ASLR(Address Space Layout Randomization)** — **스택·힙·공유 라이브러리** 등 메모리 영역의 시작 주소를 **매 실행마다 무작위화**하여 공격자가 가젯·쉘코드의 주소를 예측하지 못하게 한다.
② **DEP / NX bit(Data Execution Prevention / No-eXecute)** — **스택·힙 등 데이터 영역의 실행 권한을 제거**하여, 해당 영역에 적재된 쉘코드가 CPU에서 실행되지 않도록 차단한다.
③ **Stack Canary(Stack Guard)** — 함수 진입 시 **복귀 주소 앞에 임의의 보호값(Canary)** 을 삽입하고, 함수 종료 시 그 값이 변조되었는지를 검사하여 스택 오버플로우를 탐지한다.
④ **CSP(Content Security Policy)** — **C/C++ 컴파일러가 삽입하는 스택 보호 코드**로서, 함수가 반환되기 전에 **CPU 레지스터의 값과 스택 프레임의 값을 비교**하여 일치하지 않으면 프로세스를 종료하는 메모리 보호 기법이다.

### 문제 3 [유사 기출: 정처기 필기 2020년 3회·2022년 1회·2024년 3회 반복 — 악성코드 분류 정형 패턴]
다음 중 **악성코드(Malware)의 분류**에 대한 설명으로 옳지 **않은** 것은?

① **바이러스(Virus)** — **숙주(host) 파일이나 부트섹터에 기생**하여 자신을 복제하는 악성코드로, 숙주가 실행될 때 활성화되어 다른 파일을 감염시킨다.
② **웜(Worm)** — **숙주 파일이 필요 없으며**, **네트워크 취약점**을 이용해 스스로 다른 시스템으로 전파되는 자가복제형 악성코드이다(예: Code Red, SQL Slammer, Conficker).
③ **트로이 목마(Trojan Horse)** — **정상 프로그램으로 위장**해 사용자가 자발적으로 실행하도록 유도하며, **자가복제 기능은 없고** 백도어 설치·정보 유출 등의 악의적 기능을 수행한다.
④ **루트킷(Rootkit)** — **취약한 패스워드를 자동으로 추측**하여 시스템에 침입한 뒤 **무차별 대입(Brute-Force) 공격**으로 관리자 계정을 탈취하는 인증 공격 도구로, 침입 후의 은폐와는 무관하다.

### 문제 4 [유사 기출: 정처기 필기 2021년 3회·2023년 3회·2025년 2회 반복 — 특수 악성코드 식별 정형 패턴]
다음 설명에 해당하는 **악성코드**로 가장 적절한 것은?

> 사용자의 파일을 **임의의 대칭키 또는 비대칭키 알고리즘으로 암호화**하여 접근할 수 없게 만든 뒤, **복호화 키 제공의 대가로 가상화폐 등 금전을 요구**한다. 대표적인 사례로 2017년 전 세계적으로 확산된 **WannaCry**, **CryptoLocker**, **Locky** 등이 있다.

① **랜섬웨어(Ransomware)** — 파일·시스템 가용성을 인질로 잡고 금전을 요구하는 악성코드.
② **스파이웨어(Spyware)** — 사용자 모르게 활동·개인정보를 수집해 외부로 전송하는 악성코드.
③ **애드웨어(Adware)** — 강제 광고 노출을 통해 수익을 얻는 악성코드.
④ **로직밤(Logic Bomb)** — 특정 **조건(날짜·이벤트)** 충족 시점에 발동하는 트리거형 악성코드.

### 문제 5 [유사 기출: 정처기 필기 2020년 2회·2022년 3회·2024년 1회 반복 — Secure SDLC 모델 매칭 정형 패턴]
다음 **Secure SDLC(Secure Software Development Life Cycle) 모델**과 그 **제안 주체·특징**의 매칭으로 옳지 **않은** 것은?

① **MS-SDL(Microsoft Security Development Lifecycle)** — **마이크로소프트(Microsoft)** 가 제안한 모델로, 교육 → 요구사항 → 설계 → 구현 → 검증 → 배포 → 대응의 보안 활동을 SDLC 전 단계에 통합한 프로세스이다.
② **Seven Touchpoints** — **개리 맥그로(Gary McGraw)** 가 제안한 모델로, 코드 리뷰·위험 분석·침투 테스트·위험 기반 보안 테스트·악용 사례·보안 요구사항·외부 분석의 **7가지 보안 강화 활동**을 SDLC에 접목한다.
③ **CLASP(Comprehensive Lightweight Application Security Process)** — **OWASP**가 제안한 모델로, **개념·역할·활동·평가** 등의 뷰(View)를 기반으로 활동 중심의 경량 보안 프로세스를 제공한다.
④ **OWASP SAMM(Software Assurance Maturity Model)** — **ISO/IEC 27001**이 제정한 국제 표준으로, **정보보호 관리체계(ISMS)** 의 인증 심사에 사용되며 Secure SDLC와는 무관한 ISMS 인증 모델이다.

### 문제 6 [유사 기출: 정처기 필기 2021년 1회·2023년 2회·2025년 1회 반복 — KISA 시큐어코딩 7대 약점 카테고리 정형 패턴]
다음 중 **KISA(한국인터넷진흥원)의 「전자정부 SW 개발·운영자를 위한 시큐어코딩 가이드」가 정의하는 7대 보안약점 카테고리**에 해당하지 **않는** 것은?

① **입력데이터 검증 및 표현** — SQL 삽입, 크로스사이트 스크립트(XSS), 경로 조작, 정수 오버플로 등 외부 입력값을 충분히 검증하지 않아 발생하는 약점.
② **시간 및 상태** — TOCTOU(Time-of-Check to Time-of-Use) 경쟁 조건, 종료되지 않는 반복문 등 동시성·상태 변화 시 발생하는 약점.
③ **에러 처리** — 예외 미처리, 오류 메시지를 통한 민감정보 노출 등 오류 상황에서 비정상 동작이 발생하는 약점.
④ **물리적 접근 통제** — **출입통제·CCTV·생체인증·이중잠금 도어** 등 데이터센터 및 서버실의 **물리적 침입을 방지하는 시설·운영 통제**로, 시큐어코딩 7대 카테고리의 하나로 분류된다.

### 문제 7 [유사 기출: 정처기 필기 2022년 1회·2024년 2회·2025년 2회 반복 — OWASP Top 10 항목 정형 패턴]
다음 중 **OWASP(Open Worldwide Application Security Project)** 와 **OWASP Top 10**에 대한 설명으로 옳지 **않은** 것은?

① **OWASP**는 **웹 애플리케이션의 보안 향상**을 목적으로 하는 **비영리 오픈소스 커뮤니티**이며, **OWASP Top 10**은 약 3~4년 주기로 갱신되는 **웹 애플리케이션 10대 보안 위험** 목록이다.
② **A03:2021 Injection** 카테고리에는 **SQL 삽입, LDAP 삽입, Command 삽입**과 함께 **XSS(Cross-Site Scripting)** 도 포함되도록 2021년 개정판에서 통합되었다.
③ **A10:2021 SSRF(Server-Side Request Forgery)** 는 **서버가 사용자의 입력값을 검증 없이 사용해 내부망·메타데이터 서버 등으로 요청을 보내도록** 유도하는 공격이다.
④ **OWASP Top 10**은 **한국정보통신기술협회(TTA)** 가 한국 환경에 맞게 제정한 **국가 표준(KS) 웹 보안 점검 항목**으로, **국내에서만 사용되는 보안 점검 체크리스트**이다.

### 문제 8 [유사 기출: 정처기 필기 2021년 2회·2023년 3회·2025년 1회 반복 — 권한상승·SetUID·Rootkit 시나리오 정형 패턴]
다음 설명에 해당하는 **공격 단계 또는 보안 개념**으로 가장 적절한 것은?

> 공격자가 일반 사용자 계정으로 시스템에 침투한 뒤, **`/usr/bin/passwd`처럼 SetUID 비트(s)가 설정되어 실행 시 root 권한으로 동작하는 프로그램의 취약점**을 이용해 **일반 사용자 → root** 권한으로 상승시키는 행위이다. 이러한 공격이 성공한 이후에는 일반적으로 **루트킷(Rootkit)** 을 설치해 자신의 프로세스·파일·네트워크 연결을 은폐한다.

① **수직 권한상승(Vertical Privilege Escalation)** — 낮은 권한의 계정에서 더 높은 권한(root/admin) 계정의 권한으로 상승하는 공격.
② **수평 권한상승(Horizontal Privilege Escalation)** — 동등한 권한의 다른 사용자의 자원·기능에 접근하는 공격.
③ **세션 하이재킹(Session Hijacking)** — 인증된 세션의 토큰·쿠키를 탈취해 정상 사용자처럼 가장하는 공격.
④ **사회공학(Social Engineering)** — 기술적 취약점이 아닌 **사람의 심리적 약점**을 이용해 정보를 얻어내는 공격.

## 🔄 누적 복습 (직전 Day 44 핵심 충돌 키워드 재점검)
> Day 44(보안 기본+네트워크 통합) progress 미기록 — Day 44 학습 목표 ⑨ "자주 충돌하는 함정 매핑" 4개 항목을 자가체크 문장으로 재출제.

- **Q-R1.** **IDEA** 알고리즘은 (국산 / 국제 표준 ) **대칭키 128bit 블록 암호**이며, **국산 4대장 = "씨·아·하·리(SEED·ARIA·HIGHT·LEA)"** 에는 포함되지 (한다 / 않는다).  → 정답: **국제 표준 / 않는다**
- **Q-R2.** 국산 해시 함수는 **(SHA-160 / HAS-160)** 이며 160bit 출력이다. **SHA-160** 은 일반적으로 통용되는 명칭이 ( 있다 / 없다 — SHA-1과 혼동 주의 ).  → 정답: **HAS-160 / 없다**
- **Q-R3.** 동일 약어 **MAC** 의 두 개념: ① **접근통제 MAC** = **( Mandatory / Message ) Access Control** = 강제적 접근통제 / ② 메시지 무결성 인증의 **MAC** = **( Message / Mandatory ) Authentication Code**.  → 정답: **Mandatory / Message**
- **Q-R4.** **DSA = ( 키 교환 / 전자서명 ) 전용**, 암호화는 (가능 / 불가). **DH(Diffie-Hellman) = ( 키 교환 / 전자서명 ) 전용**, 서명·암호화는 (가능 / 불가).  → 정답: **전자서명 / 불가**, **키 교환 / 불가**

## 🔍 오답 역추적 개념 발췌
- **버퍼오버플로우 (Buffer Overflow)** — 메모리 영역의 경계를 넘어 데이터를 기록함으로써 **인접한 메모리(복귀 주소·메타데이터·함수 포인터)** 를 덮어써 임의의 코드 실행이나 비정상 종료를 유도하는 메모리 무결성 공격. **4종(Stack/Heap/Format String/Integer)** 이 시험에 반복 출제됨.
- **ASLR / DEP / Stack Canary** — 메모리 공격에 대한 **다층 방어(Defense in Depth)** 의 대표 3종. ASLR=주소 무작위화, DEP=데이터 영역 실행 금지, Canary=스택 보호값 삽입. **CSP(Content Security Policy)** 는 **웹 브라우저의 XSS 방어 헤더**이지 메모리 보호와 무관.
- **악성코드 분류 키워드** — **숙주 의존(Virus) / 자가 네트워크 전파(Worm) / 위장·복제X(Trojan) / 은폐(Rootkit) / 인증 우회 통로(Backdoor) / 암호화·몸값(Ransomware) / 정보 수집(Spyware) / 키입력 기록(Keylogger) / 광고(Adware) / 조건 발동(Logic Bomb) / 표적·장기·은밀(APT)** 11개 키워드 1자 매칭으로 출제.
- **Secure SDLC 4모델 제안 주체** — **MS-SDL**=Microsoft / **Seven Touchpoints**=Gary McGraw / **CLASP**=OWASP / **SAMM**=OWASP. ISMS·ISO/IEC 27001·BS7799는 **정보보호 관리체계** 인증 모델로 Secure SDLC와 구분.
- **KISA 시큐어코딩 7대 카테고리** — ① 입력검증·표현 ② 보안기능 ③ 시간·상태 ④ 에러처리 ⑤ 코드오류 ⑥ 캡슐화 ⑦ API 오용. **물리 보안·인적 보안은 시큐어코딩과 별도** 통제 영역(ISMS의 관리적·물리적 통제).
- **OWASP Top 10 2021 개정 포인트** — A01 접근제어 / A02 암호화 실패 / A03 인젝션(XSS 통합) / A04 안전하지 않은 설계 / A05 보안설정 오류 / A06 취약·구버전 컴포넌트 / A07 인증·식별 실패 / A08 SW·데이터 무결성 실패 / A09 로깅·모니터링 실패 / A10 **SSRF 신규 진입**.
- **권한상승(Privilege Escalation)** — 수직(상위 권한) vs 수평(동등 권한 타 사용자). **SetUID 비트**는 정상 운영(passwd·ping)에 필수지만 잘못 설정 시 수직 권한상승 통로. **Rootkit**은 권한상승 직후의 **흔적 은폐** 단계로 시퀀스 구분.

## 💡 상세 해설

### 문제 1
**정답: ④** — **정수 오버플로우(Integer Overflow)** 는 산술 결과가 정수 자료형의 표현 범위를 초과하여 **음수·작은 값으로 래핑되는 소프트웨어 약점**이며, 이후 **길이 검사 우회 → 버퍼 오버플로우 유발**의 통로가 된다. **버퍼오버플로우와 직결된 동일 분류군**으로, "CPU 캐시 라인 경계·파이프라인 중단"이라는 하드웨어 결함과는 **무관**하다. 정수 오버플로우는 KISA 시큐어코딩 7대 카테고리 중 **「입력데이터 검증 및 표현」** 항목에도 포함된다.
- ①(스택 BOF), ②(힙 BOF), ③(포맷 스트링)은 표준 정의 그대로이며 버퍼 오버플로우 4종 분류의 핵심 패턴.
- **함정**: ④의 "CPU 캐시 라인", "파이프라인 중단", "버퍼 오버플로우와 무관"이 3개 동시 오류 — 정수 오버플로우의 실제 정의는 **자료형 범위 초과 → 래핑 → 후속 메모리 손상**.

### 문제 2
**정답: ④** — ④의 설명("컴파일러가 삽입하는 스택 보호 코드 + 함수 반환 전 카나리아 비교")은 사실상 **Stack Canary(Stack Guard)** 의 정의이며, **CSP(Content Security Policy)** 는 **웹 브라우저가 외부 스크립트 로딩·인라인 스크립트 실행을 제한**해 **XSS·데이터 삽입 공격을 방어**하는 **HTTP 응답 헤더 기반 웹 보안 정책**이다(즉, 메모리 공격 대응이 아닌 **웹 클라이언트 보안**). 약어 CSP를 메모리 보호 기술처럼 위장한 **약어 충돌 함정**.
- ①(ASLR), ②(DEP/NX), ③(Stack Canary)는 표준 정의대로 모두 메모리 공격 대응 다층 방어의 핵심.
- **함정**: CSP는 **Content Security Policy(웹)** 이지 스택 보호와 무관. **SafeSEH·CFI**는 별개의 추가 방어로 시험에서 빈출.

### 문제 3
**정답: ④** — **루트킷(Rootkit)** 은 침입 **이후** 자신의 프로세스·파일·레지스트리·네트워크 연결을 **은폐**하기 위해 **커널 모듈·드라이버·시스템 콜 후킹**을 수행하는 도구이다. **무차별 대입(Brute-Force) 공격은 인증 공격의 한 유형**(보통 John the Ripper·Hydra 등의 도구)으로 루트킷과는 **별개**.
- ①(Virus=숙주 기생), ②(Worm=숙주 X·네트워크 자가전파, 대표 예시 정확), ③(Trojan=위장·자가복제 X)은 표준 분류.
- **함정**: 루트킷의 핵심 기능은 **"은폐(stealth)"** — 1자 매칭으로 암기. **백도어와 혼동 주의**: 백도어는 **재침입용 비밀 통로**(은폐는 부수적), 루트킷은 **은폐 자체가 목적**.

### 문제 4
**정답: ①** — 지문의 키워드 "**파일 암호화 + 복호화 키 대가로 금전(가상화폐) 요구 + WannaCry/CryptoLocker/Locky**" 는 **랜섬웨어(Ransomware)** 의 정형 정의이다. **Ransom(몸값) + Software**의 합성어로, **가용성(Availability) 침해**가 주된 위협이며 2017년 **WannaCry**는 SMB 취약점(EternalBlue)을 악용한 워머블(wormable) 랜섬웨어로 전 세계 확산.
- ②(스파이웨어=정보 수집·전송), ③(애드웨어=광고 강제 노출), ④(로직밤=조건 충족 시 발동)은 각자의 표준 정의.
- **함정**: 랜섬웨어가 자가전파(웜)·은폐(루트킷) 기능을 **부수적으로 동반**할 수 있어 다른 분류와 혼합돼 출제되기도 함. **"암호화 + 금전 요구"** 두 키워드가 일치하면 무조건 랜섬웨어.

### 문제 5
**정답: ④** — **OWASP SAMM(Software Assurance Maturity Model)** 은 **OWASP가 제안한 Secure SDLC 성숙도 평가 모델**로, **거버넌스(Governance)·설계(Design)·구현(Implementation)·검증(Verification)·운영(Operations)** 등 비즈니스 기능에 대해 **성숙도 0~3 단계**로 평가한다. **ISO/IEC 27001은 ISMS(정보보호 관리체계) 국제 표준**으로 SAMM과는 **완전 별개**의 인증 모델.
- ①(MS-SDL=Microsoft·7단계), ②(Seven Touchpoints=Gary McGraw·7개 보안 활동), ③(CLASP=OWASP·5뷰 경량)는 제안 주체와 특징 모두 정확.
- **함정**: SAMM의 "S"가 ISO·ISMS의 약어들과 혼동되기 쉬움. **OWASP는 SAMM, CLASP 두 모델을 모두 보유**.

### 문제 6
**정답: ④** — KISA의 「전자정부 SW 개발·운영자를 위한 시큐어코딩 가이드」 **7대 보안약점 카테고리**는 **① 입력데이터 검증 및 표현 ② 보안기능 ③ 시간 및 상태 ④ 에러처리 ⑤ 코드오류 ⑥ 캡슐화 ⑦ API 오용**이다. **물리적 접근 통제(출입통제·CCTV)** 는 **ISMS의 물리적 통제 영역**으로, 소스코드 작성 단계의 보안약점 분류와는 무관.
- ①(입력검증), ②(시간·상태), ③(에러처리)는 7대 카테고리 정확.
- **함정**: ④의 보기는 통제 분야가 잘 알려진 보안 영역이라 그럴듯해 보이지만, **시큐어코딩 = 소스코드 작성 단계의 약점 제거**이므로 물리·인적 보안은 분류 밖.

### 문제 7
**정답: ④** — **OWASP Top 10**은 **OWASP라는 국제 비영리 오픈웹 보안 커뮤니티가 발표하는 글로벌 표준 웹 보안 위험 목록**으로, **국제적으로 사용된다**. **TTA(한국정보통신기술협회) 국가 표준이나 한국 전용 점검 체크리스트가 아니다**. 국내 시큐어코딩 가이드는 별도로 **KISA가 발간**한다.
- ①(OWASP 비영리·갱신 주기), ②(2021 개정에서 XSS가 A03 Injection으로 통합), ③(A10 SSRF=내부망·메타데이터 서버 강제 요청)은 모두 사실.
- **함정**: ②는 일부 수험생이 "XSS는 별도 항목"으로 알고 있어 함정처럼 보이지만, **2021 개정판에서 XSS는 A03:2021 Injection 카테고리로 통합**된 것이 정설.

### 문제 8
**정답: ①** — 지문 "**일반 사용자 → root** 권한으로의 상승 + SetUID 악용 + 이후 Rootkit 설치"는 **수직 권한상승(Vertical Privilege Escalation)** 의 정형 시나리오이다. 공격자는 일반 셸 획득 → 커널/SetUID 익스플로잇으로 root 권한 획득 → Rootkit으로 흔적 은폐의 **3단계 시퀀스**를 반복.
- ②(수평 권한상승): 동일 권한 다른 사용자 자원 접근(다른 사용자 홈디렉터리 열람 등).
- ③(세션 하이재킹): 인증 토큰·쿠키 탈취 — 권한 자체 상승이 아닌 **인증 가장**.
- ④(사회공학): 기술적 취약점이 아닌 심리적 약점 악용(피싱·보이스피싱) — 단계 자체가 다른 공격 유형.
- **함정**: **SetUID 비트 자체가 취약점은 아님**(passwd·ping 등 정상 운영에 필수). **잘못 설정된 SetUID 프로그램의 입력 검증 결함**이 수직 권한상승의 통로.
