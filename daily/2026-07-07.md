### 문제 1 [유사 기출: 2023년 1회 1과목 — 객체지향 4대 원칙(캡슐화·상속·다형성·추상화) 정형 패턴]
객체지향(Object-Oriented) 프로그래밍의 4대 원칙에 관한 설명 중 **가장 옳지 않은 것**은?

① **캡슐화(Encapsulation)** — 데이터(속성)와 그 데이터를 처리하는 함수(메서드)를 하나의 객체로 묶고 외부로부터 은닉(Information Hiding)한다.
② **상속(Inheritance)** — 상위 클래스(Super Class)의 속성과 메서드를 하위 클래스(Sub Class)가 물려받아 재사용성을 높인다.
③ **다형성(Polymorphism)** — 동일한 메시지(Message)에 대해 객체마다 다르게 응답하는 능력으로, 오버로딩(Overloading)·오버라이딩(Overriding)이 대표 구현이다.
④ **추상화(Abstraction)** — 공통 속성을 뽑아 상위 개념으로 일반화하는 원칙으로, 상속(Inheritance)과 정반대 방향으로 특수화(Specialization)만 수행한다.

### 문제 2 [유사 기출: 2023년 2회 1과목 — GoF 디자인 패턴 23종 분류(생성 5/구조 7/행위 11) 정형 패턴]
GoF(Gang of Four) 디자인 패턴 23종 중 **생성(Creational) 패턴 5종에 속하지 않는 것**은?

① 팩토리 메서드(Factory Method) — 객체 생성을 서브클래스에 위임하여 인스턴스화의 결합도를 낮춘다.
② 추상 팩토리(Abstract Factory) — 관련된 객체군을 일관된 방식으로 생성한다.
③ 어댑터(Adapter) — 인터페이스가 호환되지 않는 클래스들을 연결하여 함께 동작하게 한다.
④ 싱글턴(Singleton) — 특정 클래스의 인스턴스가 오직 하나만 존재하도록 보장한다.

### 문제 5 [유사 기출: 2023년 2회 2과목 — AVL 트리 회전 4종(LL·RR·LR·RL) 정형 패턴]
**AVL 트리**에서 균형이 깨졌을 때 수행하는 4가지 회전 유형에 관한 설명으로 **가장 옳지 않은 것**은?

① **LL 회전** — 왼쪽 자식의 왼쪽 서브트리에 노드가 삽입되어 불균형이 발생한 경우, 오른쪽으로 한 번 회전(Single Right Rotation)한다.
② **RR 회전** — 오른쪽 자식의 오른쪽 서브트리에 노드가 삽입된 경우, 왼쪽으로 한 번 회전(Single Left Rotation)한다.
③ **LR 회전** — 왼쪽 자식의 오른쪽 서브트리에 노드가 삽입된 경우, 먼저 왼쪽 회전 후 오른쪽 회전(Left-Right Double Rotation)한다.
④ **RL 회전** — 오른쪽 자식의 왼쪽 서브트리에 노드가 삽입된 경우, 회전 없이 노드를 재배치하지 않고 그대로 둔다.

### 문제 7 [유사 기출: 2023년 1회 2과목 — 블랙박스 테스트 5종 정형 패턴]
**블랙박스(Black Box) 테스트 기법**에 관한 설명 중 **가장 옳지 않은 것**은?

① **동등 분할(Equivalence Partitioning)** — 입력 도메인을 유효/무효 분할로 나눠 각 대표 값으로 테스트한다.

③ **원인-결과 그래프(Cause-Effect Graph)** — 입력 원인과 결과 조합을 그래프로 표현해 테스트 케이스를 도출한다.
④ **기초 경로 검사(Basis Path Testing)** — 순환 복잡도 V(G)를 기반으로 프로그램의 독립 경로를 모두 실행하는 화이트박스 대표 기법이다.

### 문제 11 [유사 기출: 2023년 2회 3과목 — 트랜잭션 ACID 특성 4가지 정형 패턴]
**트랜잭션(Transaction)의 ACID 특성**에 관한 설명으로 **가장 옳지 않은 것**은?

① **원자성(Atomicity)** — 트랜잭션 내의 연산은 모두 실행되거나 모두 실행되지 않아야 한다(All-or-Nothing).
② **일관성(Consistency)** — 트랜잭션 실행 전·후 데이터베이스가 일관된(정합) 상태를 유지해야 한다.
③ **격리성(Isolation)** — 동시에 실행되는 트랜잭션들이 서로의 중간 결과에 접근하지 못하도록 한다.
④ **지속성(Durability)** — 트랜잭션이 커밋(Commit)되면 그 결과는 시스템이 고장 나도 반드시 반영되어야 하며, 이는 트랜잭션이 실패하면 자동으로 복구되지 않아야 함을 의미한다.

### 문제 13 [유사 기출: 2023년 1회 4과목 — C 비트 연산자 6종 정형 패턴]
다음 C 프로그램의 실행 결과로 가장 옳은 것은?

```c
#include <stdio.h>
int main() {
    int a = 12;   // 이진: 1100
    int b = 10;   // 이진: 1010
    printf("%d %d %d %d\n", a & b, a | b, a ^ b, a << 1);
    return 0;
}
```

① `8 14 6 24` — AND=1000(8), OR=1110(14), XOR=0110(6), 좌측 시프트 1=11000(24)
② `12 10 22 6` — 각 연산자를 산술 연산으로 해석
③ `0 22 12 24` — AND=0, OR=22, XOR=12, 좌측 시프트=24
④ `10 12 2 24` — AND·OR 순서 반전

### 문제 14 [유사 기출: 2023년 2회 4과목 — Java 상속(extends/super/final) 정형 패턴]

```java
class Dog extends Animal {

public class Test {
    public static void main(String[] args) {

        d.sound();
        d.showParent();
    }
}
```

① `Animal / 소리 / Animal / 소리` — 부모만 호출
② `멍멍 / Animal / 소리` — 자식 sound() 호출 후 super.name과 super.sound() 호출

### 문제 16 [유사 기출: 2023년 1회 4과목 — IPv4 vs IPv6 헤더 구조 비교 정형 패턴]
**IPv4와 IPv6의 헤더 구조 비교**에 관한 설명으로 **가장 옳지 않은 것**은?

① IPv4 주소는 **32비트**, IPv6 주소는 **128비트**로 구성된다.
② IPv6 헤더는 **고정 40바이트**이며, 옵션은 확장 헤더(Extension Header)로 분리하여 라우팅 효율을 높였다.
③ IPv6는 **체크섬(Checksum) 필드를 제거**하여 라우터 처리 부하를 줄였다.
④ IPv6는 IPv4보다 **주소 공간이 오히려 축소**되어 소규모 네트워크에 적합하다.

### 문제 17 [유사 기출: 2023년 2회 5과목 — CMMi 5단계(초기·관리·정의·정량관리·최적화) 정형 패턴]
**CMMi(Capability Maturity Model integration)**의 5단계 성숙도 수준에 관한 설명 중 **가장 옳지 않은 것**은?

① **레벨 1 초기(Initial)** — 프로세스가 정의되지 않고 개인 역량에 의존하며, 결과가 예측 불가능하다.
② **레벨 2 관리(Managed)** — 프로젝트 단위로 프로세스가 관리되며 요구사항·계획·품질 보증이 수행된다.
③ **레벨 3 정의(Defined)** — 조직 전체 표준 프로세스가 정의되고 프로젝트에 맞게 조정(Tailoring)된다.
④ **레벨 5 최적화(Optimizing)** — 정량적 데이터가 없더라도 경영진 직관으로 프로세스를 개선하는 단계이다.

**SQL 인젝션(SQL Injection) 공격 대응 방안**으로 **가장 관련이 적은 것**은?

### 문제 16
**정답 ④** — IPv6 주소 공간은 IPv4(32비트, 약 43억 개)의 **약 2⁹⁶배 확장(128비트, 약 3.4×10³⁸개)**. "축소되어 소규모 네트워크에 적합"은 완전 오답. IPv6 헤더는 40바이트 고정, 체크섬 필드 제거, 확장 헤더로 옵션 분리 등이 정답 특징.
- 함정 키워드: IPv6 = **주소 128비트 대폭 확장** / 헤더 40바이트 고정 / 체크섬 제거

### 문제 17
**정답 ④** — CMMi 5단계 = **1 초기(Initial) · 2 관리(Managed) · 3 정의(Defined) · 4 정량적 관리(Quantitatively Managed) · 5 최적화(Optimizing)**. 레벨 5는 **정량적 데이터·통계적 프로세스 관리(SPC) 기반의 지속적 개선**. "정량적 데이터 없이 경영진 직관으로 개선"은 CMMi 철학과 정반대.
- 함정 키워드: 레벨 4 = 정량 측정 / 레벨 5 = 정량 기반 **지속 개선**(직관 아님)

### 문제 18
**정답 ①** — 기밀성(Confidentiality) = 인가된 자만 접근, 노출 방지. 통제 수단 = **암호화·접근제어·인증**. 무결성 = 변조 방지(해시·전자서명), 가용성 = 서비스 중단 방지(이중화·백업), 부인방지 = 발신·수신 사실 부인 못하게 함(전자서명·타임스탬프)로 CIA에는 포함되지 않음(CIA는 3요소).
- 함정 키워드: 기밀성 = **암호화** / 무결성 = **해시** / 가용성 = **이중화**

### 문제 19
**정답 ④** — `X-Frame-Options: DENY`는 **클릭재킹(Clickjacking) 방어** 응답 헤더로, iframe 삽입을 차단. SQL Injection과 무관. SQL Injection 대응 = ① Prepared Statement(파라미터화 쿼리) ② 입력 검증(화이트리스트) ③ 최소 권한 원칙 ④ ORM 사용 ⑤ 오류 메시지 노출 최소화.
- 함정 키워드: SQLi ≠ Clickjacking / X-Frame-Options = 클릭재킹 방어

### 문제 20
**정답 ④** — 책임 공유 모델(Shared Responsibility Model)의 핵심은 **계층별 책임 분담**. IaaS = 물리 인프라만 CSP(클라우드 제공자) 책임, OS 이상은 사용자 책임. PaaS = OS·미들웨어까지 CSP, 앱·데이터는 사용자. SaaS = 앱까지 CSP, 사용자는 데이터·계정 관리만. "IaaS·PaaS·SaaS 모두 하드웨어·OS·앱·데이터 전체를 사용자 단독 책임"은 완전 오답(모두 사용자 책임이면 클라우드 아님).
- 함정 키워드: 책임 공유 = **계층별 분담**(IaaS↑ 사용자 책임 넓음 / SaaS↓ 사용자 책임 좁음)

## 📊 자주 틀리는 함정 5가지 (2023년 정처기 기출 전 4회독 총결산)

1. **디자인 패턴 분류 뒤섞임** — GoF 23종을 생성 5/구조 7/행위 11로 정확히 암기. **Adapter=구조**, **Command=행위**, **Prototype=생성**은 매년 오답 유도로 나옴. 특히 **Iterator(반복자)를 구조로 착각**하는 실수 주의(행위 패턴).
2. **DDL vs DCL vs TCL 명령어 혼동** — TRUNCATE는 **DDL**(구조 삭제, 롤백 불가)이지 DML 아님. GRANT/REVOKE는 **DCL**(권한), COMMIT/ROLLBACK은 **TCL**(트랜잭션). DELETE는 DML(WHERE 절 가능, 롤백 가능).
3. **Java super 키워드 동작** — 필드는 **정적 바인딩**(선언 타입, super.name=부모 필드), 메서드는 **동적 바인딩**(오버라이딩), 단 **super.method()**는 명시적으로 부모 메서드 호출. this.name vs super.name 구분 필수.
4. **IPv6 헤더 특성 오답** — IPv6 주소는 **128비트로 대폭 확장**(축소 아님), 헤더는 **40바이트 고정**(가변 아님), 체크섬 필드 **제거**(라우터 부하 감소), 옵션은 **확장 헤더**로 분리. "IPv6가 IPv4보다 헤더가 복잡해서 느리다"는 완전 오답.
5. **분산 DB 4투명성 vs 트랜잭션 ACID 혼동** — 4투명성 = **위치·중복·병행·장애**(분산 환경 관점). ACID = **원자성·일관성·격리성·지속성**(단일 트랜잭션 관점). 둘을 섞어 "격리 투명성" 같은 가짜 용어로 오답 유도 주의.
