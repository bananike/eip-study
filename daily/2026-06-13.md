## 🔍 오답 역추적 개념 발췌

### CPU 스케줄링 알고리즘 5종 핵심
- **FCFS(First Come First Served)** — 비선점, 도착 순. 단순, 호위 효과(Convoy Effect) 발생.
- **SJF(Shortest Job First)** — 비선점, 서비스시간 짧은 순. **평균 대기시간 최소**(이론상 최적), 그러나 **기아(Starvation)** 발생.
- **SRT(Shortest Remaining Time)** — SJF의 **선점형 버전**. 남은 시간 짧은 작업이 들어오면 선점.
- **RR(Round Robin)** — 선점, 시분할(Time Quantum). 응답 시간 짧음. 타임 슬라이스가 너무 작으면 컨텍스트 스위칭 오버헤드.
- **HRN(Highest Response-ratio Next)** — 비선점. SJF의 **기아 문제 해결**. 우선순위 = **`(대기시간 + 서비스시간) / 서비스시간`** = `대기시간/서비스시간 + 1`. **값이 클수록 우선**.

### 페이지 교체 알고리즘 4종 핵심
- **FIFO(First In First Out)** — 가장 먼저 들어온 페이지 교체. 큐 자료구조. **Belady's Anomaly**(프레임 늘려도 폴트 증가) 발생 가능.
- **LRU(Least Recently Used)** — **가장 오래 사용 안 한** 페이지 교체. 시간 스탬프/스택. 지역성 반영 우수.
- **LFU(Least Frequently Used)** — **가장 적게 사용한** 페이지 교체. 참조 횟수 카운터.
- **OPT(Optimal)** — **앞으로 가장 오래 사용 안 할** 페이지 교체. 이론상 최적이나 미래 참조열을 알아야 해서 구현 불가, 비교 기준용.

### 교착상태(Deadlock) 4 필요조건 — 4개 모두 만족해야 발생
- **상호 배제(Mutual Exclusion)** — 자원을 한 번에 한 프로세스만 사용
- **점유와 대기(Hold and Wait)** — 자원 점유 중에 다른 자원 요청
- **비선점(Non-preemption)** — 자원을 강제로 빼앗을 수 없음
- **환형 대기(Circular Wait)** — 프로세스들이 원형으로 자원 대기

### 교착상태 해결 4기법
| 기법 | 설명 | 예 |
|---|---|---|
| **예방(Prevention)** | 4 조건 중 하나를 사전 차단 | 모든 자원 사전 할당 → 점유와 대기 차단 |
| **회피(Avoidance)** | 안전 상태 유지하며 자원 할당 | **Banker's 알고리즘**(Dijkstra) |
| **발견(Detection)** | 자원할당 그래프로 교착 탐지 | 사이클 탐지 |
| **회복(Recovery)** | 발견 후 프로세스 종료/자원 선점 | 희생자 선정 |

### OSI 7계층 vs TCP/IP 4계층 — 프로토콜 / 장비 / PDU 매칭
| OSI 계층 | TCP/IP | 주요 프로토콜 | 장비 | PDU |
|---|---|---|---|---|
| 7 응용(Application) | 응용 | HTTP, FTP, SMTP, DNS, Telnet, SNMP | (게이트웨이) | Data |
| 6 표현(Presentation) | 응용 | SSL/TLS, JPEG, MPEG, ASCII | — | Data |
| 5 세션(Session) | 응용 | NetBIOS, RPC | — | Data |
| 4 전송(Transport) | 전송 | **TCP, UDP** | (게이트웨이) | **Segment**(TCP), **Datagram**(UDP) |
| 3 네트워크(Network) | 인터넷 | **IP, ICMP, IGMP, ARP, RARP**, 라우팅(RIP/OSPF/BGP) | **라우터(Router)**, L3 스위치 | **Packet** |
| 2 데이터링크(Data Link) | 네트워크 액세스 | Ethernet(MAC), PPP, HDLC | **브리지(Bridge), L2 스위치** | **Frame** |
| 1 물리(Physical) | 네트워크 액세스 | RS-232, V.35 | **리피터, 허브** | **Bit** |

> ⚠ **ARP/RARP**는 IP↔MAC 변환이라 OSI 2계층 또는 2~3계층 경계로 보지만, 정보처리기사 시험에서는 보통 **네트워크 계층(3계층)** 분류로 출제됨.

### TCP vs UDP 비교
| 항목 | TCP | UDP |
|---|---|---|
| 연결 방식 | **연결 지향(Connection-oriented)** | **비연결(Connectionless)** |
| 신뢰성 | 신뢰성 보장(재전송, 확인응답) | 비신뢰성 |
| 순서 제어 | O (Sequence Number) | X |
| 흐름/혼잡 제어 | O (Sliding Window) | X |
| 오버헤드 | 큼(헤더 20바이트~) | 작음(헤더 8바이트) |
| PDU | Segment | Datagram |
| 용도 | HTTP, FTP, SMTP, SSH | DNS(질의), DHCP, TFTP, RTP, VoIP |

### TCP 3-way handshake (연결 설정)
1. 클라이언트 → 서버: **SYN** (시퀀스 번호 x)
2. 서버 → 클라이언트: **SYN + ACK** (시퀀스 y, 확인 x+1)
3. 클라이언트 → 서버: **ACK** (확인 y+1)
→ 연결 종료는 **4-way handshake**(FIN → ACK → FIN → ACK).

### IPv4 vs IPv6 비교
| 항목 | IPv4 | IPv6 |
|---|---|---|
| 주소 길이 | **32비트** | **128비트** |
| 표기 | 점 10진(`192.168.0.1`) | 콜론 16진(`2001:0db8::1`), 8개 그룹 × 16비트 |
| 주소 수 | 약 43억(2³²) | 약 3.4 × 10³⁸(2¹²⁸) |
| 헤더 | 가변(20~60바이트), 체크섬 O | **고정 40바이트**, 체크섬 X(단순화) |
| 보안 | IPSec 선택적 | **IPSec 기본 내장** |
| 브로드캐스트 | O | **X (멀티캐스트/애니캐스트로 대체)** |
| 통신 방식 | 유니/멀티/브로드캐스트 | 유니/멀티/**애니캐스트(Anycast)** |

|---|---|---|---|

| B | 128~191 | /16 (255.255.0.0)     | 65,534 |

### 서브넷 마스크 계산 핵심
- **`/n`** = 앞에서 n비트가 네트워크부, 나머지 (32-n)비트가 호스트부.
- 호스트 비트가 `h`일 때 **사용 가능 호스트 수 = `2ʰ - 2`** (네트워크 주소·브로드캐스트 주소 제외).
- 서브넷 `S`개로 분할 → 추가로 `⌈log₂S⌉` 비트를 서브넷 비트로 사용. 남은 호스트 비트로 다시 `2ʰ - 2` 계산.
- 예: `/24`를 4서브넷 → 서브넷 비트 2 추가 → `/26` → 호스트비트 6 → `2⁶ - 2 = 62`.

### 라우팅 프로토콜 분류
| 구분 | 알고리즘 | 프로토콜 | 특징 |
|---|---|---|---|
| **IGP** (AS 내부) | 거리벡터(Distance Vector) | **RIP**, IGRP, EIGRP | 홉 수 기반, 주기적 광고. **RIP: 최대 15홉**(16=∞), 30초 주기 |
| **IGP** (AS 내부) | 링크상태(Link State) | **OSPF**, IS-IS | 다익스트라 **SPF**, 영역(Area) 분할, 빠른 수렴 |
| **EGP** (AS 간) | 경로벡터(Path Vector) | **BGP** | 인터넷 전체 라우팅, AS 경로 전달 |

> RIP은 거리벡터, OSPF는 링크상태, BGP는 경로벡터로 **유형이 모두 다름** — 시험 빈출 함정.

## 💡 상세 해설

### 문제 1
**정답 ④ — D**

HRN 우선순위 공식: **`(대기시간 + 서비스시간) / 서비스시간`** — **값이 클수록 먼저 실행**.

| 프로세스 | 대기 | 서비스 | (대기+서비스)/서비스 | 결과 |
|---|---|---|---|---|
| A | 5  | 20 | (5+20)/20  = 25/20  | **1.25** |
| B | 40 | 20 | (40+20)/20 = 60/20  | **3.00** |
| C | 15 | 45 | (15+45)/45 = 60/45  | **1.33** |
| D | 20 | 2  | (20+2)/2   = 22/2   | **11.00** ← 최대 |

D가 11.00으로 가장 크므로 **D가 먼저 실행**.

> HRN은 SJF의 기아 문제를 해결하려고 만든 알고리즘 — **대기 시간이 길어질수록** 우선순위가 자동으로 올라간다. D는 서비스시간(2)이 매우 짧고 대기시간(20)이 길어서 비율이 폭증.

- ① A — 1.25 (최소)
- ② B — 3.00
- ③ C — 1.33
- ④ D — **11.00 (최대) ✅**

### 문제 3
**정답 ③ — 선점(Preemption)**

교착상태 4 필요조건은 **상호배제, 점유와 대기, 비선점, 환형 대기** — 4개 모두 만족해야 발생. **선점(Preemption)이 가능하면 오히려 교착상태가 발생하지 않는다** — 자원을 강제로 빼앗을 수 있으니까. 따라서 조건은 **"비선점(Non-preemption)"** — 선점이 불가능한 상태가 조건이다.

- ① 상호 배제 — 조건 1 ✓
- ② 점유와 대기 — 조건 2 ✓
- ③ **선점 — ✗ (조건은 "비선점", 선점 자체는 교착상태 방지)** ← 정답
- ④ 환형 대기 — 조건 4 ✓

> 함정: "비선점"은 조건이지만 "선점"은 조건이 아니다 — 정반대.

### 문제 4
**정답 ④ — MAC 주소를 기반으로 프레임 전달**

네트워크 계층(3계층)은 **IP 주소로 패킷을 라우팅**. **MAC 주소를 기반으로 한 프레임 전달은 데이터링크 계층(2계층)의 기능** — L2 스위치/브리지가 담당.

- ① 라우팅 — 3계층 핵심 기능 ✓
- ② IP 주소 패킷 전달 — 3계층 ✓
- ③ ICMP — 3계층 보조 프로토콜(ping, traceroute) ✓
- ④ **MAC 프레임 전달 — 2계층 ✗** ← 정답

> 핵심 구분: **MAC = 2계층(데이터링크)**, **IP = 3계층(네트워크)**.

### 문제 5
**정답 ③ — 데이터그램 방식으로 비신뢰성·비순차적**

데이터그램(Datagram), 비신뢰성, 비순차적, 작은 오버헤드는 모두 **UDP**의 특성. TCP는 그 반대로 **세그먼트(Segment) 방식**, **신뢰성·순차성 보장**, **큰 오버헤드**.

- ① 연결 지향, 3-way handshake — TCP ✓
- ② 순서/흐름/혼잡/오류 제어 모두 — TCP ✓
- ③ **데이터그램·비신뢰성 — UDP의 특성 ✗** ← 정답
- ④ 전송 계층(4계층) — TCP ✓

### 문제 6
**정답 ④ — 브로드캐스트를 기본 통신 방식 중 하나로 채택**

IPv6는 **브로드캐스트를 제거**하고 **멀티캐스트(Multicast)와 애니캐스트(Anycast)**로 대체. 즉 정반대로 설명되어 있다.

- ① 128비트, IPv4(32비트)의 4배 ✓
- ② 16비트 × 8그룹, 콜론, 16진수 ✓
- ③ 헤더 단순화(고정 40바이트), IPSec 내장 ✓
- ④ **브로드캐스트 채택 ✗ — 정확히 반대로, 브로드캐스트 폐지·멀티캐스트/애니캐스트 사용** ← 정답

> IPv6 통신 방식 3종: **유니캐스트, 멀티캐스트, 애니캐스트** — 브로드캐스트 없음.

### 문제 7
**정답 ③ — 62**

`192.168.1.0/24`는 호스트 비트가 8개. 이를 **4개 서브넷**으로 분할:
- 4 = 2² → 서브넷 비트 **2개** 추가 → 새 프리픽스 `/26` (24+2)
- 남은 호스트 비트 = 8 - 2 = **6개**
- 사용 가능 호스트 수 = **2⁶ - 2 = 64 - 2 = 62**
  - `-2`는 **네트워크 주소(.0)**와 **브로드캐스트 주소(.63 등)** 제외분.

4개 서브넷:
| 서브넷 | 네트워크 | 브로드캐스트 | 호스트 범위 |
|---|---|---|---|
| 1 | 192.168.1.0/26   | 192.168.1.63  | .1~.62 |
| 2 | 192.168.1.64/26  | 192.168.1.127 | .65~.126 |
| 3 | 192.168.1.128/26 | 192.168.1.191 | .129~.190 |
| 4 | 192.168.1.192/26 | 192.168.1.255 | .193~.254 |

각 서브넷에 62개 호스트 사용 가능.

- ① 30 — /27(8서브넷)일 때
- ② 32 — `-2` 잊음
- ③ **62 ✅**
- ④ 64 — `-2` 잊음

### 문제 8
**정답 ③ — BGP가 AS 내부 라우팅 IGP**

BGP는 **AS(자율 시스템) 간** 라우팅을 담당하는 **EGP(Exterior Gateway Protocol)**. AS 내부 라우팅은 IGP(RIP, OSPF, IGRP, EIGRP 등). 정확히 반대.

- ① RIP — 거리벡터, 최대 15홉 ✓
- ② OSPF — 링크상태, 다익스트라 SPF ✓
- ③ **BGP — 경로벡터는 맞지만 IGP가 아니라 EGP ✗** ← 정답
- ④ IGRP — Cisco IGP, RIP 한계 극복(메트릭으로 대역폭·지연 사용) ✓

> 빈출 함정: BGP는 알고리즘이 거리벡터가 아닌 **경로벡터(Path Vector)**, 분류는 **EGP**.

## 📚 역추적 참조 (사후 검증용)

### 참조 1 — CPU 스케줄링 평균 대기시간/반환시간 계산법 (FCFS / SJF 예제)

**예제**: 동시 도착, 서비스시간 P1=24, P2=3, P3=3

#### FCFS (도착 순 P1→P2→P3)
| 프로세스 | 대기시간 | 반환시간(=대기+서비스) |
|---|---|---|
| P1 | 0  | 24 |
| P2 | 24 | 27 |
| P3 | 27 | 30 |

- 평균 대기 = (0+24+27)/3 = **17**
- 평균 반환 = (24+27+30)/3 = **27**

#### SJF (짧은 순 P2→P3→P1)
| 프로세스 | 대기시간 | 반환시간 |
|---|---|---|
| P2 | 0 | 3  |
| P3 | 3 | 6  |
| P1 | 6 | 30 |

- 평균 대기 = (0+3+6)/3 = **3**
- 평균 반환 = (3+6+30)/3 = **13**

→ SJF가 FCFS보다 평균 대기 14만큼 단축. **SJF는 평균 대기 최소가 이론적으로 증명됨**.

### 참조 2 — FIFO 페이지 교체 추적 (Belady's Anomaly 예시)

참조 문자열 `1 2 3 4 1 2 5 1 2 3 4 5`

- **3 프레임 FIFO**: 폴트 9회
- **4 프레임 FIFO**: 폴트 **10회** (프레임이 더 많은데 폴트가 더 많음!)
→ Belady's Anomaly. **LRU/OPT는 안전(스택 알고리즘)**, FIFO는 발생.

### 참조 3 — Banker's 알고리즘 (Dijkstra) 핵심
교착상태 **회피(Avoidance)** 기법. 자원 할당 전 **안전 상태(Safe State)** 검사:
1. 현재 보유 자원 + 가용 자원으로 **모든 프로세스가 종료될 수 있는 순서(Safe Sequence)** 존재?
2. 있으면 할당, 없으면 보류.

행렬: **Allocation(현재 할당), Max(최대 필요), Need(=Max-Allocation), Available(가용)**.

### 참조 4 — OSI 7계층 암기법
> "**아표세 전네 데물**" (응표세 전네 데물) — 응용/표현/세션 / 전송 / 네트워크 / 데이터링크 / 물리.
