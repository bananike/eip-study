### 문제 1 [유사 기출: 2025년 1회 1과목 — MSA 세부 패턴(API Composition · Aggregator) 정형 패턴]
**MSA(Microservice Architecture) 조회 패턴**인 **API Composition · Aggregator 패턴**에 관한 설명 중 **가장 옳지 않은 것**은?

① **API Composition** — 여러 마이크로서비스에 흩어진 데이터를 조회할 때, 조합자(Composer)가 각 서비스의 API를 호출해 결과를 메모리에서 조인(In-memory Join)하여 클라이언트에 반환하는 패턴이다.
② **Aggregator 패턴** — API Composition의 구현 형태로, 게이트웨이·전용 조합 서비스가 여러 서비스에 병렬 요청을 보내고 결과를 집계해 반환하며, 일반적으로 **BFF(Backends for Frontends)** 계층에서 자주 사용된다.
③ **트레이드오프** — 대규모 데이터셋을 서비스 간에 조인해야 할 경우 **네트워크·메모리 비용이 크고**, 이 경우 **CQRS + Materialized View**(읽기 전용 뷰)를 별도로 유지하는 대안을 고려한다.
④ **API Composition · Aggregator 패턴은 단일 서비스 내부에서 로컬 조인만 사용하는 패턴**이며, **여러 서비스 호출·병렬 요청·게이트웨이 조합·CQRS 대안은 이 패턴의 관심사와 무관**하다.

### 문제 2 [유사 기출: 2025년 2회 1과목 — Anti-Corruption Layer · Outbox 패턴 정형 패턴]
**Anti-Corruption Layer(ACL) · Outbox(트랜잭션 아웃박스) 패턴**에 관한 설명 중 **가장 옳지 않은 것**은?

① **Anti-Corruption Layer** — 레거시·외부 시스템의 도메인 모델이 자신의 바운디드 컨텍스트(Bounded Context)를 오염시키지 않도록, **번역·격리 계층**을 두어 개념·용어·데이터 구조를 변환한다(DDD 관점).
② **Outbox 패턴 문제 배경** — 서비스가 **DB 커밋 + 메시지 발행**을 함께 하는 "이중 쓰기(Dual Write)"는 원자성이 없어, DB 성공/메시지 실패(또는 반대)로 인한 데이터 불일치가 발생한다.
③ **Outbox 해법** — 도메인 이벤트를 동일 DB 트랜잭션 내 **`outbox` 테이블에 기록**하고, 별도 릴레이·CDC(Debezium 등)가 outbox 레코드를 읽어 메시지 브로커로 전송해 **at-least-once** 를 보장한다.
④ **Outbox 패턴은 DB 커밋과 메시지 발행을 분산 트랜잭션(2PC)으로 묶는 방식**이며, **ACL은 외부 시스템의 도메인을 그대로 자신의 컨텍스트에 강제 이식하는 것**이 원칙이다.

### 문제 3 [유사 기출: 2025년 3회 1과목 — Saga 패턴(Choreography vs Orchestration) 정형 패턴]
**Saga 패턴 · Choreography(협주) vs Orchestration(오케스트레이션)**에 관한 설명 중 **가장 옳지 않은 것**은?

① **Saga 패턴** — 여러 서비스에 걸친 **장기 실행 비즈니스 트랜잭션**을 로컬 트랜잭션 시퀀스로 나누고, 실패 시 **보상 트랜잭션(Compensating Transaction)** 으로 이전 단계를 되돌린다.
② **Choreography** — 중앙 조정자 없이 각 서비스가 이벤트를 구독·발행하며 자율 진행. **결합도가 낮고 확장성이 좋으나** 흐름 추적이 어렵고 순환 의존이 생길 수 있다.
③ **Orchestration** — 중앙 **Saga Orchestrator**(예: Camunda·Temporal·Netflix Conductor)가 각 단계를 명시적으로 지시. 흐름을 한눈에 파악·모니터링·재시도하기 쉬우나 오케스트레이터가 단일 결합점이 될 수 있다.
④ **Saga 패턴은 반드시 2PC와 동일한 원자성(Atomicity)을 만족**해야 하며, **Choreography는 중앙 조정자를 반드시 두어야 하고, Orchestration은 이벤트 발행 없이 동작해야 한다**.

### 문제 4 [유사 기출: 2025년 1회 1과목 — BFF · GraphQL Federation · AsyncAPI 정형 패턴]
**BFF(Backends for Frontends) · GraphQL Federation · AsyncAPI**에 관한 설명 중 **가장 옳지 않은 것**은?

① **BFF** — 웹·모바일·IoT 등 클라이언트 유형별로 **전용 백엔드 API 계층**을 두어, 클라이언트 특화 응답 형태·트래픽 최적화·인증 통합을 제공한다.
② **GraphQL Federation** — 여러 팀·도메인의 **서브그래프(Subgraph)** 를 하나의 **슈퍼그래프(Supergraph)** 로 연합하고, 게이트웨이가 쿼리를 서브그래프별로 분해·실행·병합한다(Apollo Federation).
③ **AsyncAPI** — **비동기 메시지 기반 API(이벤트·큐·MQTT·WebSocket 등)** 를 OpenAPI 스타일로 명세하기 위한 스펙으로, 발행자·구독자·채널·페이로드 스키마를 표준 문서화한다.
④ **BFF는 하나의 백엔드로 모든 클라이언트를 통합 처리하는 원칙**이며, **GraphQL Federation은 단일 스키마 파일에 모든 도메인을 하드코딩하는 방식**이고, **AsyncAPI는 REST 동기 API 전용 명세이다**.

### 문제 5 [유사 기출: 2025년 2회 2과목 — 위상 정렬(Topological Sort · Kahn · DFS) 정형 패턴]
**위상 정렬(Topological Sort)**에 관한 설명 중 **가장 옳지 않은 것**은?

① **정의** — **방향 비순환 그래프(DAG)** 의 정점들을 선행 관계가 위배되지 않도록 선형 순서로 나열하는 알고리즘으로, 빌드 시스템·과목 이수 순서·작업 스케줄링에 사용된다.
② **Kahn 알고리즘** — **진입 차수(in-degree)** 가 0인 정점을 큐에 넣고 반복적으로 제거하며 순서를 구성. 시간 복잡도 **O(V + E)**.
③ **DFS 기반 알고리즘** — DFS 완료 순서의 역순으로 위상 순서를 얻으며, 스택을 이용해 후위(post-order)에 정점을 저장한다. 시간 복잡도 **O(V + E)**.
④ **위상 정렬은 사이클이 있는 방향 그래프에서도 항상 유일한 순서를 반환**하며, **입력이 무향 그래프여도 동일하게 적용**할 수 있고, **시간 복잡도는 O(V²)** 이다.

### 문제 6 [유사 기출: 2025년 3회 2과목 — 강한 연결 요소(SCC · Tarjan · Kosaraju) 정형 패턴]
**강한 연결 요소(SCC · Strongly Connected Component)** 알고리즘에 관한 설명 중 **가장 옳지 않은 것**은?

① **정의** — 방향 그래프에서 **모든 정점 쌍(u, v)에 대해 u→v 와 v→u 경로가 모두 존재하는** 극대 부분 그래프. 컴파일러(재귀 함수 그룹)·2-SAT·소셜 네트워크 커뮤니티 탐지에 활용된다.
② **Kosaraju 알고리즘** — ① 원 그래프에서 DFS 종료 순서 스택 저장 → ② 역방향 그래프에서 스택 순서로 DFS → 각 DFS 트리가 하나의 SCC. **O(V + E)** 두 번의 DFS.
③ **Tarjan 알고리즘** — 단일 DFS 중 **`disc`(발견 시각)·`low`(도달 가능한 최저 disc)** 를 추적하며 스택으로 SCC를 즉시 추출. **O(V + E)** 로 Kosaraju 대비 메모리·상수 계수가 더 효율적이라는 평가를 받는다.
④ **SCC는 무향 그래프에서만 정의**되며, **Tarjan · Kosaraju 모두 O(V²)** 이고, **역방향 그래프·`low` 값·DFS 종료 순서 등은 SCC 계산과 무관**하다.

### 문제 7 [유사 기출: 2025년 1회 2과목 — 최단 경로(Dijkstra · Bellman-Ford · Floyd-Warshall · A*) 정형 패턴]
**최단 경로 알고리즘**에 관한 설명 중 **가장 옳지 않은 것**은?

① **Dijkstra** — **음의 가중치가 없는** 그래프에서 단일 시작점 최단 경로를 우선순위 큐로 구한다. 이진 힙 사용 시 **O((V + E) log V)**, 피보나치 힙 사용 시 **O(E + V log V)**.
② **Bellman-Ford** — 음의 가중치 간선을 허용하며 **음의 사이클 존재 여부**를 감지할 수 있다. 시간 복잡도 **O(V · E)**. V-1회 완화(Relaxation) 후 V회차에서 갱신이 있으면 음의 사이클이 존재.
③ **Floyd-Warshall** — **모든 쌍 최단 경로(All-Pairs Shortest Path)** 를 **O(V³)** 에 계산. 중간 정점을 순차적으로 허용하는 DP로, 음의 가중치는 허용하되 음의 사이클은 없어야 정의된다. **A***는 휴리스틱 함수 h(n)이 **admissible(과대추정 금지)** 하면 최적성이 보장된다.
④ **Dijkstra는 음의 가중치가 있어도 항상 최적해를 보장**하며, **Bellman-Ford는 음의 사이클을 감지할 수 없고**, **Floyd-Warshall은 O(V log V)**, **A***는 휴리스틱 조건과 무관하게 항상 최적이다.

### 문제 8 [유사 기출: 2025년 2회 2과목 — MST(Kruskal · Prim) · Fuzz · Property-Based Testing 정형 패턴]
**최소 신장 트리(MST) · Fuzz Testing · Property-Based Testing**에 관한 설명 중 **가장 옳지 않은 것**은?

① **Kruskal** — 간선을 가중치 오름차순으로 정렬한 뒤, **Union-Find(Disjoint Set)** 로 사이클을 피하며 하나씩 선택. **O(E log E)**. 희소 그래프에 유리.
② **Prim** — 시작 정점에서 확장하며 인접한 최소 가중치 간선을 추가. 우선순위 큐(이진 힙) 사용 시 **O(E log V)**. 밀집 그래프에서는 인접 행렬 O(V²) 구현이 더 유리할 수 있다.
③ **Fuzz Testing** — 무작위/변이(Mutation) 입력을 대량 주입하여 크래시·미정의 동작을 탐지. **커버리지 가이드 퍼저(AFL·libFuzzer)** 는 브랜치 커버리지를 피드백해 새 경로를 탐색한다. **Property-Based Testing**(QuickCheck·Hypothesis)은 "역함수 관계·불변식·멱등성" 같은 속성을 무작위 입력으로 검증하고, **실패 시 최소 반례를 축소(shrinking)** 한다.
④ **Kruskal·Prim 모두 사이클이 있는 방향 그래프에서만 동작**하며, **Union-Find는 MST와 무관**하고, **Fuzz Testing과 Property-Based Testing은 동일한 개념으로 shrinking·커버리지 가이드 모두 관계없다**.

### 문제 9 [유사 기출: 2025년 3회 3과목 — 2PC · 3PC · 분산 트랜잭션 정형 패턴]
**분산 트랜잭션 프로토콜 2PC(Two-Phase Commit) · 3PC(Three-Phase Commit)**에 관한 설명 중 **가장 옳지 않은 것**은?

① **2PC 절차** — ① **Prepare(투표)** : 조정자(Coordinator)가 참여자에 준비 요청 → 참여자가 Yes/No 응답 ② **Commit/Abort** : 조정자가 결과를 결정해 통보. 모두 Yes면 Commit, 하나라도 No면 Abort.
② **2PC의 한계** — **블로킹(Blocking)** — 조정자 장애 시 참여자가 Prepared 상태에서 잠금을 유지한 채 무한 대기할 수 있다. 부분 실패에 취약해 대규모 MSA에서는 Saga 등으로 대체된다.
③ **3PC** — Prepare/PreCommit/DoCommit 3단계로 확장해 조정자 장애 시에도 참여자들이 시간 초과 후 결정을 진행할 수 있게 해 **블로킹 문제를 완화**하지만, 네트워크 분할(파티션)에서는 여전히 취약하다.
④ **2PC는 조정자 장애 시에도 즉시 자동 복구되고**, **3PC는 2PC와 동일한 2단계 프로토콜**이며, **Saga는 2PC의 상위 호환 프로토콜로 항상 원자성을 보장**한다.

① **Paxos** — Leslie Lamport가 제안한 분산 합의 알고리즘으로, **Prepare/Promise/Accept/Accepted** 단계를 통해 다수결로 값을 결정. 정확하나 이해·구현이 어렵기로 유명하다.

④ **Paxos·Raft는 리더 없이 동작해야 하며 다수결 결정 없이도 합의 가능**하고, **Strong Consistency는 Eventual보다 약한 모델**이며, **Read-Your-Writes는 서버 중심 모델**이다.

}

④ **Java Virtual Threads는 OS 스레드 1:1 매핑으로 오히려 무겁고**, **Python GIL은 CPU-bound 병렬 성능을 완벽하게 제공**하며, **Rust 소유권은 런타임에 데이터 경쟁을 감지하는 기능**이다.

**SBOM 심화(SPDX vs CycloneDX) · 시크릿 관리(Vault · KMS · Sealed Secrets)**에 관한 설명 중 **가장 옳지 않은 것**은?

① **SPDX**(ISO/IEC 5962) — Linux Foundation 표준 SBOM 형식으로 **라이선스 규정 준수**에 강점. 라이선스·저작권·의존성 관계·해시를 표현.
② **CycloneDX** — OWASP 표준 SBOM 형식으로 **보안·취약점 정보 표현**에 강점. VEX(Vulnerability Exploitability eXchange)·서비스·제조·펌웨어 등 확장 프로파일 지원. **NTIA 최소 요소** 는 공급자·구성요소명·버전·고유식별자·의존성·SBOM 작성자·타임스탬프.
③ **Vault · KMS · Sealed Secrets · SOPS** — **HashiCorp Vault**(동적 시크릿·리스·PKI 인증)·**AWS KMS/Azure Key Vault/GCP KMS**(HSM·envelope 암호화·키 로테이션)·**Sealed Secrets**(공개키로 암호화된 시크릿을 Git에 안전 커밋)·**SOPS**(다중 KMS 후단 지원).
④ **SPDX·CycloneDX는 동일 회사에서 만든 사설 포맷**이며, **NTIA 최소 요소는 존재하지 않는 개념**이고, **Vault·KMS·Sealed Secrets은 모두 평문 시크릿을 Git에 그대로 저장하는 도구**이다.

### 문제 20 [유사 기출: 2025년 2회 5과목 — 랜섬웨어·APT 대응(Kill Chain · MITRE ATT&CK · 3-2-1 백업) 정형 패턴]
**랜섬웨어·APT(Advanced Persistent Threat) 대응**에 관한 설명 중 **가장 옳지 않은 것**은?

① **Cyber Kill Chain**(Lockheed Martin) — **정찰 → 무기화 → 배달 → 익스플로잇 → 설치 → C2 → 목적 달성** 7단계로 공격을 모델링. 각 단계에서 탐지·차단 대응을 매핑한다.
② **MITRE ATT&CK** — 실제 공격자의 **전술(Tactic) · 기법(Technique) · 절차(TTP)** 를 매트릭스로 정리한 지식 베이스. 방어팀이 탐지 규칙·시나리오 커버리지를 점검하는 데 활용.
3-2-1 백업 원칙 — 데이터의 **3개 사본 · 2개 서로 다른 매체 · 1개 오프사이트(Air-gap/Immutable)** 유지. 랜섬웨어에 대비해 **오프라인/불변(WORM)** 백업이 결정적.
③ **랜섬웨어·APT 대응 절차** — **Preparation(대비) → Detection(탐지) → Containment(격리) → Eradication(제거) → Recovery(복구) → Lessons Learned(사후 분석)**. NIST SP 800-61 사고 대응(Incident Response) 표준.
④ **랜섬웨어 대응에는 Kill Chain · MITRE ATT&CK · 3-2-1 백업 원칙 모두 무관**하며, **암호화된 백업은 랜섬웨어 공격에 오히려 위험하고**, **사고 대응 단계는 존재하지 않는다**.

### 문제 6
**정답 ④** — SCC는 **방향 그래프**에서 정의된다(무향은 연결 요소일 뿐). Tarjan·Kosaraju 모두 **O(V + E)**. 역방향 그래프(Kosaraju)·low 값(Tarjan)·DFS 종료 순서는 SCC 계산의 **핵심 개념**. ④는 세 사실 3중 뒤바꾸기. **함정 키워드**: "무향에서 정의·O(V²)·역방향/low 무관"은 SCC 정의 정반대.

### 문제 7
**정답 ④** — Dijkstra는 **음의 가중치 불가**(음의 가중치 있으면 Bellman-Ford 사용). Bellman-Ford는 **V번째 반복 완화 여부로 음의 사이클 감지**. Floyd-Warshall은 **O(V³)** (V log V 아님). A*는 **admissible 휴리스틱일 때만 최적 보장**. ④는 네 사실 4중 뒤바꾸기. **함정 키워드**: "Dijkstra 음가중 허용·Bellman-Ford 사이클 감지 불가·Floyd O(V log V)·A* 항상 최적"은 최단 경로 4대 알고리즘 정반대.

### 문제 8
**정답 ④** — Kruskal/Prim은 **무향 연결 가중 그래프**에 정의(방향 X). Union-Find는 **Kruskal의 사이클 검사 핵심 자료구조**. Fuzz Testing과 Property-Based Testing은 **별개 개념**(퍼즈=무작위 입력 크래시 탐지, PBT=속성 검증 + shrinking). ④는 세 사실 3중 뒤바꾸기. **함정 키워드**: "방향 그래프·Union-Find 무관·Fuzz=PBT 동일"은 MST/테스트 기법 정반대.

### 문제 9
**정답 ④** — 2PC는 **조정자 장애 시 블로킹**(자동 복구 아님)이 핵심 한계. 3PC는 **3단계 프로토콜**(2PC 확장). Saga는 **BASE·궁극 일관성** 지향으로 2PC와 원자성 특성이 다름. ④는 세 사실 3중 뒤바꾸기. **함정 키워드**: "2PC 자동 복구·3PC=2단계·Saga=2PC 상위 호환"은 분산 트랜잭션 프로토콜 정반대.

### 문제 11
**정답 ④** — CRDT는 **결합·교환·멱등 병합** 이 필수 조건(수학적 성질). G-Counter/PN-Counter/OR-Set은 대표적 CRDT 자료구조. 스트리밍 SQL은 **연속 쿼리·Windowing·Watermark**가 정의적 특성. ④는 세 사실 3중 뒤바꾸기. **함정 키워드**: "성질 없이 수렴·CRDT 자료구조 무관·스트리밍=배치·Watermark 없음"은 CRDT/스트리밍 SQL 정반대.

**정답 ④** — SPDX(Linux Foundation)·CycloneDX(OWASP)는 **서로 다른 조직 표준**. NTIA 최소 요소는 **공식 미국 상무부 지침**(존재함). Vault/KMS/Sealed Secrets는 모두 **시크릿을 안전하게 관리**하기 위한 도구(평문 Git 저장의 정반대). ④는 세 사실 3중 뒤바꾸기. **함정 키워드**: "동일 회사·NTIA 없음·평문 Git 저장"은 SBOM/시크릿 관리 정반대.

### 문제 20
**정답 ④** — Kill Chain·MITRE ATT&CK·3-2-1 백업·NIST 사고 대응 6단계는 모두 **랜섬웨어/APT 실무 표준**. **오프라인/불변(WORM) 백업**은 랜섬웨어 방어의 결정적 수단(위험이 아니라 필수). ④는 네 사실 4중 뒤바꾸기. **함정 키워드**: "Kill Chain/ATT&CK/3-2-1 무관·암호화 백업 위험·IR 절차 없음"은 랜섬웨어·APT 대응 표준 정반대.

## 📊 자주 틀리는 함정 5가지 (2025년 회독 4/4 · MSA 패턴·그래프·분산 트랜잭션·최신 언어·공급망 보안 초빈출)

1. **MSA 세부 패턴(API Composition/Aggregator·ACL·Outbox·Saga·BFF/Federation/AsyncAPI)** — API Composition은 **여러 서비스 병렬 호출 + 메모리 조인**, 대규모 데이터는 **CQRS+Materialized View**로 대안. Outbox는 **DB 트랜잭션 내 outbox 테이블 + CDC 릴레이**로 이중 쓰기 해결(2PC 회피). Saga는 **BASE·보상 트랜잭션**(2PC 원자성 아님), Choreography=이벤트 자율/Orchestration=중앙 조정자. BFF=클라이언트별 전용 백엔드, GraphQL Federation=서브그래프 연합, AsyncAPI=비동기 이벤트 명세. "로컬 조인 전용·Outbox=2PC·Saga=2PC·Choreography 중앙 조정자 필수·BFF 단일 백엔드·Federation 하드코딩·AsyncAPI=REST"는 MSA 패턴 정반대 초빈출 함정.
2. **그래프 알고리즘(위상 정렬·SCC·Dijkstra·Bellman-Ford·Floyd-Warshall·A*·MST·Fuzz vs PBT)** — 위상 정렬은 **DAG · O(V+E) · 여러 순서 존재**. SCC는 **방향 그래프 · O(V+E)** (Tarjan/Kosaraju). Dijkstra=**음가중 X**·Bellman-Ford=**음가중 O·음사이클 감지**·Floyd=**O(V³)**·A*=**admissible 조건부 최적**. Kruskal/Prim은 **무향 연결 가중 그래프** + **Union-Find**(Kruskal). Fuzz=**무작위 입력 크래시 탐지**·PBT=**속성 검증 + shrinking**(별개). "사이클 있어도 유일·무향 SCC·Dijkstra 음가중 허용·Bellman-Ford 감지 불가·Floyd O(V log V)·A* 무조건 최적·방향 MST·Fuzz=PBT"는 알고리즘 6대 핵심 정반대.
3. **분산 트랜잭션(2PC·3PC·Paxos·Raft·일관성 모델·CRDT·스트리밍 SQL·레이크하우스)** — 2PC는 **조정자 장애 시 블로킹**이 결정적 한계, 3PC는 **3단계 확장으로 블로킹 완화**, Saga는 **BASE·궁극 일관성**. Paxos/Raft는 **다수결 합의**, Raft는 **리더 선출·로그 복제 명세**. Strong ⊃ Sequential ⊃ Causal ⊃ Eventual(약한 모델 아님). CRDT는 **결합·교환·멱등** 필수. 스트리밍 SQL은 **연속 쿼리·Windowing·Watermark**. 레이크하우스 3대 포맷 모두 **ACID·시간 여행·스키마 진화** 제공. "2PC 자동 복구·3PC=2단계·Saga=2PC 상위 호환·리더 없음·다수결 없음·Strong=약한 모델·CRDT 성질 무관·스트리밍=배치·Watermark 없음·Iceberg Databricks 독점·Delta 로그 없음·Hudi 배치 전용"은 분산·데이터 표준 12중 뒤바꾸기.
4. **최신 언어 병렬 모델(pthread·Virtual Threads·GIL·Rust 소유권·WebAuthn·TLS 1.3)** — pthread mutex는 **임계구역 상호배타**로 데이터 경쟁 해결. Java Virtual Threads는 **M:N 다중화**(1:1 아님)로 확장성 향상. Python GIL은 **한 순간 한 스레드만 파이썬 바이트코드 실행** → CPU-bound는 **multiprocessing** 우회. Rust 소유권은 **컴파일 타임 검사**(불변 여러/가변 하나 XOR·Send/Sync). WebAuthn/Passkey는 **공개키 기반 · 서버는 공개키만 저장**. TLS 1.3은 **1-RTT · AEAD 필수 · CBC/RSA 제거 · (EC)DHE PFS 필수**. "뮤텍스 있어도 100000·Virtual Threads 1:1·GIL 완벽 병렬·Rust 런타임·Passkey 비밀번호 저장·TLS 1.3 RSA/CBC 필수·PFS 제거"는 최신 언어 병렬/인증 모델 7중 뒤바꾸기.
5. **위협 모델링 · 공급망 보안 · SBOM · 시크릿 · 랜섬웨어 대응** — STRIDE=**6대 위협**(Spoofing 포함), DREAD=**5축 점수 위험 평가**, PASTA=**7단계 리스크 중심**, Attack Tree=**공격 경로 트리**, LINDDUN=**프라이버시**. SLSA=**레벨 1~4**, Sigstore=**키리스 서명(OIDC·Fulcio·Rekor 투명성 로그)**, in-toto=**전체 공급망 단계 서명/검증**. SPDX(Linux Foundation)=라이선스 강점 vs CycloneDX(OWASP)=보안·VEX 강점, **NTIA 최소 요소** 존재. Vault=동적 시크릿, KMS=envelope/HSM, Sealed Secrets=Git 안전 커밋, SOPS=다중 KMS 후단. 랜섬웨어=**Kill Chain 7단계·MITRE ATT&CK·3-2-1(오프라인/불변 백업)·NIST SP 800-61 6단계 IR**. "STRIDE 5대·DREAD 열거·PASTA 1단계·Attack Tree/LINDDUN 없음·SLSA 마케팅·Sigstore 정적 키·in-toto 소스 전용·SBOM 사설 포맷·NTIA 없음·평문 Git 저장·Kill Chain/ATT&CK/3-2-1 무관·암호화 백업 위험·IR 없음"은 보안 6대 카테고리 15중 뒤바꾸기 초빈출 함정 종합판.
