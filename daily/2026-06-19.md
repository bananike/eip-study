## 🔍 오답 역추적 개념 발췌

### 1. 이상현상(Anomaly) 3종 — "삽·삭·갱"만 외우면 됨
| 종류 | 정의 | 예시 |
|---|---|---|
| **삽입(Insertion)** 이상 | 원하지 않는 정보까지 함께 삽입해야 함 | 학생 정보 입력 시 미확정 수강과목 NULL 강제 |
| **삭제(Deletion)** 이상 | 일부 정보 삭제 시 원치 않는 정보까지 함께 삭제됨 | 수강 취소 시 학생 정보까지 삭제 |
| **갱신(Update)** 이상 | 중복 데이터 일부만 갱신되어 데이터 불일치 발생 | 학과명을 한 곳만 수정해서 모순 발생 |

> **함정 단골**: "검색 이상(Retrieval Anomaly)"이라는 항목은 **이상현상에 없음** — 보기 ④번 자리에 즉시 등장.

### 2. 정규화 목적 — "중복 ↓ + 이상현상 제거 + 무결성 ↑" 3가지
- **검색 성능은 오히려 ↓** 가능성 있음 (조인 증가). "정규화 = 검색 속도 향상" 보기는 **항상 오답**.
- 성능 향상을 위해서는 **비정규화(Denormalization)** — 정규화의 반대 개념을 따로 적용.

### 3. 정규화 단계 두음 — "**도·부·이·결·다·조**"
| 단계 | 두음 | 제거 대상 | 핵심 키워드 |
|---|---|---|---|
| **1NF** | 도 | 비원자값 | **도메인 원자값**(Atomic) — 반복 그룹 X |
| **2NF** | 부 | 부분 함수 종속 | **완전 함수 종속**(복합키일 때만 의미) |
| **3NF** | 이 | 이행 함수 종속 | 비주요 속성 → 기본키 직접 종속 |
| **BCNF** | 결 | 결정자 ≠ 후보키 | **모든 결정자가 후보키** |
| **4NF** | 다 | 다치 종속(MVD) | 한 속성이 여러 독립값 |
| **5NF** | 조 | 조인 종속(PJ/NF) | 무손실 분해 불가 |

> 시험 90% 이상이 **"부·이·결"** 3개에서 출제. 1NF는 정의, BCNF는 결정자 조건만 외워두면 만점권.

### 4. 함수 종속(Functional Dependency) 3종 — X → Y
- **완전 함수 종속(Full FD)** = X의 **진부분집합**으로는 Y를 결정 **못 함**. 즉 X 전체가 있어야 Y 결정.
- **부분 함수 종속(Partial FD)** = X의 **일부**만으로 Y 결정 가능. 복합키에서만 발생. **2NF가 제거 대상**.
- **이행 함수 종속(Transitive FD)** = X→Y, Y→Z 일 때 **X→Z**. **3NF가 제거 대상**.

> 결정자(Determinant) = X→Y에서 **X**(왼쪽). 종속자(Dependent) = **Y**(오른쪽).

### 5. BCNF — "3NF + 모든 결정자가 후보키"
- 3NF를 만족해도 **결정자가 후보키가 아닌 경우** 이상현상 가능 → BCNF가 이를 해결.
- 모든 BCNF는 자동으로 3NF (역은 성립 X).
- 시험 정형 표현: **"모든 결정자가 후보키인 정규형"** — 한 문장 외우면 즉답.

### 6. 정규화 vs 비정규화
- **정규화(Normalization)** = 무결성↑, 중복↓, 이상현상 제거. 검색 성능은 **↓ 가능**.
- **반정규화(De-normalization)** = 성능↑ 위해 의도적 중복 허용, 통합. 정규화의 **반대**.
- 단골 함정: "정규화 = 성능 향상" → **X**. "정규화 = 데이터 무결성 향상" → **O**.

- **이상현상(Anomaly)** = 데이터 중복으로 인해 발생하는 부작용. 정형 3종: **삽입·삭제·갱신** ("삽·삭·갱"으로 외움).
- ①②③은 표준 3종. ④ "검색 이상"은 **이상현상 정의에 없는 함정 용어** — 검색은 단순 SELECT 동작이므로 이상이 발생할 일이 없음.
- 시험 정형 4지선다: ①②③ 3개는 정의로 채우고 ④에 가짜 항목(검색·참조·연쇄 등) 끼워 넣음. **"이상현상 = 삽·삭·갱 3개뿐"** 1:1 매칭.

### 문제 2
**정답 ④ 검색(Retrieval) 속도(성능)를 항상 향상시킨다.**
- 정규화 **3대 목적**: ① 데이터 **중복 최소화** ② 이상현상 **제거** ③ 데이터 **무결성 향상**.
- **검색 성능은 오히려 ↓ 가능** — 정규화하면 테이블이 쪼개져 **조인 횟수가 증가**하기 때문.
- 성능 향상을 원하면 **반정규화(De-normalization)** — 정규화의 반대 개념(의도적 중복·통합).
- 함정 단골: "정규화의 목적 = 검색 속도 향상" → **항상 오답**. "항상"이라는 절대 표현도 의심 신호.

### 문제 3
**정답 ③ 모든 도메인이 원자값(Atomic Value)으로만 구성되어 있어야 한다.**
- **1NF(제1정규형)** = 모든 속성 값이 **원자값(분해 불가능한 단일 값)** — 반복 그룹·다중값·집합값 X.
- 예: "전화번호" 속성에 ["010-1234", "010-5678"]처럼 다중값 들어가면 1NF 위반.
- ① = **2NF**의 조건(완전 함수 종속).
- ② = **3NF**의 조건(이행 함수 종속 제거).
- ④ = **BCNF**의 조건(결정자가 모두 후보키).
- 정규형 1:1 매칭형 함정 4지선다 — 4개 정규형 정의를 보기에 흩뿌리고 1NF만 정답.

### 문제 4
**정답 ② 제2정규형(2NF)**
- **2NF** = 1NF를 만족 + **모든 비주요(non-key) 속성이 기본키에 완전 함수 종속**.
- 즉, 부분 함수 종속(복합키의 일부만으로 비주요 속성 결정 가능한 경우)을 제거한 상태.
- 복합키일 때만 의미 있음 — 단일키이면 자동으로 2NF.
- 예: 기본키 (학번, 과목코드) → 점수 [완전 종속 — 2NF OK], 학번 → 학생명 [부분 종속 — 2NF 위반].
- ① 1NF는 원자값 조건만, ③ 3NF는 이행 종속까지 제거, ④ BCNF는 결정자 조건까지 추가.

### 문제 5
**정답 ③ 제3정규형(3NF)**
- **3NF** = 2NF를 만족 + **모든 비주요 속성이 기본키에 이행적 함수 종속이 아님**.
- **이행 함수 종속** = X→Y, Y→Z 일 때 X→Z. 즉, 비주요 속성이 다른 비주요 속성을 통해 간접적으로 기본키에 종속되는 경우 제거.
- 예: 학번 → 학과 → 학과장. "학번 → 학과장"이 이행 종속 → 분리해서 3NF로.
- ① 1NF/② 2NF는 전제 조건, ④ 4NF는 다치 종속(MVD) 제거.
- 시험 정형 표현: "2NF + 이행 종속 제거" = **3NF** 1:1 매칭.

### 문제 6
**정답 ④ 3NF를 만족하면서, 릴레이션의 모든 결정자(Determinant)가 후보키인 정규형이다.**
- **BCNF(Boyce-Codd Normal Form)** = 3NF의 **강화형**. 모든 함수 종속 X→Y에서 **X(결정자)가 반드시 후보키**여야 함.
- 3NF를 만족하더라도 결정자가 후보키가 아닌 경우 이상현상 가능 → BCNF가 이를 해결.
- 모든 BCNF는 자동으로 3NF (역은 성립 X — 3NF지만 BCNF 아닌 경우 존재).
- ① = 1NF 조건, ② = 2NF 조건, ③ = 4NF 조건(다치 종속 제거).
- 1자 외워야 할 정형 표현: **"모든 결정자가 후보키인 정규형 = BCNF"** — 즉답.

### 문제 7
**정답 ② 1NF → 2NF → 3NF → BCNF → 4NF → 5NF**
- 두음 **"도·부·이·결·다·조"** — 1NF(도메인 원자) / 2NF(부분종속 제거) / 3NF(이행종속 제거) / **BCNF(결정자 후보키)** / 4NF(다치종속 제거) / 5NF(조인종속 제거).
- **BCNF는 3NF와 4NF 사이에 위치** — 3NF 강화형이지만 4NF보다는 약함. 보기 ①(BCNF가 3NF보다 앞), ④(BCNF가 4NF보다 뒤)는 위치 오류.
- ③ 2NF/3NF 순서 뒤바뀜 — 정규형 번호는 부등호와 동일.
- 함정 단골: BCNF 위치를 3NF↔4NF 사이에서 어긋나게 배치 — **"3NF 다음 BCNF, BCNF 다음 4NF"** 위치 고정.

### 문제 8
**정답 ③ A → C는 이행 함수 종속(Transitive Dependency)이다.**
- 주어진 종속: A → B, B → C → 합성으로 **A → C** (이행 함수 종속 정의 그대로).
- **3NF가 제거 대상**으로 삼는 종속 형태.
- ① A → B는 **완전 함수 종속**(A 전체로 B 결정). 부분 함수 종속은 복합키에서만 발생 — A가 단일 속성이므로 부분 종속 불가.
- ② B → C는 **단순 함수 종속**. **다치 종속(MVD)**은 "한 속성이 여러 독립값을 결정"하는 4NF 대상이며, 표기 형식 자체가 다름(→→).
- ④ A→B→C 이행 종속이 존재 → **3NF 위반**, 당연히 BCNF도 만족 못함.
- 함수 종속 종류 판별의 정형 패턴: **단일키+추이 종속 = 이행** 1:1 매칭.
