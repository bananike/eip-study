## 🔍 오답 역추적 개념 발췌
- **빅오(Big-O) 시간복잡도 서열**: O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)
- **버블 정렬(Bubble Sort)** — 인접한 두 원소 비교·교환을 끝까지 반복. 1회전 후 가장 큰(또는 가장 작은) 원소가 맨 끝에 확정. 평균/최악 O(n²), 안정 정렬.
- **선택 정렬(Selection Sort)** — i회전마다 i번째 위치에 들어갈 (남은 부분의) 최솟값을 찾아 교환. 평균/최악 O(n²), **불안정**.
- **삽입 정렬(Insertion Sort)** — 두 번째 원소부터 정렬된 부분에 알맞은 위치로 삽입. 평균/최악 O(n²), **최선 O(n)** (이미 정렬된 경우), 안정.
- **퀵 정렬(Quick Sort)** — 피벗 기준 분할 정복. 평균 O(n log n), **최악 O(n²)**(피벗이 항상 최소/최대), **불안정**, in-place(추가공간 O(log n)).
- **합병 정렬(Merge Sort)** — 분할 정복. **항상 O(n log n)** (평균=최악), **안정**, 추가공간 **O(n)** 필요.
- **힙 정렬(Heap Sort)** — 최대힙/최소힙 구성 후 루트 추출 반복. **항상 O(n log n)**, **불안정**, in-place.
- **쉘 정렬(Shell Sort)** — 삽입 정렬 개선판. 일정 간격(gap)으로 부분 정렬 후 점차 좁힘. 평균 약 O(n^1.5).
- **기수 정렬(Radix Sort)** — 비교 없이 자릿수별 분배. O(d×n) (d=자릿수). 안정.
- **이분 탐색(Binary Search)** — 정렬 전제, 매 단계 범위 절반 → O(log n).
- **알고리즘 설계 기법 4종**: 분할 정복(Divide & Conquer), 동적 계획법(Dynamic Programming, 메모이제이션·중복 부분문제), 탐욕(Greedy, 매 단계 국소 최적), 백트래킹(Backtracking, 탐색 중 가능성 없으면 가지치기).

## 💡 상세 해설

### 문제 1
정답: ④ 선택 정렬(Selection Sort)
- 평균 시간복잡도 O(n log n) 정렬: 퀵·합병·힙(이른바 "n log n 3대장").
- 선택 정렬은 매 회전마다 남은 부분 전체를 훑어 최솟값을 찾으므로 평균/최악 모두 **O(n²)**.
- 오답 이유:
  - ① 퀵 정렬 — 평균 O(n log n) (최악 O(n²))
  - ② 합병 정렬 — 항상 O(n log n)
  - ③ 힙 정렬 — 항상 O(n log n)

### 문제 2
정답: ① 6, 7, 3, 5, 9
- 버블 정렬 1회전 추적 (인접 비교 후 큰 값을 오른쪽으로):
  - `9,6,7,3,5` → (9>6 교환) `6,9,7,3,5`
  - → (9>7 교환) `6,7,9,3,5`
  - → (9>3 교환) `6,7,3,9,5`
  - → (9>5 교환) `6,7,3,5,9`
- 1회전 후 **최댓값(9)이 맨 끝**에 확정되는 것이 버블 정렬의 정형 패턴.
- 오답 이유:
  - ② 시작 형태가 깨짐 — 인접 교환 규칙 위반
  - ③ 완전 정렬된 상태 — 1회전이 아니라 종료 결과
  - ④ 1회 교환 후 멈춘 형태 — 1회전은 끝까지 수행

### 문제 3
정답: ① 3, 8, 4, 9, 7
- 선택 정렬 1회전: 전체에서 최솟값(3)을 찾아 첫 번째 위치(8)와 **교환**.
  - `8,3,4,9,7` → (min=3을 1번 위치와 교환) `3,8,4,9,7`
- 오답 이유:
  - ② 2회전 결과 (최솟값 4를 2번 위치로)
  - ③ 완전 정렬 — 종료 결과
  - ④ 교환 규칙 위반 (3과 7이 동시에 움직임)

### 문제 4
정답: ① 5, 8, 6, 2, 4
- 삽입 정렬 1회전: **2번째 원소(5)**를 정렬된 부분(`[8]`)에 삽입.
  - 5 < 8 → 8을 한 칸 밀고 5를 앞에 → `5, 8, 6, 2, 4`
- 핵심: 삽입 정렬은 2번째 원소부터 시작, 1회전 후엔 **앞 2개만 정렬됨**.
- 오답 이유:
  - ② 2회전 결과 (6까지 정렬)
  - ③ 4회전 결과
  - ④ 변화 없음 — 5<8 비교가 누락된 오답

### 문제 5
정답: ④ "항상 안정(Stable) 정렬"이라는 설명이 틀림
- **퀵 정렬은 불안정(Unstable) 정렬**. 피벗과 멀리 떨어진 같은 값의 원소를 교환할 수 있어 상대 순서가 깨진다.
- 안정 정렬: **버블·삽입·합병·기수** / 불안정 정렬: **선택·퀵·힙·쉘**.
- 오답 이유 (모두 옳은 설명):
  - ① 분할 정복(Divide & Conquer) — 맞음
  - ② 피벗 기준 분할 — 맞음
  - ③ 평균 O(n log n), 최악 O(n²) — 맞음 (피벗이 항상 최소/최대일 때 최악)

### 문제 6
정답: ③ 분할 정복(Divide and Conquer)
- 분할(Divide) → 정복(Conquer, 재귀 해결) → 결합(Combine)의 3단계.
- 대표 예: 합병 정렬, 퀵 정렬, 이분 탐색, 큰 수 곱셈(카라츠바), FFT.
- 오답 이유:
  - ① 동적 계획법 — 부분 문제의 해를 **메모이제이션/테이블**로 저장해 중복 계산 회피 (피보나치, 0/1 배낭)
  - ② 탐욕 — 매 단계 **국소 최적(local optimum)** 선택, 전역 최적 보장 X (거스름돈, 다익스트라)
  - ④ 백트래킹 — 해를 찾는 도중 더 갈 수 없으면 **가지치기(pruning)** 후 되돌아감 (N-Queen, 미로 찾기)

### 문제 7
정답: ② O(log n)
- 매 단계 탐색 범위가 절반으로 줄어듦 → log₂n 단계.
- **전제 조건: 데이터가 정렬되어 있어야 함**.
- 오답 이유:
  - ① O(1) — 해시 테이블 평균 탐색 시간
  - ③ O(n) — 순차 탐색(Linear Search)의 시간복잡도
  - ④ O(n log n) — n log n 정렬 시간

### 문제 8
정답: ④ 삽입 정렬 — 평균 O(n) (X)
- 삽입 정렬의 **평균 시간복잡도는 O(n²)**. **최선(이미 정렬된 경우)만 O(n)**.
- 그래서 평균을 O(n)으로 쓴 ④가 잘못됨.
- 오답 이유 (모두 옳은 매칭):
  - ① 합병 정렬 — 평균=최악 O(n log n) (안정적)
  - ② 퀵 정렬 — 평균 O(n log n), 최악 O(n²) (피벗 편향 시)
  - ③ 힙 정렬 — 평균=최악 O(n log n)

### 핵심 용어
| 용어 | 정의 |
|---|---|
| 시간복잡도(Time Complexity) | 입력 크기 n에 대한 알고리즘 수행 시간 증가율 |
| 빅오 표기법(Big-O Notation) | 알고리즘 수행 시간의 점근적 상한(asymptotic upper bound) |
| 안정 정렬(Stable Sort) | 같은 키 값을 가진 원소들의 입력 순서가 정렬 후에도 유지되는 정렬 |
| 제자리 정렬(In-place Sort) | 입력 배열 외 추가 메모리를 O(1) 또는 O(log n)만 사용하는 정렬 |
| 분할 정복(Divide & Conquer) | 문제를 작은 부분으로 분할 → 재귀 해결 → 결합 |
| 동적 계획법(Dynamic Programming) | 중복 부분 문제를 메모이제이션/테이블에 저장해 재계산 회피 |
| 탐욕 알고리즘(Greedy) | 매 단계 국소 최적을 선택해 전역 해 구성 (전역 최적 보장 X) |
| 백트래킹(Backtracking) | 해 탐색 중 불가능 판단 시 즉시 되돌아가 가지치기 |
| 피벗(Pivot) | 퀵 정렬에서 분할 기준이 되는 원소 |
| 힙(Heap) | 부모-자식 간 크기 관계(최대힙/최소힙)를 만족하는 완전 이진 트리 |

### 핵심 개념
- **정렬 시간복잡도 한눈 비교 (★ 시험 빈출)**

  | 정렬 | 평균 | 최악 | 최선 | 안정성 | 비고 |
  |---|---|---|---|---|---|
  | 버블 | O(n²) | O(n²) | O(n) | 안정 | 단순·교환 다수 |
  | 선택 | O(n²) | O(n²) | O(n²) | **불안정** | 교환 횟수 적음 |
  | 삽입 | O(n²) | O(n²) | O(n) | 안정 | 거의 정렬된 데이터에 강함 |
  | 쉘 | O(n^1.5) | O(n²) | O(n) | **불안정** | 삽입 정렬 개선 |
  | 퀵 | O(n log n) | **O(n²)** | O(n log n) | **불안정** | 평균 가장 빠름·in-place |
  | 합병 | O(n log n) | O(n log n) | O(n log n) | 안정 | 추가 메모리 **O(n)** |
  | 힙 | O(n log n) | O(n log n) | O(n log n) | **불안정** | in-place·완전이진트리 |
  | 기수 | O(d·n) | O(d·n) | O(d·n) | 안정 | 비교 X·자릿수 d |

- **1회전(Pass 1) 결과 핵심 정리**
  - **버블**: 최댓값이 맨 끝으로 (또는 최솟값이 맨 앞으로) 한 개 확정
  - **선택**: 최솟값이 맨 앞으로 한 개 확정 (교환 1회)
  - **삽입**: 2번째 원소만 정렬된 부분에 삽입 → 앞 2개만 정렬됨

- **알고리즘 설계 기법 4종 비교**

  | 기법 | 부분 문제 관계 | 의사결정 | 대표 예 |
  |---|---|---|---|
  | 분할 정복 | 독립 | 모든 분할 후 결합 | 합병·퀵 정렬, 이분 탐색 |
  | 동적 계획법 | 중복 | 작은 문제부터 빌드업 | 피보나치, 배낭, LCS |
  | 탐욕 | 독립적 선택 | 매 단계 국소 최적 | 거스름돈, 다익스트라, 허프만 |
  | 백트래킹 | 트리 탐색 | 불가능 시 되돌아감 | N-Queen, 미로 |

- **탐색 알고리즘 비교**
  - 순차 탐색(Linear Search): O(n), 정렬 불필요
  - 이분 탐색(Binary Search): O(log n), **정렬 필수**
  - 해시 탐색(Hash Search): 평균 O(1), 최악 O(n) (충돌 시)
  - 보간 탐색(Interpolation Search): 평균 O(log log n), 균등 분포 가정
