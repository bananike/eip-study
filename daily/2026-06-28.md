## 🎯 오늘의 기출문제 — 2021년 정처기 필기 20문제 (5과목 골골루 · 회독 3)

### 문제 1 [유사 기출: 2021년 1회 1과목 — 결합도(Coupling) 강도 순서 정형 패턴]
다음 중 결합도(Coupling)의 강도가 **가장 약한(=가장 좋은)** 것은?

① **자료 결합도(Data Coupling)** — 모듈 간 매개변수로 **필요한 자료만** 전달. 가장 이상적·결합 강도 가장 약함.
② **제어 결합도(Control Coupling)** — 한 모듈이 다른 모듈의 **흐름을 제어하는 플래그(flag)** 를 전달.
③ **공통 결합도(Common Coupling)** — 여러 모듈이 **전역 변수(Global)** 를 공유.
④ **내용 결합도(Content Coupling)** — 한 모듈이 다른 모듈의 **내부 코드·지역 변수**를 직접 참조(가장 강·가장 나쁨).

### 문제 2 [유사 기출: 2021년 2회 1과목 — 자료흐름도(DFD) 구성 요소 정형 패턴]
**자료흐름도(Data Flow Diagram, DFD)** 의 **4가지 기본 구성 요소**에 해당하지 **않는** 것은?

① **프로세스(Process)** — 자료를 변환하는 처리·작업. 원(O) 또는 둥근 사각형으로 표기.
② **자료 흐름(Data Flow)** — 자료의 이동 방향을 화살표(→)로 표기.
③ **자료 저장소(Data Store)** — 자료가 저장되는 곳. 이중 평행선(=) 으로 표기.
④ **제어 흐름(Control Flow)** — 모듈 간 제어 신호 흐름을 점선 화살표로 표기(DFD는 자료 중심, 제어 흐름은 **제어 흐름도(CFD)/상태도** 영역).

### 문제 3 [유사 기출: 2021년 3회 1과목 — HIPO 차트 종류 정형 패턴]
**HIPO(Hierarchy Input Process Output)** 차트의 종류로 옳지 **않은** 것은?

① **가시적 도표(Visual Table of Contents, VTOC)** — 시스템 전체 기능을 계층적으로 나타낸 트리 구조 도표.
② **총체적 도표(Overview Diagram)** — 프로그램의 입력·처리·출력 전체 흐름을 한눈에 보여주는 도표.
③ **세부적 도표(Detail Diagram)** — 각 기능의 구체적 입력·처리·출력을 상세히 표현.
④ **물리적 도표(Physical Diagram)** — 하드웨어 배치·네트워크 구조를 표현하는 도표(HIPO 3종에는 포함되지 않는 가짜 항목).

### 문제 4 [유사 기출: 2021년 1회·2회 1과목 — 소프트웨어 측정(LOC vs FP) 정형 패턴]
다음 중 **기능 점수(Function Point, FP)** 기반 소프트웨어 규모 측정에 대한 설명으로 옳은 것은?

① **LOC(Line of Code)** 기반보다 **사용 언어에 종속적**이라 언어 변경 시 매번 환산이 필요하다.
② **알브레히트(Allan Albrecht)** 가 제안한 측정 방식으로, **언어·구현 기술과 독립적**이며 **외부 입력·외부 출력·외부 조회·내부 논리 파일·외부 인터페이스 파일** 5개 기능을 가중치로 측정.
③ FP는 시스템이 **완성된 후에만** 적용 가능하며 초기 개발 단계에는 사용할 수 없다.
④ FP는 객체지향 시스템에만 적용 가능하며 절차지향 시스템에는 적용이 불가하다.

### 문제 5 [유사 기출: 2021년 1회 2과목 — 해시(Hash) 충돌 해결 방법 정형 패턴]
**해시(Hash) 충돌(Collision)** 을 해결하는 방법에 해당하지 **않는** 것은?

① **개방 주소법(Open Addressing)** — 충돌 시 다른 빈 슬롯을 탐사(선형 조사·이차 조사·이중 해싱).
② **체이닝(Chaining, Separate Chaining)** — 같은 해시값을 가지는 키들을 연결 리스트로 연결.
③ **재해싱(Rehashing)** — 적재율(Load Factor)이 일정 이상이 되면 해시 테이블 크기를 늘리고 다시 해싱.
④ **버블 충돌(Bubble Collision)** — 정렬 알고리즘의 인접 비교 과정에서 발생하는 충돌(해시 충돌 해결과 무관한 가짜 용어).

### 문제 6 [유사 기출: 2021년 2회 2과목 — 알고리즘 점근 표기법(Big-O) 정형 패턴]
알고리즘 분석에서 **Big-O 표기법**의 정의로 가장 옳은 것은?

① **점근적 상한(Upper Bound)** — 알고리즘이 **최악의 경우(Worst Case)** 입력에서도 이 함수 이하로 동작함을 보장(f(n) ≤ c·g(n)).
② 점근적 하한(Lower Bound) — 알고리즘이 최소 이 함수 이상의 시간을 필요로 함(Big-Ω 표기의 정의).
③ 점근적 평균(Average Bound) — 알고리즘의 평균 수행 시간을 정확히 표현(별도 표기 없음).
④ 점근적 상하한 동시(Tight Bound) — Big-O와 Big-Ω가 동시에 성립하는 경우(Big-Θ 표기의 정의).

### 문제 7 [유사 기출: 2021년 3회 2과목 — 빌드 자동화 도구 분류 정형 패턴]
다음 중 **소프트웨어 빌드(Build) 자동화 도구**에 해당하지 **않는** 것은?

① **Jenkins** — Java 기반 CI/CD 자동화 서버. 빌드·테스트·배포 파이프라인 구성.
② **Maven** — Apache 재단의 Java 프로젝트 빌드·의존성 관리 도구. pom.xml 기반 선언형.
③ **Gradle** — Groovy/Kotlin DSL 기반 빌드 도구. Android 공식 빌드 시스템.
④ **JUnit** — Java **단위 테스트(Unit Test) 프레임워크**로, 빌드 자동화 도구가 아닌 테스트 프레임워크.

### 문제 8 [유사 기출: 2021년 1회·3회 2과목 — 검토(Review) 기법 정형 패턴]
다음 중 **공식 기술 검토(Formal Technical Review)** 기법인 **인스펙션(Inspection)** 에 대한 설명으로 가장 옳은 것은?

① **저자(Author) 주도의 비공식 검토**로, 동료들이 자유롭게 모여 결함을 토론하는 형태(워크스루 설명).
② **모더레이터(Moderator) 주도·체크리스트 기반의 공식 절차**로, 사전 준비·회의·후속 조치 등 정형 단계를 거쳐 결함을 발견·기록·추적한다.
③ 소스 코드 없이 **요구사항 명세만으로** 검토를 수행하는 정적 분석 기법(연관성 없음).
④ 실행 중인 시스템의 메모리 누수·성능 병목을 검출하는 **동적 분석 기법**(인스펙션은 정적 검토).

### 문제 9 [유사 기출: 2021년 2회 3과목 — 트랜잭션 상태(State) 5종 정형 패턴]
**트랜잭션(Transaction) 상태 전이 모델**에서 **"부분 완료(Partially Committed)"** 상태에 대한 설명으로 가장 옳은 것은?

① 트랜잭션이 시작되어 연산을 실행 중인 상태(활동·Active의 설명).
② **마지막 연산이 실행된 직후**, 변경 내용이 **아직 디스크에 반영되기 전** 상태로, 곧 Committed로 전이되거나 장애 발생 시 Failed로 갈 수 있다.
③ 트랜잭션이 정상적으로 종료되어 변경 결과가 영구히 반영된 상태(완료·Committed의 설명).
④ 트랜잭션이 실행 도중 오류로 중단된 상태(실패·Failed의 설명).

### 문제 10 [유사 기출: 2021년 1회 3과목 — 뷰(View) 갱신 제약 정형 패턴]
**관계형 데이터베이스의 뷰(View)** 에 대한 설명으로 옳지 **않은** 것은?

① 뷰는 **하나 이상의 기본 테이블에서 유도된 가상 테이블(Virtual Table)** 로, 실제 데이터를 별도로 저장하지 않는다.
② 뷰는 **ALTER 문으로 컬럼 구조를 변경할 수 없으며**, 변경하려면 DROP 후 다시 CREATE 해야 한다.
③ 모든 뷰는 **그룹 함수·DISTINCT·JOIN·UNION을 포함하더라도** 항상 INSERT·UPDATE·DELETE가 가능하다.
④ 뷰는 **논리적 데이터 독립성**을 제공하며, 사용자별로 필요한 데이터만 노출해 **보안 측면**에서도 유용하다.

### 문제 11 [유사 기출: 2021년 2회·3회 3과목 — 서브쿼리(상관/비상관) 정형 패턴]
다음 중 **상관(Correlated) 서브쿼리**에 대한 설명으로 가장 옳은 것은?

① 서브쿼리가 외부 쿼리와 **무관하게 독립적으로 한 번 실행**되고, 그 결과를 외부 쿼리가 받아 처리한다(비상관 서브쿼리의 설명).
② 서브쿼리가 **외부 쿼리의 컬럼을 참조**하여, **외부 쿼리의 각 행(row)마다 반복 실행**된다. 일반적으로 비상관 서브쿼리보다 성능이 느릴 수 있다.
③ 서브쿼리는 항상 SELECT 절에만 위치할 수 있으며, WHERE·HAVING·FROM 절에는 사용할 수 없다.
④ 상관 서브쿼리는 반드시 단일 행만 반환해야 하며, 다중 행 연산자(IN/ANY/ALL/EXISTS)와 함께 사용할 수 없다.

### 문제 12 [유사 기출: 2021년 1회·2회 3과목 — 회복(Recovery) 기법 정형 패턴]
**데이터베이스 회복(Recovery) 기법**에 대한 설명으로 옳지 **않은** 것은?

① **즉시 갱신(Immediate Update)** 기법은 트랜잭션 도중에도 변경을 디스크에 반영하므로 회복 시 **UNDO와 REDO 모두 필요**할 수 있다.
② **지연 갱신(Deferred Update)** 기법은 트랜잭션 완료 시점까지 변경을 미루므로 회복 시 **UNDO는 불필요하고 REDO만 필요**하다.
③ **검사점(Checkpoint)** 은 회복 시작점을 지정하여 **회복 시간을 단축**시키며, 검사점 이전 완료 트랜잭션은 회복 대상에서 제외된다.
④ **그림자 페이징(Shadow Paging)** 은 페이지의 사본을 유지하는 방식으로, **반드시 로그(log)를 추가로 사용**해야 하며 로그 없이는 동작하지 않는다.

### 문제 13 [유사 기출: 2021년 1회 4과목 — C 비트 연산자 정형 패턴]
다음 C 프로그램의 출력 결과로 옳은 것은?

```c
#include <stdio.h>
int main() {
    int a = 0x0F;     // 0000 1111 (15)
    int b = 0x3A;     // 0011 1010 (58)
    printf("%d %d %d", a & b, a | b, a ^ b);
    return 0;
}
```

① 10 63 53  (AND=0x0A=10, OR=0x3F=63, XOR=0x35=53)
② 15 58 53  (AND=15, OR=58, XOR=53 — AND를 잘못 본 경우)
③ 0 73 53   (AND=0, OR=합=73 — 비트 연산을 산술로 본 함정)
④ 10 53 63  (XOR과 OR을 뒤바꾼 경우)

### 문제 14 [유사 기출: 2021년 2회 4과목 — Java 예외처리(try-catch-finally) 정형 패턴]
**Java의 예외처리(try-catch-finally)** 에 대한 설명으로 옳지 **않은** 것은?

① **try 블록**에서 발생한 예외는 해당 예외의 타입과 일치하는 **catch 블록**에서 처리된다.
② **finally 블록**은 예외 발생 여부·return 문 실행 여부와 관계없이 **항상 실행**되어 자원 해제(close·release) 코드를 두는 데 주로 사용된다.
③ **Checked Exception**(예: IOException)은 컴파일러가 처리 여부를 검사하므로 **try-catch 또는 throws 선언이 필수**이다.
④ **Unchecked Exception**(RuntimeException 계열)도 **컴파일러가 처리 여부를 강제**하므로, try-catch 없이는 컴파일 자체가 실패한다.

### 문제 16 [유사 기출: 2021년 1회·2회 4과목 — CSMA/CD vs CSMA/CA 정형 패턴]
다음 중 **CSMA/CA(Carrier Sense Multiple Access / Collision Avoidance)** 에 대한 설명으로 가장 옳은 것은?

① **유선 LAN(이더넷·IEEE 802.3)** 의 매체 접근 제어 방식으로, **충돌 감지(Detection)** 후 백오프(Backoff) 후 재전송한다.
② **무선 LAN(Wi-Fi·IEEE 802.11)** 의 매체 접근 제어 방식으로, **충돌을 사전에 회피(Avoidance)** 하기 위해 **RTS/CTS·IFS·랜덤 백오프**를 사용한다.
③ 토큰을 가진 노드만 송신할 수 있는 방식으로, **토큰 링(Token Ring)** 네트워크에서 사용된다.
④ 회선을 시간 단위로 분할해 각 노드에 고정 할당하는 **TDMA** 방식의 일종이다.

### 문제 17 [유사 기출: 2021년 1회 5과목 — CMMI 5단계 정형 패턴]
**CMMI(Capability Maturity Model Integration) 성숙도 5단계** 순서로 옳은 것은?

① **Initial(초기) → Defined(정의) → Managed(관리) → Quantitatively Managed(정량적 관리) → Optimizing(최적화)** — Managed와 Defined 순서가 뒤바뀐 함정 보기.
② **Initial(초기) → Managed(관리) → Defined(정의) → Quantitatively Managed(정량적 관리) → Optimizing(최적화)** — 두문자 "초·관·정·정·최".
③ **Initial(초기) → Repeatable(반복) → Defined(정의) → Managed(관리) → Optimizing(최적화)** — CMM(구버전·5단계) 순서로 CMMI와 혼동한 함정.
④ **Incomplete(불완전) → Performed(수행) → Managed(관리) → Established(확립) → Predictable(예측) → Optimizing(최적화)** — SPICE(ISO/IEC 15504) 6단계(0~5)로 CMMI와 다른 모델.

### 문제 18 [유사 기출: 2021년 2회 5과목 — CC(Common Criteria) 평가 정형 패턴]
**공통 평가 기준(Common Criteria, CC, ISO/IEC 15408)** 의 **보증 등급(EAL, Evaluation Assurance Level)** 에 대한 설명으로 옳은 것은?

① CC의 EAL은 **EAL1~EAL5** 5단계로 구성되며, 등급이 높을수록 보증 수준이 높아진다.
② CC의 EAL은 **EAL1~EAL7** 7단계로 구성되며, **EAL1(기능 시험)** 이 가장 낮고 **EAL7(정형 검증된 설계 및 시험)** 이 가장 높다.
③ CC는 미국 국방부의 **TCSEC(오렌지북) 기준만**을 통합한 것으로, 유럽의 ITSEC과는 무관하다.
④ CC의 EAL은 **암호 모듈만 평가**하기 위한 등급으로, OS·DBMS 등 일반 IT 제품에는 적용되지 않는다.

### 문제 19 [유사 기출: 2021년 3회 5과목 — BCP/DRP의 RTO·RPO 정형 패턴]
**업무 연속성 계획(BCP)** 및 **재해 복구 계획(DRP)** 에서 사용되는 핵심 지표에 대한 설명으로 옳지 **않은** 것은?

① **RTO(Recovery Time Objective)** — **목표 복구 시간**. 재해 발생 후 시스템이 정상 가동되어야 하는 **최대 허용 중단 시간**.
② **RPO(Recovery Point Objective)** — **목표 복구 시점**. 재해 발생 시 **허용 가능한 최대 데이터 손실 시점**(=백업 주기).
③ **MTD(Maximum Tolerable Downtime)** — **최대 허용 다운타임**. 이 시간을 초과하면 비즈니스에 치명적 영향. RTO ≤ MTD 관계가 일반적.
④ **RTO와 RPO는 항상 0**이어야 하며, 0이 아닌 값을 설정하는 것은 BCP/DRP 표준에 위배된다(현실적으로 비용·기술 한계로 0 설정 불가 — 잘못된 진술).

## 💡 상세 해설

### 문제 1
**정답 ①** — **결합도 6단계 약→강**: **자료(Data) → 스탬프(Stamp) → 제어(Control) → 외부(External) → 공통(Common) → 내용(Content)** (두문자 "**자·스·제·외·공·내**"). 가장 약함=자료, 가장 강함=내용.
- 자료 결합도: 매개변수로 **필요한 자료만** 전달 — 이상적.
- 내용 결합도: 다른 모듈의 **내부 코드·변수 직접 참조** — 최악.
- 함정: 응집도와 반대로 결합도는 "낮을수록(약할수록) 좋음". 모듈 황금률 = **높은 응집도 + 낮은 결합도**.

### 문제 2
**정답 ④** — **DFD 4구성 요소**: **프로세스(O) / 자료 흐름(→) / 자료 저장소(=) / 단말(외부 entity, □)**. "제어 흐름"은 DFD가 아닌 **CFD(Control Flow Diagram) / 상태도(State Diagram)** 영역.
- 핵심: DFD는 **자료 중심**(데이터가 어디서 어떻게 흐르고 저장되는가) — 제어 로직은 별도 표기.
- 함정: "단말"을 "프로세서"로 바꿔치기 — 단말(외부 entity)은 시스템 외부 사용자·시스템.

### 문제 3
**정답 ④** — **HIPO 차트 3종**: **가시적 도표(VTOC, 트리) / 총체적 도표(Overview, 전체 흐름) / 세부적 도표(Detail, 상세)**. "물리적 도표"는 HIPO 표준에 없는 가짜 항목.
- HIPO 핵심: **Hierarchy(계층)** + **Input-Process-Output**으로 시스템 기능을 표현하는 **하향식 도표**.
- 활용: 시스템 분석·설계 단계에서 기능 분해와 입출력 명세 동시 표기.

### 문제 5
**정답 ④** — **해시 충돌 해결 방법**: **개방 주소법(선형 조사·이차 조사·이중 해싱) / 체이닝(연결 리스트) / 재해싱(테이블 확장)**. "버블 충돌"은 가짜 용어.
- 개방 주소법 vs 체이닝 비교: 개방 주소법은 **테이블 내부 해결**(군집화 문제), 체이닝은 **외부 리스트**(메모리 추가 비용 + 최악 O(n)).
- 적재율(Load Factor) α = 입력 키 수 / 테이블 크기. α가 0.7~0.75 초과 시 **재해싱(Rehashing)** 권장.

### 문제 6
**정답 ①** — **Big-O = 점근적 상한(Upper Bound)**. f(n) = O(g(n)) ⟺ ∃ c, n₀ : 모든 n ≥ n₀에 대해 **f(n) ≤ c·g(n)**.
- **Big-Ω(Omega) = 점근적 하한**(최선·최소 시간). f(n) ≥ c·g(n).
- **Big-Θ(Theta) = 점근적 상하한 동시**(타이트 한계). Big-O와 Big-Ω가 동시 성립.
- 실무: "최악의 경우" 분석 = Big-O / "최선의 경우" = Big-Ω. 일반적 표기는 **Big-O가 최악·평균 모두를 대표**해 가장 자주 출제.

### 문제 10
**정답 ③** — 뷰 갱신 함정: **모든 뷰가 INSERT/UPDATE/DELETE 가능한 것은 아님**. **갱신 불가 뷰 조건**:
- 뷰 정의에 **GROUP BY·HAVING·DISTINCT·집계 함수·UNION·JOIN(여러 테이블)·서브쿼리** 등이 포함된 경우 갱신 불가(또는 제한적).
- 단순 단일 테이블 SELECT 뷰는 갱신 가능.
- **뷰의 컬럼은 ALTER VIEW로 변경 불가** — DROP 후 CREATE만 가능(②는 옳음).
- 뷰의 효용: 논리적 데이터 독립성 + 보안(필요 데이터만 노출)(④는 옳음).

### 문제 11
**정답 ②** — **상관(Correlated) 서브쿼리**: **외부 쿼리의 컬럼을 참조** → 외부 각 행마다 서브쿼리 반복 실행. 일반적으로 비상관보다 성능 ↓.
- **비상관(Non-correlated)**: 외부 쿼리와 무관하게 한 번만 실행, 결과를 외부 쿼리가 받아 처리.
- 위치: SELECT·WHERE·HAVING·FROM 절 모두 가능(③ 함정).
- 다중 행 연산자(IN/ANY/ALL/EXISTS): 상관 서브쿼리에서도 사용 가능(④ 함정). **EXISTS는 상관 서브쿼리에서 특히 많이 사용**.

### 문제 12
**정답 ④** — **그림자 페이징(Shadow Paging)** 은 **페이지 사본 유지 방식으로 로그(log)가 불필요**한 회복 기법. ④는 정반대.
- **로그 기반(log-based) 회복**: 즉시 갱신(UNDO+REDO) / 지연 갱신(REDO만).
- **로그 없는(no-log) 회복**: **그림자 페이징** — 트랜잭션 시작 시 현재 페이지 테이블 사본 생성, 변경은 새 페이지에, 커밋 시 페이지 테이블 교체.
- 검사점(Checkpoint): **회복 시간 단축** — 검사점 이전 완료 트랜잭션은 회복 대상에서 제외(③ 옳음).

### 문제 13
**정답 ①** — 비트 연산:
- a = 0x0F = 0000 1111 = 15
- b = 0x3A = 0011 1010 = 58
- **a & b**(AND): 0000 1010 = **0x0A = 10** (둘 다 1인 비트만)
- **a | b**(OR): 0011 1111 = **0x3F = 63** (둘 중 하나라도 1)
- **a ^ b**(XOR): 0011 0101 = **0x35 = 53** (서로 다른 비트만 1)
- 함정: ③은 "OR을 산술 합 73"으로 본 오류. 비트 연산은 비트별 계산.
- 추가: **~a**(NOT, 1의 보수) / **a << n**(좌시프트, ×2ⁿ) / **a >> n**(우시프트, ÷2ⁿ).

### 문제 14
**정답 ④** — Java **Unchecked Exception(RuntimeException 계열)** 은 **컴파일러가 처리 강제 X** — try-catch 없어도 컴파일 성공.
- **Checked Exception**(IOException·SQLException·ClassNotFoundException 등): **try-catch 또는 throws 선언 필수**(컴파일러 검사).
- **Unchecked Exception**(NullPointerException·ArrayIndexOutOfBoundsException·ArithmeticException 등): **선택적 처리**.
- **finally**: 예외 발생 여부와 무관하게 항상 실행(자원 해제). **System.exit() 호출 시에만 finally도 실행 안 됨**.
- Java 7+: **try-with-resources** 문으로 AutoCloseable 자원 자동 해제.

### 문제 15
**정답 ②** — `filter(lambda x: x%2==1, nums)` → 홀수만 = [1, 3, 5] → `map(lambda x: x*x, ...)` → 제곱 = **[1, 9, 25]**.
- Python lambda: 익명 함수. `lambda 인자: 표현식`.
- map / filter / reduce(functools): 함수형 프로그래밍 3대 도구.
- map(f, iter) = iter 각 요소에 f 적용 / filter(f, iter) = f가 True인 요소만 선택 / reduce(f, iter) = 누적 계산.
- 함정: filter 조건 `x%2==1`(홀수) vs `x%2==0`(짝수) 헷갈리지 말 것.

### 문제 16
**정답 ②** — **CSMA/CA = 무선 LAN(Wi-Fi, IEEE 802.11)** 매체 접근 제어. 충돌을 **사전에 회피(Avoidance)** — RTS(Request to Send)·CTS(Clear to Send) 핸드셰이크, IFS(Inter-Frame Space) 대기, 랜덤 백오프.
- **CSMA/CD = 유선 LAN(이더넷, IEEE 802.3)** 매체 접근 제어. 충돌을 **사후 감지(Detection)** 후 백오프·재전송.
- 무선에서 CD가 어려운 이유: 송신 중에는 자신의 신호가 너무 강해 충돌 감지 불가 + 숨겨진 단말 문제(Hidden Terminal).
- 비교: **CD = 사후 감지·유선 / CA = 사전 회피·무선**.

### 문제 17
**정답 ②** — **CMMI 5단계 두문자 "초·관·정·정·최"**: **Initial(초기·1) → Managed(관리·2) → Defined(정의·3) → Quantitatively Managed(정량적 관리·4) → Optimizing(최적화·5)**.
- 함정 ①: Managed와 Defined 순서가 뒤바뀐 형태(가장 흔한 오답).
- 함정 ③: **CMM(구버전·1991)** = Initial → Repeatable → Defined → Managed → Optimizing. CMMI는 Repeatable이 사라지고 Managed가 2단계로.
- 함정 ④: **SPICE(ISO/IEC 15504) 6단계 0~5** = Incomplete → Performed → Managed → Established → Predictable → Optimizing. CMMI(SEI·5단계·1~5)와 모델·시작단계가 다름.

### 문제 18
**정답 ②** — **CC(Common Criteria, ISO/IEC 15408)** = **EAL1 ~ EAL7 7단계**. EAL1(기능 시험) → EAL7(정형 검증된 설계 및 시험).
- EAL1: 기능 시험(Functionally Tested) / EAL2: 구조 시험 / EAL3: 체계 시험 및 검사 / EAL4: 체계 설계·시험·검토(상용 SW 일반 목표) / EAL5: 준정형 설계·시험 / EAL6: 준정형 검증된 설계·시험 / EAL7: 정형 검증된 설계·시험.
- CC 배경: 미국 **TCSEC(오렌지북)** + 유럽 **ITSEC** + 캐나다 CTCPEC 통합 → **공통 평가 기준**(③ 함정).
- CC 적용: OS·DBMS·방화벽·스마트카드 등 **모든 IT 제품·시스템**(암호 모듈 한정 X — ④ 함정).

### 문제 19
**정답 ④** — RTO·RPO가 **항상 0이어야 한다는 것은 잘못된 진술**. 현실적으로 비용·기술 제약으로 0 설정 불가능 — 비즈니스 중요도에 따라 적절히 설정(예: 결제 RTO≤1시간·RPO≤5분 / 일반 업무 RTO≤24시간·RPO≤1일).
- **RTO(Recovery Time Objective)** = 목표 복구 시간. 재해 후 시스템 정상 가동까지 허용 시간.
- **RPO(Recovery Point Objective)** = 목표 복구 시점. 허용 데이터 손실 시점(=백업 주기와 직결).
- **MTD(Maximum Tolerable Downtime)** = 최대 허용 다운타임. RTO ≤ MTD 관계.
- **BCP**(Business Continuity Plan) ⊃ **DRP**(Disaster Recovery Plan) — BCP가 더 포괄적(업무 전반·DRP는 IT 시스템 복구 중심).

### 문제 20
**정답 ④** — **SHA-256은 암호학적 해시 함수**로 데이터 무결성 검증 용도. SSO 구현 핵심 기술은 **Kerberos·SAML·OAuth(+ OpenID Connect)**.
- **Kerberos**: 티켓 기반 인증, KDC·AS·TGS 구조. Active Directory 기반.
- **SAML 2.0**: XML 기반 인증·인가 정보 교환. 엔터프라이즈 웹 SSO 표준.
- **OAuth 2.0 + OpenID Connect**: 권한 위임·인증 통합. 소셜 로그인(Google·Facebook·GitHub) SSO.
- SHA-256의 용도: 비밀번호 해시 저장, 무결성 검증, 토큰 서명 보조 — SSO 자체의 인증·인가 프로토콜이 아님.

## 📊 자주 틀리는 함정 5가지 (2021년 회독 3/4 기준)

1. **결합도 자료 결합도(약·이상적)** vs **응집도 기능적 응집도(강·이상적)** — 한쪽은 약할수록 좋고 한쪽은 강할수록 좋음. 절대 헷갈리지 말 것.
2. **DFD 4구성 요소(프로세스·자료흐름·자료저장소·단말)** — "제어 흐름" 끼워넣기 함정. 제어 흐름은 CFD/상태도.
3. **CMMI 2단계 = Managed / 3단계 = Defined** — "초·관·정·정·최" 두문자 암기. CMM과 SPICE와 절대 혼동 X.
4. **CC EAL1~EAL7** — "5단계" 함정 자주 출제. 7단계가 정답.
5. **그림자 페이징 = 로그 불필요** vs **즉시/지연 갱신 = 로그 필요** — 회복 기법별 로그 사용 여부.
